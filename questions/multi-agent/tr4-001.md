# LangGraph 的状态机模型与传统 Chain（如 LangChain）有何本质区别？在处理循环逻辑时有什么优势？

LangChain 的传统 Chain 主要基于线性或简单的 DAG 结构，每个步骤的输出直接传递给下一步，难以处理复杂的循环、条件分支或状态回退。LangGraph 的核心在于将 Agent 的执行过程建模为**有向循环图**或**有限状态机（FSM）**。它维护了一个全局的共享状态，节点通过边连接，边定义了状态转移的条件。这种架构的优势在于：1. **原生支持循环**：可以轻松实现“反思-修正”的迭代过程，例如生成代码后执行，若报错则自动回退到生成节点重试，这在传统链中很难优雅实现。2. **可控性**：状态机的每一步都是可观测和可中断的，便于调试和人工干预。3. **复杂交互**：非常适合模拟多轮对话、多轮工具调用的复杂工作流，打破了线性执行的局限。
