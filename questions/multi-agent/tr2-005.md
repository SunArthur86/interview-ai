# 在 Multi-Agent 系统中，CrewAI 的“角色扮演”模式与 LangGraph 的“图状态”模式在构建协作逻辑时有什么本质区别？

CrewAI 基于角色扮演的预设逻辑，核心是将 Agent 定义为具有特定角色、目标和背景故事的“员工”，通过顺序或层级的过程来协作，适合结构化明确、分工清晰的业务流（如营销团队模拟）。其逻辑相对固定，依赖预定义的 Task 链。LangGraph 则基于有向循环图和全局状态机，核心是定义状态如何在节点间流转和转换。它更加灵活，支持复杂的条件跳转、回退和动态分支，适合构建需要根据中间结果动态改变执行路径的应用（如复杂的客服审批流）。简言之，CrewAI 更像排好的剧本，LangGraph 更像一套可编程的逻辑控制器。

### 深化内容

**实战案例**：在构建代码审计系统时，CrewAI 容易因为某一个静态任务报错导致整个流程中断，必须手动重试；而利用 LangGraph 的图结构，我实现了“代码审查失败自动回退到开发节点要求修改”的逻辑，不仅容错率高，还能根据审查结果动态决定是继续测试还是驳回。

**代码示例**：
```python
# LangGraph 条件边实现：根据状态决定下一步路由
def should_review(state: AgentState):
    if state["error_count"] > 3:
        return "escalate_to_human"  # 动态跳转人工
    return "auto_fix"              # 继续自动修复

workflow.add_conditional_edges("review_node", should_review, {
    "auto_fix": "fixer_node",
    "escalate_to_human": "human_node"
})
```

**对比表格**：

| 特性维度 | CrewAI (角色扮演) | LangGraph (图状态) |
| :--- | :--- | :--- |
| **核心逻辑** | 顺序执行的预设任务链 | 循环图与状态流转机 |
| **动态能力** | 弱，主要依赖静态配置 | 强，支持运行时条件分支和回退 |
| **适用场景** | 职责明确的流水线（如内容生成、翻译） | 复杂决策流（如客服审批、自动化运维） |
| **数据共享** | 通过 Task 输出传递 | 全局共享 State，适合多轮记忆累积 |
| **调试难度** | 较低，流程线性可视 | 较高，需追踪节点间状态变化 |

## 记忆要点

- CrewAI 是静态剧本，LangGraph 是动态状态机。
- CrewAI 适合线性流水线，LangGraph 支持循环与回退。
- CrewAI 靠 Task 传递数据，LangGraph 靠全局共享 State。
- 复杂决策选 LangGraph，明确分工选 CrewAI。
