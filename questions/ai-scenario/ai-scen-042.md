# 如何设计一个AI驱动的产品搜索与排序系统？结合语义搜索、个性化排序和LLM增强。

### 场景分析
AI驱动的产品搜索与排序系统旨在解决传统关键词搜索无法理解用户深层意图和商品语义的问题。整体流程为：用户输入 Query → 语义理解与改写 → 多路召回（向量/文本/图） → 多阶段排序（粗排/精排/重排） → LLM 增强与结果展示。

### 核心技术点
1. **语义召回**：利用 BGE/M3E 等模型将 Query 和 商品 Embedding 映射到同一向量空间，解决“口红”搜出“唇釉”的问题。
2. **LLM 增强排序**：在精排后，利用 LLM 对 Top-K 商品进行重排或生成摘要，引入逻辑推理（如“适合送女友”）。
3. **个性化**：在 Prompt 或特征中注入用户历史行为向量。

### 实战案例
某电商平台在上线向量召回初期，发现搜索“苹果”时，水果和手机混杂严重。通过引入**多路意图分支**：利用 LLM 预判用户意图（购物/资讯/生鲜），若判定为“生鲜”则强插品类过滤规则，将手机召回结果权重降权，解决了跨品类干扰问题。

### 关键代码示例 (Python: 伪代码)
```python
def search_pipeline(query, user_profile):
    # 1. Query 理解与改写
    intent = llm_classify_intent(query) # "buy_phone"
    rewrites = query_expansion_model(query)
    
    # 2. 多路召回 (向量 + 倒排)
    vector_docs = vector_store.search(query embedding, top_k=50)
    bm25_docs = es_search.search(query, top_k=50)
    
    # 3. 多路融合 (Reciprocal Rank Fusion)
    fused_candidates = rrf_fusion(vector_docs, bm25_docs)
    
    # 4. 精排模型预测
    features = extract_features(fused_candidates, user_profile)
    scores = xgboost_model.predict(features)
    
    # 5. LLM 重排与解释 (仅对Top-10，节省成本)
    top_items = sort_by_score(fused_candidates, scores)[:10]
    final_rank = llm_rerank(query, top_items)
    return final_rank
```

### 召回策略对比

| 策略 | 核心原理 | 优势 | 劣势 | 适用场景 |
| :--- | :--- | :--- | :--- | :---
| **BM25/关键词** | 精确匹配字符 | 解释性强，速度快 | 无法处理语义同义词 | 品牌、型号、长尾词精确搜
| **向量检索** | 语义空间相似度 | 解决同义词、模糊表达 | 存在语义漂移，计算成本高 | 口语化查询、以图搜物
| **图谱召回** | 实体关系关联 | 可解释性强，关联推荐 | 构建维护成本高 | 替换品、搭配推荐

### 系统优化
- **向量索引优化**：使用 HNSW 算法平衡召回率与延迟。
- **LLM 延迟优化**：重排环节仅对精排后的 Top-N（如 20 个）进行 LLM 打分，而非全量。
