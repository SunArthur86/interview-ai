# 什么是过拟合(Overfitting)?如何判断和防止

- **过拟合** = 模型'死记硬背'训练数据中的噪声和特例，导致在未见的新数据上表现差。

- **类比**：学生只背下了历年真题的答案（包括错误的题干），但没有理解原理，考试稍微换个数字就不会了。

- **判断方法**
- 训练集准确率高(95%+)，但验证集/测试集准确率低(70%)，且差距很大。
- 训练 Loss 持续下降，验证 Loss 先降后升（出现拐点）。

- **Loss 曲线示意图**
```
Loss
  │
  │      Train Loss (一直下降)
  │     ───────────────
  │    /
  │   /  Val Loss (先降后升)
  │  /   ────╱
  │ /   ╱
  │/ ╱
  └────────────────────▶ Epoch
```

- **实战案例**：在做用户流失预测时，使用 20 层全连接网络，训练集准确率 99%，但测试集只有 65%。这是典型的过拟合，原因是模型参数量（千万级）远大于数据量（万级）。将网络缩减为 3 层并增加 Dropout 后，测试集准确率反升至 82%。

- **解决方案对比**

| 方案 | 核心机制 | 适用场景 | 代价/副作用 |
| :--- | :--- | :--- | :--- |
| **L1/L2 正则化** | 在 Loss 中加入权重惩罚项 | 通用，特别是特征多时 | L1 计算稍慢；L2 可能导致欠拟合(λ过大) |
| **Dropout** | 随机丢弃神经元，抑制共适应 | 全连接层，CV/NLP | 训练时间变长（收敛慢） |
| **早停** | 监控 Val Loss，保存最佳模型 | 几乎所有场景 | 需要手动耐心调参 patience |
| **数据增强** | 造数据（旋转、裁剪、加噪） | CV(图像)、音频 | 数据增强逻辑需符合业务逻辑 |

- **代码示例**：
```python
import torch.nn as nn

class Net(nn.Module):
    def __init__(self):
        super().__init__()
        # 防止过拟合的组合拳：Linear层 + Dropout + L2正则化(weight_decay)
        self.fc = nn.Sequential(
            nn.Linear(100, 50),
            nn.ReLU(),
            nn.Dropout(p=0.5),  # 训练时随机丢弃50%神经元
            nn.Linear(50, 10)
        )
    
    def forward(self, x):
        return self.fc(x)

# Optimizer 中设置 L2 正则化 (weight_decay 对应 lambda)
# optimizer = torch.optim.Adam(model.parameters(), lr=0.001, weight_decay=1e-4)
```

- **防止方法**
1. **更多数据**：数据分布越广，模型越难死记硬背。
2. **正则化**：在 Loss 函数中加入 L1/L2 项，惩罚过大的权重，限制模型复杂度。
3. **Dropout**：训练时随机“丢弃”一部分神经元，迫使网络学习冗余特征，防止共适应。
4. **早停**：监控验证集 Loss，当不再下降时停止训练，保留最佳模型。
5. **数据增强**：通过旋转、裁剪、加噪等方式人工扩充数据集。
6. **简化模型**：减少层数或神经元数量，降低模型容量。
7. **交叉验证**：更充分地利用数据评估模型性能，减少偶然性。

- **一句话理解**：过拟合 = 模型'背题'不'学知识'，防过拟合 = 强迫模型'理解'规律而非'记忆'样本。

- ## 常见考点
1. **偏差-方差权衡**：解释过拟合是高方差，欠拟合是高偏差。
2. **L1 vs L2 正则化**：L1 倾向于产生稀疏解（特征选择），L2 倾向于让权重分布更均匀但不为零。
3. **Dropout 训练与测试时的区别**：训练时随机丢弃，测试时所有神经元保留但权重需乘以保留概率（或使用 Inverted Dropout）。
