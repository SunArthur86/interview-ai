# 神经网络是怎么'学习'的?用一句话和一个例子解释.

- **一句话**：神经网络通过'猜答案→看对错→调整'的循环不断改进。

- **例子：学认猫**
1. **猜(前向传播)**：看图片，输入像素，权重计算后输出'这是猫的概率：0.3'
2. **看对错**：实际标签是猫，但模型说 0.3 → 计算差距 = |1.0 - 0.3| = 0.7
3. **调整(反向传播)**：求导计算梯度，按梯度反方向微调权重，让下次猜得更接近 1.0
4. **重复**：迭代百万次后，参数收敛，猜得很准

- **架构图(单步训练)**
```
输入图片    权重(W)    预测值      损失(L)      梯度      更新权重(W')
 ┌───┐  ┌─────────┐  ┌──────┐   ┌───────┐   ┌─────┐   ┌─────────┐
 │ x │──▶│  f(x,W) │──▶│ 0.3  │──▶│ Loss │──▶│ ∇W  │──▶│ W - lr*∇W │
 └───┘  └─────────┘  └──────┘   └───────┘   └─────┘   └─────────┘
             ↑              │                       │
             └──────────────┴───────────────────────┘
                    反向传播
```

- **核心概念**
- **Loss(损失)**：猜的答案和正确答案的差距，衡量模型“错”得有多离谱。
- **Gradient(梯度)**：Loss 关于参数的偏导数，指示参数往哪个方向调能让 Loss 变小。
- **Optimizer(优化器)**：如 SGD/Adam，负责根据梯度计算具体的参数更新步长和方向。
- **Learning Rate(学习率)**：控制参数调整的幅度，太大导致震荡，太小收敛慢。

- **实战案例**：在图像分割项目中训练 DeepLabV3+，发现 Loss 一直卡在 0.5 不下降。排查发现学习率设置过大（0.1），导致参数在最优解附近反复跳跃震荡。使用了 `CosineAnnealingLR`（余弦退火学习率）让学习率随周期衰减，模型最终顺利收敛，IoU 提升了 15%。

- **代码示例**：
```python
import torch
import torch.nn as nn
import torch.optim as optim

# 1. 定义模型、损失函数和优化器
model = nn.Linear(10, 2) 
criterion = nn.CrossEntropyLoss()
optimizer = optim.Adam(model.parameters(), lr=0.001)

# 2. 模拟一次训练步骤
inputs = torch.randn(5, 10) # Batch size=5
labels = torch.randint(0, 2, (5,))

# 前向传播
outputs = model(inputs)
loss = criterion(outputs, labels)

# 反向传播与优化
optimizer.zero_grad() # 清空过往梯度（踩坑点：容易忘写导致梯度累加）
loss.backward()       # 计算梯度
optimizer.step()      # 更新权重 W = W - lr * gradient
```

- **一句话理解**：神经网络学习 = '反复猜→对答案→微调'直到 Loss 最小化。

- ## 常见考点
1. **局部极小值 vs 全局最小值**：非凸优化中，梯度下降可能陷入局部最优，如何解决？（如随机重启、动量项）。
2. **Batch Size 的影响**：大 Batch 梯度稳但泛化可能差，小 Batch 噪声大但有助于跳出局部最优。
3. **激活函数的作用**：如果没有非线性激活函数，多层神经网络等价于一层线性变换，无法拟合复杂函数。
