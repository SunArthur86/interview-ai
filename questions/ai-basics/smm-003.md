# Stable Diffusion的原理是什么？Latent Diffusion如何工作？

🎯 本质：Stable Diffusion是一种潜在扩散模型(Latent Diffusion Model)，在压缩的潜在空间中进行扩散和去噪，而非在像素空间，大幅降低计算成本。

📊 **Diffusion基本原理：**
前向过程（加噪）：逐步给清晰图片加高斯噪声，直到变成纯噪声
  x_0(原图) -> x_1 -> x_2 -> ... -> x_T(纯噪声)
  每步加少量噪声，T通常为1000
  公式：$q(x_t | x_{t-1}) = N(x_t; \sqrt{1-\beta_t} x_{t-1}, \beta_t I)$
  
反向过程（去噪）：训练一个UNet学习逐步去噪
  x_T(噪声) -> x_{T-1} -> ... -> x_0(生成图)
  每步预测并去除一点噪声
  目标：学习预测噪声 $\epsilon_\theta(x_t, t)$ 或直接预测 $x_0$。

**Latent Diffusion创新：**
问题：在像素空间(512x512x3)做diffusion计算量太大
解决：先用VAE将图片压缩到潜在空间(64x64x4)，在潜在空间做diffusion
  计算量减少：$512\times512\times3 / 64\times64\times4 \approx 48$倍
  **VAE细节**：使用KL正则化保持潜在空间接近标准正态分布，避免过度模糊。

**Stable Diffusion 组件：**
1. VAE Encoder/Decoder：图像 <-> 潜在空间
2. UNet：在潜在空间去噪（核心模型），包含Cross-Attention机制引入文本条件。
3. Text Encoder（CLIP）：将文本prompt编码为条件（77个Token x 768维）。
4. Scheduler：控制去噪步骤（DDIM/DPM++/Euler），决定每步如何使用预测的噪声更新 $x_t$。

**文本控制生成：**

```text
      +-----------------+
      |  Text Prompt    |
      +--------+--------+
               | (Cross-Attention Conditioning)
+--------------v-------------+    +----------------+
| Noisy Latent (z_t)         | -> | UNet (Predict |
| + Guided Noise (Optional) |    | Noise v_pred) |
+--------------+-------------+    +----------------+
               | (Predicted Noise)
+--------------v---------------------------+
| Scheduler (DDIM/Euler a=...)            |
| z_{t-1} = a * z_t - b * v_pred + noise |
+--------------+---------------------------+
               |
       (Repeat T steps)
               |
+--------------v--------+
| VAE Decoder (z_0)    |
+----------+-----------+
           |
           v
    Generated Image
```

**Classifier-Free Guidance (CFG) 原理：**
  联合训练有条件 $c$ (prompt) 和无条件 $\empty$ (空prompt) 模型。
  实际预测噪声 = $\epsilon_{uncond} + guidance\_scale \times (\epsilon_{cond} - \epsilon_{uncond})$
  作用：拉大遵循prompt与随机生成的差距。
  边界：CFG过高会导致图像过饱和、色彩失真或降低多样性。

**关键参数：**
- steps（步数）：20-50步。步数少速度快但可能构图崩坏，步数多细节好但收益递减。
- cfg_scale（引导强度）：7-12。越高越遵循prompt但可能降低艺术性。
- sampler（采样器）：DPM++ 2M Karras（速度质量平衡好），Euler a（创意性强）。
- seed（随机种子）

### 实战案例
**VAE Decode导致的RGB偏色**：在工业图像生成场景中，发现生成的图片在暗部存在明显的绿噪点。**排查**：经分析发现是微调时VAE Decoder使用了fp16精度，导致累加误差溢出。**解决**：在推理阶段强制将VAE切换为fp32精度，噪声消失但显存占用增加约20%，最后采用了"混合精度"（Encoder fp16, Decoder fp32）平衡效果。

### 采样器对比
| 采样器 | 速度 | 画面质量 | 收敛步数 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **Euler a** | 快 | 一般（偏创意） | 少 (~20) | 二次创作、风格化头像 |
| **DPM++ 2M Karras** | 中 | 高（细节丰富） | 中 (~30) | 商业海报、写实摄影 |
| **DDIM** | 快 | 较高 | 灵活 | 视频生成（帧间一致性要求高） |

### 代码示例 (CFG 去噪步骤核心逻辑)
```python
import torch

# 1. UNet 预测噪声
# noise_cond: 基于Prompt的预测噪声
# noise_uncond: 基于空Prompt的预测噪声 (通常在batch中拼接计算)
noise_cond = unet(latent, timestep, encoder_hidden_states=text_emb)
noise_uncond = unet(latent, timestep, encoder_hidden_states=uncond_emb)

# 2. 应用 Classifier-Free Guidance
guidance_scale = 7.5
noise_pred = noise_uncond + guidance_scale * (noise_cond - noise_uncond)

# 3. Scheduler 更新潜在变量 (以DDIM为例)
# prev_sample = x_t - alpha * noise_pred
latents = scheduler.step(noise_pred, t, latents).prev_sample
```
