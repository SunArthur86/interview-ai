# 大模型训练中为什么大量使用合成数据?合成数据有什么风险

- **合成数据使用场景:**
1. **领域数据不足** - 医疗/法律/低资源语言
2. **SFT数据生成** - 用强模型生成指令-回复对
3. **偏好数据生成** - 自动生成排序对
4. **推理数据** - 自动生成CoT推理路径

- **主要方法:**
- Self-Instruct: 用强模型自动生成训练数据
- Evol-Instruct: 递增复杂度的指令生成

- **风险:**
1. **模型坍缩** - 用模型生成数据训练下一代模型,多样性递减
2. **偏见放大** - 生成数据的偏差被放大
3. **幻觉传播** - 模型幻觉产生的错误数据被训练

- **缓解措施:**
- 混合真实数据(至少30%)
- 质量过滤
- 多样性约束(embedding聚类去重)

- **边界情况补充:**
  - **长尾领域**: 对于极度缺乏数据的垂直领域（如某种稀有病的诊断），合成数据中若有事实性错误，模型会"死记硬背"这些错误，且比在通用领域更难修正。
  - **评估数据污染**: 使用合成数据训练时，严禁将合成数据混入测试集，否则会造成高分假象，需严格进行数据去重。
  - **格式一致性**: 合成数据（特别是JSON等结构化数据）若格式约束不严，会导致训练出的模型在输出格式上非常松散。

- **实战案例:**
某金融问答项目中，使用GPT-4生成10万条财报问答对进行微调，结果模型在回答长尾问题时频繁出现“编造财报数据”的幻觉。引入基于Self-CheckGPT的过滤机制后，幻觉率下降了40%。

- **代码示例 (Quality Filter):**
```python
# 使用Self-CheckGPT思想过滤合成数据中的幻觉样本
def filter_hallucination(samples, teacher_model, threshold=0.5):
    clean_data = []
    for item in samples:
        # 生成多个样本的一致性检查
        others = teacher_model.generate(item['prompt'], n=3)
        # 计算当前答案与其他生成答案的相似度(BERTScore)
        score = calculate_similarity(item['response'], others)
        if score > threshold:  # 一致性高，保留
            clean_data.append(item)
    return clean_data
```

2. 在多轮对话数据合成中，如何保证上下文的一致性？避免模型在第一轮回答了A，第三轮却回答B的矛盾情况？
3. 如何区分"良性合成数据"（如格式转换）和"恶性合成数据"（如虚构事实）？针对两者应采取不同的训练权重吗？

- **## 易错点:**
1. **误区：合成数据越多越好**。低质量合成数据是"毒药"，会通过Catastrophic Forgetting机制覆盖原始知识。
2. **误区：只过滤难度**。在合成数据中，相比过滤"太简单"的样本，过滤"错误但看起来自信"的样本更为重要且更困难。

- **## 常见考点:**

2. 什么是「模型自食其子」现象？
