# LLM 作为概率模型，如何保证安全性和合规性？

LLM 本质是预测下一个 Token 的概率模型，这导致它可能输出有害、偏见或违法内容。保证安全性和合规性需要从“内”和“外”两个层面入手。

1. **模型层对齐**：在预训练后通过 RLHF（基于人类反馈的强化学习）或 DPO（直接偏好优化），注入安全价值观，让模型在底层概率分布上降低生成有害内容的可能性。
2. **系统提示词**：在 Prompt 层面显式约束模型的行为，例如“不要回答涉及暴力的问题”，这是第一道防线。
3. **输入/输出层防御**：在模型前后部署独立的“护栏”模型或分类器。输入层检测 Prompt Injection 或恶意攻击；输出层过滤敏感词或违规内容。
4. **红队测试**：模拟攻击者对抗模型，挖掘漏洞并迭代修补。
5. **数据合规**：确保训练数据的版权和隐私合规（如 GDPR），从源头剔除侵权数据。

综合策略通常组合使用，因为没有任何单一方法能实现 100% 的安全。
