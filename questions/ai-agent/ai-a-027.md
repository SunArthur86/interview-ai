# CrewAI的核心概念是什么?和AutoGen有什么设计理念上的区别

- **CrewAI核心概念:**

1. **Crew** - 一组Agent的集合(团队), 定义了整体协作框架
2. **Agent** - 有角色、目标、背景故事和工具的成员
3. **Task** - 具体任务,有描述、期望输出、指派的Agent
4. **Process** - 任务执行模式,控制流转逻辑

```text
       CrewAI Sequential Process 流程

       [Start] 
          |
          v
+-------------------+
|  Task 1           |
|  Agent: Researcher|
|  Output: Raw Data |
+---------+---------+
          | (Context Auto-passed)
          v
+-------------------+
|  Task 2           |
|  Agent: Writer     |
|  Output: Article   |
+---------+---------+
          |
          v
       [End Result]
```

- **典型代码:**
```python
researcher = Agent(
role="研究员",
goal="收集最新AI论文",
backstory="你是资深数据分析师...", # 增强人设
 tools=[search_tool]
)
writer = Agent(
role="技术作者",
goal="写一篇高质量文章",
tools=[write_tool]
)

crew = Crew(
agents=[researcher, writer],
tasks=[research_task, write_task],
process=Process.sequential # 明确的顺序流
)
result = crew.kickoff()
```

- **实战案例**: 在构建自动化研报生成系统时，我们发现Task间的上下文自动传递有时会引入冗余噪音。解决方案是显式设置Task的`context`参数，只将Researcher输出的“关键结论列表”传给Writer，而非全部原始数据。

- **代码示例 (显式上下文传递)**:
```python
# 仅传递特定输出而非整个Task对象
task_summary = Task(
    description="总结关键数据",
    expected_output="JSON格式列表",
    agent=analyst
)

task_write = Task(
    description="撰写报告",
    # 强制只使用summary的输出，忽略其他上下文
    context=[task_summary], 
    agent=writer
)
```

- **CrewAI vs AutoGen:**

| | CrewAI | AutoGen |
|--|--------|---------|
| 设计理念 | **结构化流水线** | **自由对话** |
| 控制流 | 预定义任务序列 | GroupChat动态协商 |
| 任务定义 | 显式Task对象,包含上下文依赖 | 隐式包含在对话中 |
| 可预测性 | **高** | 中 |
| 灵活性 | 中 | **高** |
| 适合 | 明确分工的生产流(SOP) | 探索性/创意任务 |

- **选择建议:**
- 任务步骤明确,类似流水线作业 → CrewAI
- 需要Agent自由协商、辩论 → AutoGen
- 需要复杂状态管理和条件分支 → LangGraph

## 常见考点
1. **Context传递**：CrewAI中Task A的输出如何自动作为Task B的输入？如何手动指定？
2. **异步执行**：CrewAI如何处理并行的Tasks？（Process.hierarchical模式下的执行逻辑）
3. **工具共享**：不同Agent如何共享工具，以及如何避免工具调用冲突？
