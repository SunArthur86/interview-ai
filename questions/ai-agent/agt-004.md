# Agent 的工作流程

### Agent 的工作流程

可理解为：**听懂 → 拆活 → 动手 → 对账 → 交卷**。

#### 详细步骤
1. **输入处理**
   - **意图识别**：分类任务类型（问答、编写代码、操作工具）。
   - **指代消解**：解析“它”、“那个”等代词所指代的上下文实体。
   - **安全过滤**：拦截 Prompt 注入或恶意指令。
   - **记忆加载**：从向量库或键值对数据库中加载相关的历史对话、用户偏好或文档碎片。

2. **任务分解**
   - 生成子任务列表（线性）或决策树（并行/条件分支）。
   - **依赖分析**：识别任务间的依赖关系（如任务 B 需任务 A 的输出）。
   - 复杂任务可配合 **Human-in-the-loop** 确认里程碑。

3. **工具选择与调用**
   - 由 LLM 或专门的**路由模型**根据工具描述选择工具。
   - **参数生成**：将自然语言转化为工具所需的 JSON/XML 参数。
   - **执行器**负责鉴权（OAuth/API Key）、限流与结果规范化。

4. **结果整合**
   - 合并多源信息，解决冲突（例如：依据时间戳，优先使用更新的数据）。
   - 将工具返回的机器数据转换为人类可读的自然语言。

5. **输出生成**
   - 面向用户的自然语言答案。
   - **引用与轨迹**：标注信息来源，输出操作链路（便于审计和调试）。

#### 常见问题：如何避免 Agent 迷失？
- **明确停止条件**：定义最大步数、Token 消耗上限或特定结束符（如 `<FINISH>`）。
- **维护任务清单**：实时更新子任务状态，记录已完成和待办项。
- **结构化输出**：强制每步输出 JSON 格式，便于程序化判断下一步。
- **强制验证**：关键步骤设置单元测试式检查或调用轻量级 LLM 进行二次审核。

#### ## 常见考点
1. **记忆机制**：如何区分短期记忆（当前上下文）与长期记忆（向量数据库）的读写策略？
2. **错误恢复**：当工具调用失败（如 API 500 错误）时，Agent 的重试与退避策略是什么？
3. **并发控制**：如果任务分解后包含无依赖的子任务，如何实现并行执行以提高效率？
