# 在LLM应用架构中，Map-Reduce 模式是如何处理长文档摘要任务的？它有什么潜在问题？

Map-Reduce 模式通过切分与汇总两阶段处理超长文本：Map阶段将长文档分块并各自生成摘要，Reduce阶段将各分块摘要拼接后再次汇总。其优势在于突破单次推理Token限制，但潜在问题是信息丢失（Map阶段过滤细节）和传递偏差（递归累积误差）。优化方案包括重叠切分保持上下文、Prompt侧重关键信息提取、结构化摘要降低遗漏风险。
