# 在LLM应用架构中，Map-Reduce 模式是如何处理长文档摘要任务的？它有什么潜在问题？

Map-Reduce 模式是一种处理超长文本（超出模型上下文窗口）的常见架构设计。在长文档摘要任务中，该模式分为两个阶段：1. Map 阶段：将长文档切分成多个较小的块，分别输入 LLM 生成每个块的摘要；2. Reduce 阶段：将所有 Map 阶段生成的摘要拼接起来，再次输入 LLM 进行汇总，生成最终的总体摘要。这种模式的优势在于突破了单次推理的 Token 限制，实现了对无限长度文档的处理。然而，其潜在问题主要在于信息丢失和传递偏差。Map 阶段的初步摘要可能会过滤掉某些细节，导致这些信息无法传递到最终阶段；随着文档长度的增加，Reduce 阶段的输入可能依然很长，甚至超出上下文限制，导致需要递归调用 Map-Reduce，这会累积误差，使得最终摘要可能遗漏全局视角或关键的低频信息。
