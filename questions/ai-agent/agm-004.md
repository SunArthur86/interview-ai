# 如何做 Token 预算分配才不容易翻车

Token 预算分配是 Prompt Engineering 和系统稳定性设计的关键。若分配不当，会导致指令被截断、工具无法识别或模型输出中断。合理的分配顺序应遵循：**指令与安全 > 功能定义 > 核心状态 > 近期上下文 > 输出预留**。

**建议优先级与分配策略**：
1.  **System 指令**：最高优先级。定义角色、安全边界、核心规则。若被截断，Agent 将“人格分裂”或产生幻觉。
2.  **安全/策略层**：过滤敏感词或硬性拒绝规则。
3.  **工具定义**：若 Agent 需要调用 API，Tools 的 Schema（名称、参数描述）必须完整包含，否则无法生成正确的函数调用。
4.  **高优先级记忆**：用户偏好、任务状态、活跃的子目标。这是当前推理的“上下文”基础。
5.  **近期对话**：通过滑动窗口保留最近的几轮对话，维持连贯性。
6.  **预留输出空间 (10%~20%)**：防止模型在生成回答或长 JSON 时因 Token 耗尽突然截断。

**优化技巧**：
- **长内容压缩**：工具返回的长文本、文档内容不要全文塞入，应提取摘要、引用 ID，或存入外部存储。
- **动态预算**：根据模型上下文窗口上限动态调整 Window 大小。

### 实战案例
某次生产环境故障，是因为随着对话进行，工具调用参数 JSON 被 Token 限制截断，导致 API 报错。我们引入了“软切断”机制：在构造 Prompt 时预先计算 Tokens，如果检测到剩余空间不足 500 Tokens，直接拒绝接受新的历史输入，强制清理旧对话，确保每次请求的输出空间绝对安全。

### 核心对比表格 (预算分配策略)

| 优先级 | 模块 | 建议占比 | 截断后果 | 处理策略 |
| :--- | :--- | :--- | :--- | :--- |
| 1 (最高) | System Prompt | ~10% | Agent 行为不可控 | **禁止截断**，硬性保留 |
| 2 | Tools / Few-Shot | ~20-30% | 无法调用工具/任务失败 | **禁止截断**，完整保留 |
| 3 | 核心状态 | ~10-20% | 逻辑混乱，遗忘目标 | **禁止截断**，通过 KV 传递 |
| 4 | 近期对话 | ~20-30% | 上下文流畅度下降 | 可截断 (滑动窗口/摘要) |
| 5 (最低) | 检索文档 | 动态 | 知识缺失 | 可截断 (Top-K 或缩短) |
| 6 (预留) | Model Output | 10-20% | 输出中断，JSON 解析失败 | **预留空间**，绝不使用 |

### 代码示例 (Token 预算动态截断)
```python
import tiktoken

def dynamic_truncate(messages, max_tokens, reserve_for_output=512):
    encoder = tiktoken.encoding_for_model("gpt-4")
    current_tokens = sum(len(encoder.encode(str(m))) for m in messages)
    
    # 如果超限，优先移除最早的对话（非 System 消息）
    while current_tokens > (max_tokens - reserve_for_output) and len(messages) > 1:
        # 假设 messages[0] 是 System，不删；从 index 1 开始删旧的
        removed = messages.pop(1) 
        current_tokens -= len(encoder.encode(str(removed)))
        
    return messages
```

## 常见考点
1.  **如果 Token 不够用了，应该先扔掉什么？**：优先丢弃“近期对话”中的最早几轮；其次压缩检索回来的长文本描述；绝对不能截断 System Prompt 和 Tool Definitions。
2.  **如何应对模型输出被截断？**：监测 `finish_reason` 是否为 `length`，如果是，需设计机制将未完成的输出重新作为输入补全，或增加预留预算。

## 记忆要点

- 分配优先级：System/Safety > Tools > 核心状态 > 近期对话 > 预留输出(10-20%)。
- System和Tools绝不可截断，否则Agent人格分裂或无法调用工具；优先丢弃最早对话。
- 优化：长内容压缩摘要，动态调整Window大小，预留空间防输出截断。
- 实战：引入软切断机制，剩余空间不足500时强制清理旧对话，确保输出空间安全。
