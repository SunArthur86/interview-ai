# 记忆系统概述

### 记忆系统概述

**定义**：Agent 为实现多轮对话、长期任务与个性化，对信息进行存储、组织、检索与更新的机制。记忆是 Agent 拥有“人格”和“历史经验”的基础。

**必要性**：
1.  **多轮一致性**：避免前后矛盾（如上文说“我住北京”，下文问天气不应查上海）。
2.  **长任务**：延续工具调用链和子目标状态。
3.  **个性化**：记住用户偏好与术语。
4.  **成本控制**：不必每次全量输入历史，按需加载，利用 RAG 压缩上下文。

**分类体系**：
1.  **短期记忆**：对话上下文，易失但访问快。通常由 LLM 的 Context Window 承载（Sliding Window 策略）。
2.  **长期记忆**：向量库、数据库、知识图谱，持久化存储。通过 RAG 检索。
3.  **情景记忆**：“发生了什么”（特定事件，如用户上周投诉过什么）。
4.  **语义记忆**：“知道什么”（通用知识，如公司政策）。

**人类记忆类比**：
*   **感觉记忆**：流式输入缓冲。
*   **工作记忆**：当前的 Scratchpad（思维链、中间变量）。
*   **长期记忆**：外挂的知识库。

**面试金句**：
*   **Q：Agent 为什么需要记忆？**
*   **A**：没有记忆就是无状态函数，无法处理多轮任务、保持一致性或提供个性化服务。记忆是 Agent 从“聊天机器人”进化为“助理”的关键。

## 常见考点
1.  **记忆存储策略**：什么时候存？（关键信息提取后存）；存什么？（而非全量存储原始对话，需 Summarize 或提取 Facts）。
2.  **记忆检索策略**：什么时候检索？（每次对话开始前）；检索多少？（根据 Token 预算动态调整 Top-K）。
3.  **窗口管理**：当对话历史超过上下文窗口时，如何处理？（滑动窗口、摘要压缩、选择性遗忘）。
