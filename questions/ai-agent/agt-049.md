# 主流多 Agent 框架

多 Agent 框架提供 Agent 抽象、消息路由、工具封装、记忆钩子、人机协作与（部分）可视化功能。版本迭代快，面试重在**设计思想**而非死记 API。

### 1. 主流框架对比

| 框架 | 背景/特点 | 典型适用 |
| :--- | :--- | :--- |
| **AutoGen** (微软) | 对话型多 Agent，可与人协作；强调可定制 Agent 与 聊天模式 | 研究原型、对话式工作流 |
| **CrewAI** | 角色 + 任务 + 团队 抽象，叙事贴近「项目组」 | 快速搭建「岗位分工」类 Demo |
| **MetaGPT** | SOP/公司角色隐喻强（PM/架构/工程师），偏软件工程过程仿真 | 代码生成流水线、教学 |
| **ChatDev** | 虚拟软件公司，多阶段聊天驱动开发 | 学术/实验、流程可视化 |
| **LangGraph** | 基于图与状态的编排，结合 LangChain 生态；强调可控循环与检查点 | 生产级需要可恢复、可调试的流程 |

**对比维度**：编排模型（图/对话/层级）、状态与持久化、人机协作、生态与供应商锁定、可观测性。

### 2. 面试问答

**Q：AutoGen 和 LangGraph 多 Agent 有什么气质差异？**

**A：** AutoGen 偏向**对话与多角色交互**的快速组合；LangGraph 偏向**显式图状态机**与检查点/分支。若强调生产可恢复与审计，LangGraph 往往更易形式化；若强调探索式对话与人机混合，AutoGen 叙事更自然。

**Q：CrewAI 的「Crew」抽象解决什么问题？**

**A：** 把角色分工 + 任务依赖 + 执行顺序从 Prompt 工程里抽成一等公民，降低心智负担，让协作结构可见、可复用。

**Q：MetaGPT 适合直接上生产吗？**

**A：** 视场景而定。它擅长结构化软件过程的演示与研究；生产需补强测试、权限、监控与成本控制，框架本身不替你完成这些。

### 3. 实战案例

某团队在做 **智能客服助手** 时，起初用 AutoGen，发现当对话轮次超过 20 轮时，Token 消耗爆炸且难以断点续接。后来迁移至 **LangGraph**，利用其 Checkpoint 功能：将用户会话状态持久化存入 Redis，每次调用只加载最近几轮对话 + 图节点状态。不仅支持随时挂起（转人工）和恢复，还将长对话成本降低了 60%。

### 4. 深度对比

| 特性 | AutoGen | LangGraph | CrewAI |
| :--- | :--- | :--- | :--- |
| **核心抽象** | ConversableAgent (对话流) | StateGraph (状态图) | Crew, Task, Agent (进程流) |
| **状态持久化** | 较弱（主要靠 History） | 强（原生 Checkpoint 支持） | 中 (依赖 LangChain 后端) |
| **控制力** | 模糊（自然语言驱动） | 精确（代码逻辑分支） | 中等（顺序/层级控制） |
| **人机协作** | 极佳（原生支持 Human 模式） | 良好 (通过 `interrupt` 机制) | 一般 |
| **生产成熟度** | 中 (适合逻辑多变的 Agent) | 高 (适合工作流引擎) | 低/中 (适合快速验证) |

### 5. 代码示例

以下对比 **AutoGen (对话式)** 与 **LangGraph (图式)** 的核心代码风格：

```python
# --- AutoGen 风格：通过对话驱动 ---
assistant = ConversableAgent("assistant", llm_config={"model": "gpt-4"})
user_proxy = ConversableAgent("user", code_execution_config=False)

# 启动对话，Agent 自动协商
result = user_proxy.initiate_chat(
    assistant, 
    message="请分析这段 Python 代码的复杂度"
)

# --- LangGraph 风格：通过图节点与边驱动 ---
from langgraph.graph import StateGraph, END

def node_analyze(state): return {"messages": ["分析结果: O(n)"]}
def node_review(state): return {"messages": ["审核通过"]}

workflow = StateGraph(AgentState)
workflow.add_node("analyze", node_analyze)
workflow.add_node("review", node_review)
workflow.add_edge("analyze", "review")
workflow.add_edge("review", END)
workflow.set_entry_point("analyze")

app = workflow.compile()
# 显式调用图执行，支持 Stream/Interrupt
app.invoke({"messages": ["请分析代码"]})
```
