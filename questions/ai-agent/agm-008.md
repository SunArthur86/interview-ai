# 只用向量检索、不做摘要可以吗

可以，但你会失去低成本的全局叙事（例如任务阶段、未决事项）。

**本质区别**：
- **摘要**：擅长提供压缩后的“主线”和“上下文预览”，Token 效率高，适合构建全局视角。
- **向量检索**：擅长提供精准的“细节证据”和“语义匹配”，适合回溯具体事实。

**最佳实践**：采用 **摘要 + 检索并存** 的分层架构。先检索摘要快速定位上下文区域，再从该区域下沉检索原始向量块，既保证效率又避免幻觉。

**## 常见考点**
- 如果纯向量检索效果已经很好，引入摘要的 ROI（投入产出比）在哪里？
- 摘要的粒度如何选取（按天、按话题、按会话）？
- 如何处理摘要与原始细节不一致的情况？
