# MCTS(蒙特卡洛树搜索)如何与LLM结合用于Agent推理

- **MCTS + LLM = 有方向的试错推理**

- **MCTS四步骤:**
1. **选择** - 用UCB公式选择最有前景的节点，平衡探索与利用
2. **扩展** - LLM根据当前上下文生成候选动作（思考/代码），增加子节点
3. **模拟** - LLM快速模拟后续推理路径或通过解释器执行代码获得反馈
4. **回传** - 根据模拟结果（如正确性、奖励信号）反向更新路径上的价值

```text
               MCTS 树搜索结构示意图

                   [Root: 初始问题]
                  /       |       \
             (选择)   (选择)     (选择)
              /          |          \
       Node A(3/5)   Node B(1/5)  Node C(5/5) <-- 高胜率，优先扩展
          |
      [扩展]  <-- LLM生成多个候选思路
       /     \
  Node D   Node E
    |        |
 [模拟]    [模拟] <-- LLM推演或代码执行
    |        |
  Reward   Reward
    |        |
 [回传]----+----> 更新父节点价值
```

- **LLM在MCTS中的角色:**

| 角色 | 作用 | 细节说明 |
|------|------|----------|
| 策略网络 | 生成候选动作(扩展) | 引导搜索向高概率区域，剪枝无效分支 |
| 价值网络 | 评估节点前景(模拟) | 预测当前状态/思路最终解决问题的概率 |
| 快速模拟 | Rollout到终局 | 通过Chain-of-Thought快速推演或解释器执行 |

- **优势:**
- 结合探索（尝试新路径）和利用（深耕高胜率路径）
- 可以发现LLM直接推理找不到的非直觉路径，纠正幻觉
- 适合数学竞赛、程序合成等需要搜索的任务

- **代表工作:** rStar-Math、AlphaCodium、ToT (Tree of Thoughts)

- **代价:** 计算量大(需要大量LLM调用),适合离线推理。需权衡搜索宽度、深度与计算成本。

- **实战案例**: 在Codeforces竞技编程中，LLM直接生成的代码在复杂边界条件上往往出错。采用MCTS后，LLM不仅生成代码，还在模拟阶段构建随机测试用例进行自测。一旦某路径发现Bug（Reward为0），搜索立即回溯至上一节点尝试新的逻辑分支，将最终Pass率从30%提升至60%。

- **代码示例 (Python - UCB选择逻辑)**:
```python
import math

def select_child(node):
    # UCB1 公式: exploitation + exploration
    best_score = -float('inf')
    best_child = None
    
    for child in node.children:
        if child.visits == 0:
            return child # 优先访问未探索节点
        
        # C值控制探索程度 (通常取 sqrt(2))
            + 1.414 * math.sqrt(math.log(node.visits) / child.visits)
        
        if ucb_score > best_score:
            best_score = ucb_score
            best_child = child
            
    return best_child
```

## 常见考点
1. **UCB公式的作用**：如何平衡Exploration（探索未访问节点）和Exploitation（利用高胜率节点）？
2. **剪枝策略**：在LLM推理成本高的情况下，如何根据阈值剪枝以降低Token消耗？
3. **奖励函数设计**：在数学题（确定答案）和创意写作（开放答案）中，如何设计模拟阶段的Reward？
