# Plan-and-Execute 与 ReAct 两种 Agent 模式有什么核心区别？在什么场景下应该选择 Plan-and-Execute？

ReAct 和 Plan-and-Execute 是两种主要的 Agent 任务规划范式。

**ReAct** 是“边想边做”， interleaving 地进行推理和行动。每一步都基于上一步的观测结果决定下一步行动。这种方式灵活、反馈快，适合路径探索或未知环境，但容易陷入局部循环或因早期错误导致后续动作全部失败。

**Plan-and-Execute** 是“先想后做”，分为两个阶段：
1. **Plan**：模型首先一次性规划出完成任务的全局步骤列表（如：1. 搜索；2. 读取；3. 总结）。
2. **Execute**：模型按顺序执行步骤，通常不再修改计划。

**选择场景**：Plan-and-Execute 适合步骤明确、逻辑链条较长的复杂任务（如编写长篇代码或旅游规划），因为它能减少执行过程中的幻觉和无效探索；而 ReAct 更适合需要动态调整策略、环境高度不确定的探索性任务。
