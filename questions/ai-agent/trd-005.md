# Function Calling 的并行调用是如何实现的？它在工程实现上有哪些挑战？

Function Calling 并行调用允许模型在一次交互中生成多个独立的工具调用请求，而不是串行等待一个工具返回后再调用下一个。

**实现原理**：模型在输出时生成结构化的 JSON 数组，包含多个 tool_calls 对象。服务端解析该数组后，并发（如使用 Python 的 asyncio 或线程池）执行所有工具函数。

**工程挑战**：
1. **参数歧义**：模型可能在一个 Call 中引用前一个 Call 的结果，但如果是并行调用，模型无法获取中间结果。需要确保模型生成的参数是独立的。
2. **异常处理**：并行调用中如果某一个工具报错，是立即终止所有调用，还是忽略错误继续执行？需要设计明确的容错策略。
3. **结果聚合与重排序**：由于网络延迟不同，工具结果返回的顺序是不确定的。Agent 必须能够根据 ID 将结果正确映射回原始的参数，并按模型的逻辑顺序重新拼装上下文，否则会导致后续推理混乱。
4. **超时控制**：必须设置合理的超时时间，防止某一个慢速工具阻塞整个响应流程。
