# MCP(Model Context Protocol)的完整架构是什么?它如何解决 N×M 工具集成问题

- **MCP (Model Context Protocol)** 是 Anthropic 提出的开放协议,标准化 LLM 与外部工具/数据的连接方式.

- **N×M 问题**
- N 个 LLM 应用 × M 个工具 = N×M 个集成点
- 每个 LLM 框架都有自己的工具定义格式
- 工具开发者要为每个框架写适配

- **MCP 架构全貌**
```text
┌───────────────────────────────────────────────────────────────────────┐
│                           LLM Application (Host)                       │
│  ┌─────────────────────────────────────────────────────────────────┐  │
│  │                     MCP Client (e.g. Claude Desktop)             │  │
│  │  ┌──────────────┐  ┌──────────────┐  ┌──────────────────────┐   │  │
│  │  │   Local Tool │  │   Resource  │  │      Prompt         │   │  │
│  │  │   Executor   │  │    Reader   │  │    Template         │   │  │
│  │  └──────┬───────┘  └──────┬───────┘  └──────────┬───────────┘   │  │
│  └─────────┼────────────────┼─────────────────────┼───────────────┘  │
└────────────┼────────────────┼─────────────────────┼──────────────────┘
             │                │                     │
             │      JSON-RPC 2.0 (Standardized Protocol)               │
             ▼                ▼                     ▼
      ┌────────────┐   ┌─────────────┐       ┌──────────────┐
      │  stdio     │   │    SSE      │       │   (Other)    │  <-- Transport Layer
      │  (Local)   │   │  (Remote)   │       │              │
      └─────┬──────┘   └──────┬──────┘       └──────────────┘
            │                 │
            ▼                 ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                        MCP Server Ecosystem                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────────────────────┐  │
│  │  File System │  │   Database   │  │   Git / GitHub Integration   │  │
│  │   Server     │  │   Server     │  │         Server               │  │
│  └──────────────┘  └──────────────┘  └──────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────────┘
```

- **核心组件**
1. **MCP Client**: 集成在宿主应用中，负责初始化连接、负载均衡及生命周期管理。
2. **MCP Host**: LLM 应用本体，决定何时调用工具。
3. **MCP Server**: 实现特定功能（如数据库查询），通过 Transport 暴露能力。
4. **Transport Layer**: 通信载体，支持 stdio（本地进程）或 SSE（远程服务）。

- **实战案例**
- **场景**：企业内部有 PostgreSQL 和 GitLab 两个数据源。以前需要分别为 LangChain 和 AutoGPT 开发两套适配器；使用 MCP 后，只需开发一个 GitLab Server 和一个 PG Server，所有支持 MCP 的客户端（Claude Desktop, Cline 等）即可直接调用。
- **踩坑**：使用 stdio 传输时，如果 Server 进程未正确处理 `SIGTERM`，会导致宿主应用关闭时出现僵尸进程，长期占用内存。

- **代码示例 (Python Server 实现)**
```python
# mcp_server_example.py
from mcp.server.fastmcp import FastMCP

# 实例化 MCP Server
mcp = FastMCP("MyTools")

# 定义工具：通过装饰器自动生成 JSON Schema
@mcp.tool()
def calculate_bmi(weight_kg: float, height_m: float) -> float:
    """计算 BMI 指数"""
    return weight_kg / (height_m ** 2)

# 定义资源：暴露给 LLM 的静态或动态数据
@mcp.resource("uri://config/app")
def get_config() -> str:
    return "{\"env\": \"production\", \"version\": \"1.0.0\"}"

if __name__ == "__main__":
    mcp.run()  # 启动 stdio 通信服务
```
