# 什么是 Agent 的上下文压缩？有哪些触发阈值策略？

Agent 的上下文压缩是指随着对话历史和工具调用记录的增长，为了控制上下文窗口大小和推理成本，对记忆数据进行有损或无损的压缩处理。

**压缩方法**：
1. **摘要**：使用 LLM 将过去的一系列交互和工具执行结果总结为一段简短的摘要，替代原始详细日志。
2. **提取关键实体**：保留对后续任务最重要的参数、文件名或数据，丢弃过程性描述。
3. **分层记忆**：将短期记忆（详细）转换为长期记忆（向量库或摘要）。

**触发阈值策略**：
1. **长度阈值**：当当前 Token 数量接近模型限制（如 80%）时触发压缩。
2. **轮次阈值**：每进行 N 轮对话后，对历史记录进行一次归档和压缩。
3. **事件触发**：当某个重要阶段完成（如“订单已创建”）时，将该阶段的所有交互压缩为一条状态记录。合理的阈值设计需要在信息保留率和推理成本之间寻找平衡点。
