# Plan-and-Execute Agent和ReAct有什么区别？

ReAct：边想边做，每步 Thought→Action→Observation 循环。适合简单、动态的任务。

**Plan-and-Execute**：
1. **Planner**：LLM 先基于目标制定完整的步骤列表（多步骤规划）
2. **Executor**：逐步执行每个子任务，通常不进行深度推理，只负责调用
3. **Re-Plan**（可选）：根据执行结果和中间反馈调整后续计划

**架构对比图**：
```text
ReAct 模式 (串行交互):
User -> LLM -> Tool -> LLM -> Tool -> LLM -> Answer
(决策紧耦合，每一步都消耗大模型推理成本)

Plan-and-Execute 模式 (规划与执行分离):
     ┌─────────┐
     │ Planner │ (一次性生成计划: 1->2->3)
     └────┬────┘
          ▼
   ┌──────────────┐
   │ Step List:   │
   │ 1. Search...  │
   │ 2. Calc...   │
   │ 3. Summarize │
   └──────┬───────┘
          ▼
     ┌─────────┐
     │Executor │ ──▶ Tool 1
     │(轻量LLM)│ ──▶ Tool 2
     │ or Rule │ ──▶ Tool 3
     └────┬────┘
          ▼
       Answer
```

**优势**：
- **推理成本优化**：Planner 只需调用一次（通常用强模型），Executor 步骤可用更小、更快的模型甚至规则引擎
- **全局视野**：在执行前就有完整路径，减少陷入局部死胡同的概率
- **并行化**：若计划中的步骤无依赖关系，Executor 可并行执行

**劣势**：
- **脆弱性**：如果初始计划有误，执行过程很难补救（除非引入 Re-Plan 机制）
- **延迟问题**：必须等所有计划生成后才能开始第一步执行
- **缺乏反馈**：Executor 通常不修正计划，遇到错误容易直接失败

**对比表格**：

| 维度 | ReAct | Plan-and-Execute |
| :--- | :--- | :--- |
| **执行方式** | 串行推理 + 执行 | 先全局规划，后执行 |
| **模型调用** | 高频调用强模型 | Planner(强) + Executor(弱/规则) |
| **纠错能力** | 强（每步可调整） | 弱（依赖 Re-Plan 机制） |
| **并行能力** | 差（天然串行） | 好（执行阶段可并行） |
| **适用场景** | 探索性任务、复杂逻辑 | 流程化任务、成本敏感场景 |

- **实战案例**
- **场景**：数据分析任务“分析2023年销售数据并生成图表”。使用 Plan-and-Execute，Planner 先拆解为“读取数据→计算增长率→生成图表”三步，Executor 直接调用 Python 脚本执行，耗时仅为 ReAct 的 1/3，且避免了 ReAct 可能出现的“读取数据后忘记计算直接尝试画图”的幻觉。
- **踩坑**：Planner 生成的计划如果包含模糊指令（如“优化数据”），Executor 若只是轻量模型，极易直接报错或执行无意义操作，需在 Planner Prompt 中明确要求输出具体函数名或参数。

- **代码示例 (LangChain Plan-and-Execute)**
```python
from langchain.chains import PlanAndExecute
from langchain_openai import OpenAI
from langchain_experimental.plan_and_execute import PLANNER_PROMPT

# 模型分离：强模型规划，弱模型执行
model_planner = OpenAI(temperature=0, model="gpt-4")
model_executor = OpenAI(temperature=0, model="gpt-3.5-turbo-instruct")

# 实例化 Agent
agent = PlanAndExecute(
    planner=model_planner,
    executor=model_executor,
    verbose=True
)

# 执行任务
agent.run("谁是美国2020年总统选举的获胜者？他的年龄多大了？")
```
