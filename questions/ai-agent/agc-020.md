# 为什么「让模型自己选工具」可能不如「路由器 + 规则」

## 答案
**对比分析**：
*   **路由器 + 规则**：适合域窄、路径稳定的场景（如特定 FAQ 分类）。优势是成本极低（甚至无需 LLM）、速度快、行为确定、易于单元测试；劣势是规则维护成本高，无法处理泛化意图。
*   **模型自选**：适合开放域、复杂意图。优势是灵活、泛化能力强；劣势是成本高、存在不确定性（可能选错）、延迟增加。
*   **混合策略**（最佳实践）：简单意图（如“查天气”、“查余额”）走关键词或意图识别规则路由；复杂边缘 Case 走模型判别。

**边界情况**：
*   **歧义意图处理**：当用户说“打开它”时，路由器若仅基于当前轮次判断会失败，需结合对话历史的多轮上下文进行路由，这对简单的规则路由是挑战。
*   **工具定义漂移**：当工具的描述更新不及时，模型路由器可能基于过时描述选错工具，而规则路由直接绑定函数签名，不受描述文本影响。
*   **高并发下的路由延迟**：如果模型路由器使用的是大模型（如 GPT-4），并发高峰期会成为系统瓶颈，需考虑模型路由器本身的扩缩容和排队机制。

**实战案例**：在一个支付接入 Agent 中，最初让模型自己选“支付宝”或“微信”工具，结果当用户说“用那个绿色的支付”时，模型因理解偏差选错工具。后来增加了一条规则路由：检测关键词“绿色/微信” -> 强制路由 WechatTool，准确率瞬间提升至 100% 且成本下降。

**补充细节**：
*   **模型路由的不稳定性**：模型可能因为 Prompt 微小波动或 Temperature 设置导致在工具 A 和工具 B 之间反复横跳。
*   **规则路由的参数匹配**：规则不仅能路由，还能在路由前预先提取参数，减少模型后续的解析负担。

**架构模式对比图**：

```text
模式 A: 纯模型路由 (灵活/慢/贵)
User ──► LLM (Function Calling) ──► 决策调用 Tool A/B/C

模式 B: 规则路由 (死板/快/省)
User ──► Regex/Classifier ──► 匹配规则 X ──► 调用 Tool A

模式 C: 混合路由 (推荐)
User ──► [1. 规则过滤器] ──(未命中)──► [2. 模型路由器] ──► Tool
            (命中则直接返回)          (处理复杂意图)
```

**代码示例（混合路由逻辑）：**
```python
# Python: 优先规则，兜底模型的路由器
def route_intent(user_query):
    # 1. 规则层：高频固定意图
    if "余额" in user_query:
        return "check_balance_tool"
    elif re.search(r"转.*账", user_query):
        return "transfer_tool"
    
    # 2. 模型层：复杂意图兜底
    response = llm_client.chat(
        model="gpt-4o-mini", # 使用小模型降本
        messages=[{"role": "user", "content": f"Classify intent: {user_query}"}]
    )
    return map_to_tool(response.content)
```

**对比表格：路由策略对比**

| 维度 | 规则路由 | 模型自选 (LLM Routing) | 混合路由 |
| :--- | :--- | :--- | :--- |
| **响应速度** | 毫秒级（正则匹配） | 500ms+（生成式推理） | 接近规则（命中率高时） |
| **Token 成本** | 0 | 较高（每次请求消耗） | 低（仅未命中时消耗） |
| **维护难度** | 高（需穷举意图） | 低（自然语言描述） | 中（兼顾两者） |
| **确定性** | 100%（输入同则输出同） | 波动（受 Temperature 影响） | 高（核心路径确定） |

## 易错点
1.  **路由层与业务逻辑强耦合**：在路由器中直接编写了过多的业务处理代码（如参数校验逻辑），导致路由器变得臃肿且难以复用。
2.  **忽视置信度阈值**：在使用模型路由时，没有设置置信度阈值，导致模型对模糊意图强行分类，调用错误的工具，此时应返回“澄清问题”而非强制路由。

## 面试追问
1.  当规则和模型判断冲突时，以谁为准？（答：通常优先规则兜底，或设计置信度阈值，低置信度时强制走人工或默认回复）。
2.  模型作为路由器时，Prompt 怎么设计能提高准确率？（答：提供每个工具的详细描述、输入输出示例、Few-Shot 引导）。
3.  路由器这一步本身带来的延迟如何优化？（答：使用小参数模型（如 3B/7B）专门做路由，或使用 BERT 等分类模型替代生成式 LLM）。
