# Few-shot 示例顺序会影响结果吗

会。这是 LLM 中的“最近偏差”或“位置偏差”表现。模型可能更关注开头或结尾的示例，或者受示例顺序隐含的某种逻辑关系影响。工程上不应依赖某种“神秘的最佳顺序”，而应通过明确的规则（如按随机顺序、按难度排序）和严格的格式约束来增强鲁棒性；建议通过 A/B 测试对比不同顺序（如正序、逆序、随机）的效果差异。

## 常见考点
1. **偏差类型**：除了位置偏差，还有哪些常见偏差？（Majority bias 多数标签偏差、Recency bias 近因偏差）。
2. **缓解措施**：如何消除顺序影响？（每次请求动态打乱示例顺序，或者使用集成投票机制）。
3. **格式干扰**：示例中的格式不一致会导致什么问题？（可能导致模型输出格式混乱，需严格保持示例模板一致性）。
