# LangGraph的核心概念是什么?它如何用图结构管理Agent状态

- **LangGraph:** 用有向图(StateGraph)编排Agent工作流.

- **核心概念:**

1. **State** - 全局状态对象(TypedDict/Pydantic)
2. **Node** - 图节点,每个节点是一个函数/Agent
3. **Edge** - 边,定义节点间转移
4. **Conditional Edge** - 根据状态动态选择下一节点

- **示例架构:**
```
START → research → draft → review
↑       ↓
revise ← critique
```

- **代码示例:**
```python
graph = StateGraph(AgentState)
graph.add_node("research", research_node)
graph.add_node("draft", draft_node)
graph.add_node("review", review_node)
graph.add_edge("research", "draft")
graph.add_conditional_edges("review", should_revise,
{True: "draft", False: END})
graph.add_edge(START, "research")
```

- **优势:**
1. **循环支持** - LangChain Chain不支持循环,LangGraph原生支持
2. **状态管理** - 全局状态自动传递
3. **可持久化** - Checkpoint支持中断/恢复
4. **人机协作** - Interrupt机制支持人工审批

- **边界情况：**
  - **状态并发冲突**：当同一Graph被多个并发请求触发时，如果State包含可变对象且处理不当，会导致状态污染。LangGraph通过Thread ID隔离，但自定义State时需注意不可变性。
  - **无限循环检测**：Conditional Edge 的逻辑缺陷可能导致图在两个节点间死循环。需在运行时配置最大步数或超时熔断。
  - **大序列化开销**：State中若包含大量数据（如Base64图片），每次Checkpoint持久化会产生显著的IO延迟和存储成本。

- **状态流转详细架构图:**
```
+-------------------------------------+
|            LangGraph Runtime        |
|                                     |
|  ┌─────────────────────────────┐   |
|  │    Checkpointer (Memory)     │   |
|  └──────────────┬──────────────┘   |
|                 │ saves/loads      |
|                 ↓                  |
|  ┌─────────────────────────────┐   |
|  │      State (TypedDict)       │◄───┼─── Global Shared State
|  │  { messages, next_action }  │   |
|  └──────────────┬──────────────┘   |
|                 │ passes           |
|      ┌──────┬───┴───┬──────┐      |
|      ▼      ▼       ▼      ▼      |
|   [Node A][Node B][Node C][End]   |
|      │      │       │             |
|      └──►(Edges)◄───┘             |
|            │                      |
|    Conditional Edge Function       |
|    (decides next node based       |
|      on current state)            |
+-------------------------------------+
```

- **实战案例：**
在开发金融研报生成器时，使用了LangGraph的`interrupt`功能。当Agent需要调用高权限的交易数据库时，触发暂停，等待人工输入OTP验证码后才继续执行，有效避免了自动化的安全风险。

- **对比表格：LangChain Chain vs LangGraph**
| 特性 | LangChain Chain (Legacy) | LangGraph |
| :--- | :--- | :--- |
| **拓扑结构** | 线性/简单分支 | 有向循环图 (Cyclic Graph) |
| **状态管理** | 隐式传递 | 显式 StateGraph (TypedDict) |
| **可控性** | 黑盒 | 白盒，每一步可观测 |
| **人机交互** | 困难 | 原生支持 Interrupt/Resume |

## 面试追问
1. LangGraph 的 Checkpointer 如果在数据库中存储了大量的历史状态，如何进行冷热数据分离或清理策略，以避免性能下降？
2. 当图的结构非常复杂（例如几十个节点）时，如何进行调试和可视化？有没有推荐的可观测性工具集成方案（如 LangSmith）？
3. 在分布式部署 LangGraph 应用时，如何保证 `interrupt` 后的 Resume 请求能够路由到正确的 Worker 进程上？（涉及到状态共享与亲和性）

## 易错点
1. **State 更新逻辑错误**：在 Node 函数中直接修改 State 字典而不是返回新的字典，导致某些版本或特定配置下状态更新不生效。推荐使用 `reducer` 模式或不可变更新。
2. **忽略了 `END` 节点**：在 Conditional Edge 中忘记处理所有可能的分支情况，导致某些状态流转没有对应的出口，程序卡死或报错。
