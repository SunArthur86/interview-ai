# 工具路由（Tool Routing）

### 工具路由

**背景**：当工具数量过多（几十至上百个）时，全量发给模型会导致 Token 耗费高、上下文窗口溢出、模型注意力分散导致误选率高（“迷失在中间”现象）。路由用于缩小候选集。

**策略**：
1.  **向量检索**：将工具描述 embedding，Query 检索 Top-K 工具。速度快，语义匹配好。
2.  **分类模型**：训练轻量分类器（BERT/小 LLM）直接预测工具类别。准确率高，但需训练数据。
3.  **层级目录**：先选组（如 `db_group`），再选具体工具。适合结构化明显的工具集。
4.  **语义路由**：基于规则或 LLM 判断意图类型，映射到不同的子 Agent 或工具集。

**路由流程图**：
```text
User Query
   │
   ▼
┌──────────────┐
│  Router Layer│ (Embedding or Classifier)
└──────┬───────┘
       │
       │ Select Top-K Candidates
       │ (e.g., Weather_Tool, News_Tool)
       ▼
┌──────────────┐
│ LLM + Tools  │ (Only K tools in context)
└──────────────┘
```

**面试金句**：
*   **Q：向量路由选不对怎么办？**
*   **A**：增大 Top-K、混合关键词检索（BM25）、加一层 LLM 二分类、保留通用搜索兜底（Re-Rank 机制）。

### 代码示例（向量路由）
```python
import numpy as np
from openai import OpenAI

def route_tools(query: str, tools: list, client: OpenAI, top_k: int = 3):
    # 1. 向量化工具描述
    texts = [f"{t.name}\n{t.description}" for t in tools]
    doc_emb = client.embeddings.create(input=texts, model="text-embedding-3-small")
    doc_vecs = [d.embedding for d in doc_emb.data]
    
    # 2. 向量化 Query
    q_emb = client.embeddings.create(input=[query], model="text-embedding-3-small")
    q_vec = q_emb.data[0].embedding
    
    # 3. 计算余弦相似度并返回 Top-K
    scores = np.dot(doc_vecs, q_vec)
    top_indices = np.argsort(-scores)[:top_k]
    return [tools[i] for i in top_indices]
```

## 常见考点
1.  **召回率与准确率权衡**：Top-K 设为多少合适？如何平衡“漏掉工具”和“给模型太多干扰”？（通常根据经验值，如 5-10 个，并监控未匹配率）。
2.  **工具描述优化**：为了提升向量检索效果，工具描述应该怎么写？（包含具体的使用场景示例、同义词，而非简单的功能定义）。
