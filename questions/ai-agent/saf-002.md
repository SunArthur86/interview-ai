# CrewAI框架是什么？它如何实现多Agent协作？

🎯 **本质**：CrewAI是一个以角色为核心的多Agent协作框架，每个Agent有明确的角色、目标、背景故事和工具集，通过任务分配实现协作。

📊 **CrewAI核心概念**：

1. **Agent（智能体）**
  每个Agent定义：
  - role：角色（如“研究员”、“写手”、“审稿人”）
  - goal：目标（如“收集最新AI论文”）
  - backstory：背景故事（影响LLM的行为方式）
  - tools：可用工具列表
  - llm：使用的语言模型

2. **Task（任务）**
  定义具体的工作项：
  - description：任务描述
  - expected_output：预期输出格式
  - agent：负责的Agent
  - context：依赖的其他任务输出

3. **Process（流程）**
  Sequential（顺序）：Agent按顺序依次执行任务
  Hierarchical（层级）：有管理Agent分配任务给子Agent

4. **Crew（团队）**
  将Agents和Tasks组合成一个团队
  指定执行流程

**边界情况**：
1. **死锁与依赖**：在Hierarchical模式下，如果Manager Agent分配的任务依赖链过长，或者子Agent任务失败未返回，如何进行全局超时控制。
2. **资源竞争**：多个Agent同时调用同一个独占性工具（如文件写入）时，CrewAI默认不处理锁机制，需要在外部工具层实现互斥。
3. **上下文传递限制**：Task之间的Context传递如果包含大量数据，可能会迅速消耗Token，需要设计中间产物的摘要或过滤机制。

**实战案例**：
在使用CrewAI自动化生成行业研报时，如果让Researcher和Writer同时写，Writer会编造数据。最佳实践是强制Sequential流程：Researcher检索 -> Data Analyst清洗数据 -> Manager审核 -> Writer写作，通过中间产物的交接来抑制幻觉。

**代码示例**：
```python
from crewai import Agent, Task, Crew, Process

researcher = Agent(
    role="资深研究员",
    goal="发现前沿技术趋势",
    backstory="你是一位在科技领域工作20年的专家...",
    tools=[search_tool, scrape_tool],
    verbose=True
)

task1 = Task(
    description="搜索2024年大模型的最新进展",
    expected_output="包含5个关键发现的总结报告",
    agent=researcher
)

crew = Crew(
    agents=[researcher, writer],
    tasks=[task1, task2],
    process=Process.sequential,
    verbose=True
)
crew.kickoff()
```

## 面试追问
1. CrewAI的Process主要支持顺序和层级，如果需要实现更复杂的网状协作（如Agent A完成后同时触发 B 和 C，等 B C 都完成后再触发 D），你会如何改造或限制？
2. 在Hierarchical模式下，Manager Agent本身也是LLM，如果它做出了错误的任务拆解，下游Agent能否感知并纠错？如何设计容错？
3. 如何监控CrewAI内部各个Agent的Token消耗？如果某个Agent陷入死循环或输出冗长，有Cost Control机制吗？

## 易错点
1. **过度依赖Auto模式**：认为设定好Role和Goal后Agent就能自动完美协作，实际上Task的Description（特别是Context的传递）对结果影响巨大。
2. **忽视Task的异步性**：在默认Sequential流程中，任务是串行的，不要误以为CrewAI会自动优化并行执行不相关的Task，必须手动配置或切换框架。
