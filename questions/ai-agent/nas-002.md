# RAGFlow、Dify、FastGPT 等开源 RAG 框架有什么区别?如何选型

- **主流开源 RAG 框架对比**

| 框架 | 定位 | 核心优势 | 适用场景 | 底层架构倾向 |
|------|------|----------|----------|--------------|
| **RAGFlow** | 深度文档理解 RAG | 基于 DeepDoc 的精准文档解析(表格/图片/布局) | 复杂文档(PDF/合同/论文) | 重解析，轻编排 |
| **Dify** | LLM 应用开发平台 | 可视化工作流 + Agent + RAG 全栈 | 快速构建 AI 应用 | 平台化，SaaS 化 |
| **FastGPT** | 知识库问答 | 工作流编排 + 函数调用 | 企业知识库客服 | 基于 Node.js/HTTP 服务 |
| **LangChain** | LLM 开发框架 | 最灵活的组件化设计 | 定制化 RAG 开发 | Python/JS SDK 库 |
| **LlamaIndex** | 数据框架 | 专注数据连接和索引 | 企业数据湖 RAG | 数据为中心的索引 |

- **技术架构深度对比**
```text
┌──────────────────────┐  ┌───────────────────────┐  ┌──────────────────────┐
│    LangChain (SDK)   │  │    Dify (Platform)    │  │   RAGFlow (Focus)    │
├──────────────────────┤  ├───────────────────────┤  ├──────────────────────┤
│ • Chains / Runnables │  │ • Visual Workflow     │  │ • DeepDoc (OCR/Layout)│
│ • Integrations (100+)│  │ • RAG Pipeline UI     │  │ • Hybrid Search       │
│ • Memory management  │  │ • Model/Key Mgmt      │  │ • Knowledge Graph     │
└──────────┬───────────┘  └───────────┬───────────┘  └──────────┬───────────┘
           │                          │                         │
           ▼                          ▼                         ▼
    High Code Complexity         Low Code / Ops           High Data Accuracy
```

- **选型决策**

1. **文档复杂度高**(PDF 表格、扫描件)→ **RAGFlow**
- DeepDoc 引擎支持布局识别、表格还原、OCR
- 切片质量显著高于朴素按字数切分

2. **快速构建产品**(工作流 + 多 Agent)→ **Dify**
- 拖拽式工作流设计
- 内置 prompt 模板、模型管理、监控

3. **需要深度定制**(自定义检索逻辑)→ **LangChain**
- 最灵活,但需要更多开发工作
- 适合有工程团队的企业

4. **企业知识库**(客服/FAQ)→ **FastGPT**
- 开箱即用的知识库管理
- 支持函数调用增强

- **实战案例**
- **场景**：某律所需要处理大量扫描版合同 PDF。使用 LangChain 的 naive splitter 导致切片错乱（表头和内容分离）；切换到 RAGFlow 后，DeepDoc 成功识别版面，将表格整体作为一个语义单元索引，检索准确率从 40% 提升至 85%。
- **踩坑**：Dify 在处理千万级向量数据时，其内置的 Weaviate/Elasticsearch 配置若未调优，会导致导入阶段内存溢出（OOM），建议生产环境使用外部向量库并自行管理连接池。

- **代码示例 (LangChain 自定义 RAG 管道)**
```python
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

# 自定义检索逻辑（例如：只检索最近24小时的数据）
def custom_retriever(query):
    # 假设 vectorstore 已配置 filter
    return vectorstore.similarity_search(
        query, 
        k=5, 
        filter={"timestamp": {"$gte": "2023-10-01"}}
    )

# 构建链路
rag_chain = (
    {"context": custom_retriever, "question": RunnablePassthrough()}
    | prompt | llm | StrOutputParser()
)

# 执行
print(rag_chain.invoke("最新的股价是多少？"))
```
