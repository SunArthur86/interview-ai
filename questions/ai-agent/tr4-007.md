# Semantic Kernel 的 Planner（规划器）是如何实现复杂任务分解的？它和 LangChain 的 Agent 有何不同？

Semantic Kernel (SK) 的 Planner 利用 LLM 来理解用户的意图，并将其分解为一系列可执行的函数调用链。SK 强调**原生代码集成**，Skills/Plugins 是通过 C# 或 Python 代码定义的强类型函数。Planner 会根据函数的语义描述和参数 schema 来构建一个执行计划。与 LangChain 的 Agent 相比，SK 的优势在于其**类型安全**和**企业级集成能力**：LangChain 更侧重于链式调用和 Prompt 模板的编排，灵活性高但调试复杂；SK 则更像是一个编排层，能够更自然地利用现有的业务逻辑代码库。此外，SK 的 Handlebars 模板允许更复杂的变量传递和计划结构化生成，适合需要精确控制执行流程的企业级应用。
