# Plan-and-Execute 框架

### 1. 概念解释
Plan-and-Execute 分为两个阶段：
1. **Planner（规划）**：先制定全局计划，进行任务分解。
2. **Executor（执行）**：按计划逐步执行，执行过程中根据反馈决定是否重规划。

### 2. 核心流程与架构

```text
┌────────────────────────────────────────────────────────────┐
│                        User Goal                           │
└────────────────────┬───────────────────────────────────────┘
                     ▼
        ┌──────────────────────────────┐
        │      Planner (LLM)           │
        │  1. Decompose Task           │
        │  2. Generate Steps (List)    │
        └──────────────┬───────────────┘
                       ▼
        ┌──────────────────────────────┐
        │    Plan: [Step1, Step2...]   │
        └──────────────┬───────────────┘
                       ▼
┌────────────────────────────────────────────────────────────┐
│                    Executor Loop                           │
│  ┌──────────────┐      ┌──────────────┐                    │
│  │   Step N     │ ───▶ │    Tool      │                    │
│  │ (From Plan)  │      │   Execution  │                    │
│  └──────┬───────┘      └──────┬───────┘                    │
│         │                    │                             │
│         ▼                    ▼                             │
│  ┌──────────────┐      ┌──────────────┐                    │
│  │   Update     │ ◀─── │    Result    │                    │
│  │   State      │      │              │                    │
│  └──────┬───────┘      └──────────────┘                    │
│         │                                                  │
│         ▼ (If Error/New Info)                              │
│  ┌──────────────┐      (Re-plan: Adjust/Modify)            │

```

### 3. 与 ReAct 的对比
| 维度 | ReAct | Plan-and-Execute |
| :--- | :--- | :--- |
| **规划方式** | 隐式、逐步贪心 | 显式、先全局后局部 |
| **灵活性** | 高，随时可改 | 中，依赖重规划机制 |
| **Token 消耗** | 较低（无独立 Plan 阶段） | 较高（需携带完整 Plan 上下文） |
| **适用场景** | 路径不确定、探索性强 | 任务结构清晰、步骤多、需全局统筹 |

### 4. 优缺点
- **优点**：减少短视行为，计划可审计，全局视野好，易于并行执行无依赖的步骤。
- **缺点**：初始计划错误可能影响全局，两阶段可能增加端到端延迟。

#### ## 常见考点
1. **重规划触发**：在什么情况下 Executor 应该中断并请求 Planner 重新生成计划？
2. **动态调整**：如果 Plan 中包含条件分支（If/Else），Executor 如何处理？
