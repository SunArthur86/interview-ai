# AutoGen / CrewAI 等多 Agent 框架

### 1. 概念解释
多 Agent 框架（如 AutoGen, CrewAI）将任务分配给多个不同“角色”的 Agent。它们通过对话或消息传递协作完成任务，模拟人类团队的协作方式。

### 2. 核心设计
- **角色定义**：每个 Agent 有特定的 System Prompt（设定背景和目标）、工具集和配置（如温度参数）。
- **协作模式**：
  - **对话驱动**：像群聊一样互相发送消息，基于内容决定谁下一个发言。
  - **流程驱动**：像流水线一样按顺序传递工作，一个 Agent 的输出是另一个的输入。

### 3. 多 Agent 交互架构 (ASCII 图示)

```text
         模式 A: 对话驱动          模式 B: 层级/流水线

   +--------+                      +--------+
   | User   |                      | User   |
   +---+----+                      +---+----+
       |                              |
       v                              v
   +-------+  Msg    +-------+    +-------+   Task   +-------+
   |Agent A+-------->+Agent B+    |Manager|--------->|Agent 1|
   +-------+         +-------+    +-------+          +-------+
       ^                 |            ^                 |
       |      Msg        |            |  Result         | Result
       +-----------------+            |                 v
                                     |             +-------+
                                     +-------------+Agent 2|
                                                   +-------+
```

### 4. 优势与挑战
- **优势**：
  - **分工明确**：如写代码 Agent vs 审查代码 Agent，利用角色互补提高质量。
  - **专家模型**：不同的 Agent 可以指定使用不同的 LLM（如一个用 GPT-4 规划，一个用 GPT-3.5 摘要），优化成本与性能。
  - **并行探索**：多个 Agent 可以同时尝试不同的解决路径。
- **挑战**：
  - **协调成本高**：Token 消耗随 Agent 数量指数级增长。
  - **社会惰性**：可能出现“互相附和”或推诿。
  - **死循环**：没有明确的终止条件时，对话可能无限进行。

### 5. 防止“互相附和”与死循环的策略
- **角色设置**：引入独立的**批评者** 或 **魔鬼代言人** 角色，专门负责挑错。
- **外部验证**：强制要求使用外部工具验证（如单元测试、代码执行结果、RAG 检索证据），而不只是互相采信。
- **结构化对话**：使用 Speaker Selection 机制（基于 LLM 预测下一个发言者）或 Group Chat Manager 控制轮次。
- **终止条件**：设置最大轮次，或者定义特定的终止 Token（如 "TERMINATE"）。

### 6. 实战深化
#### 6.1 实战案例
在构建自动发帖系统时，如果只有“写手”和“审核员”，审核员往往倾向于通过（社会惰性）。引入“律师”Agent 后，律师会专门审查内容是否有版权风险或违规词。这种三足鼎立的架构显著降低了内容事故率，且多 Agent 间的辩论往往能单 Agent 遗漏的边缘情况。

#### 6.2 关键代码示例

```python
# AutoGen 伪代码：配置多 Agent 辩论模式
from autogen import AssistantAgent, UserProxyAgent, GroupChat, GroupChatManager

# 定义角色：包含 Devil's Advocate (批评者)
coder = AssistantAgent(name="Coder", system_message="You write Python code.")
reviewer = AssistantAgent(name="Reviewer", system_message="You review code for bugs.")
critic = AssistantAgent(name="Critic", system_message="You are a devil's advocate. Find logic flaws and security risks.")

# 定义群聊管理器，控制发言顺序
groupchat = GroupChat(agents=[coder, reviewer, critic], messages=[], max_round=10)
manager = GroupChatManager(groupchat=groupchat, name="Manager")

# 启动任务
user_proxy = UserProxyAgent(name="User", human_input_mode="NEVER")
user_proxy.initiate_chat(
    manager, 
    message="Write a secure function to process user input."
)
```
