# Tokenization

Tokenization 将原始文本切分为模型可处理的 Token 序列，每个 Token 对应词表中的 ID。

### 1. 常见算法

*   **BPE (Byte Pair Encoding)**：
    *   原理：从字符级开始，迭代合并频率最高的字节对。
    *   特点：平衡词表大小和 OOV（未登录词）问题，子词组合灵活。GPT 系列常用。
*   **WordPiece (BERT)**：
    *   原理：类似 BPE，但合并准则基于最大化训练数据的语言模型似然，而非单纯频率。
*   **SentencePiece**：
    *   特点：将空格也作为特殊字符处理，支持多语言（尤其是无空格语言如日文）。底层支持 BPE 和 Unigram。

### 2. Tokenizer 训练流程图解

```text
原始语料库
    │
    ▼
预处理 (Unicode 归一化)
    │
    ▼
预切分 (单词级/字符级)
    │
    ▼
统计词频
    │
    ▼
迭代合并 (BPE: 选最大频对)
    │
    ▼
生成词表 & Merge Rules
```

### 3. 面试问答

**Q：BPE 和 WordPiece 主要区别？**

**A：** 都是子词合并思路。BPE 常用最高频合并；WordPiece 选择能使训练数据概率提升最大的合并（似然最大化）。

**Q：为什么 LLM 常用子词而不是纯词？**

**A：** 纯词级词表巨大且稀疏，难以处理未登录词（OOV）。子词级可以用有限词表组合出任意单词，且词表大小可控。

**Q：字节级 BPE (Byte-level BPE) 的优缺点？**

**A：**
*   **优点**：词表小，能处理任意字符（包括乱码、特殊符号），完全解决 OOV。
*   **缺点**：同样的文本 Token 数量会变多，增加算力成本和上下文窗口压力。

## 常见考点
1.  **特殊 Token**：`<BOS>`, `<EOS>`, `<PAD>`, `<UNK>` 的含义及使用场景（如填充截断策略）。
2.  **分词器的不可逆性**：从文本转 ID 再转回文本，空格或特殊符号可能会发生变化（如 SentencePiece）。
3.  **多语言分词**：不同语言在统一词表下的 Token 效率差异（中文通常比英文切分更碎）。
