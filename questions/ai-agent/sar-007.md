# 多模态RAG（Multimodal RAG）是什么？如何处理图文混合检索？

**多模态 RAG (Multimodal RAG)** 是指能够处理和检索包括文本、图像、音频、视频等多种模态数据，并将其作为上下文输入给大模型以增强生成能力的架构。

### 如何处理图文混合检索？

1.  **统一向量空间 (CLIP 策略)**
    *   **原理**：使用像 CLIP (Contrastive Language-Image Pre-training) 这样的对齐模型，将图片和文本映射到同一个特征向量空间。
    *   **检索**：用户可以用文本搜图片（“找一张红色的猫”），也可以用图片搜图片，甚至图片搜文本。
    *   **实现**：离线时提取图片特征存入向量库；在线时将 Query 对齐到同一空间进行检索。

2.  **多模态大模型直接理解**
    *   **原理**：利用像 GPT-4o 或 LLaVA 这种原生支持多模态输入的 LLM。
    *   **流程**：RAG 检索阶段可以分离（用 CLIP 检索图片，用 BM25 检索文本），但在 Prompt 构建阶段，将图片转为 Base64 或 URL，和文本摘要一起喂给 LLM。
    *   **优势**：LLM 能理解图片内容（如 OCR 识别、图表分析）并回答。

3.  **图文混合索引策略**
    *   **图片描述增强**：离线处理时，先利用视觉模型（如 BLIP）生成图片的详细描述文本，将描述文本作为图片的 Metadata 索引到向量库。检索时用文本搜描述，命中率更高。
    *   **表格解析**：对于文档中的表格，不能只截图，需识别为 HTML/Markdown 结构化文本，确保能检索到具体的单元格数据。

### 实战案例
*   **电商搜索优化**：用户上传一张家具图片，系统通过 CLIP 检索向量库中风格相似的“沙发的图片和商品详情文本”，LLM 综合分析后推荐搭配建议，而不仅仅是返回相似图。
*   **工业维修手册**：维修员上传设备故障照片，RAG 系统检索出该部件的 PDF 原理图（图片）和维修步骤（文本），LLM 对照图片指出故障点。

### 关键代码示例 (Python - 使用 CLIP 检索)
```python
import clip
import torch
from PIL import Image

# 加载模型
device = "cuda" if torch.cuda.is_available() else "cpu"
model, preprocess = clip.load("ViT-B/32", device=device)

# 1. 离线：图片向量化
image = preprocess(Image.open("product.jpg")).unsqueeze(0).to(device)
with torch.no_grad():
    image_features = model.encode_image(image) # 存入向量库

# 2. 在线：图文混合检索
text = clip.tokenize(["a red sofa"]).to(device)
with torch.no_grad():
    logits_per_image, logits_per_text = model(image, text)
    probs = logits_per_image.softmax(dim=-1).cpu().numpy()
```

### 图文处理策略对比
| 策略 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **CLIP 向量检索** | 图文映射到同一空间 | 速度快，支持跨模态 | 细粒度内容（如小字）识别差 | 以图搜图、风格匹配 |
| **视觉描述生成** | 图片转文本摘要再检索 | 语义理解深，利用成熟的文本 RAG | 增加离线推理成本，依赖描述质量 | 电商详情、医疗影像 |
| **端到端 VLM** | LLM 直接看图 | 理解能力最强，能推理 | 推理成本极高，延迟高 | 复杂图表分析、诊断 |

## 记忆要点

- 本质区别：RAG解决知识（外挂数据库），微调解决能力（内化模型参数）
- RAG优势：知识实时更新、私有数据访问、幻觉低、可溯源、成本低
- 微调优势：改变输出风格/格式、学习领域术语、降低推理延迟
- 最佳实践：先RAG满足知识需求，效果不足再微调提升特定能力
