# RAG 知识库更新后如何避免旧索引污染

- **索引污染**指知识库更新后,旧版本的向量仍残留在索引中,导致检索到过期或错误信息.

- **污染来源**
1. 文档更新后旧向量未删除
2. 文档删除后向量未清理
3. 版本冲突:新旧版本同时被检索
4. 元数据未同步更新

- **解决方案**
1. **文档级 ID 管理**:每个文档/切片分配唯一 ID,更新时先删旧 ID 再插入新 ID
2. **版本号标注**:向量存储时带 version 字段,检索时过滤旧版本
3. **增量更新管道**:监听文档变更 → diff 检测 → 自动更新索引
4. **TTL 过期机制**:给向量设过期时间,定期清理
5. **软删除 + 定期重建**:标记删除,低峰期全量重建索引
6. **双索引策略**:新旧索引并存,灰度切换后删除旧索引

- **增量更新流程架构**

- **实战案例**
- **场景**：在一个电商 FAQ 机器人中，促销政策文档每日更新。若仅采用追加模式，用户提问“今日活动”时，模型可能检索到昨日的已过期优惠向量，导致回答错误。
- **踩坑**：依赖“软删除”进行检索过滤时，若忘记在 `delete` 语句中同步删除对应的倒排索引或过滤索引，会导致虽然向量标记为删除，但检索时仍会被 Top-k 算法捞出，增加无效计算和幻觉风险。
