# 注意力机制详解

注意力本质上是一种软寻址机制：根据 Query 与各个 Key 的相似度得到权重，再对 Value 加权求和。

### 1. 核心原理

*   **缩放点积**：$\text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$。除以 $\sqrt{d_k}$ 是为了防止点积过大导致 softmax 进入饱和区，梯度消失。
*   **Multi-Head Attention (MHA)**：将 $d_{model}$ 拆分为多个头，每个头独立计算注意力，最后拼接。类似于 CNN 的多通道。
*   **Grouped Query Attention (GQA)**：折中方案。多组 Query 共享一组 Key/Value。减少 KV Cache 显存，平衡推理速度与质量（Llama 3 采用）。
*   **Multi-Query Attention (MQA)**：所有头共享同一组 K/V，推理最快，但可能损质量。

### 2. Flash Attention (核心优化)

*   **问题**：标准实现需要物化完整的 $N \times N$ 注意力矩阵，显存占用大，且频繁在 HBM（慢）和 SRAM（快）之间读写。
*   **解法**：
    1.  **Tiling**：将 Q, K, V 分块，在 GPU SRAM 内完成计算。
    2.  **IO-aware**：减少 HBM 读写次数。
    3.  **Recomputation**：反向传播时重算 Attention，省去存储中间激活值。

### 3. 数据流与算子融合图解

```text
标准 Attention:
[HBM Load Q,K,V] -> [SRAM Compute QK^T] -> [HBM Write S(N*N)] -> 
[HBM Load S, V] -> [SRAM Compute Softmax] -> [HBM Write O] 
(大量 HBM 读写)

Flash Attention:
[HBM Load Q,K,V Blocks] -> [SRAM: Compute & Keep Softmax Stats] -> 
[Accumulate Output] -> [HBM Write Final O]
(SRAM 分块计算，无需存储巨大 S 矩阵)
```

### 4. 面试问答

**Q：简述 Flash Attention 为什么能快和省显存？**

**A：** 通过分块在片上完成注意力计算，大幅减少对慢速全局显存的读写；避免存储完整的 $N \times N$ 注意力矩阵（或显著降低峰值），并融合算子提高吞吐。

**Q：GQA 相对 MHA 推理上主要省在哪里？**

**A：** KV Cache 体积大幅减小（多组 Query 共享 KV），降低了内存带宽压力和显存占用，Decode 阶段受益明显。

## 常见考点
1.  **Softmax 饱和问题**：为何要除以 $\sqrt{d_k}$？解释梯度消失与方差的关系。
2.  **注意力头数**：头数太少导致信息瓶颈，太多导致每个头维度下降影响表达能力。
3.  **因果掩码实现**：在代码中如何通过下三角矩阵或加法掩码实现未来信息的屏蔽。

### 实战案例
在 Llama-2-70B 推理服务化中，发现 Decode 阶段吞吐受限于显存带宽。通过将模型从 MHA 权重量化为 GQA 格式（虽然略有精度损失，但在可接受范围内），KV Cache 占用减少了 60%，使得单卡 Batch Size 翻倍，整体吞吐提升 1.8 倍。

### 代码示例
模拟 GQA 的 KV Cache 结构（Key/Value 头数少于 Query 头数）:

```python
import torch

class GQA_Cache:
    def __init__(self, num_heads, num_kv_heads, head_dim):
        self.num_heads = num_heads  # 例如 32
        self.num_kv_heads = num_kv_heads # 例如 8 (GQA)
        self.head_dim = head_dim
        self.k_cache = torch.zeros((1, num_kv_heads, 0, head_dim))
        self.v_cache = torch.zeros((1, num_kv_heads, 0, head_dim))
        self.repeats = num_heads // num_kv_heads

    def update(self, new_k, new_v):
        self.k_cache = torch.cat([self.k_cache, new_k], dim=2)
        self.v_cache = torch.cat([self.v_cache, new_v], dim=2)
        
    def get_kv(self):
        # 关键：通过 repeat 扩展 KV 以匹配 Q 的头数
        return self.k_cache.repeat_interleave(self.repeats, dim=1), \
               self.v_cache.repeat_interleave(self.repeats, dim=1)
```

### 对比表格

| 变体 | KV Cache 大小 | 推理速度 | 模型表达能力 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **MHA** | 大 (1x) | 慢 | 强 | 训练、追求极致精度 |
| **GQA** | 中 (1/Group) | 中 | 中 | 通用大模型推理 (如 Llama 3) |
| **MQA** | 小 (1/Heads) | 快 | 弱 | 极低延迟需求场景 (如 PaLM) |
