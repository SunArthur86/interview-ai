# 记忆检索策略

检索策略决定在生成下一轮回复前，从记忆池中召回哪些信息。单一策略往往不足，需混合使用。

### 常见策略
1.  **基于时间**：优先召回最近对话。适合排障、联调等强时效任务。
2.  **基于相关性**：通过向量相似度召回。适合用户换说法的开放域问答。
3.  **基于重要性**：优先召回高权重信息（如用户核心目标）。避免早期硬约束被淹没。

### Generative Agents 多因子融合
斯坦福 Generative Agents 论文提出综合打分公式：
$$ Score = w_{rel} \cdot \widehat{rel} + w_{rec} \cdot \widehat{recency} + w_{imp} \cdot \widehat{importance} $$
其中：
*   $\widehat{rel}$: 相关性（向量余弦）
*   $\widehat{recency}$: 近期性（时间衰减函数，如 $e^{-\alpha \Delta t}$）
*   $\widehat{importance}$: 重要性（通常由 LLM 生成时打分 1-10）

### 实现难点
*   **去重**：同一事实的不同表述需语义对齐，避免冗余占用上下文。
*   **限长**：按得分排序后进行 Token 装箱，或分层注入（摘要+细节）。

### 实战案例
*   **优化经验**：在客服场景中，直接按 Score 返回 Top 5 可能导致 5 条都是关于“退款政策”的重复内容，而忽略了用户当前具体的“订单号”。实战中常加入 **MMR (Maximal Marginal Relevance)** 算法，在保证相关性的同时，强制增加结果之间的多样性（如 1 条政策 + 1 条历史记录 + 1 条工具日志）。

### 对比表格：召回策略
| 策略 | 核心逻辑 | 适用场景 | 常用权重配置 |
| :--- | :--- | :--- | :--- |
| **时间优先** | 时间倒序 | 连续任务、纠错对话 | Recency=1.0, Rel=0.2 |
| **语义优先** | 向量相似度 TopK | 知识库问答 | Rel=1.0, Recency=0.1 |
| **重要性优先** | 加权打分 | 长目标追踪 | Imp=0.6, Rec=0.2, Rel=0.2 |
| **混合策略** | 多因子加权 | 通用智能体 | Rel=0.5, Rec=0.3, Imp=0.2 |

### 代码逻辑（Python）
```python
import numpy as np

def hybrid_score(memories, query_vec, weights={'rel': 0.5, 'rec': 0.3, 'imp': 0.2}, time_decay=0.1):
    now = time.time()
    scored_memories = []
    
    for m in memories:
        # 1. 归一化相关得分
        rel_score = np.dot(m.vector, query_vec) / (np.linalg.norm(m.vector) * np.linalg.norm(query_vec))
        
        # 2. 计算时间衰减 (例如：半衰期策略)
        rec_score = np.exp(-time_decay * (now - m.timestamp))
        
        # 3. 获取归一化重要性 (假设写入时已存 0-1 之间)
        imp_score = m.importance
        
        # 4. 加权求和
        final_score = (
            weights['rel'] * rel_score + 
            weights['rec'] * rec_score + 
            weights['imp'] * imp_score
        )
        scored_memories.append({'memory': m, 'score': final_score})
        
    # 按分数降序排列
    return sorted(scored_memories, key=lambda x: x['score'], reverse=True)
```
