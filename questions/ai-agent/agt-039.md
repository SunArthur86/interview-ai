# 情景记忆与语义记忆

情景记忆与语义记忆源于认知心理学，在 Agent 中对应不同的存储与检索策略。

1.  **情景记忆**：关于具体事件（时间、地点、参与者）。例如「昨天下午用户要求改 PDF」。
    *   **实现**：带时间戳的对话片段、工具日志。
    *   **特点**：高度个人化，随时间衰减。
2.  **语义记忆**：关于一般性知识、规则。例如「公司退货政策是 7 天」。
    *   **实现**：知识库条目、FAQ、图谱三元组。
    *   **特点**：共享、稳定，更新频率低。

### 为什么要区分？
*   **检索特征**：情景检索偏时间邻近；语义检索偏概念相似。
*   **治理策略**：情景隐私级别高且易过期；语义需长期维护且多租户共享。

### 实现策略
*   **情景记忆**：通常存储在向量数据库中，检索时强依赖时间衰减权重，防止过时信息干扰当前任务。
*   **语义记忆**：可以是向量库，也可以是知识图谱。对于规则类信息，KG 的结构化查询更精准；对于通用知识，向量库更灵活。

### 实战案例
*   **工程场景**：在构建 HR 助手时，用户问“我有多少年假？”。这属于**情景记忆**（需查该用户的个人余额）；而用户问“年假怎么请？”，这属于**语义记忆**（查公司制度）。混淆两者会导致通用错误回复或隐私泄露。架构设计时，必须将两者的 Collection 物理隔离，并对情景记忆增加严格的 ACL 校验。

### 对比表格：记忆类型差异
| 维度 | 情景记忆 | 语义记忆 |
| :--- | :--- | :--- |
| **数据来源** | 用户交互历史、Agent 行为日志 | 官方文档、手册、Wiki |
| **共享性** | 私有 | 全局共享 |
| **时效性** | 强依赖时间 (需要衰减) | 相对静态，仅在政策变更时更新 |
| **检索逻辑** | "用户最近做了什么" | "通用规则是什么" |
| **存储建议** | Vector DB (带 Timestamp) | Graph DB / Vector DB (全局索引) |

### 代码模型（Python）
```python
@dataclass
class MemoryObject:
    type: str # "episodic" or "semantic"
    content: str
    metadata: dict
    
    def get_search_filter(self):
        if self.type == "episodic":
            # 情景记忆：强过滤，必须匹配用户，且通常是近期
            return {"must": [{"key": "user_id", "match": {"value": self.metadata["user_id"]}}]}
        else:
            # 语义记忆：无用户过滤，仅匹配业务领域
            return {"must": [{"key": "domain", "match": {"value": self.metadata["domain"]}}]}
```
