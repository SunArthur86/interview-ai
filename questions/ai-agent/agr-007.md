# RRF(Reciprocal Rank Fusion)为什么比加权分数融合更鲁棒

**SAME**

## 记忆要点

- RRF使用倒数排名融合(1/(k+rank))，不依赖分数绝对值，只看相对排序。
- 加权融合受不同检索器分数量纲影响(如0-1 vs 0-100)，需复杂归一化。
- RRF对异常分数鲁棒，且参数k(通常60)通用性强，无需针对数据集微调。
- 结论：RRF消除了分数校准的难题，在多路异构检索器融合中更稳定。
