# 为什么需要多智能体

多智能体系统指多个相对独立的 Agent 在协作协议下共同完成复杂任务。

### 为什么需要多智能体？
**单 Agent 的瓶颈**：
1.  **注意力漂移**：长提示词下，模型易忽略中间的约束。
2.  **推理链断裂**：复杂任务中，单次调用难以进行长程规划与回溯。
3.  **能力局限**：单一 System Prompt 难以同时兼顾严谨规划、创意发散与代码执行。

**多 Agent 的优势**：
1.  **专业化**：每个 Agent 专注于特定领域（如代码审查），Prompt 更短更稳。
2.  **并行化**：独立子任务可并行执行，缩短墙钟时间。
3.  **容错性**：某一 Agent 失败可隔离重试，不致全盘崩溃。

### 协作模式与架构
```text
      [ 用户请求 ]
           │
           ▼
    ┌──────────────┐
    │  管理者 Agent │
    │  (拆解任务)   │
    └──────┬───────┘
           │
     ┌─────┴─────┬────────────┐
     ▼           ▼            ▼
[ 研究员 ]  [ 程序员 ]    [ 审核员 ]
(并行/串行)  (执行代码)    (质量把关)
     │           │            │
     └─────┬─────┴────────────┘
           │
           ▼
    [ 结果汇总/迭代 ]
```

### 代码逻辑（示意）
```python
roles = [
    ("analyst", "你只输出需求要点。"),
    ("coder", "你只输出代码。"),
    ("reviewer", "你只审查代码安全。")
]
for name, hint in roles:
    agent = SimpleAgent(name, hint)
    result = agent.run(task)
```

### 实战案例
在自动化测试生成场景中，单 Agent 往往在生成测试代码后忽略边界条件检查。引入“代码生成员”+“测试专家”双 Agent 后，前者专注写代码，后者专门针对前者输出构造 Edge Case 输入，测试覆盖率从 40% 提升至 85%。

### 代码示例（动态并行调用）
```python
import concurrent.futures

def run_multi_agent(task):
    # 定义专家
    agents = [ResearchAgent(), CoderAgent(), ReviewerAgent()]
    results = {}
    
    # 并行执行独立任务
    with concurrent.futures.ThreadPoolExecutor() as executor:
        futures = {executor.submit(agent.run, task): agent for agent in agents}
        for future in concurrent.futures.as_completed(futures):
            agent = futures[future]
            results[agent.name] = future.result()
    
    return ManagerAgent().synthesize(results)
```

### 对比表格
| 维度 | 单 Agent | 多 Agent |
| :--- | :--- | :--- |
| **Context 利用** | 效率随长度下降，易分心 | 每个 Agent 拥有纯净上下文 |
| **错误恢复** | 需重新生成全量内容 | 隔离失败 Agent，定点重试 |
| **Token 成本** | 单次长请求（高 Hidden Cost） | 多次短请求（可控） |
| **适用场景** | 简单问答、翻译 | 复杂工作流、代码审查、科研 |

## 常见考点
1.  **通信机制**：多 Agent 之间如何通信？（如：自然语言消息传递、共享的黑板机制/消息队列、或直接函数调用）。
2.  **冲突解决**：当两个 Agent 的意见不一致时（如程序员和审核员吵架），如何解决？（通常引入裁判 Agent 或由 Manager Agent 依据最终目标进行裁决）。
3.  **成本控制**：多 Agent 会成倍增加 Token 消耗，如何优化？（例如：Agent 间使用更紧凑的协议格式而非自然语言，或设置最大迭代轮数）。
4.  **动态规划**：Agent 的数量和角色是静态配置的还是动态生成的？（高级系统会根据任务难度动态生成新的子 Agent）。
