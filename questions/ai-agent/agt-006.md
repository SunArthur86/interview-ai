# Agent 的应用场景

### 1. 智能客服
**场景**：工单查询、订单状态、退换货政策。
**技术要点**：
- **知识库对齐**：确保 RAG 检索准确。
- **身份鉴权**：验证用户权限，防止越权查询。
- **升级策略**：识别无法解决的情况，无缝转接人工。

> **实战案例**：某电商大促期间，用户询问“怎么退空调”，Agent若仅检索通用退款流程会漏掉“大件物流上门取件”的关键信息，需针对 SKU 级别构建差异化知识库。

**代码示例**：
```python
def check_permission(user_id, order_id):
    # 检查用户是否拥有该订单的查看权限
    if not db.order_exists(order_id, user_id):
        raise PermissionError("无权访问该订单")
    return db.get_order_status(order_id)
```

### 2. 代码研发
**场景**：读仓库、跑测试、提交 PR。
**技术要点**：
- **沙箱环境**：隔离执行环境，防止恶意代码影响宿主。
- **Diff 审查**：人工或自动审查代码变更。
- **最小权限**：限制代码仓库的操作权限。

> **实战案例**：在生成数据库迁移脚本时，Agent 曾因幻觉生成了 `DROP TABLE` 语句，必须在沙箱中配置“只读数据库”并在执行前增加 SQL 关键词黑名单校验。

**代码示例**：
```python
import docker

def run_tests_in_sandbox(repo_path):
    client = docker.from_env()
    # 挂载代码卷，限制网络，仅执行测试命令
    container = client.containers.run(
        "python:3.9-slim", 
        command=f"bash -c 'cd /app && pip install -r requirements.txt && pytest'",
        volumes={repo_path: {'bind': '/app', 'mode': 'ro'}},
        network_disabled=True,
        remove=True
    )
```

### 3. 数据分析
**场景**：自然语言转 SQL（NL2SQL）、自动制图、结果解读。
**技术要点**：
- **数据治理**：确保数据来源合规、口径统一。
- **行级权限**：精确控制数据访问范围。
- **结果校验**：对生成结果进行逻辑或统计验证。

> **实战案例**：在 BI 场景中，用户问“上季度销售额”，Agent 直接汇总了全公司数据，未按用户所属部门过滤，必须在生成 SQL 阶段注入动态上下文（如 `WHERE dept_id = {current_user.dept}`）。

**代码示例**：
```python
# 简单的结果校验逻辑
def validate_sql_result(df, query_context):
    if df.empty:
        return {"status": "warning", "msg": "查询结果为空，请确认筛选条件"}
    # 防止全表扫瞄导致的数据量过大
    if len(df) > 10000:
        return {"status": "error", "msg": "数据量超过展示上限，请增加聚合维度"}
    return {"status": "success", "data": df.head(100).to_dict()}
```

### 4. 自动化运维
**场景**：日志查询、服务重启、故障开单。
**技术要点**：
- **审批流**：关键操作需经过人工或自动审批。
- **灰度发布**：逐步滚动更新，降低风险。
- **一键回滚**：操作失败时快速恢复状态。

> **实战案例**：某生产环境 CPU 告警，Agent 误判为应用异常并尝试重启容器，实际上是上游流量激增。解决方案是增加“熔断机制”，在重启前强制要求查询该实例的 QPS 历史趋势。

**代码示例**：
```python
def restart_service_with_approval(service_name):
    # 预检查：如果负载过高，拒绝自动重启
    current_load = monitor.get_cpu_usage(service_name)
    if current_load > 90:
        return {"action": "reject", "reason": "CPU过高，可能由流量突增引起，拒绝自动重启"}
    
    # 执行重启
    k8s_api.restart_deployment(service_name)
    return {"action": "executed", "status": "restarting"}
```

### 5. 知识管理
**场景**：文档摘要、自动标签、关联检索。
**技术要点**：
- **溯源机制**：标注信息来源，便于查证。
- **版本控制**：维护知识的历史版本。
- **权限控制**：按密级或部门隔离知识。

> **实战案例**：企业内网文档常出现“V2.0”等非标准化版本号，导致检索匹配困难。工程上需在文档入库前，通过正则提取并规范化版本元数据，构建结构化索引。

**代码示例**：
```python
from datetime import datetime

def index_document(doc):
    # 增加元数据增强，确保溯源
    metadata = {
        "source_url": doc.url,
        "author": doc.author,
        "indexed_at": datetime.now().isoformat(),
        "dept_access_level": doc.level  # 用于权限过滤
    }
    vector_store.add(text=doc.content, metadata=metadata)
```
