# ReAct框架的核心思想是什么?Thought-Action-Observation循环如何工作

- **ReAct = Reasoning + Acting**

让LLM交替进行推理和行动,使用工具后观察结果.

- **工作流程:**
```
**用户：** 北京明天天气如何?

**Thought：** 我需要查询北京的天气预报
**Action：** weather_api(city="北京", date="tomorrow")
**Observation：** 晴, 25°C, 东南风3级

**Thought：** 我拿到了天气信息,可以回答了
**Action：** finish(answer="北京明天晴,25°C,东南风3级")
```

- **优势:**
1. **减少幻觉** - 基于工具返回的真实数据
2. **可追溯** - 推理链可审查
3. **可纠错** - 工具失败后可重试或换方案

- **局限:**
- Token消耗大(每次都带完整推理历史)
- 延迟高(多轮工具调用)
- 需要精心设计工具描述

- **实战案例**：在运维告警处理中，当服务器API报错时，LLM不会凭空捏造原因，而是通过ReAct调用日志查询工具，发现是磁盘满，再自动调用清理脚本，避免了人工排查的滞后。

- **代码示例**：
```python
# 模拟 ReAct 循环解析伪代码
prompt = """
Question: {input}
Thought: {agent_scratchpad}
"""

while True:
    response = llm.predict(prompt) # LLM生成 Thought 和 Action
    thought, action, action_input = parse_react(response)
    if action == "Finish": break
    
    observation = tools[action].run(action_input) # 执行工具
    prompt += f"\nObservation: {observation}" # 将结果回填
```
