# AutoGen的GroupChat如何工作?发言人选择策略有哪些

- **AutoGen GroupChat架构:**

多个Agent在一个共享对话中协作,由GroupChatManager管理.

```text
       AutoGen GroupChat 交互流

    +-------------------------+
    |   GroupChatManager      |
    | (管理发言顺序 & 上下文) |
    +-----------+-------------+
                | Select Next
       +--------+--------+-------+
       |                 |       |
   [Agent A]         [Agent B] [Agent C]
   (Coder)          (Reviewer) (Tester)
       |                 |       |
       +--------+-------+-------+
                | Broadcast
                v
       [Shared Conversation History]
```

- **核心组件:**
1. **Agents** - 各有角色和工具, `system_message`定义人设
2. **GroupChatManager** - 核心调度器,维护`messages`列表,决定下一轮谁说话
3. **Speaker Selection** - 决定下一个发言者,支持多种策略

- **发言人选择策略:**

| 策略 | 机制 | 适用场景 | 性能消耗 |
| :--- | :--- | :--- | :--- |
| **auto** (默认) | LLM作为裁判，根据对话历史解析 | 动态协商、不确定的流程 | 高 (每轮需额外LLM调用) |
| **round_robin** | 按列表顺序固定轮询 | 固定流水线 (如 写->审->测) | 低 (无计算开销) |
| **manual** | 程序暂停，等待人工输入 | 调试、关键决策介入 | 依赖人 |
| **random** | 随机挑选 | 创意生成、头脑风暴 | 低 |
| **custom** | 自定义Python函数逻辑 | 需要特定规则路由 (如关键词触发) | 中 (取决于函数逻辑) |

- **典型配置:**
```python
manager = GroupChatManager(
groupchat=GroupChat(
agents=[planner, coder, reviewer],
messages=[],
max_round=20,  # 防止死循环
speaker_selection_method="auto"
)
)
```

- **实战案例**: 在代码审查场景中，Coder和Reviewer常陷入“改好-未改好”的死循环。实战中我们会引入带有`termination_msg`的Admin Agent，或自定义选择函数：如果连续3轮发言人相同，强制切换到第三方Agent。

- **代码示例 (自定义选择策略)**:
```python
def custom_speaker_selection(last_speaker, groupchat):
    # 如果发言人是Coder且包含[已修复]，强制切换给Tester
    if last_speaker.name == "Coder" and "[已修复]" in last_speaker.last_message():
        return next(agent for agent in groupchat.agents if agent.name == "Tester")
    return None # 回退到默认auto

manager = GroupChatManager(groupchat=groupchat, speaker_selection_method=custom_speaker_selection)
```

- **适用场景:**
- 代码开发:Planner→Coder→Tester→Reviewer
- 研究任务:Researcher→Analyst→Writer
- 决策:多个专家Agent辩论

## 常见考点
1. **终止条件**：如何定义对话结束？（例如`max_round`、检测到“TERMINATE”关键词）
2. **上下文窗口限制**：多轮对话后History如何处理？（Summary、滑动窗口）
3. **循环问题**：两个Agent陷入无限争论（如Coder说改好了，Reviewer说不行），如何设计机制打破僵局？
