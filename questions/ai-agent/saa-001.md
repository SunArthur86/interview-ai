# ReAct框架的原理是什么？

ReAct（Reasoning + Acting）让LLM交替进行推理和行动。这种范式结合了思维链和决策执行能力。

**循环流程**：
1. **Thought**：LLM分析当前状态，推导下一步行动
2. **Action**：调用外部工具（搜索、计算器、API等）
3. **Observation**：获取工具返回的执行结果
4. 重复上述步骤，直到得出最终答案

**架构流程图**：
```text
┌─────────────┐
│   用户问题   │
└──────┬──────┘
       ▼
┌─────────────┐     ┌──────────────┐
│   Thought   │────▶│    Action    │────▶ 外部环境(API)
│  (LLM推理)   │     │  (调用工具)    │     (执行查询/计算)
└──────┬──────┘     └──────┬───────┘     └──────┬───────┘
       ▲                   ▼                    │
       │           ┌──────────────┐             │
       │           │ Observation  │◀────────────┘
       │           │  (获取结果)   │
       └───────────┴──────┬───────┘
                       ▼
                 (是否完成?) ──No──┐
                       │ Yes      │
                       ▼          │
                ┌─────────────┐   │
                │  Final Answer◀──┘
                └─────────────┘
```

**Prompt 模板示例**：
```
Question: 北京今天的天气怎么样？
Thought 1: 我需要查询北京的实时天气。
Action 1: Search["北京天气"]
Observation 1: 北京今天晴，气温25°C。
Thought 2: 已经获取到信息，现在可以回答用户了。
Action 2: Finish["北京今天晴，25°C"]
```

**核心优势**：
- **可解释性强**：推理过程（Thought）完全透明，便于调试
- **纠错能力**：如果工具返回错误，LLM可以在下一步Thought中修正策略
- **泛化性**：无需针对特定任务微调模型

**局限性**：
- **线性陷阱**：单线程执行，无法并行处理任务
- **幻觉累积**：中间某一步推理错误可能导致后续一连串错误
- **上下文消耗**：长任务会快速占用Token窗口

- **实战案例**
- **场景**：开发运维助手查询服务器状态。ReAct 能够先思考“需要确认服务器 IP”，再调用工具获取 IP，最后用 IP 查询负载，逻辑清晰。但在查询一组无关服务器的健康状态时，因为必须串行执行，耗时远超并发脚本。
- **踩坑**：如果工具返回的 `Observation` 是极长的报错日志（如 5000 字符堆栈），极易撑爆上下文窗口或导致 LLM 注意力分散，需要在工具层对结果进行摘要截断。

- **代码示例 (Python 简易 ReAct Loop)**
```python
import re

# 模拟工具
import random

def search_engine(query):
    return f"Search result for {query}: The current price is ${random.randint(100, 200)}."

class ReActAgent:
    def __init__(self, llm):
        self.llm = llm
    
    def run(self, question):
        prompt = f"Question: {question}\nThought:"
        history = prompt
        
        for _ in range(5): # 限制最大步数
            response = self.llm.generate(history)
            
            # 解析 Action: Search[...]
            action_match = re.search(r'Action\:\s*(\w+)\[(.*?)\]', response)
            if action_match:
                tool, arg = action_match.groups()
                if tool == "Search":
                    obs = search_engine(arg)
                    print(f"Action: {tool}[{arg}]")
                    print(f"Observation: {obs}")
                    history += f" {response}\nObservation: {obs}\nThought:"
            else:
                return response
        return "Max steps reached"
```
