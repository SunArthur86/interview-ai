# 文档解析与预处理

### 文档解析与预处理

**2.1 常见解析工具**
- **PDF**：PyPDF2 (简单), pdfplumber (表格), Unstructured (统一分区)
- **Word**：python-docx
- **HTML**：BeautifulSoup4, readability-lxml (去噪)

**面试 Q5：PDF 解析常见坑有哪些？**
A：1）扫描件无文本层，必须 OCR；2）多栏、脚注、页码混入正文；3）表格被拆成无序碎片；4）字体编码异常导致乱码。对策：版面分析、表格专用解析、清洗规则。

**实战案例**：处理一份财报 PDF 时，pdfplumber 把跨页表格解析成了两段独立的文本，导致检索“净利润”时数值与表头分离。后来改用 Table-Transformer 专门提取表格结构才解决。

**2.2 OCR 处理**
常用方案：云服务（Azure/Globe/阿里云 OCR），开源（Tesseract, PaddleOCR 中文友好）。

**面试 Q6：OCR 结果对 RAG 有什么影响？**
A：OCR 会引入错字、断行、丢标点，导致检索词不匹配与 embedding 语义偏移。需在清洗阶段做规范化，并考虑关键词+向量混合检索。

**实战案例**：一份扫描合同中“10%”被 OCR 识别为“1 0 %”，导致向量检索无法匹配到关键词检索的结果。通过引入基于规则的“纠错正则”映射才找回召回率。

**代码示例 (PaddleOCR)**：
```python
from paddleocr import PaddleOCR

ocr = PaddleOCR(use_angle_cls=True, lang="ch")
result = ocr.ocr("scan.png", cls=True)
text_lines = [line[1][0] for line in result[0]]
full_text = "\n".join(text_lines)
```

**2.3 表格和图片处理**
- **表格**：转为 Markdown/HTML 或 JSON 行记录，保留结构。
- **图片**：用 Caption 模型（如 BLIP）生成描述文本入库。

**面试 Q7：表格为什么难做 RAG？**
A：表格语义依赖行列关系与表头；固定长度切分易切断行列。应用表头感知分块、SQL/结构化检索或专用表格问答模型辅助。

**实战案例**：在对比两款手机参数时，简单的文本分块把“摄像头”参数切成了两行，模型回答时混淆了两个机型。改用 HTML 格式保留 Table 标签后，模型理解力大幅提升。

**2.4 数据清洗与标准化**
要点：统一编码（UTF-8）、去空白去噪、去重、PII 脱敏。

### 流程示意：文档处理管道
```text
原始文档
   │
   ├──> [检测] 是扫描件?
   │         │
   │         ├── Yes ──> OCR 引擎 ──> 文本层提取
   │         │
   │         └── No ───> 文本层提取
   │
   ▼
版面分析
   │
   ├──> 识别: 标题, 段落, 表格, 图片
   │
   ├──> [表格] ──> 结构化解析 (转 Markdown/CSV)
   │
   └──> [图片] ──> 多模态模型 ──> 图片描述文本
   │
   ▼
清洗标准化
   │
   └──> 正则去噪, 特殊字符替换, 编码统一
```

### 常见考点
1. **去重策略**：如何在海量文档中有效识别并去除高度重复的内容（如版权声明、页眉页脚）？
2. **元数据提取**：在解析阶段除了正文，还应提取哪些元数据（如标题、作者、时间）以增强检索过滤能力？
3. **版面分析模型**：除了传统的规则，常用的基于深度学习的版面分析模型（如 LayoutLM）有哪些优势？
4. **隐私合规**：在解析阶段如何识别并处理 PII（个人敏感信息）以防止入库泄露？

### 代码示例：基础清洗与解析
```python
import re
import unicodedata
from pypdf import PdfReader

def normalize_text(s: str) -> str:
    s = unicodedata.normalize("NFKC", s)
    s = re.sub(r"\s+", " ", s).strip()
    return s

def extract_pdf_pypdf(path: str) -> str:
    reader = PdfReader(path)
    parts = [page.extract_text() or "" for page in reader.pages]
    return normalize_text("\n".join(parts))
```
