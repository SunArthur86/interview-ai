# 如何评测记忆系统好坏

### 离线评估
1.  **检索质量**：基于标注集计算 Recall@K（召回率）和 Precision@K（精确率）。
2.  **记忆一致性**：检查新写入的记忆是否与旧记忆冲突（Contradiction Rate），如“用户喜欢猫”与“用户对猫过敏”共存。
3.  **摘要质量**：使用 LLM-as-a-Judge 评估摘要是否保留了关键信息且无幻觉。

### 在线评估
1.  **任务成功率**：最终任务是否完成（如预约是否成功）。
2.  **用户干预率**：用户手动纠正 Agent 错误记忆的频率。
3.  **Token 效率**：检索到的记忆中有多少 Token 被模型实际引用（利用 Attention 权重分析）。
4.  **延迟**：记忆检索耗时（P99）。

### Bad Case 归因分析
```text
Bad Case 发生
      │
      ├─ 检索错 (Recall低) ─> 向量模型不匹配 / 切片策略不合理 / 查询扩展失效
      │
      ├─ 排序错 (Precision低) ─> Reranker 模型能力不足 / 缺乏时间衰减权重
      │
      ├─ 写入错 (污染) ─> 提取信息时产生幻觉 / 未去重
      │
      └─ 摘要错 (失真) ─> 压缩率过高 / 丢失了关键实体/意图
```

### ## 常见考点
- 如何构建记忆系统的标注数据集？（答案：根据对话日志反推“当时如果有哪条记忆，任务会完成得更好”）
- 如何判断记忆是否过时？（答案：结合时间衰减权重，或显式询问用户确认）
- Recall 和 Precision 在记忆系统中哪个更重要？（答案：通常 Recall 更重要，因为 Precision 可以交给 LLM 在上下文中过滤，但漏掉关键信息往往无法挽回）
