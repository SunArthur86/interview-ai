# 只做强相关性检索会有什么问题

只做强相关性检索会导致：
1. **漏检**：无法召回有效但表述不相似的“硬约束”（如“禁止做X”可能被描述为“只能做Y”）。
2. **偏差**：过度偏向语义“像”但内容错误的片段，引入噪声。

**解决方案**：引入**时间维度**（新鲜度）与**重要性维度**（关键权重）补足单纯的相关性得分。

**多路融合策略**：
- **RRF（Reciprocal Rank Fusion）**：对多路召回结果进行倒数排名融合，对异常评分不敏感。
- **Learning to Rank**：训练模型根据相关性、时间、重要性等特征综合排序。
- **加权融合**：$Score = w_1 \cdot Sim + w_2 \cdot TimeDecay + w_3 \cdot Importance$。

**边界情况**：
- **负向查询**：用户问“如何避免XXX”，强相关检索可能召回“如何做XXX”的教程（语义极其相似），导致给出相反建议。需引入“负向关键词过滤”或意图识别。
- **陈旧但权威**：某些法规或合同虽然时间久远，但重要性极高。简单的时间衰减可能导致其被新但非权威的信息覆盖。需对特定类目（如法律、核心配置）进行“时间免疫”或加权。
- **特征归一化失效**：当使用加权融合时，如果 `vec_sim` 是 0-1 之间，而 `doc_importance` 是 1-100 之间，权重参数会完全失效。必须进行 Min-Max 或 Z-Score 归一化。

**实战案例**：
电商客服系统中，用户搜索“退货”。单纯语义检索会返回高相似度的“退货政策说明”，却漏掉了昨天刚刚发布的“由于系统升级暂停退货服务”的紧急公告。引入时间衰减权重后，公告被优先召回，避免了客诉。

**代码示例**：
```python
def hybrid_score(vec_sim, doc_timestamp, doc_importance, current_time):
    # 时间衰减：半衰期为7天
    time_decay = 0.5 ** ((current_time - doc_timestamp) / (7 * 24 * 3600))
    # 重要性加权：人工打分或PageRank
    importance_weight = doc_importance.get('weight', 1.0)
    
    # 线性加权融合，权重需AB测试调优
    final_score = 0.7 * vec_sim + 0.2 * time_decay + 0.1 * importance_weight
    return final_score
```

**## 面试追问**
1. 如果向量数据库本身的分数分布发生了偏移（例如切换了 Embedding 模型），基于阈值的融合策略如何快速自适应？
2. 在多路召回（向量+关键词+图）中，如果某一路召回结果为空，你的融合算法（如 RRF）是否需要特殊处理？
3. 如何通过离线评估指标（如 NDCG）来反推在线融合公式中的最佳权重组合？

**## 易错点**
- **时间衰减一刀切**：对所有文档都应用时间衰减。实际上，对于“定义”、“原理”类静态知识，时间衰减应设为 0 或极低；对于“新闻”、“日志”类动态知识才应用高衰减。
- **权重的静态固化**：上线后固定 `w1, w2, w3` 不再调整。正确的做法是建立特征重要性监控，随着数据分布变化（如疫情期间政策频发），动态调整时间权重。

**## 常见考点**
- 具体的融合公式中，$w_1, w_2, w_3$ 权重如何通过 AB 测试确定？
- 什么是“负向检索”，如何避免检索到“与用户意图相反”的内容？
- 如何处理“长期记忆”与“短期上下文”的冲突？
