# Agent如何实现并行工具调用?有哪些架构挑战

- **并行工具调用场景:**

**用户：** 「比较北京和上海的天气、房价和就业机会」
→ Agent需要调用6个API(2城市×3类别)
→ 并行执行可大幅减少延迟

- **实现方式:**

1. **LLM原生并行**
- 模型一次输出多个tool_calls
- 应用并行执行后一起返回

2. **应用层并行**
- Agent框架管理并行
- asyncio/gather实现

3. **Plan阶段并行检测**
- Planner识别无依赖的任务
- 标记为可并行

- **架构挑战:**
1. **依赖管理** - B依赖A的结果,不能并行
2. **结果聚合** - 并行结果需要正确合并
3. **错误处理** - 部分工具失败怎么办
4. **上下文管理** - 并行结果如何组织在prompt中
5. **超时控制** - 最慢的工具决定整体延迟

- **并行执行与数据流图:**
```
[User Query]
    │
    ▼
┌─────────────────┐
│ LLM Planner     │
│ (Identify Tasks)│
└────────┬────────┘
         │
         ▼
    ┌────┴─────┐
    │  Barrier  │ (Dependency Check: A depends on B?)
    └────┬─────┘
         │ No Dependencies
    ┌────┴──────────────────────┐
    │                         │
    ▼                         ▼
[Task A: Weather]         [Task B: Hotel]
    │                         │
    ▼                         ▼
[API Call A]             [API Call B]
    │                         │
    └────────────┬────────────┘
                 │
                 ▼ (Wait All / Wait Any)
         ┌───────────────┐
         │ Result Aggregator│
         │ (Map Tool -> Result)│
         └───────┬───────┘
                 │
                 ▼
         [LLM Synthesize]
```

- **## 常见考点**
1. **OpenAI 并发格式**：OpenAI 的 `tool_calls` 是一个数组，如何在 Python 中使用 `asyncio.gather` 高效处理这些调用并匹配返回的 `tool_call_id`？
2. **部分失败处理**：如果并行调用了5个工具，其中2个超时失败，LLM 在 Synthesize 阶段应该如何被 Prompt 指导？是告知失败还是忽略？
3. **资源竞争**：如果并行工具都需要访问同一个受限资源（如数据库连接），如何设计信号量或锁来防止竞态条件？
