# Embedding 模型升级后旧向量怎么办

旧向量与新模型不在同一向量空间，不可直接混合比较，否则余弦相似度计算失效（即语义不匹配）。

### 对比表格：升级策略权衡
| 策略 | 优点 | 缺点 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **全量重算** | 效果无损，架构最清晰 | 耗时长，成本高，需停机或双写 | 原始文本保留完整，数据量中等 |
| **向量空间对齐** (如 Matryoshka) | 无需重算旧数据，速度快 | 存在精度损失，需训练对齐矩阵 | 原始文本丢失，或计算资源极度受限 |
| **双塔并行 (灰度)** | 风险低，可随时回滚 | 维护两套库，逻辑复杂，成本翻倍 | 高可用要求，且 V1 向量数据量巨大 |

### 处理方案架构
```text
阶段一: 双写期
┌─────────┐         ┌──────────────┐
│ 新数据   │ ──┬───> │ Embedding V1 │ (旧逻辑)
└─────────┘   │    └──────────────┘
              │           │
              └───┬───────┘
                  │           ▼
                  │      ┌──────────────┐
                  └────> │ Embedding V2 │ (新逻辑) ──> 写入新库/新字段
                         └──────────────┘

阶段二: 查询与回填
┌─────────┐    查询 V1      ┌──────────────┐
│ 用户Query│ ────────────> │ 向量库 (V1)   │ ──> 若未命中
└─────────┘                 └──────────────┘      │
                                                │ 离线重算 V2
                                                ▼
                                         ┌──────────────┐
                                         │ 向量库 (V2)   │ (全量重写完成后切换)
                                         └──────────────┘
```

### 关键细节
1.  **模型版本标识**：在向量记录的 Metadata 中强制存储 `embedding_model_version` 字段。
2.  **灰度发布**：先对新数据使用新模型，旧数据保留旧向量。查询时，分别检索 Top-K V1 和 Top-K V2 结果，在应用层进行 Rerank 混合，逐步验证 V2 效果。
3.  **全量重嵌入**：后台异步任务读取所有原始文本，用新模型重新向量化并覆盖。注意对在线服务的影响，需限流。
4.  **兼容性处理**：切换期间，若用户 Query 使用 V2 向量，必须确保检索参数中指定 `filter: model_version='v2'`，避免跨空间比对。

### 实战案例
在从 bge-small 升级到 bge-large 时，未做隔离直接混合查询，导致检索到的全是语义接近但无关的旧数据（因为不同维度的向量空间度量标准不同）。最终采用“蓝绿部署”：新建 Collection 存 V2 数据，通过应用层 Routing 流量，待全量迁移完成后切换 DNS。

### 代码示例 (Python - 混合检索与元数据过滤)
```python
# 查询时强制通过元数据过滤，防止跨模型比对
def search_hybrid(query_vector, top_k=5):
    # 查询新模型数据
    results_v2 = v2_client.search(
        collection_name="memories_v2",
        query_vector=query_vector,
        limit=top_k,
        expr="model_version == 'v2'" # 关键：确保空间一致性
    )
    
    if len(results_v2) < top_k:
        # 回退：如果 V2 数据不足，可查询 V1（需在应用层做 Rerank 混合，此处省略）
        pass 
    return results_v2
```

### ## 常见考点
- 如果原始文本丢失了，怎么升级模型？（答案：无法完美解决，只能用旧向量通过矩阵变换或一个跨模型对齐模型，但效果有损）
- 如何加速全量重embedding过程？（答案：利用 GPU 批处理，或者利用 MapReduce 架构分片重算）
