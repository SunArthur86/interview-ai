# 在 Multi-Agent 系统中，什么是多 Agent 的资源限制与状态同步问题？如何解决冲突？

在 Multi-Agent 系统（尤其是多角色协作或分层架构）中，多个 Agent 可能同时访问或修改共享资源（如全局记忆、数据库或外部工具），这就带来了资源限制和状态同步的挑战。

**问题**：
1. **冲突**：两个 Agent 同时写入同一个键值对，导致数据覆盖。
2. **一致性**：Agent A 读取了过时的状态，导致决策基于错误信息。
3. **资源争抢**：并发的 API 调用超出速率限制。

**解决方案**：
1. **锁机制**：引入分布式锁，确保同一时间只有一个 Agent 能修改关键资源。
2. **消息队列与事件总线**：将 Agent 的操作异步化，通过队列串行化处理写入请求。
3. **版本控制**：为共享状态增加版本号或时间戳，写入时检查冲突（类似于 Optimistic Concurrency Control）。
4. **中心化管理器**：设置一个“控制者”Agent，专门负责资源调度和状态仲裁，避免子 Agent 直接竞争。
