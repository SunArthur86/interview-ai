# Tool Use / Tool Calling

### Tool Use / Tool Calling

**定义**：Tool 是 Agent 的“能力单元”，封装了名称、描述、参数 Schema 和执行函数。它是连接大模型（LLM）逻辑世界与物理/数字世界（API、数据库、硬件等）的桥梁。

**核心原理**：
模型本身无法执行代码或网络请求。Tool Calling 机制通过特殊 Prompt 注入工具的定义，让模型输出结构化的 JSON（包含 `name` 和 `arguments`），而非普通文本。宿主程序解析该 JSON 并执行实际逻辑，将结果回传给模型继续推理。

**执行流程图**：
```text
┌─────────┐   1. Query+Tool_Schemas   ┌───────────┐
│   User  │ ──────────────────────> │   LLM     │
└─────────┘                         └─────┬─────┘
                                          │
                                    2. Generate Tool Call
                                    (JSON: name, args)
                                          │
                                          ▼
┌─────────┐   3. Execute Function    ┌───────────┐
│  Host   │ <────────────────────── │ Executor  │
│  (App)  │   ─────────────────────> │ (Function)│
└────┬────┘   4. Return Result       └───────────┘
     │
     │ 5. Inject Result into Context
     ▼
┌─────────┐   6. Final Answer       ┌───────────┐
│   User  │ <────────────────────── │   LLM     │
└─────────┘                         └───────────┘
```

**最佳实践**：
1.  **描述**：是模型选择工具的关键。应包含功能、何时用、何时不用的示例。使用 `Few-Shot` 示例显著提升准确率。
2.  **参数设计**：少而精，优先枚举，避免自由文本，日期统一 ISO 格式。参数名应具有自解释性。
3.  **返回值**：优先 JSON，超大结果需摘要或分页，错误信息结构化。返回给模型的内容应经过压缩，去除无关 HTML/噪音。
4.  **错误处理**：区分可重试（429、5xx）与不可重试（4xx）错误，敏感操作需人工确认。

**面试金句**：
*   **Q：Tool 与普通函数的区别？**
*   **A**：Tool 多了面向模型的元数据（描述、Schema）和统一执行接口（日志、超时、权限）。普通函数直接由代码逻辑调用，Tool 由 LLM “意念” 驱动调用。
*   **Q：工具返回 10MB 日志怎么办？**
*   **A**：不应直接返回。应摘要、写入对象存储返回链接、或提供 grep 工具缩小范围。直接塞入上下文会导致 Token 爆炸且淹没注意力。

### 代码示例
```python
from langchain_core.tools import tool

@tool
def search_product(query: str, category: str = "") -> str:
    """在电商站内搜索商品。用户要找商品、比价时用；不要用于闲聊。
    Args:
        query: 搜索关键词
        category: 可选类目
    """
    return f"Results for {query} in {category}" 
```

## 常见考点
1.  **并行调用**：当模型返回多个 `tool_calls` 时，如何识别无依赖关系并进行并行执行以降低延迟？
2.  **Tool Choice 参数**：`auto`（模型决定）、`none`（强制不调用）、`{"type": "function", "function": {"name": "my_tool"}}`（强制调用特定工具）的使用场景。
3.  **流式处理**：在 Stream 模式下，如何解析增量生成的 Tool Call 参数（如 `arguments_fragment` 拼接）。
