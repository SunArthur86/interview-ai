# Reflexion 机制在 Agent 中是如何工作的？它与简单的 Self-Correction 有什么区别？

Reflexion 是一种赋予 Agent 自我反思能力的框架，旨在通过动态的“反思-修正”循环来提升任务成功率。

**工作原理**：
1. Agent 尝试执行任务并获得结果（包括 Reward 或 Success/Fail 信号）。
2. **自我评估**：如果任务失败，Agent 会生成一段“反思文本”，描述错误的原因（如“计算错误”或“参数缺失”）。
3. **记忆更新**：这段反思文本被存储在 Agent 的长期记忆（ episodic memory）中。
4. **重试**：在下次尝试同一任务时，Agent 会检索相关的反思内容作为上下文，从而避免重蹈覆辙。

**与简单 Self-Correction 的区别**：简单的 Self-Correction 通常是在同一次上下文窗口内进行的，模型意识到错误后立即尝试修改，容易受限于上下文长度和短期记忆。Reflexion 则将反思持久化，形成了跨任务的长期学习机制，利用历史经验来指导未来的行为，更适合解决复杂的、需要多次试错的推理任务。
