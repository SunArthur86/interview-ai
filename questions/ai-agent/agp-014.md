# Function Calling 与「输出 JSON」二选一

Function Calling 与「输出 JSON」二选一

看生态与框架；Function Calling 强在动作空间清晰；纯 JSON 适合简单结构化且无工具场景。可混用。

**关键细节与原理**：
1. **Function Calling (FC)**：
   - **本质**：模型并非直接执行代码，而是输出一个表示“调用某个函数”的特定结构化对象（通常由模型微调过以识别该模式）。
   - **优势**：模型经过训练，能更精准地理解何时该调用工具、调用哪个工具以及参数是什么。它是一种“显式的意图触发”。
   - **局限性**：受限于平台支持的 Function 定义数量，且无法输出非函数定义的任意结构（尽管通常可以用 detail 描述）。
2. **纯 JSON Mode**：
   - **本质**：模型仅输出符合 JSON 格式的文本，不强制关联函数执行语义。
   - **优势**：极度灵活，适合提取实体、生成配置文件等数据转换任务，不依赖外部工具定义。
   - **劣势**：模型可能“幻觉”出不存在的字段，或者格式不够严格。
3. **混用场景**：使用 Function Calling 触发核心业务逻辑（如查库），在返回结果的描述部分或者非工具类的输出要求上，配合 JSON Mode 确保数据结构化。

**实战案例**：
在做客服工单提取时，初期用 JSON Mode 让模型直接抽取字段，结果经常将“明天上午”这种模糊时间幻觉成具体的 `"time": "10:00"`；后来改用 Function Calling，将时间作为 `string` 参数传给专门的时间解析工具，解决了合规性问题。

**代码示例**：
```python
# Python: 定义 Function Calling 的 Tools Schema
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名称"},
                    "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
                },
                "required": ["city"]
            }
        }
    }
]

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "北京今天热吗？"}],
    tools=tools,
    tool_choice="auto" # 允许模型自主决定是否调用
)
```

**对比表格**：

| 维度 | Function Calling | 纯 JSON Mode / JSON Schema |
| :--- | :--- | :--- |
| **核心目的** | 触发外部动作/工具调用 | 数据结构化提取/格式化输出 |
| **输出内容** | 函数名 + 参数对象 | 任意定义的 JSON 结构 |
| **语义理解** | 模型理解“何时调用” | 仅理解“数据格式”，无动作语义 |
| **灵活性** | 受限于 API 定义的 Function 列表 | 极高，可定义任意深度的嵌套 JSON |
| **错误处理** | API 层面强制校验参数类型 | 依赖 Prompt 约束，易产生格式错误 |

**常见考点**：
1. 如果用户的问题不需要调用工具，Function Calling 模型会输出什么？（通常会输出一个特定的字段如 `null` 或直接返回文本回复，视 API 规范而定）
2. 如何处理 Function Calling 参数类型校验失败的情况？（拦截错误并反馈给模型重试，或者由客户端填默认值）
3. Function Calling 的参数是放在 System Prompt 还是 Messages 中？（通常通过 API 参数传递，模型内部会将其映射为特殊的系统指令，不需用户手动拼 Prompt）
