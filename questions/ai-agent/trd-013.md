# 什么是 Human-in-the-loop (HITL) 的审批模式？在 Agent 应用中如何实现？

Human-in-the-loop（人机回环）审批模式是指在 Agent 执行关键或高风险操作前，暂停流程，请求人类介入确认，只有在获得批准后才继续执行。这是解决 Agent 幻觉和不可控性的重要安全机制。

**实现方式**：
1. **状态挂起**：当 Agent 决策需要调用“高风险工具”（如“发送邮件”、“删除数据库”、“转账”）时，不立即执行。
2. **信号发送**：Agent 返回一个特殊的响应结构，包含需要人类审批的动作内容（如：`action: "send_email"`, `status: "pending_approval"`, `content: "..."`）。
3. **前端交互**：客户端识别到该状态，弹出一个确认对话框或 UI 组件，供用户点击“同意”或“拒绝”。
4. **回调恢复**：用户操作后，前端将结果通过 API 回传给后端。Agent 根据“同意”信号执行工具，或根据“拒绝”信号重新规划路径。

**应用场景**：金融交易、代码部署、自动化营销等容错率低的场景。
