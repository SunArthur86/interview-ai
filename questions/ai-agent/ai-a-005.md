# 多Agent系统(MAS)的常见协作模式有哪些?如何设计角色分工

- **AutoGPT架构:**

```
目标输入 → Agent循环 {
Thought → Reasoning → Plan → Action → Observation
} → 目标完成
```

- **核心组件:**
1. **AI(LLM)** - 大脑,做推理和决策
2. **Memory** - 向量数据库存储历史
3. **Tools** - 搜索、文件、代码执行等
4. **Workspace** - 持久化文件/状态

- **死循环原因:**
1. **目标模糊** - Agent不知道何时算完成
2. **工具失败** - 工具返回错误,Agent反复重试
3. **上下文遗忘** - 长对话后忘记原始目标
4. **决策僵化** - 每次推理路径太相似

- **解决方案:**
1. **最大迭代数限制** - 硬性终止
2. **完成检测器** - 专门的判断是否完成的机制
3. **反思机制** - 每N步反思进度
4. **温度调节** - 提高采样多样性避免重复
5. **状态压缩** - 定期总结中间结果

- **补充细节：**
  - **内存管理**：采用长短时记忆结合，向量检索相关历史减少输入。
  - **自我反思**：引入“Critic”角色，对执行结果打分，决定是继续还是结束。
