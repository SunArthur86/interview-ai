# Reflexion 框架

### 1. 概念解释
Reflexion 框架的核心在于“反思”。它不只是尝试，而是在每次失败后生成语言化的反思文本，作为记忆存储，指导下一次尝试。这是一个典型的**试错-反思-改进**的强化循环。

### 2. 核心机制
- **Actor (智能体)**：执行任务，生成输出或动作。
- **Evaluator (评估器)**：评估结果（如单元测试结果、人类打分、Self-Check）。
- **Reflector (反思器)**：根据评估反馈生成具体的、语言化的反思文本。

### 3. 工作流与数据流

```text
┌──────────────────────────────────────────────────────────┐
│                       Task Input                         │
└─────────────────────┬────────────────────────────────────┘
                      ▼
         ┌────────────────────────────┐
         │         Actor (LLM)        │
         │   (Prompt + History)       │
         └─────────────┬──────────────┘
                       ▼
               ┌───────────────┐
               │    Output    │
               └───────┬───────┘
                       ▼
         ┌────────────────────────────┐
         │      Evaluator             │
         │  (Unit Test / Heuristic)   │
         └─────────────┬──────────────┘
                       │
            ┌──────────┴──────────┐
            ▼                     ▼
      (Success)              (Failure)
            │                     │
            ▼                     ▼
      ┌─────────┐      ┌──────────────────────┐
      │  End    │      │    Reflector (LLM)    │
      └─────────┘      │  (Input: Error Trace) │
                       └──────────┬───────────┘
                                  ▼
                        ┌──────────────────────┐
                        │  Reflection Text     │
                        │  (e.g., "Forgot to   │
                        │   convert unit")    │
                        └──────────┬───────────┘
                                   │
                                   ▼ (Append to Actor Context)
                        ┌──────────────────────┐
                        │    Actor (Retry)     │◀─────┐
                        └──────────────────────┘      │
                                   │                   │
                                   └───────────────────┘
```

### 4. 适用场景
- **编程任务**：编译报错或测试用例失败能提供精确的 Feedback。
- **数学/逻辑推理**：答案可验证，容易发现推理漏洞。
- 需多轮试错且易犯系统性错误（如总是忽略某个约束条件）的场景。

#### ## 常见考点
1. **反思内容**：Reflector 生成的反思文本应包含哪些要素才能有效指导 Actor 的下一次尝试？
2. **记忆管理**：如果任务多次重试仍失败，如何避免反思文本无限堆积导致 Context 溢出？
3. **评估准确性**：Evaluator 如果误判（例如测试用例本身有误），会导致什么后果？如何缓解？
