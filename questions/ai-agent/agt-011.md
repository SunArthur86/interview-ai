# Reflexion 框架

### 1. 概念解释
Reflexion 框架的核心在于“反思”。它不只是尝试，而是在每次失败后生成语言化的反思文本，作为记忆存储，指导下一次尝试。这是一个典型的**试错-反思-改进**的强化循环。

### 2. 核心机制
- **Actor (智能体)**：执行任务，生成输出或动作。
- **Evaluator (评估器)**：评估结果（如单元测试结果、人类打分、Self-Check）。
- **Reflector (反思器)**：根据评估反馈生成具体的、语言化的反思文本。

### 3. 工作流与数据流

```text
┌──────────────────────────────────────────────────────────┐
│                       Task Input                         │
└─────────────────────┬────────────────────────────────────┘
                      ▼
         ┌────────────────────────────┐
         │         Actor (LLM)        │
         │   (Prompt + History)       │
         └─────────────┬──────────────┘
                       ▼
               ┌───────────────┐
               │    Output    │
               └───────┬───────┘
                       ▼
         ┌────────────────────────────┐
         │      Evaluator             │
         │  (Unit Test / Heuristic)   │
         └─────────────┬──────────────┘
                       │
            ┌──────────┴──────────┐
            ▼                     ▼
      (Success)              (Failure)
            │                     │
            ▼                     ▼
      ┌─────────┐      ┌──────────────────────┐
      │  End    │      │    Reflector (LLM)    │
      └─────────┘      │  (Input: Error Trace) │
                       └──────────┬───────────┘
                                  ▼
                        ┌──────────────────────┐
                        │  Reflection Text     │
                        │  (e.g., "Forgot to   │
                        │   convert unit")    │
                        └──────────┬───────────┘
                                   │
                                   ▼ (Append to Actor Context)
                        ┌──────────────────────┐
                        │    Actor (Retry)     │◀─────┐
                        └──────────────────────┘      │
                                   │                   │
                                   └───────────────────┘
```

### 4. 实战深化
#### 4.1 实战案例
在实际的 Python 数据分析任务中，Agent 可能会因为 Pandas 的 `SettingWithCopyWarning` 而生成逻辑正确但运行报错的代码。Reflexion 的 Evaluator 捕获堆栈信息后，Reflector 不会只报错，而是会生成具体的反思建议：“建议在修改切片数据前显式使用 `.copy()`”，从而在第二轮直接修正，比简单的重试效率高很多。

#### 4.2 关键代码示例

```python
# Python 伪代码：Reflector 生成反思并更新上下文
if not evaluator.check_result(actor_output):
    error_trace = get_error_trace(actor_output)
    # Reflector 是一个独立的 LLM 调用，专注于分析错误
    reflection_text = reflector_llm.invoke(
        f"Task failed. Error: {error_trace}. "
        f"What should be done differently next time? Be specific."
    )
    # 将反思文本作为记忆存入 Actor 的 Prompt 历史中
    actor_memory.append(f"[PREVIOUS REFLECTION]: {reflection_text}")
    # Actor 重试
    new_output = actor.invoke(task_input, memory=actor_memory)
```
