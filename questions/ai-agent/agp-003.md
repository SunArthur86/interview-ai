# 你如何迭代优化 Prompt

(1) **建立固定评测集**：构建包含典型与边缘案例的 Golden Dataset，并制定量化评分标准（如准确率、格式合规率）；(2) **错误分类分析**：将失败日志归类（理解错误、知识幻觉、格式错误、安全合规问题）；(3) **控制变量迭代**：采用“一次改一个变量”的策略，如单独修改指令、更换 Few-shot 示例或调整分隔符；(4) **数据驱动记录**：记录每个 Prompt 版本的改动内容及对应的评测指标，避免凭直觉调整导致效果波动。

**实战案例**：优化一个电商情感分析 Agent 时，发现“讽刺”识别率极低。通过分析 Bad Cases，发现模型将“这手机真‘好用’，一天充三次电”误判为正面。在 Prompt 中加入“反语/反讽识别模式”的 Few-shot 示例后，该类别 F1 分数从 0.2 提升至 0.85。

**代码示例**：
```python
# 使用框架记录迭代实验（如 LangSmith 或自建）
import json

experiment_log = {
    "version": "v4.1",
    "change_desc": "增加反语识别 Few-shot 示例",
    "metrics": {
        "accuracy": 0.92,
        "sarcasm_recall": 0.85 # 关注特定指标提升
    },
    "prompt_template": system_prompt_v4 # 存储完整快照
}

with open("prompt_experiments.jsonl", "a") as f:
    f.write(json.dumps(experiment_log) + "\n")
```

**优化闭环流程：**
```text
     ┌─────────────┐
     │  现有 Prompt │
     └──────┬──────┘
            │
            ▼
     ┌─────────────┐
     │  运行评测集  │───────┐
     └──────┬──────┘       │ (获取 Bad Cases)
            │             │
            ▼             │
     ┌─────────────┐       │
     │  错误归因分析 │◄──────┘
     └──────┬──────┘
            │
            ▼ (假设生成: 改指令/换样例/加CoT)
     ┌─────────────┐
     │   修改 Prompt │
     └──────┬──────┘
            │
            ▼ (回归测试)
     ┌─────────────┐       是
     │ 指标提升?   ├────────┤ (发布) 
     └──────┬──────┘       │ 否 (回滚/重试)
            │             │
            └─────────────┘
```

## 边界情况
1. **灾难性遗忘**：在针对某一类 Bad Case（如讽刺识别）优化时，可能会破坏模型在其他常见类别（如简单正面评价）上的表现，需每次迭代都运行全量回归测试。
2. **数据分布漂移**：随着时间推移，线上真实用户的提问方式可能发生变化（如出现新梗），导致旧的 Golden Dataset 失效，需定期注入最新的线上 Case 到评测集。
3. **模型版本升级**：当底层 LLM 升级（如从 GPT-4-Turbo 到 GPT-4O）时，原有的“最优 Prompt”可能不再是最优，甚至由于新模型对指令理解更敏感而产生过拟合，需重新评估。

## 面试追问
1. **过拟合问题**：如果 Prompt 在固定评测集上表现很好，但上线效果差怎么办？（评测集需定期更新，增加真实流量分布的数据，使用留出集验证）。
2. **CoT 触发**：什么时候引入 CoT（思维链）？（逻辑推理复杂任务，简单任务引入反而可能降低准确率并增加延迟）。
3. **评估指标**：如何评估主观任务（如创意写作）？（使用 LLM-as-a-Judge 进行打分，或设计基于细粒度维度的评分表）。

## 易错点
1. **盲目堆砌指令**：发现一个 Case 做错了，就直接在 Prompt 里加一句针对性指令，导致 Prompt 臃肿不堪、指令权重失衡，最终模型“不知道听谁的”。
2. **忽视温度参数**：在优化 Prompt 时未固定 Temperature 等生成参数，导致无法区分效果提升是源于 Prompt 改进还是源于模型采样的随机性。
