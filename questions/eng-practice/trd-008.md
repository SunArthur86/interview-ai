# 在 AI Agent 工程化中，如何优化 Token 成本？有哪些具体的压缩策略？

Agent 往往涉及长上下文和循环对话，Token 消耗巨大。优化策略主要分为“压缩输入”和“控制输出”两部分。

**压缩输入策略**：
1. **上下文窗口滑动与裁剪**：保留最近的 N 条消息和系统 Prompt，丢弃老旧对话。对于 Agent，可以使用摘要策略，将多轮的历史 Tool 执行结果总结为一句话。
2. **系统 Prompt 极简化**：移除冗余的角色描述，使用更紧凑的指令。
3. **工具结果清洗**：许多工具（如搜索引擎）返回大量无关 HTML 或广告，必须在注入 Prompt 前进行清洗，只保留核心语义内容。
4. **低精度向量检索**：在 RAG 环节，虽然不直接省 Prompt Token，但通过精准检索减少注入的文档块数量。

**控制输出策略**：
1. **最大步数限制**：限制 ReAct 循环的最大迭代次数，防止陷入死循环消耗 Token。
2. **强制停止**：一旦获得最终答案，立即截断生成，防止模型废话连篇。
