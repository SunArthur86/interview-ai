# 怎么做的数据预处理和数据清洗

**Situation：** 企业文档质量参差不齐：有 OCR 错误、格式混乱、重复内容、过时信息等。低质量数据直接影响检索和生成质量。

**Task：** 建立数据预处理管道，确保入库数据的质量。

**Action：** 
1. **数据清洗管道：**
   原始文档 → 格式转换 → 文本提取 → 清洗 → 去重 → 质量评估 → 分块 → 入库

2. **清洗规则：**
   - **去除噪音：** 页眉页脚、水印文字、无意义空白。
   - **格式标准化：** 统一编码（UTF-8）、统一标点（全角/半角）。
   - **OCR 纠错：** 基于语言模型的拼写纠错（如“己术”→“技术”）。
   - **敏感信息脱敏：** 手机号、身份证号等自动打码。

3. **去重策略：**
   - **完全重复：** MD5 hash 去重。
   - **近似重复：** MinHash + LSH 算法，Jaccard 相似度 > 0.85 视为重复。
   - **跨版本去重：** 同一文档的不同版本，只保留最新版本。

4. **质量评估打分：**
   - 文本长度（太短的 chunk 信息量不足）。
   - 语义完整性（句子是否完整）。
   - 信息密度（关键词密度、实体密度）。
   - 分数低于阈值的 chunk 标记为低质量，不参与检索。

**Result：** 
- 数据清洗后，检索准确率提升 8%。
- 去重减少了 15% 的存储空间。
- 低质量 chunk 过滤后，LLM 生成的幻觉率降低 12%。

### 深化补充

**实战案例：**
在处理 PDF 合同时，发现正则表达式提取的表格数据经常错位（因为合并单元格的存在）。引入了 `pdfplumber` 的可视化调试功能，通过分析 PDF 的布局对象，自定义了基于坐标的表格提取逻辑，解决了复杂表格结构丢失的问题，将表格数据的结构化准确率从 60% 提升至 95%。

**代码示例（Python - 使用 Datasketch 进行近似去重）：**
```python
from datasketch import MinHashLSH, MinHash

data = ["doc text 1...", "doc text 2...", "doc text 3..."]

# 创建 LSH 索引
lsh = MinHashLSH(threshold=0.85, num_perm=128)

for i, text in enumerate(data):
    # 创建 MinHash 对象
    m = MinHash(num_perm=128)
    for d in text.split(' '):
        m.update(d.encode('utf8'))
    
    # 查询是否已存在相似文档
    if len(lsh.query(m)) == 0:
        lsh.insert(i, m)
        print(f"Document {i} is unique.")
    else:
        print(f"Document {i} is duplicate of {lsh.query(m)}")
```

**对比表格（固定大小分块 vs 语义分块）：**

| 维度 | 固定大小分块 | 语义分块 |
| :--- | :--- | :--- |
| **切分依据** | 字符数或 Token 数硬切分 | 句子边界、段落结构或语义向量相似度 |
| **上下文完整性** | 可能截断句子或逻辑，破坏语义 | 保持语义单元完整，但 chunk 长度差异大 |
| **检索效果** | 简单快速，但可能召回不相关碎片 | 召回更精准，但计算开销较大 |
| **适用场景** | 通用性强，作为 Baseline | 对逻辑连贯性要求高的问答系统 |

## 常见考点
1. **分块策略与清洗结合**：清洗后的文本直接按固定长度切分可能会破坏语义，如何优化？（答：在清洗后引入语义分块，如基于递归字符切分或语义断句，确保 Chunk 边界完整）
2. **表格数据处理**：文档中的表格提取后往往变成乱序文本，如何处理？（答：使用专门的表格解析工具（如 Table-Transformer）保留行列结构，或转换为 HTML/Markdown 格式作为上下文）
3. **增量更新清洗**：源文档发生微小修改时，是否需要全量重新清洗和 Embedding？（答：监控文档指纹（MD5/ETag），仅对变更的文档部分进行局部重处理，但需注意上下文关联块的重新索引）

## 记忆要点

- 清洗管道：去噪、格式统一、OCR 纠错、敏感信息脱敏。
- 去重策略：MD5 去完全重复，MinHash+LSH 去近似重复。
- 质量评估：基于长度、完整性、信息密度打分，过滤低质量 Chunk。
- 分块优化：语义分块优于固定切分，保持上下文逻辑完整。
