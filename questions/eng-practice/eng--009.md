# 微调后指标提升,但线上回复风格变差,你怎么排查和解决

- **典型场景**: 离线 BLEU/accuracy 提升，但用户反馈回复'生硬'、'格式乱'、'不自然'。这通常是 **Reward Hacking** 或 **过拟合** 导致的模型对齐退化。

- **排查步骤**

  1. **定位变化范围**
     - **人工抽检**: 对比微调前后的输出（同一 prompt），重点看边缘案例。
     - **定量分析**: 
       - 统计平均回复长度变化（是否变短或变长导致截断）。
       - 计算词汇多样性（Distinct-N），检查是否复读机。
       - 分析困惑度分布，是否出现极高的困惑度峰值。
     - **Log 分析**: 检查训练后期 Loss 是否震荡（可能 Learning Rate 过大）。

  2. **检查训练数据** - **最常见原因**
     - **风格不一致**: 微调数据是否混入不同风格的语料（如正式文档 vs 聊天记录）。
     - **长尾分布**: 是否包含大量短回答（导致模型变话痨）或极短回答（导致模型变惜字如金）。
     - **数据污染**: 是否存在 JSON 解析错误、特殊字符未清洗（导致格式乱码）。
     - **标注质量**: 人工标注数据是否存在幻觉或逻辑错误。

  3. **检查训练超参**
     - **过拟合**: 验证集 Loss 持续上升，训练集 Loss 下降。→ 降低模型容量、增加 Dropout、减少 Epoch。
     - **灾难遗忘**: 通用 NLU 能力（如分类、实体识别）下降。→ 降低学习率、使用正则化、混合预训练数据。
     - **学习率**: Warmup 太短或峰值太高，导致破坏预训练权重。
     - **权重衰减**: L2 正则化是否过大，导致模型过度保守。

  4. **检查评测指标**
     - **指标缺陷**: BLEU/ROUGE 基于词重叠，无法反映语义通顺度；Accuracy 无法反映风格。
     - **对齐偏差**: 离线评测集分布可能与真实用户 Query 分布不一致（Out-of-Distribution）。
     - **改进评测**: 引入基于 LLM 的打分（如 GPT-4 作为 Judge）或人工满意度评分，侧重“帮助性”、“安全性”、“语气”。

- **解决方案**
  1. **数据清洗与配比**: 
     - 剔除风格不符的数据。
     - **混合数据策略**: 在微调数据中混入 10%-20% 的通用预训练语料，缓解遗忘。
  2. **算法层面**: 
     - **LoRA 替代 FFT**: 只训练少量参数，最大程度保留原模型的通用能力。
     - **DPO (Direct Preference Optimization)**: 如果 SFT 导致风格变差，使用偏好数据（好回答 vs 坏回答）进行对齐，比 SFT 更能精准优化风格。
  3. **训练策略**: 
     - **两阶段训练**: 第一阶段只训任务能力（如写代码），第二阶段只训风格（如语气助词、格式），避免冲突。
     - **学习率退火**: 最后 10% steps 使用线性衰减余量，稳定权重。
  4. **发布保障**: 
     - **在线 A/B Testing**: 灰度发布，实时监控点赞率、完读率、截断率。
     - **安全护栏**: 增加后处理规则（如强制去重、长度限制），兜底模型输出。

**实战案例**: 某金融客服模型微调后，虽然意图识别准确率提升，但回复变得过于简短且缺乏礼貌用语。排查发现 SFT 数据中 80% 为简短的 KBQA (问答对)，缺少对话上下文。**解决**: 引入 DPO，人工构造“简短回答 vs 详尽礼貌回答”的偏好对，训练后模型风格显著改善，且保留了准确率。

**代码示例 (DPO 风格修复)**:
```python
# 使用偏好对数据进行 DPO 训练片段
from trl import DPOTrainer

dpo_dataset = [
    {
        "prompt": "怎么查余额？",
        "chosen": "您可以通过APP首页点击【我的】查看余额。", # 有礼貌、有指引
        "rejected": "点我的看。" # 过于简短
    }
]

trainer = DPOTrainer(
    model, ref_model, args=training_args, 
    beta=0.1, # 控制偏离参考模型的程度
    train_dataset=dpo_dataset
)
trainer.train()
```
