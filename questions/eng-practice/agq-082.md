# LoRA 的原理是什么?为什么低秩有效

**LoRA 核心思想：**
在微调时不修改原始模型的权重 $W$，而是添加一个低秩分解的增量 $\Delta W$。
\[ W' = W + \Delta W = W + B \cdot A \]
其中 $W \in \mathbb{R}^{d \times k}$， $B \in \mathbb{R}^{d \times r}$， $A \in \mathbb{R}^{r \times k}$， $r \ll \min(d, k)$。

**参数量对比：** 例如 $d=4096, r=8$ 时，参数量从 $4096 \times 4096 = 16.7\text{M}$ 降到 $2 \times 4096 \times 8 = 65\text{K}$ (减少 99.6%)。

**为什么低秩有效？**
- **内在维度假说:** 预训练模型在微调时，参数变化实际上存在于一个低维子空间中。
- 实验表明，大模型微调的权重变化矩阵的有效秩远小于其原始维度。
- 现实任务通常是特定的下游任务，不需要调整模型所有的特征方向，只需在特定方向上调整即可。

**LoRA 实现细节：**
- $A$ 矩阵用随机高斯初始化，$B$ 矩阵初始化为零 (保证训练开始时 $\Delta W=0$，即模型初始状态与预训练模型完全一致)。
- **缩放系数**：在推理时，通常将 $\Delta W$ 乘以一个缩放系数 $\frac{\alpha}{r}$，其中 $\alpha$ 是超参数（通常设为 16 或 32），$r$ 是秩。这样在调整 $r$ 时无需重新调整学习率。
- 应用位置：实践中只在 Attention 的 $Q$、$V$ 投影矩阵上应用 LoRA 效果最好，也有方案尝试在 MLP 层或全层应用。
- 推理优化：可以将 $B \cdot A$ 合并到 $W$ 中 ($W_{new} = W + \frac{\alpha}{r}BA$)，推理架构无需改变，**零额外延迟**。

```text
           原始路径              LoRA 增量路径
      Input(x) ───► [ W ] ────┐
                            │  (+) ──► Output
             Input(x) ───► [ B·A ] ──┘
                            ▲
                     (训练时更新 B 和 A)
```

**与其他方法对比：**

| 方法 | 参数量 | 显存占用 | 推理延迟 | 训练稳定性 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Full Fine-tuning** | 100% | 极高 | 无 | 极高 | 资源充足，追求极致效果 |
| **LoRA** | < 1% | 低 | **零延迟** (合并后) | 高 | 通用微调，性价比首选 |
| **Adapter** | 3-5% | 中 | **有增加** (插入层) | 中 | 老旧框架，不支持权重合并 |
| **Prompt Tuning** | < 0.1% | 最低 | 无 | 低 (难收敛) | 极端资源受限，多任务切换 |
| **QLoRA** | < 1% | **极低** (4-bit) | 无 (需解压) | 中 | 单卡微调大模型 (如 65B) |

**QLoRA：** 结合量化 (4-bit NormalFloat) 和 LoRA。使用双重量化、分页优化器等技术，进一步降低显存需求（如在 48GB 显存上微调 65B 模型），使得在消费级 GPU 上微调大模型成为可能。

**实战案例**：我们在医疗领域微调 70B 模型时，原本需要 8 张 A800（80G）做全量微调。切换到 QLoRA 后，仅用 2 张 A800 就跑起来了。但在合并权重时发现，如果将 $r$ 设为 64 以上，模型在医学实体识别上反而出现轻微退化，说明过高的秩带来了冗余噪声。

**代码示例 (LoRA Layer - PyTorch)**：
```python
class LoRALayer(nn.Module):
    def __init__(self, in_dim, out_dim, rank=8, alpha=16):
        super().__init__()
        self.lora_A = nn.Parameter(torch.randn(rank, in_dim))
        self.lora_B = nn.Parameter(torch.zeros(out_dim, rank))
        self.scaling = alpha / rank

    def forward(self, x):
        # 原始权重 W 通常在外部线性层处理，这里只计算增量
        return (x @ self.lora_A.T @ self.lora_B.T) * self.scaling
```

**## 常见考点**
1. **初始化策略**：为什么 $B$ 初始化为 0 而 $A$ 随机初始化？
   - *答案*：为了确保训练初始时刻 $\Delta W = 0$，使模型完全等同于预训练模型，保证训练的稳定性。
2. **缩放因子 $\alpha$**：$\alpha$ 的作用是什么？
   - *答案*：$\alpha$ 用于控制更新的步幅。实际更新量为 $\frac{\alpha}{r} x B A$。调整 $r$ 时保持 $\alpha$ 不变可以近似保持更新量级稳定，无需重调学习率。
3. **推理合并**：如何将 LoRA 权重合并？
   - *答案*：$W_{final} = W_{frozen} + (\frac{\alpha}{r} \times B \times A)$，这是矩阵加法，对推理框架透明。
4. **秩 $r$ 的选择**：$r$ 是不是越大越好？
   - *答案*：不是。实验发现 $r$ 在 4-16 之间往往足够，过大的 $r$ 可能导致过拟合且增加计算量，但提升微乎其微。

## 记忆要点

- 核心原理：冻结原权重 W，增加低秩分解 BA，W' = W + BA
- 参数量：r<<d，参数减少 99% 以上，显存大幅降低
- 初始化：A 随机高斯，B 初始化为零，保证训练初始状态一致
- 推理优势：BA 可合并入 W，实现零额外推理延迟
