# 模型幻觉怎么控制的

**Situation：** LLM 在没有足够信息时会"编造"内容(幻觉),在企业场景中这是严重的问题(如编造不存在的产品功能或价格).

**Task：** 多层次控制模型幻觉,确保回答基于事实.

**Action：** 
1. **RAG 基础(最重要):**
   - 所有回答都基于检索到的知识片段,要求 LLM 只使用提供的上下文信息.
   - Prompt 中明确声明:"只根据以下参考信息回答,如果信息不足,请明确说明无法回答."

2. **引用机制:**
   - 要求 LLM 在回答中标注引用来源:[参考1] 、[参考2] .
   - 后处理检查引用是否真实存在于检索结果中.
   - 没有引用的事实性陈述标记为"待验证".

3. **幻觉检测(输出侧):**
   - **NLI(自然语言推断)检查:** 检查回答中的每个关键陈述是否能从 context 中推断出来.
   - **关键实体核验:** 提取回答中的数字、日期、名称等关键实体,与 context 对比.
   - **自我一致性检查:** 让 LLM 对同一问题生成 3 次回答,如果答案不一致,标记为低置信度.

4. **降级策略:**
   - 检索结果相关性低(rerank score < 0.4) → 回复"知识库中暂无相关信息".
   - 检测到可能的幻觉 → 在回答中添加提示"该信息仅供参考,建议核实".

5. **温度控制:**
   - 事实性回答场景 temperature=0(确定性最高).
   - 创意性场景 temperature=0.7.

**RAG 幻觉防护全流程：**
```text
User Query
     │
     ▼
[Retrieval] ──> Context Chunks (Knowledge Base)
     │                   │
     ▼                   ▼
[Generation] <─── Prompt Instruction ("Only use context")
     │
     ├── Output Text
     │
     ▼
[Post-Hoc Verification Layer]
     │
     ├── 1. Citation Check: Are [Ref IDs] valid? ──┐
     ├── 2. NLI Check: Does Answer ⊇ Context?      │(If Fail)
     └── 3. Entity Extraction & Match               │
          │                                        │
          ▼                                        │
     (High Confidence)                             │
          │                                        │
          └────────────────────────────────────────┘
                   │
                   ▼
            Return to User
```

**Result：** 
- 幻觉率从无防控的 18% 降低到 3%.
- 关键业务场景(报价、合同条款等)的幻觉率 < 0.5%.
- 用户对回答准确性的信任度提升 35%.

**实战案例**：医疗问答系统中，模型曾将未收录的新药名称混淆为旧药并编造用法。后引入基于知识图谱的实体核验层，若回答中的药物名未在图谱中，则强制拒绝回答，消除了此类医疗合规风险。

**代码示例**：
```python
# 事实性Prompt模板示例
SYSTEM_PROMPT = """
你是一个严谨的助手。请仅基于以下 [Context] 回答问题。
如果 [Context] 中没有答案，请直接回答“我不知道”，严禁编造。

[Context]:
{context_str}

[Question]:
{query}
"""
```

**常见考点**
1. **检索不到时的处理**：如果知识库里真没有答案，直接让模型编造还是拒绝？（答案：坚决拒绝，Prompt 中需强化“若未找到相关信息请回答不知道”）
2. **NLI 成本**：对每句话做 NLI 验证会增加一次模型调用，如何优化？（答案：仅对关键实体、数字或高风险场景进行验证，或使用小模型）
3. **Prompt Engineering 细节**：如何让模型严格遵守引用格式？（答案：使用 Few-shot 示例，并在 System Prompt 中强约束）
4. **错误传播**：如果检索源本身就是错误的（脏数据），RAG 能解决幻觉吗？（答案：不能，RAG 依赖于数据质量，数据清洗是基础）

## 记忆要点

- RAG基础约束：Prompt明确要求仅基于上下文回答，无信息时直接拒绝回答。
- 引用机制溯源：要求回答标注引用ID，后处理验证引用真实性，无引用标记待验证。
- 输出侧检测：使用NLI推断检查、关键实体核验或自我一致性检查发现幻觉。
- 参数与降级：事实性回答设Temperature为0，检索相关性低时直接降级拒答。
