# Token 计数和费用统计是怎么实现的

**Situation：** LLM API 按 token 计费，需要精确统计每次调用的 token 消耗，用于成本控制和计费。

**Task：** 实现精确的 token 计数和费用统计系统。

**Action：** 
1. **Token 计数方案：**
   - **输入 Token：** 使用对应模型的 tokenizer 在客户端预计算。
     - OpenAI：使用 `tiktoken` 库。
     - Claude：使用 Anthropic 的 token counter API。
     - 本地模型：使用 HuggingFace tokenizer。
   - **输出 Token：** 从 API 响应的 `usage` 字段获取（最准确）。

2. **费用计算：**
   - 按模型的百万 token 价格进行计算。

3. **统计维度：**
   - **按租户统计：** 每个客户的 token 消耗和费用。
   - **按功能统计：** 意图识别、RAG 生成、摘要等各模块的消耗。
   - **按模型统计：** 不同模型的使用量和费用占比。
   - **按时间统计：** 日/周/月的消耗趋势。

4. **预算告警：**
   - 每个租户设置月度 token 预算。
   - 消耗达到 80% 时提醒，100% 时限制（或降级到小模型）。

**Result：** 
- Token 计数误差控制在 2% 以内。
- 月度成本报告自动生成，精确到每个功能模块。
- 通过分析发现“会话摘要”模块是 token 消耗大户（占 25%），优化后降低了 40%。

### 深化补充

**实战案例：**
在统计“输入 Token”时，发现直接使用 `tiktoken` 计算长文本（>32k）的速度较慢，阻塞了请求处理。优化方案是：先对文本进行快速 SHA256 Hash 缓存查询，如果文本内容未变化（如重试请求），直接读取缓存的 Token 数，避免了重复计算 CPU 密集型的分词操作。

**代码示例（Python）：**
```python
import tiktoken

class TokenCounter:
    def __init__(self, model_name="gpt-4o"):
        self.encoder = tiktoken.encoding_for_model(model_name)

    def count_tokens(self, text: str) -> int:
        # tiktoken 计算逻辑
        return len(self.encoder.encode(text))

    def calculate_cost(self, input_tokens, output_tokens):
        # 价格字典 (示例)
        pricing = {"input": 2.5, "output": 10.0} # per 1M
        cost = (input_tokens * pricing["input"] + output_tokens * pricing["output"]) / 1_000_000
        return cost
```

**对比表格（客户端估算 vs 服务端计费）：**

| 维度 | 客户端估算 | 服务端计费 |
| :--- | :--- | :--- |
| **数据来源** | 本地 Tokenizer 库 | API 响应 `usage` 字段 |
| **准确度** | 高，但可能与微调后的服务器分词器有细微差异 | 100% 准确（作为账单依据） |
| **用途** | 实时预算拦截、请求前预估 | 最终账单结算、成本归因 |
| **成本** | 占用客户端 CPU 计算 | 无（随响应返回） |

## 常见考点
1. **缓存计费**：如果使用了 Redis/Semantic Cache，命中缓存的请求是否计费？如何统计？（答：通常不向 LLM 厂商付费，但内部可能需要按“请求次数”计费；统计时需区分“LLM 调用成本”和“计算资源成本”）
2. **流式 Token 统计**：流式输出时，如何实时统计已输出的 token 数？（答：流式过程中无法预先知道总 token，只能在流结束时根据 API 返回的 usage 字段汇总，或通过本地 tokenizer 实时估算用于进度展示）
3. **非对等计费**：某些特殊字符（如某些生僻字或 emoji）在不同模型中 token 数差异很大，如何避免估算偏差？（答：以服务器端返回的 usage.total_tokens 为准进行结算，本地估算仅用于实时预算控制预警）

## 记忆要点

- 计数方案：输入用 tiktoken 预估，输出以 API usage 字段为准。
- 费用计算：按模型百万 token 单价，分租户/功能/模型维度统计。
- 预算控制：月度预算限制，80% 预警，100% 限流或降级。
- 缓存优化：文本 Hash 缓存 Token 数，避免重复计算。
