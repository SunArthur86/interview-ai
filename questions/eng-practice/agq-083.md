# RAG 和微调怎么选?各自优劣

**回答：**

1. **RAG (检索增强生成)：**
   - **优势：** 
     - 知识可实时更新 (改知识库即可，无需重训模型)。
     - 可解释性好 (有来源引用，可追溯)。
     - 减少幻觉 (模型基于检索到的事实回答)。
     - 无需 GPU 训练资源，仅需推理资源。
   - **劣势：** 
     - 受检索质量影响大 (检索器的好坏决定上限)。
     - 上下文窗口限制 (检索内容过多可能被截断或导致“迷失中间”现象)。
     - 推理延迟增加 (需要额外的检索步骤和更长的上下文编码)。
     - 无法改变模型本身能力 (如逻辑推理、语言风格)。

2. **微调 (SFT/LoRA)：**
   - **优势：** 
     - 内化知识，不依赖外部检索 (推理速度快)。
     - 可以改变行为和风格 (如说话方式、格式约束)。
     - 推理无额外延迟 (模型权重固定)。
     - 特定任务精度高 (如分类、SQL 生成)。
   - **劣势：** 
     - 知识更新需重新训练 (时效性差)。
     - 需高质量标注数据 (数据准备成本高)。
     - 需 GPU 资源 (训练成本高)。
     - 可能导致灾难性遗忘 (忘记预训练知识)。
     - 可能产生幻觉 (如果学到了错误的知识)。

3. **选择策略：**
   - **知识频繁更新 / 需要引用来源 / 知识库巨大：** 推荐用 RAG。
   - **特定领域术语 / 风格对齐 / 模型能力提升 (如思维链)：** 推荐用微调。
   - **数据量小 (< 1000 条) / 预算有限：** 推荐用 RAG (微调容易过拟合)。
   - **黑盒 API 调用场景：** 只能使用 RAG (无法微调模型权重)。
   - **最佳实践：** RAG + 微调 (互补)。

4. **实际项目中的做法：**
   - 主体方案用 **RAG**：处理知识库问答、文档检索。
   - 辅以 **微调**：使用 LoRA 让模型更好地遵循指令格式、生成专业风格，或学习如何更好地利用检索到的上下文。
   - 效果叠加：微调后的模型 + RAG，通常比纯 RAG 效果更好。

```text
决策树：

需求：模型需要掌握新知识？
├─ 是：知识是否频繁变化？
│   ├─ 是 ──> 【RAG】 (时效性优先)
│   └─ 否 ──> 是否需要改变模型行为/风格？
│       ├─ 是 ──> 【微调】 或 【RAG + 微调】
│       └─ 否 ──> 【微调】 (内化知识，速度快)
└─ 否：需改变格式/风格？
    └─ 是 ──> 【微调】
```

**对比表格：**

| 维度 | RAG (检索增强) | SFT (有监督微调) |
| :--- | :--- | :--- |
| **知识时效性** | 实时更新 (秒级) | 滞后 (需重新训练) |
| **外部知识准确率** | 高 (有据可依) | 低 (容易产生幻觉) |
| **推理延迟** | 高 (需检索+长Context) | 低 (模型直接生成) |
| **训练成本** | 低 (只需训练检索器) | 高 (需反向传播) |
| **模型行为控制** | 弱 (依赖 Prompt) | 强 (内化为权重) |
| **数据依赖** | 文档库 (非结构化) | 高质量问答对 (结构化) |

**实战案例**：在为一个法律大模型做选型时，我们发现 RAG 在引用法条时非常准确，但无法像律师一样进行严谨的“三段论”推理。最终我们采取了“RAG + 微调”方案：用 RAG 提供法条依据，同时用 5000 条真实的法律判决书微调模型，使其学会用律师的口吻和逻辑进行推理，最终采纳率提升了 25%。

**## 常见考点**
1. **混合架构**：RAG 和微调能否同时使用？有什么注意事项？
   - *答案*：可以。先微调模型学会如何更好地利用 Context（例如减轻“忽略上下文”的问题），再挂载 RAG 系统。注意微调数据中要包含类似 RAG 的输入格式。
2. **知识边界**：微调能否注入事实性知识？
   - *答案*：可以，但效率低且容易产生幻觉。微调更擅长学习“形式”和“能力”，而 RAG 更擅长“事实”。对于事实性知识，RAG 通常更可靠。
3. **检索局限**：RAG 在什么情况下效果很差？
   - *答案*：需要跨文档推理、全局总结、或者检索到的文档包含噪声且模型无法区分时，RAG 效果会下降。

## 记忆要点

- RAG 优势：知识实时更新、可解释性强、减少幻觉，适合知识密集
- 微调优势：内化知识、改变风格/行为、推理快，适合能力对齐
- 选择策略：知识频繁变选 RAG，需改风格/格式选微调
- 最佳实践：RAG 处理事实，微调学习指令遵循，两者互补
