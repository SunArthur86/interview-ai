# 请设计一个日均 1000 万次调用的企业 AI 中台架构,月度成本控制在 50 万以内.

- **千万级日调用 AI 中台设计**

- **容量分析**
- 日 1000 万次调用 -> 平均 116 QPS,峰值 500+ QPS
- 假设每次调用平均 2000 token(1000 in + 1000 out)
- 日消耗 200 亿 token -> 月 6000 亿 token
- 纯 API 成本(GLM-4-Plus 约 0.05元/千token)= 月 300 万 -> 远超预算

- **实战案例**：在处理某电商大促的智能客服时，发现大量重复的“查物流”查询直接消耗了昂贵的 GPT-4 类模型。通过引入基于 Redis 的精确缓存和基于本地 Embedding 的语义缓存，将缓存命中率提升至 45%，单日 Token 成本直接下降 60%，成功将峰值响应延迟从 2s 降至 50ms。

- **成本优化策略(核心挑战)**

1. **模型分级路由(省 60%)**
- 70% 简单问题 -> GLM-4-Flash(0.001元/千token)
- 20% 中等问题 -> GLM-4-Air(0.01元/千token)
- 10% 复杂问题 -> GLM-4-Plus(0.05元/千token)
- 加权后约 0.008元/千token -> 月 48 万

2. **语义缓存(省 20-30%)**
- 高频相似问题直接返回缓存
- 缓存命中率 25% -> 再省 12 万/月

3. **自部署 Embedding(省 5%)**
- BGE-M3 本地部署,不再调用 Embedding API
- 节省约 2 万/月

4. **Prompt 精简(省 5%)**
- 上下文裁剪 + 历史摘要
- 减少 30% 输入 token

- **综合月成本**
- API:~36 万(路由+缓存优化后)
- 服务器:~8 万(4台 GPU 推理 + 6台 CPU 服务)
- 存储:~2 万(Milvus + Redis + PostgreSQL)
- 网络+CDN:~2 万
- 其他:~2 万
- **总计 ~50 万**

- **代码示例**（Python 路由逻辑）
```python
# router_service.py
def route_query(query):
    complexity = model.predict_complexity(query) # 轻量级分类器
    if complexity < 0.3:
        return "flash_model_endpoint" # 便宜模型
    elif complexity < 0.8:
        return "air_model_endpoint"   # 中等模型
    else:
        return "plus_model_endpoint"  # 昂贵模型
```

- **技术选型对比**
| 组件 | 选型 A (高可用/高性能) | 选型 B (低成本/平衡) | 推荐理由 |
|------|------------------------|----------------------|----------|
| 向量数据库 | Milvus (分布式) | Pgvector (单机扩展) | 亿级向量检索Milvus性能更稳 |
| 模型部署 | vLLM (高吞吐) | Ollama (轻量) | vLLM P99延迟更优 |
| 缓存 | Redis Cluster | Redis Standalone | 避免单点故障 |

- **架构亮点**
- 模型路由器:用轻量分类器(FastText)预判问题复杂度
- 多级缓存:Redis(精确匹配)+ Milvus(语义匹配)
- 自动扩缩容:基于 QPS + Token 消耗率
- 成本告警:日消耗超预算 5% 自动通知
