# FDE 如何在一周内交付一个让客户买单的 AI PoC(概念验证)

- **一周交付 PoC 的 FDE 方法论**

- **Day 1(周一):需求锁定**
- 上午:客户深度访谈(1-2小时),问 5 个核心问题
- 下午:梳理需求文档(1 页),发客户确认
- 晚上:确定技术方案和测试数据集

- **Day 2-3(周二-周三):核心开发**
- 搭建 MVP pipeline,用客户真实数据
- 聚焦核心场景(不要做边缘 case)
- 技术栈:Python + Streamlit + GLM-4 API
- 目标:核心功能跑通

- **Day 4(周四):优化 + 测试**
- 调优 Prompt(对 10-20 个真实问题优化)
- 界面美化(虽然只是 PoC,但客户看脸)
- 准备演示脚本(5 个最亮眼的场景)

- **Day 5(周五):演示 + 复盘**
- 现场演示(30分钟展示 + 30分钟讨论)
- 拿到客户反馈:满意?还需要什么?
- 讨论:从 PoC 到生产的路径和预算

- **关键原则**
1. **用客户真实数据**,不要用通用 demo 数据
2. **解决最痛的 1 个问题**,不要贪多
3. **演示 > 文档**,让客户亲手操作
4. **准备好'下一步'**,演示成功后立刻谈生产化方案

---

- **实战案例**：某制造业客户要做“AI 质检报告分析”，原本计划两周开发。我们在 Day 1 意识到他们只需要“自动判断合格/不合格”这一个动作，于是砍掉了复杂的图表生成功能，只保留 OCR + 判断逻辑，Day 3 就交付了可用版，客户当场签单。

- **代码示例**：
```python
# Streamlit 极简 PoC 结构
import streamlit as st
from client import LLMClient

st.title("智能合同审核 PoC")
contract_text = st.text_area("请粘贴合同内容", height=200)

if st.button("审核风险"):
    with st.spinner("AI 正在分析..."):
        # 直接调用后端或 LLM API
        result = LLMClient.audit(contract_text)
        # 高亮显示结果
        st.error(f"发现 {result['risk_count']} 处风险")
        st.json(result['details'])
```

- **对比表格**：PoC vs 生产环境
| 关注点 | PoC (概念验证) | Production (生产环境) |
| :--- | :--- | :--- |
| **核心目标** | 验证想法可行性，打动客户 | 稳定性、高并发、安全性 |
| **技术栈** | 快速开发 | 考虑长远维护和性能 |
| **数据处理** | 采样测试，甚至静态 Mock | 全量清洗，实时同步 |
| **错误处理** | 打印日志，忽略异常 | 完整的 Retry、Fallback、告警 |
| **UI/UX** | 能用即可 | 响应式设计、无障碍访问 |
| **代码质量**| 只要能跑 | 完整的 Test/CI/CD |
