# 如何为客户建立 AI 系统的自动化评估流水线

- **AI 自动化评估流水线设计**

- **评估维度**
1. **Faithfulness (忠实度)**: 回答是否基于检索到的上下文，没有幻觉。**原理**：通过 LLM 将答案拆解为原子事实，逐条核对检索上下文的支持度。
2. **Answer Relevancy (相关性)**: 回答是否切题。**原理**：基于生成的答案反向生成问题，计算反向生成问题与原始问题的余弦相似度。
3. **Context Precision (检索精度)**: 检索结果是否包含正确答案。**原理**：计算检索到的文档中，相关文档的排序靠前程度（如 NDCG 指标）。
4. **Context Recall (检索召回)**: 所有相关信息是否都被检索到。**原理**：计算标准答案中的声明能在检索到的上下文中找到的比例。

- **实现方案**

1. **黄金测试集构建**
- 从客户真实问题中选 200-500 条，需覆盖高频场景和长尾 Corner Case。
- 人工标注标准答案和评分，确保数据质量的信度（Kappa 系数 > 0.8）。
- 每次迭代回归测试，建立版本基线。

2. **自动化评估工具**
- **RAGAS**: 开源 RAG 评估框架，集成上述指标计算逻辑。
- **LLM-as-Judge**: 用 GPT-4/Claude 3 Opus 评估输出质量，需设计精细的 Prompt 模板，采用 CoT（思维链）减少评分波动。
- **人工抽检**: 每天随机抽 20 条人工复核，作为自动化指标校准的 Ground Truth。

3. **CI/CD 集成**
- 每次 Prompt/配置变更 -> 自动触发评估流水线。
- 评分下降超阈值（如 > 5%）-> 阻止上线并报警。

4. **线上监控**
- 用户点赞/点踩率（显式反馈）。
- 转人工率（模型兜底阈值触发的频率）。
- 用户追问率（追问多说明没答好或用户困惑）。

- **实战案例**
在某金融客户项目中，我们曾发现仅依赖 Faithfulness 指标无法捕获模型“过度保守”导致拒答率上升的问题，因此后续增加了“Helpfulness（有用性）”作为辅助指标，并结合线上用户点击率进行加权修正。

- **关键代码示例 (Python/RAGAS)**
```python
from ragas import evaluate
from ragas.metrics import faithfulness, answer_relevancy, context_precision

def run_eval(dataset):
    results = evaluate(
        dataset,
        metrics=[faithfulness, answer_relevancy, context_precision]
    )
    # 自动判定是否阻断上线
    if results["faithfulness"].mean() < 0.85:
        raise Exception("Faithfulness drop detected, blocking deployment")
    return results.to_pandas()
```
