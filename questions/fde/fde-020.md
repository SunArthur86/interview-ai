# 客户期望 AI 系统准确率 99%,但实际只有 85%,你怎么管理客户预期

- **FDE 客户期望管理 SOP**

- **Step 1: 重新定义'准确率'**
- 客户说的'99%准确'可能指不同含义:事实正确率?格式正确率?用户满意度?
- 拆解为可衡量的维度:回答正确率(85%) + 格式合规率(95%) + 用户满意度(80%)

- **Step 2: 对标行业基准**
- 展示行业数据:GPT-4 的幻觉率也有 3-5%
- 人类客服的错误率也有 5-10%
- 85% 的自动正确率已经是行业优秀水平

- **Step 3: 分阶段提升路线图**
- 当前:85%(PoC 阶段)
- 1个月后:90%(数据补充 + Prompt 调优)
- 3个月后:93%(Rerank + Fine-tune)
- 目标:95%(生产可用阈值)

- **Step 4: 设计兜底机制**
- AI 不确定时自动转人工
- 人工抽检 + 反馈闭环
- 'AI + Human' 模式比 '纯 AI' 更实际

- **Step 5: 关注客户真正在意的**
- 有时候客户说'要 99%',其实在意的是'别出大错'
- 设计安全护栏:敏感场景必须人工确认
- 用'零重大错误'替代'99%准确率'作为目标

- **预期管理沟通流程**

- **## 常见考点**
1. **准确率评估方法**：如何证明 85% 这个数字？需要建立金标准数据集，通过自动化测试或人工抽检来计算 Confusion Matrix（准确率、召回率、F1）。
2. **SLA 定义细节**：如果必须签 99% 的 SLA 怎么办？将 SLA 定义为"系统可用性"或"非敏感问题准确率"，或者通过"自动转人工成功"来算作"成功解决"。
3. **长尾问题处理**：客户往往会拿长尾难题（Corner Case）来质疑整体准确率。需强调通过 Bad Case 分析迭代优化，并设定明确的 Scope（边界）。

- **实战案例**：某电商客户坚持要求 99% 的“退货原因分类准确率”。经分析发现，对于“尺码不符”等标准问题 AI 已达 98%，但拉低整体分数的是“其他/奇葩”原因。我们通过调整 KPI 为“核心分类准确率 98% + 长尾问题自动识别并标记待人工”，成功说服客户验收。

```python
import numpy as np

def generate_with_guardrail(question: str, threshold: float = 0.7):
    # 1. 获取模型回复及置信度 (部分模型支持 logprobs)
    response = llm.generate(question, return_logprobs=True)
    confidence = calculate_confidence(response.logprobs)
    
    # 2. 置信度兜底：不确定时转人工或声明不知
    if confidence < threshold:
        return {
            "answer": "抱歉，这个问题比较复杂，我无法直接回答，已为您转接人工客服。",
            "action": "ESCALATE_TO_HUMAN",
            "confidence": confidence
        }
    
    # 3. 敏感词兜底
    if check_sensitive_keywords(response.text):
        return {"answer": "该问题涉及敏感信息，请咨询人工。", "action": "BLOCK"}
        
    return {"answer": response.text, "action": "AI_AUTO"}
```
