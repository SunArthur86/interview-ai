# 如何把一个 AI 应用快速部署到客户的服务器上

- **AI 应用客户部署 Checklist**

- **1. 容器化(最重要)**
```dockerfile
FROM python:3.11-slim
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

- **2. docker-compose 一键启动**
```yaml
version: '3'
services:
  app:
    build: .
    ports:
      - "8000:8000"
    environment:
      - API_KEY=${API_KEY}
    depends_on:
      - redis
  redis:
    image: redis:7-alpine
```

- **3. 部署步骤**
```bash
# 客户服务器上
git clone <repo>
cd <project>
cp .env.example .env  # 填入 API Key
docker-compose up -d
# 验证
curl localhost:8000/health
```

- **4. 常见坑**
- 客户服务器是 ARM 架构 → Docker buildx 跨平台构建
- 客户没有公网 → 用内网 IP + 反向代理
- 客户防火墙限制 → 只开放必要端口
- GPU 驱动版本不匹配 → 提前确认 CUDA 版本

- **FDE 经验**:永远准备一个 `install.sh` 脚本,客户只需一行命令就能装好.

- **实战案例**：曾遇到客户内网环境极度封闭，无法拉取 Docker Hub 镜像。我们使用了 `docker save` 将镜像打包成 tar 文件，通过 U 盘拷贝进内网，再用 `docker load` 导入，配合离线安装脚本成功部署。

- **代码示例**：
```bash
#!/bin/bash
# 一键离线部署脚本 install.sh
check_dependencies() {
    command -v docker >/dev/null 2>&1 || { echo "Docker 未安装"; exit 1; }
    command -v docker-compose >/dev/null 2>&1 || { echo "Docker Compose 未安装"; exit 1; }
}
load_images() {
    echo "加载离线镜像..."
    docker load -i images/app_image.tar
}
start_service() {
    docker-compose up -d
}
check_dependencies && load_images && start_service
```

- **对比表格**：云端 SaaS vs 私有化部署
| 维度 | 云端 SaaS | 私有化部署 |
| :--- | :--- | :--- |
| **上线速度** | 分钟级 (配置账号即可) | 天级 (依赖环境/硬件) |
| **数据安全** | 数据出域，依赖供应商合规 | 数据完全本地留存 (合规首选) |
| **运维成本** | 厂商负责 | 客户 IT 团队负责 (更新/备份) |
| **定制化** | 仅支持配置级修改 | 支持代码级深度修改 |
| **适用场景** | 初创企业/非敏感数据 | 政府/金融/大型企业 |

- 常见坑：ARM架构需跨平台构建，内网环境用U盘传镜像。
- 架构设计：App+VectorDB+Redis全容器化，Nginx做反向代理。
- 离线方案：docker save打包，内网docker load导入，解决网络隔离。
