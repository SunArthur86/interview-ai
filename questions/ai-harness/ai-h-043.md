# Ragas框架如何评估RAG系统?有哪些核心指标?如何部署

**Ragas 评估框架核心解析**

**1. 评估指标体系与原理**

**检索质量指标：**
- **Context Precision (上下文精确度)**：检索到的上下文中相关信息的比例，衡量是否包含噪声。
- **Context Recall (上下文召回率)**：金标准答案所需的信息是否被检索到，实际上是判断 Ground Truth 中的断言能否由检索到的 Context 推导出来。
- **Context Relevancy (上下文相关性)**：检索到的上下文与用户问题的直接相关性，通过计算句子相似度得出。

**生成质量指标：**
- **Faithfulness (忠实度)**：答案是否严格基于检索到的文档，检测幻觉。原理是将Answer拆解为多个原子声明，逐一验证这些声明是否能在Context中找到依据（NLI任务）。
- **Answer Relevancy (答案相关性)**：生成的答案是否直接回应了用户的问题。通过将Answer反向生成问题，计算生成的问题与原始问题的Embedding相似度。

**Ragas 评估工作流图：**
```
[ 输入数据集 ]
    │ (Question, Context, Answer, Ground Truth)
    ▼
┌───────────────────────────────────────┐
│         LLM as a Judge (评估器)        │
├───────────────────────────────────────┤
│ 1. Faithfulness:  Answer -> Claims    │
│                 Context + Claims -> NLI│
│                                       │
│ 2. Recall:      GT -> Sentences       │
│                 Context + Sentences -> NLI│
└─────────────────────────────────────────┘
    │
    ▼
[ 评估分数报告 (0-1) ]
```

**2. 代码实现示例**

```python
from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_precision,
    context_recall
)

# eval_dataset 需包含: question, contexts, answer, ground_truth
result = evaluate(
    dataset=eval_dataset,
    metrics=[
        faithfulness,
        answer_relevancy,
        context_precision,
        context_recall
    ]
)

print(result.scores)
```

**3. 评估数据集构建**
- **人工标注**：专家编写 Question + Ground Truth，质量最高但成本高。
- **LLM 自动生成**：基于文档通过大模型自动生成问答对，常用“Evolution”模式（增加问题难度和多样性）。
- **日志提取**：从真实用户日志中清洗并脱敏提取，需人工抽检质量。

**4. 最佳实践**
建议建立 CI/CD 集成的持续评估 Pipeline，每次 RAG 配置（如 Chunk 策略、Embedding 模型）变更后自动运行评估，确保效果不回退。

**实战案例与部署**

**实战案例**：某金融 RAG 系统上线后，用户投诉回答偶尔不准确。使用 Ragas 评估发现 `Faithfulness` 分数低（0.65），原因在于检索到了过期的政策文档。将向量数据库从 Milvus 迁移至支持混合检索（Vector+Keyword）的 Qdrant 并增加时间过滤后，`Faithfulness` 提升至 0.92。**踩坑**：`Context Recall` 计算非常消耗 Token（需将 Context 和 GT 分词做 NLI），建议使用小模型（如 Llama-3-8B）作为 Judge 以降低 80% 的评估成本。

**关键代码（自定义 Judge 以降低成本）：**
```python
from langchain_openai import ChatOpenAI

# 使用成本较低的小模型作为 Judge
judge_llm = ChatOpenAI(model="gpt-3.5-turbo-0125")

result = evaluate(
    dataset=eval_dataset,
    metrics=[context_recall],
    llm=judge_llm  # 注入自定义 Judge
)
```

## 面试追问
1. Ragas 评估 Faithfulness 时使用 NLI（自然语言推断）任务，如果 Judge 模型本身能力不足，如何保证评估的客观性？
2. 在没有 Ground Truth 的真实生产环境中，如何利用 Ragas 进行评估（如仅使用 Question 和 Context）？

## 易错点
1. **混淆 Answer Relevancy 和 Context Relevancy**：前者评估“生成的答案是否解决了问题”（可能包含幻觉但相关性高），后者评估“检索到的文档是否包含答案的信息”（检索质量）。
2. **忽略评估本身的成本**：Ragas 是“用LLM评估LLM”，高频全量评估成本极高，通常只应用于黄金测试集或抽样验证，不适合实时流水线。

- 核心原理：LLM as a Judge，利用 NLI 任务验证 Answer 与 Context 的一致性。
- 数据构建：人工标注（高质量）或 LLM 自动生成（Evolution 模式）。
