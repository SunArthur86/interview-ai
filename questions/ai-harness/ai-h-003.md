# DSPy的编程模型和LangChain有什么本质区别

- **本质区别:**

| | LangChain | DSPy |
|--|-----------|------|
| 范式 | **命令式**(手写prompt) | **声明式**(定义输入输出) |
| Prompt | 手动编写和调优 | **自动搜索最优prompt** |
| 示例选择 | 手动 | **自动选择few-shot** |
| 适配模型 | 需重写prompt | **自动适配** |

- **DSPy核心模式:**
```python
# 1. 定义Signature(做什么)
class QA(dspy.Signature):
    """回答问题"""
    question = dspy.InputField()
    answer = dspy.OutputField(desc="简洁准确的答案")

# 2. 选择Module(怎么做)
qa = dspy.ChainOfThought(QA)

# 3. 编译优化(自动找最优prompt+示例)
optimizer = dspy.BootstrapFewShot(metric=accuracy)
compiled_qa = optimizer.compile(qa, trainset=data)

# 4. 使用
result = compiled_qa(question="什么是注意力机制?")
```

- **DSPy vs 手写Prompt:**
- 手写:依赖人工经验,换模型可能失效
- DSPy:自动优化,换模型只需重新compile
- 可版本管理、可复现、可A/B测试

**实战案例**
在构建一个复杂的情感分析 Pipeline 时，传统方法需要花费 2 天微调 Prompt。引入 DSPy 后，只需定义好 `Input/Output` 签名和评估指标，DSPy 自动在 15 分钟内生成了效果优于人工调优的 Few-shot Prompt，且准确率提升了 5%。

**代码示例（自定义 Metric 与 Teleprompter）**
```python
import dspy

# 定义评估指标（验证逻辑是否匹配）
def validate_logic_answer(example, pred, trace=None):
    return pred.answer.lower() == example.answer.lower()

# 定义程序：将长文本转换为逻辑步骤
class LogicGen(dspy.Module):
    def __init__(self):
        super().__init__()
        self.gen = dspy.ChainOfThought("context -> reasoning_steps")

    def forward(self, context):
        return self.gen(context=context)

# 配置优化器：自动尝试不同的 Few-shot 示例
teleprompter = dspy.BootstrapFewShot(metric=validate_logic_answer, max_labeled_demos=4)
optimized_logic = teleprompter.compile(LogicGen(), trainset=trainset)
```

## 常见考点
1. **Teleprompter (优化器) 的作用**：DSPy 中的 `BootstrapFewShot` 或 `BayesianSignatureOptimizer` 是如何工作的？它们本质上是在验证集上尝试不同的 Few-Shot 示例组合或 Prompt 指令，以最大化 Metric（如准确率）。
2. **Metric 定义**：DSPy 如何知道 Prompt 好坏？必须定义一个 Trace 评估函数，例如判断 `answer == ground_truth` 或使用 LLM 作为 Judge 来打分。
3. **Self-Reflection 模块**：DSPy 提供了 `dspy.ReAct` 和 `dspy.MultiChainComparison`，它们与标准 Prompt 工程中的同名模式有何不同？它们是以模块化代码形式存在，可以被 Teleprompter 自动调优参数。
4. **DSPy 的局限性**：DSPy 极大地降低了 Prompt 工程成本，但它主要关注逻辑任务的准确性。对于创意写作或需要严格特定语气（Tone）的任务，自动 Prompt 可能不如人工调优的精细。
