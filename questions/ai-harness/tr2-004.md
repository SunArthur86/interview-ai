# MCP (Model Context Protocol) 提出的三层架构（传输/会话/能力）分别解决什么问题？它如何促进 AI 工具生态的标准化？

MCP 的三层架构包括：1) **传输层**：定义了数据如何在 Host 和 Server 之间传输（如基于 SSE 或 WebSocket），解决通信协议的兼容性问题；2) **会话层**：管理请求的生命周期、上下文状态和采样，解决多轮对话和状态同步问题；3) **能力层**：定义了 Prompts、Resources（资源）、Tools（工具）的具体 JSON Schema，解决能力发现和调用的互操作性问题。MCP 通过这种解耦，使得任何支持 MCP 的 LLM 应用都能“即插即用”地接入数据源和工具，无需为每个工具编写定制化代码，从而极大地降低了 AI 应用开发的碎片化成本。

**实战案例**：在构建企业级知识库助手时，利用 MCP 将私有 GitLab 仓库作为 Resources 接入，解决了传统 API 集成中因鉴权差异导致的连接超时问题。

**代码示例** (TypeScript - 工具注册):
```typescript
// MCP Server 定义工具能力
server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [{
    name: "fetch_git_log",
    description: "Get latest commit from repo",
    inputSchema: {
      type: "object",
      properties: { repo_id: { type: "string" } },
      required: ["repo_id"]
    }
  }]
}));
```
