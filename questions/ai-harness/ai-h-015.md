# 生产RAG系统的10个常见问题及解决方案

- **10大问题与方案:**

| # | 问题 | 症状 | 解决方案 |
|---|------|------|---------|
| 1 | 检索不准 | 答非所问 | 加Reranker + 查询改写 |
| 2 | 检索太多 | 上下文过长 | 减少Top-K + 压缩chunk |
| 3 | 检索太少 | 答案不全 | Multi-Query扩大召回 |
| 4 | 分块不合理 | 语义割裂 | 语义分块/结构化分块 |
| 5 | 幻觉 | 编造答案 | 提示「只基于文档」+ 引用标注 |
| 6 | 多语言 | 混合检索差 | BGE-M3多语言模型 |
| 7 | 实时更新 | 知识过时 | 增量索引 + 通知机制 |
| 8 | 大规模 | 检索慢 | 分区索引 + 缓存 |
| 9 | 多轮对话 | 上下文丢失 | 对话历史改写查询 |
| 10 | 评估难 | 不知效果如何 | Ragas/TruLens自动评估 |

- **优化优先级:**
1. 数据质量(分块+解析)
2. 检索质量(Reranker+查询扩展)
3. 生成质量(Prompt+引用)
4. 评估体系(Ragas+人工抽检)

**RAG 系统全链路优化点:**
```text
[数据 ingestion]
   ↓
清洗/解析 → [高级分块] → 多阶段索引
                ↑ (关键点1)
[用户 Query]
   ↓
Query Routing → [查询扩展] → [混合检索] → [Reranker] → 上下文压缩
                      ↑ (关键点2)        ↑ (关键点3)
                                             ↓
[LLM Generation] ←───── Prompt Engineering + Citations
   ↓
[Evaluation] (Ragas/Ground Truth)
```

- **实战案例**：在某法律问答系统中，用户提问"合同中关于违约责任的条款"，单纯的向量检索往往漏掉"赔偿"或" breach "等不同表述的段落。引入 **HyDE (Hypothetical Document Embeddings)** 技术后，先让LLM生成一段虚构的"理想答案"，再用这个虚构答案去检索，召回率提升了40%。

- **关键代码**：
```python
from llama_index.core.query_engine import TransformQueryEngine
from llama_index.core.indices.query.query_transform import HyDEQueryTransform

# 包装原始查询引擎
hyde_transform = HyDEQueryTransform(include_original=True)
hyde_query_engine = TransformQueryEngine(
    query_engine, 
    query_transform=hyde_transform
)

# 查询时会先生成假设文档，再用假设文档去检索
response = hyde_query_engine.query("项目交付延期后的具体罚款比例是多少？")
```

- **对比表格**：

| 症状 | 根因 | 轻量级方案 | 重度方案 |
| :--- | :--- | :--- | :--- |
| **检索不准** | 语义/关键词不匹配 | 混合检索 (Vector+BM25) | Reranker (Cohere/BGE-Reranker) |
| **检索太多** | 噪声大 | 降低 Top-K (e.g., 10→5) | LongContextLLMLingua (上下文压缩) |
| **回答空洞** | 知识库缺失 | 数据回溯/补充文档 | Agent (调用搜索API) |

## 常见考点
1. **评估指标**：Ragas 框架中的 Faithfulness（忠实度）和 Answer Relevance（相关性）是如何计算的？（答：忠实度通常通过 LLM 判断生成答案是否完全由检索片段支持；相关性通过 LLM 判断答案是否解决了用户问题）
2. **上下文压缩**：当检索到的文档很长但只有一两句有用时，除了 Reranker 还有什么办法？（答：使用 LlamaIndex 的 Context Post-processor 或 LongLLMLingua 进行上下文压缩，剔除噪声）
3. **Citations（引用）**：如何强制模型在回答时标注引用来源？（答：在 Prompt 中明确要求输出格式，如 `[statement] (source_id)`，并在最终输出时校验 source_id 是否在检索结果中）

## 记忆要点

- 检索不准：加Reranker精排 + 查询改写/扩展(HyDE/Multi-Query)
- 检索太多：减少Top-K + 上下文压缩，或使用LongContext模型
- 幻觉问题：Prompt限制“只基于文档” + 引用标注(Citations)
- 评估体系：使用Ragas框架自动评估Faithfulness(忠实度)和Answer Relevance
