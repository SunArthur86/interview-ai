# 针对LLM的模型水印有哪些技术路线？它们对生成文本的质量和推理速度有何影响？

主流技术路线包括基于绿红名单的 Logits 调整和基于后处理的关键词植入。Logits 调整在采样时，利用伪随机数发生器对特定词表的 token 概率进行轻微增减，使其在统计分布上呈现出特定的模式，且不改变语义。关键词植入则是在生成的文本中强制包含特定的隐形字符或词汇。Logits 调整对文本质量影响微乎其微，因为仅微调概率分布，几乎不影响推理速度（仅需计算 Hash 函数）。它确保了生成的每一段文本都能被验证是否源自该模型，是目前防御模型滥用和版权侵犯的重要手段。

## 易错点
1. **混淆水印鲁棒性与不可见性**：往往认为调整 Logits 幅度越大水印越鲁棒，但这会导致生成文本质量急剧下降（如出现重复词或语义不通），需在两者间寻找平衡点。
2. **误认为所有后处理水印都不可见**：关键词植入容易被发现或破坏，实际应用中，基于 Logits 的统计水印才是隐形的主流，而基于修改文本结构或字符的后处理手段通常牺牲了一定的文本可读性。

## 边界情况
1. **短文本与低熵场景**：当生成的文本极短（如“好的”）或本身熵值很低（如代码片段）时，嵌入足够的统计特征非常困难，导致水印检测准确率大幅下降。
2. **对抗性攻击**：攻击者可能对水印文本进行轻微的改写、翻译或添加噪声，这些操作可能在保留原意的情况下破坏水印的统计特性，导致水印失效。

## 面试追问
1. **在分布式生成场景下（如多个 GPU 并行生成），如何保证水印 Key 的一致性和生成分布的统计特性不变？**
2. **如果攻击者通过蒸馏攻击获取了模型的私有副本，这种水印是否还能有效证明版权归属？**
3. **如何量化评估水印对模型生成质量（如 Perplexity）的具体影响？**

## 记忆要点

- 两大路线：绿红名单Logits调整(隐形主流)、后处理关键词植入(易破坏)。
- Logits调整：微调采样概率分布，不改变语义，质量影响极小。
- 性能影响：几乎无延迟，仅需计算Hash函数。
- 核心作用：验证文本来源，防御模型滥用和版权侵权。
- 易错点：水印鲁棒性与不可见性需平衡；短文本/低熵场景检测率下降。
