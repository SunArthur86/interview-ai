# Few-shot Learning

### 3. Few-shot Learning（少样本学习）

#### 3.1 概念解释
- **Zero-shot**：不给示例，只给指令与任务说明。
- **One-shot**：给 1 个输入输出样例。
- **Few-shot**：给少量（通常 2～8 个）样例，示范期望行为。

#### 3.2 原理详解
**In-context learning**：模型在不更新权重的情况下，从前文示例中「推断」映射关系。示例越多，对格式与边界的约束越强，但占用更多 token，且可能过拟合到示例风格。

从注意力机制角度看，Few-shot 示例实际上改变了 Query（当前问题）与 Key/Value（示例中的模式）之间的注意力分布，使得模型更关注示例中特定的输出模式。

**Few-shot 工作流程图**
```text
┌─────────────────────────────────────────────────────────────┐
│                  Prompt Construction                        │
├─────────────────────────────────────────────────────────────┤
│  Instruction: "Classify the sentiment of the review:"      │
├──────────────────┬──────────────────┬──────────────────────┤
│ Example 1        │ Example 2        │ Example 3            │
│ Input: Good      │ Input: Bad       │ Input: Okay          │
│ Output: Positive │ Output: Negative │ Output: Neutral      │
├──────────────────┴──────────────────┴──────────────────────┤
│  Query: "The movie was spectacular."                       │
├─────────────────────────────────────────────────────────────┤
│                   Model Inference                           │
│  (Model calculates attention between Query and Examples)    │
├─────────────────────────────────────────────────────────────┤
│  Completion: "Positive"                                    │
└─────────────────────────────────────────────────────────────┘
```

#### 3.3 Few-shot 一定比 Zero-shot 好吗？
不一定。若任务简单且指令已足够清晰，Zero-shot 更省 token；若输出格式复杂或边界情况多，Few-shot 往往更稳。若示例质量差或与测试分布不一致，反而有害。

**边界情况**：当示例中的噪声过多（如包含无关信息），或者模型容量较小无法捕捉长上下文中的规律时，增加示例可能导致性能下降。

#### 3.4 示例选择策略
**原则**：优先选覆盖典型模式、边界情况、易错反例；避免高度重复的同质示例。

**实践建议**：
1. 包含「最难的合法输入」与「应拒绝/应降级」的样例。
2. 若有多类意图，每类至少一例。
3. 示例输出风格与生产环境一致（尤其是 JSON 键名与大小写）。

#### 3.5 示例排列顺序的影响
研究表明（因模型与任务而异）：首尾位置往往印象更深；中间示例可能被「平均化」。

**工程做法**：
- 把最重要约束放在指令段，而非只依赖最后一个示例。
- 由简到繁或先典型后边界，避免第一个示例过难导致整体跑偏。

#### 3.6 动态 Few-shot
**概念**：不是固定写死例子，而是每次请求前用检索（向量相似度、BM25）从示例库中选出与当前用户输入最相近的 k 条。

**价值**：相当于「小样本集合上的 RAG」，提升示例与用户问题的分布匹配度，减少干扰。这有助于解决固定示例无法覆盖长尾领域的问题。

**实战案例**：在多语言客服系统中，硬编码的英语示例对中文、西语查询帮助甚微。我们维护了一个包含 20 种语言的“优质问答库”，通过 Embedding 检索 Top-3 相似语料的问答对作为动态示例填入 Prompt，使得非英语意图识别准确率提升了 15%。

**代码示例**：
```python
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer('all-MiniLM-L6-v2')
examples = ["查快递", "退货申请", "投诉建议"] # 预定义示例库
emb_examples = model.encode(examples)

def get_dynamic_few_shot(query):
    emb_query = model.encode(query)
    # 计算相似度并取最相关的 1 个示例
    scores = util.cos_sim(emb_query, emb_examples)
    best_idx = scores.argmax()
    return f"Example: {examples[best_idx]} -> Intent: Shipping_Info"
```
