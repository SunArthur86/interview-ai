# 异步一定能提高 Agent 吞吐吗

异步处理不一定会提高吞吐，其效果取决于系统的瓶颈类型：

1.  **I/O 密集型场景（有效）**：
    *   适用：Agent 需要频繁调用外部 LLM API、向量数据库检索、查询业务数据库。
    *   原理：在等待网络 I/O 时释放线程处理其他请求。**异步可以显著提高吞吐**。
2.  **CPU/GPU 密集型场景（无效或有限）**：
    *   适用：本地运行 LLM 推理，完全受限于 GPU 算力。
    *   原理：Async/Coroutines 主要解决等待问题，不增加算力。若 GPU 已满载，异步只是把请求排队，反而增加上下文切换开销。此时需要**水平扩展（多副本）**。
3.  **长尾延迟与批处理**：
    *   对于本地推理，单纯的异步不够，需配合 **Continuous Batching**（如 vLLM 技术）或 **动态批处理**，将异步到达的请求打包成 Batch 送入 GPU，提高利用率。
4.  **副作用风险**：
    *   **无界并发**：若异步不加限制，下游（如 LLM Vendor）会被瞬时海量请求打爆（触发 429），导致整体系统雪崩。必须配合 **信号量** 或 **限流器** 控制最大并发数。

### 实战案例
我们曾开发了一个 Python 异步 Agent 服务，初期未加并发控制。在遭遇突发流量时，瞬间发起了 5000 个并发请求到 OpenAI，直接导致 IP 被封禁 429 错误。后来引入 `asyncio.Semaphore(100)` 限制最大并发数，并在业务层配合请求队列，才实现了吞吐的稳定提升而非雪崩。

### 代码示例 (Python 并发控制)
```python
import asyncio

# 限制最大并发数为 100，防止打爆上游 API
semaphore = asyncio.Semaphore(100)

async def call_llm_api(prompt):
    async with semaphore:
        # 模拟网络 IO 请求
        await asyncio.sleep(1) 
        return "response"

async def handle_request(req):
    # 并发发起多个任务，但受 semaphore 控制
    tasks = [call_llm_api(p) for p in req.prompts]
    return await asyncio.gather(*tasks)
```

## 常见考点
1.  **Python 的 `asyncio` 能利用多核 CPU 吗？**（追问：不能，需要结合 `multiprocessing` 或多 K8s Pod。Asyncio 仅解决单核并发效率）
2.  **在 Node.js 或 Python Agent 中，如何防止异步任务阻塞事件循环？**（追问：避免在协程中执行同步重计算，CPU 密集任务扔到线程池或独立服务）
3.  **如何控制异步的最大并发度？**（追问：`asyncio.Semaphore` 的使用，或者消息队列的 `prefetch_count`）

### 边界情况补充
*   **异常传播**：在 `asyncio.gather` 中如果某个子任务抛出异常且未处理，默认可能会导致整个批次失败（取决于 `return_exceptions` 参数），进而吞吐量瞬间跌零。需要做好异常隔离。
*   **超时熔断**：外部 API 调用可能在异步挂起中无限等待，必须为每个异步调用设置 `wait_for` 超时时间，防止连接泄漏。
*   **内存暴涨**：异步架构下，吞吐提升意味着内存中驻留的上下文 变多。如果单个请求处理逻辑涉及大文件加载，高并发下会导致 OOM Kill，需注意流式处理。

## 易错点
1.  **异步文件读写未使用库**：在 Python 中使用普通的 `open()`/`read()` 代替 `aiofiles`，导致伪异步，阻塞整个 Event Loop。
2.  **过度并发导致上下文切换开销**：理论上异步可以支持万级并发，但在 Node.js 或 Python 中，当并发数超过数千（具体取决于业务复杂度）时，调度协程的开销本身就会成为瓶颈，反而降低吞吐。

## 记忆要点

- I/O 密集型：异步通过重叠等待时间显著提升吞吐，如调用外部 API 或 DB。
- CPU/GPU 密集型：异步无效，需水平扩展副本或批处理（如 vLLM）提升算力。
- 风险控制：必须配合 Semaphore 限制最大并发，防止打爆上游导致雪崩。
- 伪异步陷阱：避免在协程中使用同步 IO（如普通文件读写），阻塞事件循环。
