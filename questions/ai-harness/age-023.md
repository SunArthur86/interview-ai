# 如何做「关键路径」与「非关键路径」分级

### 核心理念：风险分级，差异化治理

#### 1. 定义分级标准
- **关键路径**：涉及资金交易、数据删除、权限变更、核心业务决策（如医疗诊断、法律建议）。
  - *特征*：错误容忍度极低，不可逆，合规性要求高。
- **非关键路径**：闲聊、草稿生成、内容摘要、内部知识检索、辅助性代码生成。
  - *特征*：允许一定偏差，用户体验为主，追求速度和低成本。

#### 2. 关键路径：强一致性与人机协同
- **模型选择**：最强推理能力模型（如 GPT-4, Claude 3.5 Sonnet），低 Temperature (0~0.1) 确保确定性与逻辑性。
- **HITL (Human In The Loop)**：
  - **Pre-check**：关键操作前要求用户二次确认。
  - **Post-review**：生成结果后人工审核（或仅标记为待审核状态）。
- **审计**：全量记录输入、输出、中间推理步骤，保存至不可篡改存储（如 WORM 存储），用于合规审计。

#### 3. 非关键路径：高吞吐与容错
- **模型选择**：小模型或低成本模型（如 GPT-4o-mini, Llama-3-8B），较高 Temperature (0.7~1.0) 增加创造性。
- **超时策略**：设置宽松但合理的超时，避免阻塞用户，超时则直接降级或返回占位符。
- **缓存策略**：对高频相似 Query 开启激进的语义缓存。
- **降级策略**：服务不可用时返回友好的兜底话术，而非错误堆栈。

#### 实战案例
在电商自动客服中，简单的“查物流”被误判为“修改订单”（关键路径），导致直接调用高成本模型且触发人工审核。解决方法是增加一个轻量级的 Intent Router 层，先用 3.5-turbo 判意图，确认为“写操作”才进入关键路径流程，降低成本 60%。

#### 代码示例
```python
def dispatch_agent(query: str):
    # Step 1: Lightweight Intent Classification
    intent = classify_intent(query) # Returns 'critical' or 'casual'
    
    if intent == 'critical':
        # Critical Path: High Accuracy, HITL
        agent = CriticalAgent(model='gpt-4', temp=0.1)
        draft = agent.run(query)
        return send_to_human_review(draft) # Block output until reviewed
    else:
        # Non-Critical Path: Fast, Cached, Best Effort
        if cached := semantic_cache.get(query):
            return cached
        agent = CasualAgent(model='gpt-3.5-turbo', temp=0.8)
        return agent.run(query) or fallback_response()
```

#### 对比表格：关键路径 vs 非关键路径
| 维度 | 关键路径 | 非关键路径 |
| :--- | :--- | :--- |
| **目标** | 准确性、合规性、可追溯 | 速度、体验、低成本 |
| **模型选择** | SOTA (GPT-4/Claude Opus) | 轻量模型 (Mini/Llama-8B) |
| **Temperature** | 0.0 ~ 0.1 (确定性) | 0.7 ~ 1.0 (创造性) |
| **容错机制** | 人工审核、强约束 Prompt | 降级兜底、自动重试 |
| **典型场景** | 医疗诊断、金融转账、代码生成 | 闲聊、摘要、翻译、搜索 |

## 记忆要点

- 关键路径：涉及资金/权限，用高精模型、低 Temp、强审计及人工审核（HITL）。
- 非关键路径：闲聊/摘要，用轻量模型、高 Temp、语义缓存及快速降级。
- 核心差异：关键路径追求准确与合规，非关键路径追求速度与低成本。
- 路由策略：增加 Intent Router 层，先判意图再分发，避免过度治理。
