# Claude Code 的架构是怎样的?它的 Agent Loop 是如何设计的

**Claude Code 架构与 Agent Loop 设计**

**核心定位：**
Claude Code 是 Anthropic 推出的 CLI AI 编程助手，其架构是现代 Agent 设计的参考标杆。

**1. Agent Loop 设计**
- **循环逻辑**：接收用户指令 -> LLM 推理 -> 工具调用 -> 观察结果 -> 继续推理/结束。
- **迭代限制**：最大 25 轮迭代，防止无限循环。
- **上下文管理**：支持上下文压缩，超长历史时自动摘要。
- **请求模式**：每轮都带完整上下文（无状态请求），依赖 Prompt Caching 优化。

**2. 核心工具集**
- **文件操作**：Read (读取)、Write (写入)、Edit (编辑)、Glob (查找)。
- **命令执行**：Bash 命令执行环境。
- **代码搜索**：Grep 语义或文本搜索。
- **MCP 协议**：通过 Model Context Protocol 连接外部工具（如 GitHub、数据库）。

**3. 系统提示词 关键设计**
- **角色定义**：明确身份为 Claude Code，Anthropic 的 AI 助手。
- **工具使用规则**：优先搜索再修改、改动后必须验证。
- **安全约束**：禁止执行破坏性命令、高风险操作需用户确认。

**4. 上下文管理策略**
- **Prompt Caching**：利用缓存减少重复计算，降低成本和延迟。
- **自动压缩**：当上下文超过阈值时自动压缩历史记录，保留关键摘要。

**设计哲学：**
采用“无状态请求 + 前缀缓存 + 上下文压缩”的简单架构，而非复杂的有状态 Agent。

### 实战案例
在使用 Claude Code 修改一个遗留的 Python 项目时，它尝试安装 `pip install requests`，但触发了 `pip` 版本冲突错误。Agent Loop 成功识别了错误日志中的 `pip install --upgrade pip` 建议，自动执行了修复命令并重试任务，最终完成了依赖安装，展示了闭环修复能力。

### Agent Loop 核心代码逻辑
这是一个简化版的 Agent Loop 实现，展示了**状态检测**和**工具分发**的核心逻辑：

```python
import json

def run_agent_loop(client, user_message, max_iterations=25):
    messages = [{"role": "user", "content": user_message}]
    
    for _ in range(max_iterations):
        response = client.messages.create(
            model="claude-3-5-sonnet-20241022",
            max_tokens=4096,
            tools=[{"type": "bash_20241022", "name": "run_command"}, ...],
            messages=messages
        )
        
        # 检查是否停止
        if response.stop_reason == "end_turn":
            break
            
        # 处理工具调用
        for block in response.content:
            if block.type == "tool_use":
                tool_result = execute_tool(block.name, block.input)
                messages.append({"role": "assistant", "content": response.content})
                messages.append({"role": "user", "content": [{
                    "type": "tool_result", 
                    "tool_use_id": block.id, 
                    "content": str(tool_result)
                }]})
```
