# 性能优化

### 性能优化

**核心目标**：解决 Agent 系统中的串行 LLM 调用、高延迟网络 IO 和上下文膨胀问题。

**原理详解**：

1. **异步处理**：
   - 使用 `asyncio` 或消息队列，避免 I/O 等待阻塞线程。
   - **并行工具调用**：无依赖的工具调用并发执行，大幅降低总耗时。

2. **流式输出**：
   - 利用 SSE (Server-Sent Events) 边生成边返回，降低首字延迟 (TTFB)。
   - 需处理流式中断情况，保证最终状态一致性。

3. **并发控制**：
   - 使用 `Semaphore` 限制对同一模型的并发请求数，防止触发 429 (Rate Limit)。

4. **连接复用**：
   - HTTP 客户端开启 `keep-alive`，数据库使用连接池，减少握手开销。

5. **缓存分层**：
   - **L1 (进程内)**：LRU 缓存，极低延迟。
   - **L2 (Redis)**：分布式共享，支持 TTL。
   - 读路径：L1 -> L2 -> 回源模型。

#### 实战案例
某旅游规划 Agent 原本串行调用“查航班”、“查天气”、“查酒店”，总耗时 15秒。通过分析 DAG（有向无环图）发现这三者无依赖关系，改为 `asyncio.gather` 并行调用后，耗时降至 5秒（由最慢的接口决定），用户体验显著提升。

#### 代码示例
```python
import asyncio

async def call_tool(tool_name: str, query: str):
    # 模拟工具调用 IO 耗时
    await asyncio.sleep(1) 
    return f"Result for {tool_name}"

async def parallel_agent_executor():
    # Agent 决策需要调用的工具列表（并行）
    tools_to_call = ["search_flights", "check_weather", "book_hotel"]
    
    # 并行执行，无需等待上一个完成
    results = await asyncio.gather(
        *[call_tool(name, "user query") for name in tools_to_call]
    )
    
    return results
```

#### 优化策略对比
| 优化手段 | 优化维度 | 适用场景 | 潜在风险 |
| :--- | :--- | :--- | :--- |
| **异步/并行** | 吞吐量 & 延迟 | 多步骤无依赖 Agent | 增加代码复杂度，需处理竞态条件 |
| **流式输出 (SSE)** | 首字延迟 (TTFT) | 生成式对话 | 前端需处理增量渲染，难以中途取消 |
| **L1/L2 缓存** | 吞吐量 & 成本 | 高频重复问题 | 缓存一致性难题，可能返回过期数据 |
| **连接复用** | 网络延迟 | 高并发短连接 | 占用更多长连接资源，需配置合理超时 |
