# vLLM vs SGLang vs TGI vs Triton--推理引擎如何选择

- **推理引擎对比:**

| 引擎 | 开发者 | 核心技术 | 优势 |
|------|--------|---------|------|
| **vLLM** | UC Berkeley | PagedAttention (KV Cache分页) | **吞吐量最高** (显存利用率高) |
| **SGLang** | UC Berkeley | RadixAttention (前缀树缓存) | **复杂推理最快** (多轮对话/Agent) |
| **TGI** | HuggingFace | Continuous Batching | 易用/兼容HF生态/功能全 |
| **TensorRT-LLM** | NVIDIA | TensorRT + Fusion Kernels | **单卡延迟最低** (GPU算子优化) |
| **Ollama** | Ollama | llama.cpp | **本地部署最简** (GGUF/CPU) |

- **核心技术原理:**
- **vLLM (PagedAttention)**: 借鉴操作系统虚拟内存理念，将KV Cache切分为固定大小的Block，不再受限于连续显存，解决显存碎片问题，极大提高Batch Size。
- **SGLang (RadixAttention)**: 将KV Cache组织成前缀树，自动识别并共享公共前缀（如System Prompt），对于大量并发相同Prompt的场景（如Coze/Dify后端），显存和计算效率极高。

- **选择建议:**
```
高并发通用服务 (100+ QPS, 聊天)
→ vLLM (生态最成熟，吞吐量天花板)

结构化生成 / 多轮Agent / 长上下文
→ SGLang (RadixCache复用前缀，Token消耗少)

企业级部署 / 模型微调 / 监控
→ TGI (Hugging Face官方，功能最全，稳定)

极致延迟 (<100ms TTFT) / NVIDIA硬件
→ TensorRT-LLM (算子融合，F16/INT8极致优化)

MacBook / 本地开发 / 边缘设备
→ Ollama (GGUF量化，一行命令启动)
```

- **实战案例:**
在一个日均千万调用的客服机器人后端迁移中，我们最初使用 TGI，但在高并发下显存利用率低导致排队严重。切换到 vLLM 后，通过启用 PagedAttention 和调整 `gpu_memory_utilization`，在相同的 4x A100 集群上，QPS 从 45 提升至 110，且 P99 延迟降低了 40%。

- **代码示例:**
```python
# vLLM 启动关键参数 (Python)
from vllm import LLM, SamplingParams

# 实战中必须限制显存占用，防止OOM
llm = LLM(
    model="meta-llama/Llama-2-7b-hf",
    tensor_parallel_size=4,         # 多卡并行
    gpu_memory_utilization=0.90,    # 预留10%显存给CUDA内核
    max_model_len=4096,
    enforce_eager=True              # 调试时开启，生产环境通常用False(CUDA Graph)
)

sampling_params = SamplingParams(temperature=0.7, top_p=0.95, max_tokens=100)
outputs = llm.generate(["Hello, my name is"] * 10, sampling_params)
```

- **## 常见考点**
1. **Continuous Batching (连续批处理)**: 解释从Static Batching到Continuous Batching的演进，以及如何处理不同Sequence结束时间不一致的问题。
2. **KV Cache**: 什么是KV Cache，为什么它是大模型推理显存占用的主要来源，PagedAttention如何解决显存浪费。
3. **Decoding瓶颈**: 推理分为Prefill（首字生成，计算密集）和Decode（后续生成，内存带宽密集），不同引擎针对这两个阶段的优化策略。
4. **Speculative Decoding (推测解码)**: vLLM/TGI中如何利用小模型辅助大模型加速生成。

## 记忆要点

- vLLM：PagedAttention分页管理KV Cache，显存利用率最高，适合高并发通用服务。
- SGLang：RadixAttention前缀树缓存，复用公共Prompt，适合多轮对话和Agent。
- TGI易用全，TensorRT-LLM延迟最低；Ollama适合本地部署。
