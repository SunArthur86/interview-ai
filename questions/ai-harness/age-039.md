# Agent 的 Memory 系统如何设计?短期记忆和长期记忆怎么区分

**Agent Memory 设计** 是生产级 Agent 的核心难题之一.

**短期记忆:**
*   **存储位置:** LLM 上下文窗口或 KV Cache.
*   **包含内容:** 当前对话历史、工具调用结果、中间推理步骤.
*   **生命周期:** 单次任务或单轮对话.
*   **管理策略:** 滑动窗口截断、基于重要性的摘要、Token 预算动态分配.

**长期记忆:**
*   **存储位置:** 外部存储系统(向量数据库、关系数据库、图数据库).
*   **包含内容:** 用户偏好、历史决策、学到的知识、项目结构索引.
*   **生命周期:** 跨会话持久化.
*   **管理策略:** 语义检索、时间衰减、定期更新与归档.

**数据流向示意图:**
```text
┌──────────────────────────────────────────────────────────────┐
│                       Agent Core                              │
│  ┌──────────────┐      ┌──────────────┐                      │
│  │   LLM Inference      │   Controller │                      │
│  │  (Working Mem)      │              │                      │
│  └───────┬──────┘      └──────┬───────┘                      │
│          │   Write/Read         │ Write/Read                 │
│          ▼                      ▼                            │
│  ┌──────────────────────────────────────┐                    │
│  │     Memory Interface (Read/Write)    │                    │
│  └──────────────┬───────────┬───────────┘                    │
│                 │           │                                 │
└─────────────────┼───────────┼─────────────────────────────────┘
                  │ Write     │ Search/Retrieve
                  ▼           ▼
          ┌──────────────┐ ┌──────────────┐
          │ Vector Store │ │  SQL/Graph   │
          │ (Semantic)   │ │ (Structured) │
          └──────────────┘ └──────────────┘
```

**设计要点:**
1.  **记忆写入:** 什么信息值得记住?通过反思机制提取关键洞察,避免存入噪声.
2.  **记忆检索:** 如何高效找到相关记忆?混合检索(关键词+向量) + 时间衰减 + 重排序.
3.  **记忆更新:** 如何处理冲突信息?基于时间戳的版本控制或置信度比较.
4.  **遗忘机制:** 如何淘汰过时记忆?基于访问频率(LRU)和信息熵值.

**常见实现:** MemGPT、LangChain Memory、Letta(前 MemGPT).

### 实战深化

**1. 实战案例:**
曾遇到一个私有知识库问答 Agent，系统错误地将用户随口说的“以后不要这样了”当成了业务规则写入长期记忆，导致后续正确查询被屏蔽。解决方案是增加“记忆审核层”，只有 LLM 判定为高置信度的事实或指令才入库，且支持用户显式“撤回”记忆。

**2. 代码示例 (Python - 反思式写入):**
```python
def write_memory(user_input: str, agent_response: str):
    # 1. 提取: 仅提取重要的结构性信息，忽略闲聊
    reflection_prompt = f"Extract facts/preference from: {user_input}"
    insights = llm_generate(reflection_prompt)
    
    if not insights: return # 过滤噪声

    # 2. 写入: 混合存储 (向量用于检索，KV用于精确匹配)
    vector_db.insert(embed(insights), metadata={"type": "preference"})
    kv_store.put(f"user_pref_{date}", insights)
```

**3. 记忆类型对比:**

| 特性 | 短期记忆 | 长期记忆 |
| :--- | :--- | :--- |
| **存储介质** | Context Window / KV Cache | Vector DB / SQL / Graph DB |
| **访问速度** | 极快 (纳秒/毫秒级) | 慢 (毫秒/秒级，涉及 I/O) |
| **容量限制** | 极小 (通常 < 128k tokens) | 极大 (几乎无限) |
| **持久性** | 易失性 (会话结束即丢) | 持久化 (永久保存) |
| **主要作用** | 维持对话连贯性、推理上下文 | 积累经验、个性化、知识库 |
| **成本** | 高 (按 Token 计费) | 低 (存储便宜，检索产生少量 Token 成本) |

## 常见考点
*   向量数据库检索精度不够时，如何优化召回率？（如：Hybrid Search, Re-ranking）
*   如何解决“记忆污染”问题（错误信息被存入长期记忆）？
*   RAG 和 Long-term Memory 的界限在哪里？
