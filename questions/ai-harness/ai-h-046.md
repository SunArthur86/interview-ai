# LLM应用的A/B测试如何设计?统计显著性如何保证

**LLM应用A/B测试设计与统计**

**1. 定义指标 (增强)**
- **主指标**: 
  - **业务指标**: 任务成功率、点击率 (CTR)、转化率。
  - **用户主观**: 评分 (1-5)、CSAT (客户满意度)、 thumbs_up/down 比例。
- **辅助指标**: 
  - **性能**: 端到端延迟 (TP50, TP99)、首字生成时间 (TTFT)。
  - **成本**: 平均 Token 消耗、单次调用 API 成本。
- **护栏指标**: 
  - **安全**: 拒答率、幻觉率 (需用 LLM-as-a-Judge 评估)、合规违规次数。

**2. 样本量计算 (公式推导)**
```
# 双样本比例检验的最小样本量 (每组)
n = (Z_α/2 + Z_β)² * (p1(1-p1) + p2(1-p2)) / (p1 - p2)²

# 简化工程近似公式 (80% 功效, 95% 置信度)
n ≈ 16 * σ² / Δ²

参数说明：
- σ² = p * (1-p) (方差，p 为基准转化率)
- Δ = MDE (最小可检测效应 Minimum Detectable Effect)

例: 基准 p=0.5 (σ=0.5), 期望提升 10% (Δ=0.05)
→ n ≈ 16 * 0.25 / 0.0025 = 1600 per group
```

**3. 流量分配与实验架构**
```
用户请求 → 分桶层
├─ 哈希(User_ID) % 100
│  ├─ [0-49]: 版本 A (Control)
│  └─ [50-99]: 版本 B (Treatment)
├─ 白名单机制 (强制分流)
└─ 灰度发布: 1% → 5% → 50%
```
- **分桶粒度**: 必须基于 **User_ID** 而非 Request_ID，保证同一用户体验一致性。
- **SRM (Sample Ratio Mismatch) 检验**: 启动后需检查分流比是否符合 50/50，卡方检验 p-value < 0.05 说明分桶系统有 Bug。

**4. 统计检验代码与解读**
```python
from scipy import stats
import numpy as np

def test_significance(conversions_a, total_a, conversions_b, total_b):
    # 1. Z-test for proportions
    count = np.array([conversions_a, conversions_b])
    nobs = np.array([total_a, total_b])
    z_stat, p_value = stats.proportions_ztest(count, nobs)
    
    # 2. 计算置信区间
    p1 = conversions_a / total_a
    p2 = conversions_b / total_b
    se = np.sqrt(p1*(1-p1)/total_a + p2*(1-p2)/total_b)
    ci_low = (p2 - p1) - 1.96 * se
    ci_high = (p2 - p1) + 1.96 * se
    
    return {
        "p_value": p_value,
        "is_significant": p_value < 0.05,
        "uplift": (p2 - p1) / p1,
        "ci": (ci_low, ci_high)
    }
```

**5. 常见陷阱 (深度解析)**
- **多重检验问题**: 
  - 场景：同时测了 10 个指标，发现其中一个显著。
  - 对策：使用 **Bonferroni 校正** (α = 0.05 / 10 = 0.005) 或 **False Discovery Rate (FDR)** 控制。
- **新奇效应**: 用户因为新模型好玩而暂时活跃，数据需观察 7-14 天稳定后再结论。
- **辛普森悖论**: 
  - 场景：整体看 B 比 A 好，但细分到“移动端”和“桌面端”都是 A 好。
  - 原因：流量分布不均（如 B 分到了更多高质量的移动端用户）。
  - 对策：必须进行 **分层分析**。

**## 常见考点**
1. **功效**: 如果没测出显著差异，是因为模型没效果，还是样本量不够？（答：计算 Statistical Power，通常要求 0.8 以上）
2. **非参数检验**: 如果指标数据不符合正态分布（如延迟分布长尾），用什么检验？（答：Mann-Whitney U test 或 Bootstrap）
3. **LLM 特有的评价**: 如何自动化评估 A/B 两组的回答质量？（答：使用 LLM-as-a-Judge，如 GPT-4 打分，再比较分数分布）
4. **P-hacking**: 如何避免“试探性”地挖掘显著数据？（答：提前注册实验，只看主指标，不事后找理由）
