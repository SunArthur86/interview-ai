# Prompt版本管理最佳实践?如何做Prompt的A/B测试

- **Prompt版本管理方案:**

| 方案 | 适用 | 优势 |
|------|------|------|
| Git + 文件 | 小团队 | 简单直观 |
| LangSmith | LangChain | 内置版本+评估 |
| Promptflow | 企业 | 微软开源方案 |
| Langfuse | 开源 | 自托管+实验 |
| DSPy | 优化型 | **自动优化** |

- **A/B测试流程:**
1. 定义指标(准确率/用户满意度/延迟)
2. 创建Prompt变体(v1当前, v2改进版)
3. 流量分割(50/50 或 10%金丝雀)
4. 收集数据(N>=100样本)
5. 统计检验(t-test / 序贯检验)
6. 决策(v2显著优于v1则全量切换)

- **A/B 测试流量分配架构:**

```text
User Request
    │
    ▼
┌───────────────┐
│  Router/UA    │ (Hash(UserID) % 100)
└───────┬───────┘
        │
   ─────┴─────
   │          │
   ▼          ▼
Group A     Group B
(v1 Prompt) (v2 Prompt)
   │          │
   └────┬─────┘
        ▼
  Evaluation Store
```

- **实战案例：** 某电商客服上线了新的 Prompt，但导致“退换货”意图识别率骤降。通过引入 Prompt 中间件，按 User ID 尾号进行灰度，发现新 Prompt 虽然提升了语气友好度（人工评分+10%），但因为指令过长增加了 Token 消耗且干扰了分类，最终决定保留旧版并仅微调语气部分。

- **关键代码示例 (带版本的 Prompt 模板):**
```python
from jinja2 import Template

PROMPT_REGISTRY = {
    "v1": Template("你是客服助手。请回答：{{user_input}}"),
    "v2": Template("你是专业客服助手，语气要热情。指令：{{user_input}}")
}

def get_prompt(version: str, **kwargs):
    if version not in PROMPT_REGISTRY:
        raise ValueError(f"Unknown version: {version}")
    return PROMPT_REGISTRY[version].render(**kwargs)
```

- **版本管理策略对比：**

| 策略 | 优点 | 缺点 |
| :--- | :--- | :--- |
| **Git + Config** | 极低门槛，无需依赖 | 缺乏运行时关联分析，难回滚 |
| **Prompt Engineering IDE (e.g. PromptLayer)** | 图形化对比，自动日志 | 依赖外部服务，有迁移成本 |
| **代码化**

- **关键原则:**
1. Prompt和代码分离,每次修改有版本号+变更说明
2. 生产prompt需要审批
3. 可以快速回滚

- **## 常见考点**
1. 如何处理 Prompt 中的变量插值版本？（使用 Jinja2 模板版本化而非仅版本化纯文本）
2. 什么是“Prompt 飘移”，如何监控？（通过计算 Embedding 余弦相似度监控语义变化）
3. 在 DSPy 中，版本管理是如何工作的？（DSPy 自动优化 Prompt 中的“指令”部分，优化后的签名需单独版本化）
