# 什么是 Golden Set 和 LLM-as-Judge?如何在生产中构建 AI 评测体系

- **AI 评测体系** 是确保 Prompt 和模型变更不引入回归的核心手段.

- **Golden Set(黄金测试集):**
  - 人工标注的高质量测试用例(问题 + 标准答案 + 评分标准)
  - 覆盖核心场景和边界情况,通常 100-1000 条
  - 每次 Prompt 或模型变更后跑一遍
  - 用通过率衡量变更安全性

- **LLM-as-Judge:**
  - 用 GPT-4 或 Claude 评估另一个模型的输出质量
  - 评分维度:正确性、完整性、相关性、安全
  - 优势:比人工评估快 100 倍,比规则评估灵活
  - 风险:评判模型自身有偏差

- **评测流程:**
  1. 构建 Golden Set(人工标注)
  2. 每次变更跑 Golden Set
  3. 用 LLM-as-Judge 自动打分
  4. 低分样本人工复核
  5. 变更通过率 > 95% 才上线

- **Trace 回放:** 用历史真实请求的 Trace 数据,在新版本上回放,确保不影响线上体验.

### 实战深化

**实战案例:** 在优化 RAG 系统检索算法时，通过 LLM-as-Judge 运行 500 条 Golden Set，发现准确率提升了 5%。但在上线后，用户投诉增多。原因是 Judge 模型偏好信息详尽的回答（即使包含幻觉），而用户更看重简洁准确。通过调整 Judge 的 Prompt 强调“准确性优先”解决了该问题。

**代码示例 (Python - LLM-as-Judge 评估):**
```python
eval_prompt = """
You are an AI judge. Rate the following Answer based on the Reference Answer.
Score: 0 (Bad) to 5 (Perfect).
Focus on Factuality.

Reference: {reference}
Answer: {answer}

Respond with JSON: {{"score": <int>, "reason": "<string>"}}
"""

def evaluate_output(reference, answer):
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": eval_prompt.format(reference=reference, answer=answer)}],
        response_format={"type": "json_object"}
    )
    return json.loads(response.choices[0].message.content)
```

**评测方式对比表格:**

| 维度 | Golden Set (黄金集) | Trace 回放 | 在线 A/B 测试 |
| :--- | :--- | :--- | :--- |
| **数据来源** | 人工构造的典型/边界 Case | 生产环境历史真实流量 | 生产环境实时流量 |
| **执行速度** | 快 (可控) | 较慢 (数据量大) | 慢 (需长期运行) |
| **覆盖广度** | 聚焦核心场景 | 覆盖长尾场景 | 全量真实场景 |
| **评估成本** | 低 (仅 API 费用) | 中 (计算资源) | 高 (分流导致业务风险) |
| **主要用途** | 开发阶段回归测试 | 回归测试/数据挖掘 | 验证最终效果/商业指标 |
| **结果可信度** | 高 (人工标准) | 中 (依赖无监督评估) | 最高 (真实用户反馈) |

## 常见考点
1. **LLM-as-Judge 的准确性如何保证?**
   可以使用“黄金参考集”来校准 Judge 模型。即让 Judge 评估一批已知分数的数据,计算其与真实分数的相关性。此外,可采用“多数投票法”让多个不同的 Judge 模型评估同一条数据,取平均分.

2. **如何评估开放式生成任务(如创意写作)?**
   这类任务很难有唯一标准答案。通常采用“相对评估”模式,让 Judge 模型对比 V1 和 V2 的输出,判断哪个更好,而不是打绝对分数.

3. **自动化评测与真实用户体验的偏差?**
   自动评测可能过分关注“事实正确性”,而忽略了用户体验中的“风格”、“交互流畅度”等软性指标,需结合人工复核。
