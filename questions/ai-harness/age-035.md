# Agent Skill 是什么?如何设计可复用的 Agent 技能

- **Agent Skill** 是 Agent 的可复用能力单元,封装了特定任务的 prompt、工具、知识.

- **设计原则:**

- **1. 单一职责:** 每个 Skill 只做一件事(如搜索文档、生成代码、分析数据),避免 Skill 之间的职责重叠.

- **2. 声明式接口:** Skill 定义清晰的输入输出 schema,Agent 通过接口调用而非硬编码.

- **3. 版本管理:** Skill 需要版本控制,因为 prompt 和工具定义会迭代更新.生产环境需要支持灰度发布和回滚.

- **4. 上下文治理:** 每个 Skill 应控制注入的上下文量,避免 Skill 组合时上下文溢出.

- **评估指标:** Skill 的独立调用成功率、平均执行时间、Token 消耗、与基线 Prompt 的效果对比.

- **实践案例:** Claude Code 的 Superpowers 插件系统、Hermes Agent 的 Skill 系统都是 Agent Skill 的工程化实践.

- **Agent Skill 内部结构图:**

```text
┌─────────────────────────────────────┐
│           Agent Skill Definition     │
├─────────────────────────────────────┤
│  Meta:                              │
│    - Name: "Web_Searcher"           │
│    - Version: "v1.2.0"              │
├─────────────────────────────────────┤
│  Interface (Schema):                │
│    Input:  { query: str, limit: int }│
│    Output: { results: list[Item] }   │
├─────────────────────────────────────┤
│  Implementation:                    │
│    - System Prompt: "你是搜索专家..." │
│    - Tools: [Google_API, Parser]     │
│    - Knowledge: [Search_Tips.pdf]    │
├─────────────────────────────────────┤
│  Config:                             │
│    - Timeout: 10s                    │
│    - Max_Tokens: 2000                │
└─────────────────────────────────────┘
```

- **对比表格:**
| 维度 | 函数/传统 API | Agent Skill |
| :--- | :--- | :--- |
| **执行主体** | 机器代码 (确定逻辑) | LLM (概率逻辑) |
| **输入输出** | 强类型 | 弱类型/Schema 描述 |
| **维护成本** | 低 (逻辑固定) | 高 (Prompt 调优) |
| **容错性** | 严格抛异常 | 具备一定的“理解”和纠错能力 |
| **可组合性** | 硬编码调用 | 动态编排 |

- **实战案例:** 在构建客服系统时，设计了一个 `Refund_Skill`。初期没有约束输入 Schema，导致 Agent 经常传入“我想退款”这种模糊意图而非 `order_id`，Skill 调用成功率仅 30%。引入 Pydantic 强类型校验后，模型在调用前会自动引导用户提供 `order_id`，成功率提升至 95%。

- **代码示例:**
```python
from pydantic import BaseModel, Field

# 定义 Skill 的强类型输入 Schema
class SearchInput(BaseModel):
    query: str = Field(description="搜索关键词，必须包含产品名称")
    limit: int = Field(default=5, ge=1, le=10)

class WebSearchSkill:
    def __init__(self):
        self.description = "用于搜索最新技术文档"
        # 将 Schema 暴露给 Agent Framework
        self.args_schema = SearchInput 

    def run(self, input: SearchInput):
        return google_search_api(input.query, input.limit)
```

- **## 常见考点:**
1. **Schema 强类型**: 如何使用 Pydantic 或 JSON Schema 严格约束 Skill 的输入输出，以减少 LLM 的解析错误？
2. **Skill 注册与发现**: 在微服务或插件化架构中，Skill 是如何注册到 Agent 的？如何处理 Skill 冲突（同名或功能重复）？
3. **上下文传递**: Skill 之间如何传递中间结果？是通过全局状态机还是显式的参数传递？
4. **组合模式**: 多个 Skill 组合时，如何处理依赖关系（如 Skill B 依赖 Skill A 的输出）？是使用 DAG 还是手动编排？
