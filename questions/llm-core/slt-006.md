# BPE Tokenizer的原理是什么？

BPE（Byte Pair Encoding）是子词分词算法，是目前LLM最主流的Tokenizer（GPT, Llama, Qwen等均使用其变体）。

**核心思想：**
将单词拆分为最小的字符单元，然后迭代地统计并合并频率最高的相邻子词对，直到达到预设的词表大小。这使得常用词保持独立，罕见词被拆分为有意义的词根和词缀。

**训练过程（Greedy Merge）：**
1.  **准备语料**：准备大量的文本语料，将所有单词拆分为字符序列（包含结束符</w>）。
2.  **统计频率**：统计语料中所有相邻字节/字符对的出现频率。
3.  **合并**：找到频率最高的一对（例如 'e' 和 'r'），将其合并为一个新的子词（'er'），加入词表。
4.  **更新语料**：用新子词替换语料中所有对应的字符对。
5.  **循环**：重复步骤2-4，直到词表大小达到设定值（如32k或100k）。

**示例：**
初始词表：{'l', 'o', 'w', 'e', 'r', 's', 't'}
语料：'low', 'lower', 'lowest'
1. 'e', 'r' 频率高 -> 合并为 'er'
2. 'l', 'o' 频率高 -> 合并为 'lo'
3. 'lo', 'w' 频率高 -> 合并为 'low'
最终可能拆解为：'low', 'est', 'er'。

**BPE vs WordPiece vs Unigram：**
- **BPE**：基于频率统计的贪婪合并，最常用。
- **WordPiece（BERT）**：基于最大化训练数据的似然概率来选择合并。
- **Unigram（T5, ALBERT）**：从大词表开始，不断删除导致似然下降最小的词，保留小词表。

**技术细节与问题：**
- **多语言支持**：GPT-2/3 BPE对多语言支持较差（文本效率低）。Llama等后续模型使用**SentencePiece**（将空格视为字符_）或**BBPE (Byte-level BPE)**，直接在字节层面操作，解决生僻字和多语言乱码问题。
- **数字敏感度**：传统BPE容易将数字切散（如 '123' -> '1', '2', '3'），导致算术能力弱。改进方法（如Llama 3）将单独的数字片段加入词表以提升算术能力。
- **词表大小权衡**：词表越大，编码效率越高（序列越短），但模型Embedding层参数量越大，训练数据变得稀疏。

### 实战深化

**实战案例**：
在做金融类大模型Tokenization时，我们发现常规BPE将长ID（如“688981”）切分得很碎，导致模型难以精准检索。通过在训练语料中人为注入特定格式的高频数据，强制BPE将常见股票代码合并为单个Token，显著提升了RAG检索的准确率。

**代码示例 (Python)**：
```python
import re
from collections import defaultdict

def get_stats(vocab):
    pairs = defaultdict(int)
    for word, freq in vocab.items():
        symbols = word.split()
        for i in range(len(symbols)-1):
            pairs[symbols[i], symbols[i+1]] += freq
    return pairs

def merge_vocab(pair, vocab):
    v_out = {}
    bigram = re.escape(' '.join(pair))
    p = re.compile(r'(?<!\S)' + bigram + r'(?<!\S)')
    for word in vocab:
        w_out = p.sub(''.join(pair), word)
        v_out[w_out] = vocab[word]
    return v_out
```

**对比表格**：

| 特性 | BPE (Byte Pair Encoding) | WordPiece (BERT) | Unigram (T5/ALBERT) |
| :--- | :--- | :--- | :--- |
| **构建方向** | 自底向上（从字符开始合并） | 自底向上（选择最大化似然的合并） | 自顶向下（从大词表开始剪枝） |
| **合并/剪枝标准** | 统计相邻字节对最高频率 | 最大化训练语言模型的可能性 | 最小化语言模型损失的变化量 |
| **复杂度** | 低，速度快 | 中，需计算LM似然 | 高，需反复训练LM评估 |
| **常用模型** | GPT系列, Llama, RoBERTa | BERT, DistilBERT | T5, mBART, ALBERT |
| **处理未知词** | 拆分为字符级序列 | 拆分为字符级序列 | 使用[UNK]或拆分为子词 |

## 记忆要点

- 核心思想：自底向上，迭代合并频率最高的相邻字符对直到词表满。
- 效果：常用词独立，罕见词拆为词根词缀，平衡词表大小与语义粒度。
- 对比：BPE看频率（GPT/Llama），WordPiece看似然（BERT），Unigram做剪枝（T5）。
