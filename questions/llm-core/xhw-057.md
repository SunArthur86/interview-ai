# Cross-Attention交叉注意力

### Cross-Attention (交叉注意力)

#### 1. 定义
Cross-Attention 是指 Query (Q) 来自一个序列，而 Key (K) 和 Value (V) 来自另一个序列的注意力机制。它主要用于融合两个不同模态或不同来源的信息。

#### 2. Transformer 中的应用
在标准的 Transformer Encoder-Decoder 架构（如原始的 Transformer 用于机器翻译）中：

- **Decoder 端**：使用 Masked Self-Attention 处理已生成的目标序列。
- **Cross-Attention 层**：
    - **Q**：来自 Decoder 的上一层输出（代表目标语言）。
    - **K, V**：来自 Encoder 的最终输出（代表源语言）。

这使得 Decoder 在生成每一个词时，都能去“查阅”源语言句子中最相关的部分。

#### 3. 计算公式
$$Attention(Q_{dec}, K_{enc}, V_{enc}) = \text{softmax}(\frac{Q_{dec}K_{enc}^T}{\sqrt{d_k}})V_{enc}$$

#### 4. 架构数据流图

```text
Encoder (Source)              Decoder (Target)
┌──────────────┐             ┌──────────────────┐
│  Source Seq  │             │  Target Seq (t)  │
│  Embeddings  │             │  Embeddings      │
└──────┬───────┘             └──────┬───────────┘
       │                            │
   Self-Attn                   Masked Self-Attn
       │                            │
       v                            v
┌──────────────┐             ┌──────────────────┐
│ Encoder Out  │───────┬────>│   Linear (Q)     │<─ Decoder Output
│ (用作 K, V)  │       │     └────────┬─────────┘
└──────────────┘       │              │
                       │    ┌─────────▼─────────┐
                       │    │ Cross-Attention   │
                       │    │ (Q, K_enc, V_enc) │
                       │    └─────────┬─────────┘
                       │              │
                       └──> K/V <─────┘
```

#### 5. 应用场景
- **机器翻译**：源语言 -> 目标语言。
- **图像描述**：图像特征 (CNN 输出) 作为 K/V，文本作为 Q。
- **多模态大模型**：文本 Q 去查询图像/视频编码后的 K/V。

#### 6. 实战补充

**实战案例**：在 Stable Diffusion 的去噪过程中，Cross-Attention 机制让文本提示词精准控制图像生成；若 Cross-Attention 权重被错误缩放，生成的图像将忽略文本提示或出现错误的特征混合。

**代码示例**：
```python
def cross_attention(q, k, v):
    # q: [batch, target_len, d_model]
    # k, v: [batch, source_len, d_model]
    scores = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(d_k)
    attn_weights = F.softmax(scores, dim=-1)
    output = torch.matmul(attn_weights, v)
    return output
```

## 常见考点
1.  **Self-Attention 和 Cross-Attention 的主要区别是什么？计算量上呢？**
    *   **回答思路**：Self-Attention 的 Q、K、V 均来自同一序列，计算复杂度是 $O(N^2)$（N 为序列长度）；Cross-Attention 中 Q 来自序列 A（长 N），K/V 来自序列 B（长 M），计算复杂度是 $O(N \times M)$。在机器翻译中，Decoder 的序列长度 N 通常小于 Encoder 的 M，因此计算量受限于两者乘积。
2.  **在 RAG（检索增强生成）中，使用的是什么注意力机制？**
    *   **回答思路**：本质上是 Cross-Attention。Prompt 和问题被处理成 Query，而检索到的文档片段经过 Encoder 编码后作为 Key 和 Value 传入模型的 Cross-Attention 层（或通过拼接输入，但在架构上属于 Fusion 机制）。
