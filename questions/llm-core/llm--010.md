# 在大模型强化学习对齐中,如何解决信用分配(Credit Assignment)问题?token 级别和序列级别的奖励有什么不同

- **信用分配问题**: 给定一个序列级奖励(如'这个回答好/差')，如何确定序列中哪些 token 贡献了正向/负向价值? 这是 RLHF 中的核心难题。

- **实战案例**：在训练数学模型时，我们发现虽然模型最终答案正确，但推理过程包含幻觉。使用 GAE 时会出现“沾光”现象，即错误推理步骤因为结果正确反而被增强，后期引入 PRM 才解决了步骤级纠偏问题。

- **序列级奖励**
- 整个回答获得一个标量 reward。
- 优点: 标注成本低(人类只需整体评分)。
- 缺点: 无法区分序列中哪部分好哪部分差，梯度更新可能波及到无辜的 token。

- **Token 级奖励**
- 序列中每个 token 获得独立 reward。
- 优点: 精确指导哪些 token 该加强/减弱。
- 缺点: 标注成本极高，通常无法直接获得。

- **主流信用分配方法对比**

| 方法 | 优势估计来源 | 是否需 Value Model | 训练稳定性 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **GAE** | 基于时序差分 (TD) | 是 | 高 (偏差-方差平衡) | 通用 RLHF，对长序列效果好 |
| **GRPO** | 组内相对排名 | 否 | 中 (方差较大) | 资源受限、快速验证的场景 (如 DeepSeek-V3) |
| **PRM** | 步骤级监督信号 | 否 (需 RM) | 高 (信号强) | 复杂逻辑推理、数学任务 |

1. **GAE (Generalized Advantage Estimation)**
- PPO 标准方法，用 Value Function 估计每个 token 的价值。
- 公式：$A_t = \sum_{l=0}^{\infty}(\gamma \lambda)^l \delta_{t+l}$。
- 需要 value model(额外参数和计算)，通过偏差和方差的平衡来估计优势。

2. **GRPO 的组内相对优势**
- 同一 prompt 采样 G 个回答，用组内排名作为优势 (e.g., 第1名得正优势，最后一名得负优势)。
- 不需要 value model，简化了训练流程，但组大小 G 影响估计质量。

3. **过程奖励模型 (Process Reward Model)**
- 对推理过程的每一步打分(如 OpenAI 的 Math PRM)。
- 精确但需要大量步骤级标注数据，常用于数学推理（如 GSM8K）。

4. **隐式 token 归因**
- 用梯度或注意力权重推断哪些 token 对结果贡献最大。

- **关键代码片段 (GAE 计算)**
```python
# PyTorch 实现 GAE 计算
def compute_gae(rewards, values, dones, gamma=0.99, lambda_=0.95):
    advantages = []
    last_advantage = 0
    # 逆序遍历时间步
    for t in reversed(range(len(rewards))):
        delta = rewards[t] + gamma * values[t+1] * (1 - dones[t]) - values[t]
        last_advantage = delta + gamma * lambda_ * (1 - dones[t]) * last_advantage
        advantages.insert(0, last_advantage)
    return torch.tensor(advantages, dtype=torch.float32)
```

## 记忆要点

- 信用分配定义：将序列级 Reward 合理分配给每个 Token，确定贡献。
- 序列级 Reward：标注成本低，但无法区分好坏步骤，易误伤。
- Token 级 Reward：指导精确，但标注成本极高，需 PRM 等辅助。
- 主流方案：GAE (Value 估计)、GRPO (组内排名)、PRM (步骤打分)。
