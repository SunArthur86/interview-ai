# 什么是 Embedding(向量化)?它在 AI 中有什么用

- **Embedding** 是将文本(或图片)转换为固定维度的向量(如 768 维浮点数组),使语义相近的内容在向量空间中距离也近.

- **原理细节**
  - **映射机制**: 神经网络将高维稀疏的 One-hot 编码（词汇表大小）映射到低维稠密的实数向量空间。
  - **几何意义**: 向量的方向代表语义特征，向量的模长通常代表信息量或强度（取决于归一化方式）。
  - **距离度量**: 常用余弦相似度衡量方向一致性，欧氏距离衡量绝对距离。

- **用途**
  1. **语义搜索**:'苹果手机' 和 'iPhone' 向量距离近。
  2. **RAG 检索**:从知识库中找最相关的文档。
  3. **聚类分类**:相似内容自动分组。
  4. **推荐系统**:基于向量相似度推荐。

- **直觉理解**
  - king - man + woman ≈ queen
  - 向量空间中'语义关系'可以用方向表示。

```text
       语义向量空间示意图
           
           |          (King)
           |           *
           |          / 
           |         /   
    (Man)  *--------/----* (Queen)
           |       /    /
           |      /    /
           |     /    /
      (Woman)*    /   /
           |    /   /
           |   /  /
           |  / /
           | / / 距离近 = 语义相似
           |//
``` 

- **常用模型**:BGE、text-embedding-ada-002、E5、M3E

- **一句话理解**:Embedding = 把文字变成'坐标',让计算机用距离来衡量'语义相似度'.

- **Embedding 模型选型对比**

| 模型 | 特点 | 适用场景 |
| :--- | :--- | :--- |
| **text-embedding-3** | OpenAI 官方，多语言支持好，效果稳定 | 通用场景，追求快速集成 |
| **BGE-M3** | 支持多语言、多功能（稠密/稀疏/多向量） | 跨语言检索，复杂语义匹配 |
| **E5** | 基于对比学习，指令微调增强 | 需要特定指令引导的检索 |
| **M3E** | 中文语境优化，轻量级 | 资源受限环境，纯中文应用 |

- **实战案例**：在做“电商商品搜索”时，直接用商品标题做 Embedding 效果很差（如“iPhone 15”和“苹果手机”距离远）。改进方案是将用户搜索词和商品标题拼接成指令格式（如 Query: `为这个查询生成表示：手机`），检索准确率提升了 25%。

```python
from sentence_transformers import SentenceTransformer, util

# 加载支持指令微调的 E5 模型
model = SentenceTransformer('intfloat/multilingual-e5-large')

#实战：添加指令前缀以区分 Query 和 Document
query_embedding = model.encode('query: 如何更换显卡', normalize_embeddings=True)
doc_embedding = model.encode('passage: 拆开机箱侧板，找到PCIe插槽...步骤如下', normalize_embeddings=True)

# 计算余弦相似度
similarity = util.cos_sim(query_embedding, doc_embedding)
print(f"相似度得分: {similarity.item():.4f}")
```

## 常见考点
1.  **Word2Vec 与 Transformer 产生的 Embedding 有什么区别？** （静态 vs 上下文动态 contextualized，如 BERT 生成的是基于上下文的 Embedding）。
2.  **向量的维度如何选择？** （维度越高表达能力越强，但计算和存储成本越大，存在边际效应递减）。
3.  **检索时用 Cosine Similarity 还是 Dot Product？** （如果向量已经 L2 归一化，两者等价；Dot Product 计算更快）。
4.  **什么是 Matryoshka Representations（俄罗斯套娃表征）？** （一种支持灵活截断维度的 Embedding 技术，适配不同存储和计算需求）。
