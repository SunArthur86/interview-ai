# 大海捞针测试和概率探针分别是什么

### 1. 大海捞针

**定义**：大海捞针测试着重考察模型在超长上下文（Long Context）里的信息提取和理解能力。通过在长文本中随机插入关键信息（“针”）形成 Prompt，检测模型是否能从大量冗余信息（“干草堆”）中准确提取出这些关键信息。

**测试方法**：
1.  准备一段较长文本（数千词以上），混合各种冗余或无关信息。
2.  随机插入关键细节（如特定的 UUID、日期、陈述）。
3.  要求模型回答必须依赖文中关键信息的问题，评估其召回准确性。

**常见变体**：
*   **单一信息检索**：文章中只藏了一个关键信息，评估精确回忆能力。
*   **多信息检索**：埋了多个分散的关键信息，需要综合检索。
*   **多信息推理**：检索多个关键信息后，需进一步推理整合才能得出结果。
*   **祖先追溯挑战 (ATC)**：类似复杂的家谱推理，需要多层逻辑串联，去掉无关文本设定，要求模型综合运用所有内容进行推理。

### 2. 概率探针

**定义**：精准观察模型是否学到了某个事实或对特定句子的概率分布变化。适用于评估模型的知识内化程度或价值观对齐情况。

**实施步骤**：
1.  人工设计或挑选带有特定知识/立场的句子（如“北京是中国的首都”、“台湾属于中国”等）。
2.  查看模型对特定 Token 的预测概率或整句的 PPL。
3.  **对比观察**：
    *   事实准确性：$P(“北京” \mid “中国的首都是”)$ 是否持续增加？
    *   PPL 对比：PPL('正确事实') 是否持续下降，且 PPL('错误事实') 是否上升？
    *   偏好对齐：$PPL('A观点') > PPL('B观点')$ 是否符合预期？

**核心价值**：探针通常需手动构造测试集，重点观察指标随训练迭代的**变化趋势**，而非绝对大小，从而判断模型是否“学会”了特定知识。

### 3. 实战案例与代码

**实战案例**：在训练 128k 长窗口的模型时，**大海捞针**测试发现“针”放在文档开头和结尾时准确率 100%，但放在中间 50%-70% 的位置时准确率骤降。这暴露了 Rotary Position Embedding (RoPE) 在中段位置编码的插值能力不足（Attention Sinking 问题），迫使我们调整位置编码插值策略。对于**概率探针**，常用于检测安全微调后的模型是否遗忘：例如观察“如何制造武器”的 PPL 是否显著上升，而“如何做饭”的 PPL 保持稳定。

**代码示例**：
```python
# 伪代码：概率探针检查特定知识
def probe_knowledge(model, tokenizer, prompt, target_token):
    inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
    with torch.no_grad():
        outputs = model(**inputs)
    
    # 获取最后一个 token 的 logits
    next_token_logits = outputs.logits[0, -1, :]
    
    # 计算 target_token 的概率
    target_id = tokenizer.convert_tokens_to_ids(target_token)
    log_probs = F.log_softmax(next_token_logits, dim=-1)
    prob = torch.exp(log_probs[target_id]).item()
    
    return prob
```
