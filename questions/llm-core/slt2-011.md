# 投机采样（Speculative Decoding）是什么？

投机采样用一个小模型快速生成候选token，大模型并行验证。

**流程**：
1. 小模型自回归生成k个候选token（速度快）
2. 大模型一次性并行计算这k个token的logits（并行，只需一次forward）
3. 对比大小模型输出：
   - 如果小模型生成的token与大模型一致 → 接受
   - 如果不一致 → 从第一个不一致处拒绝，用大模型的分布重新采样

**优势**：
- 数学上等价于直接用大模型采样（无损）
- 小模型猜对的部分不需要大模型自回归计算
- 典型加速2-3倍

**变体**：
- Medusa：多个头同时预测多个未来token（不需要小模型）
- EAGLE：用LLM自身的隐状态预测
- Lookahead Decoding：基于Jacobi迭代并行解码

### 实战案例
在客服机器人系统中，使用Medusa投机采样策略后，由于Draft模型（或额外预测头）的接受率通常在60%-70%之间，P99延迟显著降低。但在处理代码生成或数学逻辑推理任务时，由于Draft Model猜测能力下降，接受率跌破30%，此时反而增加了推理开销，因此需要根据任务场景动态开关此功能。

### 代码示例 (验证逻辑示意)
```python
# 投机采样验证伪代码 (Python)
import torch

def verify_draft(target_model, draft_tokens, prompt_ids):
    # 1. 构造大模型输入: Prompt + Draft Tokens
    full_input = torch.cat([prompt_ids, draft_tokens])
    
    # 2. 大模型并行计算 (一次 Forward)
    logits = target_model(full_input).logits  # shape: [seq_len, vocab_size]
    
    # 3. 逐位验证
    accepted_len = 0
    for i in range(len(draft_tokens)):
        # 获取大模型在对应位置的预测概率
        target_prob = torch.softmax(logits[i + len(prompt_ids)], dim=-1)
        target_token = torch.argmax(target_prob)
        
        if target_token == draft_tokens[i]:
            accepted_len += 1
        else:
            # 拒绝：从该位置开始按大模型分布重新采样
            next_token = torch.multinomial(target_prob, num_samples=1)
            return accepted_len, next_token
    return accepted_len, None
```

### 对比表格：Speculative Decoding 变体对比
| 算法 | Draft 机制 | 优势 | 挑战 |
| :--- | :--- | :--- | :--- |
| **Classic Speculative** | 独立小模型 (如 7B Draft -> 70B Target) | 架构灵活，可跨模型 | 需要加载两个模型，显存开销大，需对齐 |
| **Medusa** | 在原模型上增加多个预测头 | 单模型推理，显存占用低，易部署 | 需要训练Medusa头，额外参数 |
| **EAGLE** | 利用LLM中间特征 线性预测 | 验证阶段计算量极小，速度极快 | 依赖模型内部结构，迁移性稍弱 |

```text
投机采样流程图：

Prompt: [The capital of France is]

1. Draft Model (Small) 自回归生成 k=3 个候选:
   Input: [The capital of France is]
   Output Draft: [Paris][,][a] (逐步生成)

2. Target Model (Large) 并行验证:
   Input: [The capital of France is Paris, a]
   Forward Pass 一次计算所有位置的 Logits

3. 验证与采样:
   Pos 1: Draft [Paris] vs Target [Paris] -> 匹配
   Pos 2: Draft [,]    vs Target [,]    -> 匹配
   Pos 3: Draft [a]    vs Target [beautiful] -> 不匹配

4. 最终结果:
   Output: [The capital of France is Paris,] + Target采样出的[beautiful] ...
```

## 常见考点
- **接受率**：投机采样的加速比关键在于“接受率”。如果小模型与大模型差异过大，接受率低，反而会因为频繁的验证失败导致性能下降。因此 Draft Model 通常需要与 Target Model 对齐。
- **树注意力**：为了在验证阶段一次性处理 k 个候选 token，需要对大模型的 Attention Mask 进行特殊构造，使其能并行计算前缀和所有候选分支的概率。
- **无偏性证明**：为什么说它是无损的？因为在拒绝发生时，我们使用的是 Target Model 在当前位置的真实分布进行重新采样，这在数学上保证了最终输出分布与 Target Model 自回归生成的分布完全一致。
