# LLM数据合成工作有哪些

LLM数据合成旨在利用强模型（如GPT-4）生成高质量数据来训练弱模型，主要分为数学、代码、指令跟随三大方向。

### 1. 数学推理数据合成
核心逻辑：通过问题改写、逆向生成、分步增强来扩充数据多样性。

- **Self-Correction & Consistency**：先生成答案，再反向生成问题，验证一致性（如 **MathGenie**, **MetaMath**）。
- **Evol-Instruct**：通过“进化”提示让模型生成更难的问题（如 **WizardMath**）。

### 2. 代码生成数据合成
核心逻辑：利用编译器反馈保证正确性。

- **Execution Feedback**：
  1. 模型生成代码 -> 2. 编译器执行/单元测试 -> 3. 通过的保留，未通过的反馈重试（如 **CodeRL**, **Self-Instruct**）。
- **Evolutionary**：从简单种子开始，通过交叉变异生成复杂指令（如 **Genetic Instruct**）。

```text
┌─────────┐      ┌──────────┐      ┌──────────────┐
│ 种子数据 │ ---> │ LLM 生成 │ ---> │ 执行/解释器  │
└─────────┘      └──────────┘      └──────────────┘
                                             │
                ┌──────────────┐             │
                │ 过滤/反馈    │ <-----------┘ (通过/报错)
                └──────────────┘
                      │
                      v
              ┌──────────────┐
              │ 高质量合成数据│
              └──────────────┘
```

### 3. 指令微调数据合成
- **Self-Instruct**：利用模型自己生成指令和输入，利用人工生成的少量种子作为启动。
- **Back-translation**：将内容翻译成其他语言再翻译回来，增强多样性。

**实战案例**：
在训练行业垂类模型时，由于开源的高质量指令数据稀缺，我们采用了“Magpie”思路，利用 Qwen-72B 模型自动生成海量（约 50w 条）行业相关的问答对。结果发现，直接合成会导致模型语气单一且出现幻觉，最终通过引入基于 BERT-score 的多样性过滤和人工抽检 5% 的机制，将合成数据的有效率提升至可用水平。

**代码示例 (基于 Self-Instruct 风格的简化版)**：
```python
import json

prompt = """Generate 5 diverse user instructions related to 'Database Tuning'. 
Format: JSON list with keys 'instruction', 'input', 'output'."""

# 调用强模型生成 (伪代码)
response = strong_model.generate(prompt)
data = json.loads(response)

# 简单的长度与格式过滤
valid_data = [item for item in data if len(item['output']) > 50 and 'SQL' in item['output']]
```

**常见考点**
1.  **数据质量 vs 数量**：合成数据中，如果基准模型有幻觉，如何清洗？（答案：基于代码执行、模型投票或更高级别的Teacher模型过滤）。
2.  **模型坍塌**：完全用合成数据训练下一代模型会有什么后果？（答案：长期会导致模型多样性降低，出现 Model Collapse）。
3.  **蒸馏的合规性**：商业使用合成数据（特别是基于闭源模型生成的）有哪些法律风险？

## 记忆要点

- 三大方向：主要分为数学推理、代码生成和指令跟随数据合成。
- 数学方向：通过问题改写、逆向生成或进化指令增加难度。
- 代码方向：核心依赖编译器执行反馈，通过测试的保留，报错的重试。
- 指令方向：利用Self-Instruct基于种子自我生成，需重点防范模型坍塌和多样性降低。
