# 大模型中常用的激活函数有哪些?为什么LLaMA选择SwiGLU

- **主流激活函数演进:**

| 激活函数 | 公式 | 代表模型 |
|---------|------|----------|
| ReLU | max(0,x) | 早期模型 |
| GELU | x·Phi(x) | BERT, GPT-2 |
| SiLU/Swish | x·sigma(x) | 部分现代模型 |
| SwiGLU | Swish(xW)⊙(xV) | LLaMA, Qwen |

- **SwiGLU详解:**
**传统FFN：** FFN(x) = ReLU(xW₁)W₂
**SwiGLU：** FFN(x) = (Swish(xW) ⊙ xV)W₂

- 引入门控机制: xV作为门控信号调制Swish(xW)的输出
- 需要三个权重矩阵(W, V, W₂),参数量增加约50%
- 为保持总参数量不变,通常将隐藏层维度缩小(如从4d变为${8d \over 3}$)

- **为什么选择SwiGLU:**
1. 效果好 - 多个基准测试上优于ReLU和GELU
2. 门控机制 - 允许模型动态选择信息通路,增强表达能力
3. 平滑梯度 - SiLU的平滑性有助于深层网络训练

- **架构细节补充:**
SwiGLU 本质上是一个带有门控的 FFN 变体。
在标准 Transformer 中，维度变化通常为：d_model -> 4*d_model -> d_model。
在 LLaMA 使用 SwiGLU 时，为了补偿增加的参数量（3个矩阵 vs 2个矩阵），隐藏层维度通常调整为 $d_{ff} = \frac{8}{3} d_{model}$ 或 $\frac{2}{3} \times 4d_{model}$，这样总参数量与使用 ReLU 的标准 FFN 持平。

```text
Standard FFN Block        LLaMA SwiGLU Block
┌──────────────┐          ┌───────────────────────────────┐
│   Input (d)  │          │         Input (d)              │
└──────┬───────┘          └───────────┬───────────────────┘
       │                              │
       ▼                              ▼
┌──────────────┐          ┌─────────────────────┐   ┌───────────────────┐
│  Linear (d→4d)│          │ Linear (d→8d/3) [W] │   │ Linear (d→8d/3) [V]│
└──────┬───────┘          └──────────┬──────────┘   └──────────┬──────────┘
       │                             │                         │
       ▼                             ▼                         ▼
┌──────────────┐               ┌──────────┐              ┌──────────┐
│    ReLU      │               │  Swish   │              │ Identity │
└──────┬───────┘               └─────┬────┘              └────┬─────┘
       │                             │                         │
       └──────────┬──────────────────┘                         │
                  │ (Element-wise Multiply / Gate)             │
                  ▼                                            │
            ┌─────────────────────┐                           │
            │ Element-wise Prod    │◄──────────────────────────┘
            └──────────┬──────────┘
                       ▼
            ┌─────────────────────┐
            │  Linear (8d/3→d)    │
            └──────────┬──────────┘
                       ▼
               Output (d)
```

- **实战案例:** 在微调 LLaMA 3 处理长文本任务时，如果尝试将 SwiGLU 替换回标准 ReLU FFN（通常为了提升推理速度），会发现模型在复杂逻辑推理任务上的准确率下降 3%-5%，且 Loss 收敛变慢，证明门控机制对复杂语义理解至关重要。

- **代码示例:**
```python
def swiglu_ffn(x, c_fc, c_gate, c_proj):
    # x: [batch, seq_len, d_model]
    # LLaMA 3 implementation style
    gate = torch.nn.functional.silu(c_gate(x)) # 门控分支
    value = c_fc(x)                         # 值分支
    x = gate * value                        # 元素级相乘
    return c_proj(x)                        # 投影回 d_model
```

## 记忆要点

- SwiGLU引入门控机制，公式为Swish(xW)⊙(xV)W₂，增强表达能力
- 参数量增加50%，通常将隐藏层维度缩至8d/3以保持总量不变
- SwiGLU优于ReLU/GELU，平滑梯度且允许动态选择信息通路
