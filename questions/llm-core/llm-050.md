# SFT后会出现哪些问题

🦄在深度学习和自然语言处理领域,监督微调是训练大型语言模型的重要步骤.尽管SFT能够使预训练模型适应特定任务,但在SFT之后可能会出现一些问题.

**SFT之后可能面临以下几个主要问题：**

1.  **过拟合**
2.  **灾难性遗忘**
3.  **模型偏差增加**
4.  **泛化能力下降**
5.  **鲁棒性降低**

---

### 1. 过拟合
*   **定义**：模型在训练数据上表现良好,但在新数据(测试集或实际应用中)上性能较差.
*   **根源分析**：
    *   **数据规模有限**：微调数据集通常比预训练数据集小很多.
    *   **模型复杂度高**：大型模型具有大量参数,容易记住训练数据的细节.
    *   **缺乏正则化**：如果在微调过程中没有适当的正则化措施,模型可能过度拟合训练数据.
*   **影响**：泛化能力差,实际应用受限.
*   **解决方案**：数据增广、正则化技术(早停、权重衰减)、交叉验证.

### 2. 灾难性遗忘
*   **定义**：模型在新任务训练中,遗忘了之前学到的知识.
*   **根源分析**：
    *   **参数更新**：微调时,模型参数朝着新任务的最优方向调整,可能偏离了预训练阶段的知识.
    *   **任务冲突**：新任务的目标可能与预训练任务不一致,导致模型需要“舍弃”部分旧知识.
*   **影响**：通用能力下降,知识遗失.
*   **解决方案**：冻结部分参数、使用小学习率、渐进式微调.

### 3. 模型偏差增加
*   **定义**：模型的预测结果偏向某些特定的模式或群体,导致不公平或不准确的结果.
*   **根源分析**：
    *   **数据偏差**：微调数据可能存在偏差,如代表性不足或标注不公平.
    *   **过度适应特定任务**：模型可能强化了特定任务的数据模式,忽略了其他可能性.
*   **影响**：公平性问题,可信度降低.
*   **解决方案**：数据审查、偏差检测、模型校正.

### 4. 泛化能力下降
*   **定义**：模型在未见过的数据上保持良好性能的能力.
*   **根源分析**：
    *   **过度专注于特定任务**：微调可能让模型只擅长于微调任务的数据分布.
    *   **数据多样性不足**：微调数据集的覆盖范围有限.
*   **影响**：适用范围缩小,灵活性降低.
*   **解决方案**：多任务学习、使用多样化的数据集.

### 5. 鲁棒性降低
*   **定义**：模型应对噪声、异常值或对抗样本的能力.
*   **根源分析**：
    *   **微调数据质量问题**：噪声数据或异常样本可能影响模型稳定性.
    *   **过拟合风险**：过度拟合导致模型对数据微小变化敏感.
*   **影响**：易受攻击,性能不稳定.
*   **解决方案**：数据清洗、对抗训练、增加噪声训练.

### 6. 额外补充：指令遵循退化
*   **定义**：模型虽然学会了回答特定领域的问题，但丧失了理解复杂指令格式的能力，例如无法再处理 JSON 输出要求或忽视提示词中的否定词。
*   **根源**：SFT 数据格式单一，或指令不够多样化。
*   **解决**：混合训练数据，保持指令格式的多样性。

## 常见考点
1. **如何缓解灾难性遗忘**：除了常规的小学习率，面试中常提到 **LoRA (Low-Rank Adaptation)**。通过只训练极少量的适配器参数，保留原始模型参数不变，可以极大程度减少遗忘。此外，**Elastic Weight Consolidation (EWC)** 也是经典方法。
2. **SFT 与 RLHF 的关系**：SFT 往往是 RLHF 的前置步骤。SFT 阶段如果过拟合严重的指令格式，会导致后续 RLHF 阶段模型探索能力变差，难以生成多样化的回复供 Reward Model 打分。
3. **过拟合与泛化的权衡**：大模型的一个特性是“悖论般的泛化”，即有时候插值训练数据（甚至 memorize）反而能带来更好的泛化性能（Grokking 现象），但在 SFT 阶段通常还是表现为传统的过拟合风险。

---

**实战深化**

*   **实战案例**：在做金融研报摘要微调时，团队仅使用了 5,000 条高质量的“标题+摘要”对进行全参数微调。结果模型虽然在验证集 Loss 很低，但面对用户简单的“你好”竟然回复“本季度营收增长...”，且完全丧失了代码生成能力。这是典型的**指令跟随退化**与**灾难性遗忘**。随后采用了 10% 的通用指令数据混合微调才得以恢复。

*   **代码示例 (LoRA)**：
    ```python
    from peft import LoraConfig, get_peft_model
    
    # 配置 LoRA 以缓解灾难性遗忘，仅训练旁路参数
    config = LoraConfig(
        r=16,           # 低秩维度，参数量极少
        lora_alpha=32,
        target_modules=["q_proj", "v_proj"],
        lora_dropout=0.05
    )
    model = get_peft_model(base_model, config)
    # 此时打印可见，绝大多数 parameters requires_grad=False
    ```
