# FIM (Fill in the Middle)

- **FIM (Fill in the Middle)**:
  - **目标**: 让Decoder-only模型具备在文本中间生成内容的能力（如代码补全）。
  - **方法**: 将数据分为Prefix, Suffix, Middle。训练时拼接待预测的Middle在Suffix后，用特殊符分隔：
    `<PRE> prefix <SUF> suffix <MID> middle <EOS>`。
  - **原理**: 利用因果掩码使Middle只能看到Prefix和Suffix，从而学习填空。
  - *架构流程*:
    ```text
    输入: <PRE> Code_A <SUF> Code_C <MID>
              │
              ▼
    模型生成 -> Code_B <EOS>
    (Attention Mask: Code_B 只能 attend to <PRE>...<MID> 和 Code_C)
    ```

- **实战案例:** 在IDE代码补全插件中，普通模型只能根据光标前的代码进行续写。应用FIM技术后，模型能理解光标后的大括号`}`，从而准确推断出中间缺失的函数逻辑，将补全的准确率在真实场景下提升了40%。

- **代码示例:**
```python
# PSM (Prefix-Suffix-Middle) permutation strategy
def apply_fim(sample, suffix_ratio=0.5):
    prefix, suffix = sample[:int(len(sample)*suffix_ratio)], sample[int(len(sample)*suffix_ratio):]
    # Simplified: normally split at random point
    return {"text": f"<PRE> {prefix} <SUF> {suffix} <MID>", "target": suffix}
```

- **涌现能力**:
  - **In-context Learning**: 模型通过Prompt示例学习新任务。
  - **Chain of Thought (CoT)**: 模型通过逐步推理解决复杂问题。
  - **解释**:
    1. 评价指标不平滑（如Exact Match）导致突变现象。
    2. 复杂任务由多个子任务组成，综合阈值触发涌现。
    3. Grokking（顿悟）：规模/数据量达临界点后突然理解规律。

- **复读机问题**:
  - **现象**: 模型过度复制输入文本，缺乏创造性。
  - **原因**: 训练数据重复、自监督目标局限、缺乏多样性。
  - **解决**: 多样化数据、调整采样温度、引入噪声、后处理去重。

- **Self-consistency & Best-of-N**:
  - **Self-consistency**: 生成多条CoT路径，通过**多数投票**选最终答案。
  - **Best-of-N**: 生成N个回答，通过Reward Model打分，选**最高分**。

- **对齐税**:
  - **定义**: RLHF对齐过程中导致模型预训练能力（如NLU基准性能）下降。
  - **缓解 (PPO-ptx)**: 在PPO目标中增加预训练数据的梯度更新（混合预训练Loss），防止遗忘。

## 常见考点
1. **FIM 掩码构造**: 手写 Attention Mask 矩阵，如何确保 Middle 只看 Prefix 和 Suffix 而不看未来的 Middle。
2. **涌现本质**: “涌现”是模型能力突变还是评测指标的伪象？
3. **RLHF 过拟合**: RLHF 训练中 Reward Hack 和对齐税的具体表现及权衡。
