# 什么是损失缩放(Loss Scaling)?为什么混合精度训练需要它

- **损失缩放** 是混合精度训练中解决 FP16 梯度下溢的关键技术。

- **实战案例**：在训练 BERT 模型时，开启 FP16 后 Loss 一直卡在 0.1 不动。检查梯度发现全为 0。通过开启 PyTorch 的 `GradScaler` 并将 `init_scale` 设为 $2^{16}$，梯度恢复，模型开始正常收敛。

- **问题背景**
- FP16 (Float16) 的表示范围有限，最小正常数约为 $6.1 \times 10^{-5}$。
- 在深度网络反向传播中，梯度往往非常小（尤其是网络浅层或使用 Sigmoid/Tanh 激活函数时）。
- 很多梯度值小于最小正常数时会被截断为 0。
- 梯度为 0 → 参数不更新 → 模型停止学习。

- **损失缩放原理**

1. **前向传播**: 正常计算，得到 loss (FP16)。
2. **损失放大**: loss $\times$ scale_factor (如 $2^{16} = 65536$)。
3. **反向传播**: 放大后的 loss 产生放大的梯度（链式法则放大），从而使得梯度在 FP16 范围内，避免下溢为 0。
4. **梯度缩小**: 更新权重前，梯度 $\div$ scale_factor 恢复真实梯度值 (转换为 FP32 更新)。

```python
# PyTorch AMP (自动混合精度) 伪代码
scaler = torch.cuda.amp.GradScaler(init_scale=2**16)

for data, target in dataloader:
    optimizer.zero_grad()
    
    # 开启自动混合精度上下文
    with torch.cuda.amp.autocast():
        output = model(data)      # 部分算子用 FP16
        loss = criterion(output, target)
    
    # 1. 放大 Loss
    # 2. 反向传播 (计算放大后的梯度)
    scaler.scale(loss).backward()
    
    # 3. 梯度 Unscale (缩小) 并 检查 inf/nan
    # 4. 更新参数 (FP32)
    scaler.step(optimizer)           
    
    # 5. 动态调整 scale_factor
    scaler.update()                 
```

```text
Loss Scaling 数据流示意图：

正常流程:
Loss (FP16, 极小) ──► Backward ──► Grad (Underflow -> 0)

Loss Scaling 流程:
Loss (FP16) ──► [ × Scale ] ──► Big Loss ──► Backward
                                         │
                                         ▼
                              Big Grad (FP16, 正常范围)
                                         │
                                         ▼
                              [ ÷ Scale ] ──► True Grad (FP32)
                                         │
                                         ▼
                              Optimizer Update (FP32 weights)
```

- **混合精度训练数据类型对比**

| 特性 | FP16 (Float16) | BF16 (BFloat16) | FP32 (Float32) |
| :--- | :--- | :--- | :--- |
| **位宽** | 16 bit | 16 bit | 32 bit |
| **指数位** | 5 bit | **8 bit** (与FP32同) | 8 bit |
| **尾数位** | 10 bit | 7 bit | 23 bit |
| **数据范围** | 窄 ($6\times10^{-5}$ ~ 65000) | **广** ($\approx$ FP32范围) | 最广 |
| **精度** | 较高 | 较低 | 最高 |
| **Loss Scaling** | **必须** | 通常**不需要** | 不需要 |

- **动态损失缩放**
- **目的**: 自动找到最大的、不会导致梯度溢出的 scale_factor。
- **机制**:
- 如果梯度出现 inf/nan (溢出) → scale_factor 减半 (如 /2)，跳过本次更新。
- 连续 N 步 (如 2000 iterations) 无 inf/nan → scale_factor 翻倍 (如 *2)。
- 自动适应梯度的量级，无需手动调参。

- **bfloat16 vs float16**
- **float16 (FP16)**: 1符号位 + 5指数位 + 10尾数位。
- 动态范围小 ($6 \times 10^{-5}$ to 65505)，容易出现溢出或下溢，需要 loss scaling。
- 精度较高（尾数位多），适合推理。
- **bfloat16 (Brain Float)**: 1符号位 + 8指数位 + 7尾数位。
- 动态范围与 FP32 一致，极大减少了溢出/下溢风险，通常不需要 Loss Scaling。
- 精度略低，但对深度学习训练收敛影响较小，是 LLM 训练的首选（如 Llama, GPT-4）。
