# 解释一下Prompt-tuning

💫 **Prompt-tuning** 是一种专注于优化提示来提升预训练语言模型在下游任务性能的技术。它通过对提示部分进行训练，而不修改模型参数，从而实现对大规模语言模型的高效适应。Prompt-tuning 能够以较少的参数调整，获得接近甚至超过传统微调方法的性能，尤其适用于大规模预训练模型（如 GPT-3、T5 等）。

**1. 背景和来源**
Prompt-tuning 的概念源于利用预训练语言模型的填空能力来解决下游任务。例如，对于文本分类，可以将输入转换为一种填空问题，如“这篇文章的情感是 **MASK**”。Prompt-tuning 进一步发展了这一思想，通过训练一个可优化的提示，使得提示能够引导模型更好地完成特定任务。Prompt-tuning 的思想最早可以追溯到 2021 年由 Lester 等人提出的 *The Power of Scale for Parameter-Efficient Prompt Tuning*。这篇文章展示了 Prompt-tuning 在大规模模型（尤其是 100 亿参数以上模型）中的有效性，并提出了使用可训练的连续提示向量代替离散文本提示。

**2. 主要观点**
Prompt-tuning 的核心思想是在输入序列前插入一个可训练的提示，这些提示不再是离散的文本，而是连续的嵌入向量。通过对提示向量进行训练，可以有效引导模型输出更符合任务目标的结果。

*   **参数高效：** 仅优化提示向量，而保持模型的其他参数冻结。相比传统的微调方法，Prompt-tuning 在参数更新上具有显著的优势。
*   **大模型适应性强：** Prompt-tuning 在大规模预训练模型（如数十亿参数模型）上表现尤为出色，能够充分利用大模型的知识进行任务适应。
*   **任务转换能力强：** 通过修改或优化提示，可以在多个任务之间快速切换，而无需对每个任务都进行全参数训练。

**3. 技术细节与架构**
Prompt-tuning 的实现过程包括以下几个关键步骤：

1.  **连续提示的初始化：** 在传统的 Prompt 工程中，提示通常是手工设计的自然语言文本。Prompt-tuning 将这些提示替换为连续的可优化向量，即可训练的嵌入。这些提示向量的长度 $m$ 由用户指定（提示长度对性能影响较大），提示的维度 $d$ 等于模型的嵌入层维度。提示向量可以被随机初始化或者从现有的嵌入向量中初始化。
2.  **输入序列的构造：** 给定一个任务输入序列 $X = [x_1, x_2, ..., x_n]$，其中 $x_i$ 表示第 $i$ 个 token 的嵌入向量，Prompt-tuning 在输入序列的前面添加提示向量 $P = [p_1, p_2, ..., p_m]$。扩展后的输入序列表示为 $X_{ext} = [p_1, p_2, ..., p_m, x_1, x_2, ..., x_n]$。
3.  **冻结模型参数，仅训练提示向量：** 在 Prompt-tuning 的训练过程中，保持预训练模型的所有参数不变，只对提示向量的参数进行优化。优化目标通常是任务相关的损失函数，如分类任务的交叉熵损失或生成任务的语言模型损失。
4.  **在生成任务中的应用：** Prompt-tuning 不仅可以应用于分类任务，还适用于各种生成任务（如机器翻译、文本摘要）。在这些任务中，提示向量能够引导生成模型朝着特定的方向进行输出，使得生成结果更符合任务需求。

```text
输入处理架构图：

[Input Text] -> [Tokenizer] -> [Input Embeddings (x1...xn)]
                                            |
[Soft Prompt Embeddings (p1...pm)] <-------+  (Concatenate)
       |
       v
[Extended Input: P + X] -> [Frozen LLM (θ)] -> [Output Logits] -> [Loss]
       ^                                     |
       |-------(Backpropagation)-------------|
               (Only update P, Freeze θ)
```

**4. 数学表述**
假设给定输入序列 $X$，Prompt-tuning 的输入扩展为 $X_{ext} = [P; X]$，其中 $P$ 是长度为 $m$ 的提示向量。Prompt-tuning 的目标是优化提示向量 $P$ 的参数，使得模型的输出能够更好地完成任务。训练过程中优化的目标可以表示为：
$$ \min_P L(f(X_{ext}; \theta), y) $$
其中 $L$ 表示损失函数，$f$ 为语言模型，$\theta$ 为冻结的预训练模型参数，$y$ 为任务标签。

**5. 优势与局限**

**优势：**
*   **参数效率高：** 仅需优化提示向量，显著减少了需要训练的参数量（通常 < 0.1%）。
*   **迁移性强：** 可以快速适应不同任务，尤其是在大规模模型中效果更为明显。
*   **存储友好：** 不同的任务只需存储各自的小规模 Prompt 向量，无需存储整个模型副本。

**局限：**
*   **依赖提示长度：** 提示向量的长度对性能有影响，寻找最优长度需要调优。
*   **低参数模型表现较差：** 在参数量较小的模型（如 BERT-Base）上，Prompt-tuning 的性能往往不如全量微调。
*   **收敛速度：** 有时比全量微调收敛得更慢，因为只调整了极小部分的参数。

## 常见考点
1.  **Prompt-tuning 与 Prefix-tuning 的区别是什么？**（Hint：Prefix-tuning 会在每一层都插入可训练向量，而 Prompt-tuning 仅在输入层插入。）
2.  **为什么 Prompt-tuning 在小模型上效果不如大模型？**（Hint：大模型具备更强的知识和模式提取能力，小模型更依赖参数调整来适应任务。）
3.  **Prompt-tuning 中 Prompt 的初始化方式有哪些？**（Hint：随机初始化、从词汇表 embedding 中采样、或使用领域相关文本的 embedding。）

## 记忆要点

- 仅在输入层拼接可训练的连续向量，冻结模型所有参数。
- 依赖模型规模，仅在超大模型（如GPT-3）上效果媲美全量微调。
- 不同任务只需存储各自的Prompt向量，存储成本极低。
- 局限性：小模型表现差，且占用输入序列长度。
