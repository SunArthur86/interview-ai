# 为什么需要增量预训练

### 1. 什么是增量预训练
**增量预训练** 是指在已经完成初始预训练的基础模型上，使用特定领域或新增的数据继续进行预训练的过程。

### 2. 为什么需要增量预训练？
*   **知识时效性：** 通用预训练模型的数据截止于过去，无法理解新出现的词汇、事件或知识。
*   **领域适配：** 通用模型在医疗、法律、代码等垂直领域的专业术语和逻辑上可能表现不佳。
*   **成本考量：** 相比于从头训练一个大模型，在已有基础上继续训练的成本要低得多。

### 3. 核心挑战与目标
*   **目标：** 注入新知识、新领域风格，同时**不遗忘**通用能力（避免灾难性遗忘）。
*   **策略：** 使用混合数据（新数据+少量旧数据）或调整学习率，在保持原有通用语言能力的前提下，使模型分布向新领域偏移。

### 4. 流程地位
通常位于“基础预训练”之后，“指令微调（SFT）”之前。

### 6. 实战案例：金融模型构建
在构建金融领域的 LLM 时，如果仅使用通用模型，它可能无法准确理解“做多”、“掉期”等术语。如果直接用 50万条金融研报做 SFT，模型可能会“一本正经地胡说八道”（幻觉）。正确的路径是：先用 100GB 金融语料做增量预训练，让模型学会金融世界的“语言”和“事实”，再用 5万条金融问答做 SFT，教模型如何“说话”和“回答问题”。

### 7. 代码示例：混合数据加载策略
在增量预训练中，防止灾难性遗忘的关键在于数据配比。通常使用 `ConcatDataset` 或自定义 Sampler 来混合新旧数据。

```python
from torch.utils.data import Dataset, DataLoader

class MixedDomainDataset(Dataset):
    def __init__(self, general_data, domain_data, domain_ratio=0.8):
        self.general_data = general_data
        self.domain_data = domain_data
        self.domain_ratio = domain_ratio
        # 调整采样数量，例如 20% 通用，80% 领域
        self.len_general = int(len(general_data) * (1 - domain_ratio))
        self.len_domain = int(len(domain_data) * domain_ratio)

    def __len__(self):
        return self.len_general + self.len_domain

    def __getitem__(self, idx):
        if idx < self.len_general:
            return self.general_data[idx % len(self.general_data)]
        else:
            return self.domain_data[(idx - self.len_general) % len(self.domain_data)]

# 实战中需确保 General Data 覆盖足够广泛以维持通用能力
```

### 8. 常见考点
1.  **数据混合比例**：在做增量预训练时，新数据和旧数据（通用数据）应该按什么比例混合？（通常保留 10% - 20% 的通用数据，防止模型严重遗忘通用的语言能力）。
2.  ** catastrophic forgetting**：如何定量评估是否发生了灾难性遗忘？（在基准测试集（如 MMLU, C-Eval）的通用任务上评估性能，看是否显著下降）。
3.  **与 SFT 的区别**：既然 SFT 也能用领域数据，为什么还需要增量预训练？（增量预训练是“注入知识”（事实、术语），SFT 是“对齐格式”（指令遵循）。SFT 很难教会模型新的客观事实，容易产生幻觉）。
