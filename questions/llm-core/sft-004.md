# 微调LLM的数据如何准备？有哪些数据质量最佳实践？

🎯 **本质**：微调数据的质量远比数量重要——1000条高质量数据通常胜过10万条低质量数据。

📊 **数据准备流程**：

**1. 数据收集**
  - **来源**：人工编写（精准但慢）、GPT-4生成（Self-Instruct/蒸馏）、真实用户交互日志（最贴合场景）、开源数据集。
  - **多样性**：必须覆盖不同话题、难度、指令长度、输出风格。
  - **数量参考**：通用SFT约5K-50K条，领域特定（如医疗、法律）约1K-10K条即可。

**2. 数据格式**
  - **Chat格式（推荐）**：
    `json
    {
      "messages": [
        {"role": "system", "content": "你是一个助手"},
        {"role": "user", "content": "什么是RAG?"},
        {"role": "assistant", "content": "RAG是..."},
        {"role": "user", "content": "请举例"},
        {"role": "assistant", "content": "例如..."}
      ]
    }
    `
  - **Alpaca格式**：`{"instruction": "...", "input": "...", "output": "..."}`（适合单轮指令）。

**3. 数据质量标准**
  - **准确性**：答案必须正确（最重要），事实性错误会误导模型。
  - **完整性**：回答要全面，避免“太长不看”式的截断。
  - **一致性**：相同类型的问题应保持相同的回答格式和风格。
  - **多样性**：避免重复模式（如同一个模板生成1000条），防止模型死记硬背。
  - **格式**：JSON有效，无特殊编码问题（如UTF-8乱码）。

**4. 数据清洗**
  - **去重**：
    - 精确去重（MD5）。
    - 语义去重（MinHash, LSH, Embedding Cosine Similarity > 0.9 删一个）。
  - **去噪**：去除HTML标签、乱码、截断的文本、过多重复标点。
  - **长度过滤**：
    - 过短（<10 tokens）通常信息量不足。
    - 过长（>模型最大上下文，如>4096/8192）导致截断，需特殊处理。
  - **安全过滤**：移除PII（个人隐私）、有害内容、 Bias 内容。

**5. 数据混合**
  - **通用能力保持**：混入10-20%通用SFT数据（如OpenHermes, ShareGPT）。
  - **防止灾难性遗忘**：纯领域数据微调会导致模型丧失通用对话能力。
  - **多任务混合**：不同任务比例需平衡，避免模型偏向某一类指令。

**高质量数据生成方法**：
1. **Self-Instruct**：用强模型（GPT-4）根据少量种子生成指令-回答对。
2. **Evol-Instruct**：通过“增加约束”、“增加推理步骤”、“增加难度”等指令让模型逐步进化问题。
3. **GPT-4辅助生成**：用GPT-4生成回答，人工审核或使用Judge模型筛选。
4. **合成数据**：基于规则或模板批量生成（如SQL生成），需人工抽样检查质量。

**实战案例**：在某金融垂直领域微调中，直接使用GPT-4生成的数据训练导致模型喜欢“幻觉”编造法规。引入专家Review并修正前500条数据作为Golden Set，后续使用此Set进行Few-shot生成，数据质量提升了40%，模型幻觉率大幅下降。

**代码示例**：
```python
# 基于SimHash进行快速语义去重的关键逻辑
from datasketch import MinHash, MinHashLSH

def deduplicate_dataset(data_list, threshold=0.8, num_perm=128):
    # 创建LSH索引
    lsh = MinHashLSH(threshold=threshold, num_perm=num_perm)
    minhashes = {}
    
    clean_data = []
    for idx, text in enumerate(data_list):
        # 计算MinHash签名
        m = MinHash(num_perm=num_perm)
        m.update(text.encode('utf8'))
        
        # 查询是否存在近似重复
        if not lsh.query(m):
            lsh.insert(idx, m)
            minhashes[idx] = m
            clean_data.append(text)
    return clean_data
```

## 常见考点
1. **如何防止灾难性遗忘？**：在微调数据中混合一定比例（如10%-20%）的通用或预训练数据，持续训练通用能力。
2. **如何构建高质量的领域数据？**：通常采用“知识蒸馏+人工修正”的方式，先让GPT-4根据领域文档生成QA，再由专家审核。
