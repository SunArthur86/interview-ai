# Infini-attention 和 Ring Attention 分别解决什么问题?它们的原理是什么

- **两种扩展注意力窗口到无限长度的方案**

- **Infini-attention (Google, 2024)**
  **核心思想：** **压缩记忆 + 局部注意力**
  
  **原理细节：**
  - 将输入序列分段，每段内部使用标准的**因果自注意力**（捕获局部细粒度依赖）。
  - 引入一个全局的**压缩记忆**（通常存储在线性层参数中，通过 Delta Rule 更新：$M \leftarrow M + \sigma(Q K^T)$），捕获长期历史信息。
  - **输出融合**: $O = \text{softmax}(\frac{QK^T}{\sqrt{d}})V + Q M^T$。即局部注意力结果与记忆检索结果线性相加。
  
  **公式：** $A_{infi} = \sigma(A_{dot}) + (1-\sigma) \cdot A_{mem}$
  (其中 $\sigma$ 是 Sigmoid 门控，平衡局部注意力和长期记忆检索)。

  **优点：** 
  - **显存 O(1)**: 历史信息压缩在固定大小的记忆矩阵中，不随序列长度线性增长。
  - **无限上下文**: 理论上可处理无限长度输入。
  - **训练友好**: 兼容标准 Transformer 训练流程，无需特殊的分段并行策略。

  **架构图**:
```
Segment 1      Segment 2      Segment 3 (Current)
┌───────┐      ┌───────┐      ┌───────┐
│Local  │      │Local  │      │Local  │
│Attn   │      │Attn   │      │Attn   │
└───┬───┘      └───┬───┘      └───┬───┘
    │              │              │
    ▼              ▼              ▼
Update Mem    Update Mem    Retrieve Mem
    │              │              │
    └───────► [Compressed Memory] ◄────┘
                         │
                         ▼
                 Final Output = Local + Mem
```

- **Ring Attention (UC Berkeley, 2023)**
  **核心思想：** **跨 GPU 环形通信 + 块级注意力**
  
  **原理细节：**
  - **序列切分**: 将超长序列按 Block 切分，分布到环形拓扑连接的多个 GPU 上。
  - **流水线通信**: 每个 GPU 计算当前 Block 的 Attention，随后将自己的 KV Block 发送给 Ring 中的下一个 GPU，同时接收上一个 GPU 的 KV Block。
  - **累加计算**: 每收到一个新的 KV Block，就更新一次注意力分数（Softmax 的累加是可以分块进行的）。
  - **Flash Attention 结合**: 在计算局部 Attention 时使用 Flash Attention 优化 IO，通信与计算重叠以掩盖延迟。

  **优点：** 
  - **精确的长上下文**: 没有信息压缩，支持百万级上下文（1M+ tokens），注意力计算结果与非分段模型一致。
  - **模型无关**: 不修改模型结构（如 Attention 公式、FFN），纯系统级优化。
  
  **缺点：** 
  - **资源受限**: 需要的 GPU 数量与序列长度/上下文窗口大小成正比。
  - **通信开销**: 环形通信增加了整体延迟，尽管可以通过计算重叠掩盖，但仍受限于带宽。

  **架构图**:
```
 GPU 0  ──────────────► GPU 1  ──────────────► GPU 2 
  │  (KV0)             │  (KV1)             │  (KV2)
  │     Send KV0 ──────│───── Send KV1 ─────│───── Send KV2
  │◀──── Recv KV2      │◀──── Recv KV0      │◀──── Recv KV1
  │                    │           
```

- **## 深化实战**
  1. **实战案例**：在做长文本摘要（如分析 100 页财报）时，Ring Attention 需要 8 卡 A100 才能跑通，成本极高且受限于集群带宽；而 Infini-attention 只需要单卡即可处理，但发现摘要往往会遗漏文档中间的细节（因为记忆被压缩了）。工程上常采用折中：Ring Attention 用于高精度的 RAG 索引构建，Infini-attention 用于低成本的对话式长文本阅读。
  2. **代码示例 (Ring Attention 伪代码)**:
```python
# 环形注意力核心：分块计算 Softmax 并累加
for ring_step in range(num_devices):
    # 1. 接收上一步的 KV_Block
    k_block, v_block = receive_kv_from_prev_device()
    
    # 2. 结合本地 KV 进行 Attention 计算
    # q: [local_batch, heads, len, dim]
    # k, v: [global_batch, heads, len, dim]
    attn_weights = torch.matmul(q, k_block.transpose(-2, -1)) / scale
    attn_scores = softmax(attn_scores) # 注意：这里需要全局 max 进行数值稳定性
    
    # 3. 累加输出 (利用数学等价性)
    local_output += torch.matmul(attn_scores, v_block)
    
    # 4. 将本地 KV 发送给下一设备
    send_kv_to_next_device(local_k, local_v)
```
