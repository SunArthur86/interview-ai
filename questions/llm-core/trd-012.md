# 在 RAG 系统中，如何利用 BM25、向量检索和重排序来提升召回率？

单一检索方式往往难以兼顾语义理解和关键词匹配，混合检索是提升 RAG 召回率的主流方案。

**1. 混合检索**：
- **向量检索**：擅长语义匹配，能找到“同义不同字”的文档，但可能忽略专有名词（如具体型号代码）。
- **BM25（稀疏检索）**：基于关键词匹配（TF-IDF 变种），擅长精确匹配专有名词，但无法处理语义变形。
- **策略**：并行执行两种检索，然后通过倒数排名融合（RRF）或加权分数合并结果列表。

**2. 重排序**：
- **作用**：初检阶段为了速度（从百万级文档中选 Top 50），牺牲了一定精度。Rerank 模型（如 Cross-Encoder）会对这 Top 50 文档进行精细化的两两打分，计算量大但精度极高。
- **流程**：Retriever（粗排） -> Reranker（精排） -> LLM。

这种组合策略先用混合检索保证广度（不漏掉关键信息），再用 Rerank 保证精度，确保进入 LLM 上下文的信息是最相关、最噪音最小的。
