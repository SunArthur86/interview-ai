# 什么任务适合上RLHF,什么任务sft效果更好

1. **适合 RLHF 的任务**：
   - **Reward 明确且复杂的任务**：典型的是数学、物理、化学、代码 等有客观标准答案的任务。
   - **逻辑推理链**：需要多步推导且每一步都有对错之分的任务，RLHF 能提供精细的反馈。
   - **长尾对齐**：当 SFT 无法覆盖所有 Corner Case（边界情况）时，RLHF 可以通过探索发现并奖励正确行为，泛化能力更强。

2. **适合 SFT (而非 RLHF) 的任务**：
   - **创意生成类**：文案写作、角色扮演、开放式问答。这类任务 Reward 主观性强，难以定义标准 Reward，且极度依赖长文本生成能力，做好预训练和 SFT 即可。
   - **特定格式/指令遵循**：如 JSON 输出、特定 API 调用。这类任务有确定的“标准范式”，SFT 通过模仿学习效率极高。
   - **知识注入**：需要模型记住特定新知识时，SFT 效果通常优于 RLHF，因为 RLHF 优化的是偏好而非事实记忆，容易导致幻觉。
   - **某些情况下可辅以 DPO**：对于创意类任务，SFT 是核心，偶尔使用 DPO 调整风格或微调偏好，但效果不如逻辑类任务显著。

**实战案例**：在做 Code LLM 优化时，我们发现单纯用 SFT 教模型“写注释”效果很好，但在解决复杂算法题（如 LeetCode Hard）时，SFT 容易过拟合训练集的解法，引入 RLHF（基于测试用例的 Reward Model）后，模型在未见题上的通过率提升了 15%。

**对比表格**：

| 维度 | SFT (Supervised Fine-Tuning) | RLHF (Reinforcement Learning from Human Feedback) |
| :--- | :--- | :--- |
| **核心机制** | 模仿学习，拟合专家数据分布 | 强化学习，优化 Reward Model 给出的评分 |
| **数据依赖** | 高质量、结构化的问答对 | 成对/排序数据，更容易标注 (相对优劣) |
| **训练稳定性** | 极高，就是标准分类/生成任务 | 较低，容易 Reward Hacking 或模式崩溃 |
| **适用场景** | 注入知识、特定格式、风格迁移 | 复杂逻辑推理、代码正确性、安全性对齐 |
| **资源消耗** | 低 (只需前向传播+Loss计算) | 高 (需训练 RM，PPO 策略需多次 Rollout) |

**## 常见考点**
1. 为什么代码和数学任务特别适合 RLHF？（思维链奖励信号明确）
2. RLHF 中的 Reward Hacking 在创意写作任务中表现如何？（较少，因为缺乏客观标准）
3. 在资源有限的情况下，如何选择 SFT 还是 DPO？（SFT 是基础，DPO 是锦上添花）
4. SFT 能否替代 RLHF 完全解决幻觉问题？（不能，SFT 主要学分布，RLHF 主要学偏好）

## 记忆要点

- RLHF适合：有客观标准的复杂任务，如数学、代码、逻辑推理
- SFT适合：创意生成、特定格式遵循、知识注入等模仿类任务
- 核心区别：RLHF优化结果与过程反馈，SFT拟合专家数据分布
