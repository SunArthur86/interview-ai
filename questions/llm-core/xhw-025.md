# 为什么大模型用GQA

大模型（LLM）广泛使用 GQA（Grouped-Query Attention）是为了在推理速度和模型性能之间取得更好的平衡，特别是在内存带宽和推理速度成为瓶颈的情况下。

### 1. Multi-Head Attention (MHA)
MHA 是 Transformer 的基础。每个注意力头都有独立的 Query、Key 和 Value 投影矩阵。
- **优势**：捕捉多样化的特征，不同头关注不同的模式和依赖关系。
- **劣势**：计算量和内存占用高。每个头都要维护 KV 矩阵，导致显存占用大，推理时 KV Cache 带宽需求高。

### 2. Multi-Query Attention (MQA)
MQA 为了解决 MHA 的带宽问题，让所有头共享同一个 Key 和 Value 矩阵（Query 矩阵仍然独立）。
- **优势**：极大地减少了 KV Cache 的大小，提升了推理速度，降低了内存带宽压力。
- **劣势**：由于共享 KV，模型表达能力受限，性能通常比 MHA 有所下降。

### 3. Grouped-Query Attention (GQA)
GQA 是 MHA 和 MQA 的折中方案。它将查询头分为若干组，每组内的头共享一对 KV 矩阵。
- **原理**：假设有 $H$ 个 Query 头，将其分为 $G$ 组。每组共享一个 KV 矩阵。当 $G=H$ 时，即为 MHA；当 $G=1$ 时，即为 MQA。
- **优势**：
  - **平衡性能与速度**：相比 MQA，保留了更多的特征表达能力（性能接近 MHA）；相比 MHA，大幅减少了 KV Cache 显存和带宽需求（速度接近 MQA）。
  - **灵活性**：可以通过调整组数 $G$ 来在精度和效率之间灵活权衡。
- **总结**：GQA 专为推理加速设计，有效缓解了自回归生成中的内存带宽瓶颈，是大模型推理的标配技术。

### 4. 实战案例与代码
**实战案例**：在 Llama-3-70B 的落地部署中，若使用 MHA，单请求的 KV Cache 显存占用高达 100GB+，难以进行高并发推理；切换为 GQA (G=8) 后，KV Cache 显存占用降低至约 20GB，使得单卡 A100 可支持的并发 Batch Size 翻倍，吞吐量提升 40% 以上。

**代码示例 (PyTorch)**：
```python
# GQA 实现核心逻辑：通过 repeat 操作实现 Query 对共享 KV 的 attention
def scaled_dot_product_gqa(q, k, v, group_size):
    # q: [batch, seq_len, num_heads, head_dim]
    # k, v: [batch, seq_len, num_kv_heads, head_dim]
    
    if group_size > 1: # 如果是 GQA 或 MQA
        k = k.unsqueeze(3).repeat(1, 1, 1, group_size, 1) # 扩展 K
        k = k.view(k.shape[0], k.shape[1], -1, k.shape[4]) # Reshape
        v = v.unsqueeze(3).repeat(1, 1, 1, group_size, 1) # 扩展 V
        v = v.view(v.shape[0], v.shape[1], -1, v.shape[4]) # Reshape
    
    # 后续执行标准的 attention 计算
    attn = torch.matmul(q, k.transpose(-2, -1)) / math.sqrt(q.size(-1))
    return torch.matmul(attn, v)
```

### 架构对比表

| 特性 | MHA (Multi-Head) | MQA (Multi-Query) | GQA (Grouped-Query) |
| :--- | :--- | :--- | :--- |
| **KV Cache 大小** | $2 \times L \times d_{model}$ | $2 \times L \times d_{model} / H$ | $2 \times L \times d_{model} / G$ |
| **推理显存带宽** | 极高 (瓶颈) | 低 | 中低 (显著优化) |
| **模型表达能力** | 最强 | 较弱 (有精度损失) | 强 (接近 MHA) |
| **计算复杂度** | $O(L^2 \cdot d_{model})$ | $O(L^2 \cdot d_{model})$ | $O(L^2 \cdot d_{model})$ |
| **典型应用** | BERT, GPT-2 | PaLM, T5 (部分) | Llama-3, Mistral, Gemma |

## 常见考点
1. **KV Cache 的具体计算**：计算推理时显存占用公式，特别是 GQA 相比 MHA 减少显存的倍数（约为 $1/G$ 或 $(G+H-1)/H$，取决于实现细节）。
2. **训练与推理的一致性**：GQA 在训练时如何处理，在推理时如何复用 KV Cache，以及如何从 MHA Checkpoint 转换为 GQA（Up-training）。
3. **GQA 数量选择**：如何根据模型大小选择 $G$（例如 Llama-3-8B 用 G=8，70B 用 G=8），这背后的 trade-off 是什么？
