# MHA、MQA、GQA 三者的区别是什么?为什么 GQA 是最优选择

### MHA、MQA、GQA 三者的区别

注意力机制的 KV Head 复用演进:

- **MHA (Multi-Head Attention):**
  - **结构**：Q、K、V 各有 N 个头（如 32 个），每个 Q 头对应独立的 K/V 头。
  - **优点**：表达能力强，模型效果最好。
  - **缺点**：KV Cache 显存占用巨大（与头数 N 线性相关），Decode 阶段访存带宽压力大。

- **MQA (Multi-Query Attention):**
  - **结构**：Q 仍有 N 个头，但 K/V 只有 1 个头，所有 Q 头共享同一组 K/V。
  - **优点**：KV Cache 显存降低 N 倍，推理速度显著提升。
  - **缺点**：表达能力受限，模型质量下降明显，难以训练。

- **GQA (Grouped-Query Attention):**
  - **结构**：折中方案。Q 有 N 个头，K/V 有 G 个头（1 < G < N）。每 N/G 个 Q 头共享一组 K/V。例如 Llama 2 中，Q=32, KV=8，即每 4 个 Q 头共享一组 KV。
  - **优点**：质量接近 MHA，KV Cache 显存大幅降低，推理速度显著提升，是效果和效率的最佳平衡点。

#### 结构对比图

```text
MHA (Multi-Head)        GQA (Grouped)           MQA (Multi-Query)
Q: [1|2|3|4]           Q: [1|2|3|4]             Q: [1|2|3|4]
K: [1|2|3|4]           K: [1 | 2]               K: [1]
V: [1|2|3|4]           V: [1 | 2]               V: [1]
                       (Groups: 2)             (Groups: 1)
```

#### 对比表格

| 特性 | MHA (Multi-Head) | MQA (Multi-Query) | GQA (Grouped-Query) |
| :--- | :--- | :--- | :--- |
| **KV Cache 大小** | N × SeqLen × d (最大) | 1 × SeqLen × d (最小) | G × SeqLen × d (折中) |
| **显存带宽压力** | 极高 (Decode瓶颈) | 极低 | 低 (接近MQA) |
| **模型表达能力** | 强 (SOTA基准) | 弱 (易损精度) | 强 (接近MHA) |
| **训练稳定性** | 稳定 | 较差 | 稳定 (需Upcheck) |
| **适用场景** | 研究基准、高精任务 | 极致吞吐场景 | 主流生产环境 (Llama3/Mistral) |

#### 为什么 GQA 是最优选择
1. **性能无损**：实验表明，只要分组数 G 适中（如 N 的 1/4 或 1/8），GQA 的模型效果与 MHA 几乎无异。
2. **推理加速**：显存占用减少意味着更大的 Batch Size，同时 KV Cache 的读取量减少，极大缓解了 Decode 阶段的显存带宽瓶颈。
3. **工业界标准**：Llama 2-70B、Llama 3 全系列、Mistral、Gemini 等主流大模型均采用 GQA，已成为推理优化的标配。

#### 实战案例
在将 70B 参数模型从 MHA 改造为 GQA（KV Head 数量缩减为 1/4）后，推理服务的并发吞吐量提升了 3 倍，且 Perplexity（困惑度）仅增加了 0.1，几乎不影响生成质量，同时显著降低了长文本场景下的 OOM 风险。

#### 关键代码 (PyTorch/投影层定义)
```python
class GQAAttention(nn.Module):
    def __init__(self, embed_dim, num_heads, num_kv_heads):
        super().__init__()
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads
        self.head_dim = embed_dim // num_heads
        
        # Q 投影保持全头数，KV 投影缩减
        self.q_proj = nn.Linear(embed_dim, num_heads * self.head_dim, bias=False)
        self.k_proj = nn.Linear(embed_dim, num_kv_heads * self.head_dim, bias=False)
        self.v_proj = nn.Linear(embed_dim, num_kv_heads * self.head_dim, bias=False)

    def forward(self, x):
        B, T, C = x.shape
        q = self.q_proj(x).view(B, T, self.num_heads, self.head_dim)
        k = self.k_proj(x).view(B, T, self.num_kv_heads, self.head_dim)
        
        # 关键：将 KV 扩展以匹配 Q 的头数 (repeat_interleave)
        # 例如 KV Heads=8, Q Heads=32，则每个 KV Head 复制 4 次
        k = k.repeat_interleave(self.num_heads // self.num_kv_heads, dim=2)
        return q, k
```

#### 常见考点
1. **具体分组比例**：如何确定 GQA 的分组数？（经验法则通常是保持 10%-25% 的 KV 头数，或者通过消融实验确定）。
2. **推理性能瓶颈**：MQA/GQA 主要解决的是计算瓶颈还是显存带宽瓶颈？（主要解决 Decode 阶段的 Memory Bandwidth 瓶颈）。
3. **KV Cache 形状变化**：从 MHA 切换到 GQA，KV Cache 的张量形状发生了什么变化？（例如从 [Batch, NumHeads, SeqLen, HeadDim] 变为 [Batch, NumKVHeads, SeqLen, HeadDim]）。
4. **Training 与 Inference 的差异**：GQA 在训练时是否需要特殊处理？（需要针对 Multi-Head 的 Query 和 Grouped 的 Key/Value 进行正确的 Attention Score 计算和广播机制）。
