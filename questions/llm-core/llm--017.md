# 什么是模型微调(Fine-tuning)?为什么不直接用预训练模型

- **微调** 是在预训练模型基础上,用特定领域数据继续训练,使其适应下游任务.

- **为什么不直接用预训练模型?**
  1. 预训练模型是'通才',缺乏领域深度
  2. 不会遵循指令格式(如输出 JSON)
  3. 回复风格不符合业务需求
  4. 缺乏私有知识

- **微调方式**
  - **全量微调**:更新所有参数(效果好,成本高)
  - **LoRA**:只训练低秩适配器(成本低,效果接近)
  - **QLoRA**:4bit 量化 + LoRA(极致省显存)

- **一句话理解**:预训练模型 = 大学毕业生;微调 = 入职培训.

- **实战案例**：在某医疗问答项目中，为了在单张 24G 显存的 3090 显卡上微调 7B 参数模型，我们尝试了全量微调导致 OOM（显存溢出）。切换到 QLoRA（4bit量化）后，显存占用从 48GB 降低到 14GB，且训练速度提升了 1.5 倍，最终模型在医疗术语上的准确率提升了 20%。

- **微调方式对比表**：
| 维度 | 全量微调 | LoRA | QLoRA |
| :--- | :--- | :--- | :--- |
| **参数更新量** | 100% (所有参数) | <1% (仅 Adapter) | <1% (仅 Adapter) |
| **显存需求** | 极高 (需要多卡 A100) | 低 (单卡可运行) | 极低 (消费级显卡) |
| **训练速度** | 慢 | 快 | 稍慢 (涉及量化计算) |
| **模型效果** | **最佳** (基准) | 接近全量 (95%+) | 接近全量 (90%+) |
| **部署复杂度** | 低 (原始权重) | 中 (需合并权重) | 中 (需合并权重+反量化) |
| **适用场景** | 追求极致效果/重新预训练 | 垂类领域适配/风格迁移 | 资源受限/本地实验 |

- **代码示例 (PEFT/LoRA 配置)**：
```python
from peft import LoraConfig, get_peft_model

peft_config = LoraConfig(
    r=16,            # LoRA 秩，越大参数越多
    lora_alpha=32,   # 缩放因子
    target_modules=["q_proj", "v_proj"], # 只微调注意力层
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM"
)
model = get_peft_model(model, peft_config)
model.print_trainable_parameters() # 打印可训练参数占比
```

- 原因：预训练模型缺乏领域深度、不懂指令格式、无私有知识。
- 全量微调：更新所有参数，效果最佳但成本极高。
