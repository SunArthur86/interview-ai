# 什么是投机解码(Speculative Decoding)？它如何加速LLM推理？

🎯 **本质**：投机解码用一个小模型(草稿模型)快速生成候选token，然后用大模型并行验证，在不损失质量的前提下加速推理。

🧒 **直观类比**：
写作文时，先让实习生（小模型）写一段草稿（快但可能粗糙），然后主编（大模型）一次性检查。如果草稿大部分正确，主编只需稍作修改或直接通过，效率远高于主编逐字撰写。

🏗️ **工作流程图**：
```text
[输入]
   │
   ▼
┌─────────────────────┐
│  Draft Model (小模型) │  
│  串行生成 N 个Token   │  Token: [t1, t2, t3, t4]
└─────────────────────┘
   │ 输出序列
   ▼
┌─────────────────────┐
│   Target Model (大)  │
│   并行处理 + 验证     │  输入: [Prefix + t1, t2, t3, t4]
└─────────────────────┘
   │
   ▼
┌─────────────────────┐
│  验证与采样          │
│  - t1: 验证通过       │
│  - t2: 验证通过       │  
│  - t3: 不匹配 -> 采样 │ -> 新Token t3_new
│  - 后续丢弃          │
└─────────────────────┘
   │
   ▼   
[输出] Prefix + t1 + t2 + t3_new (循环继续)
```

⚙️ **核心细节**：
1.  **并行验证机制**：大模型并非“判断对错”，而是以 t1, t2, t3... 为条件，并行输出这些位置的Logits分布。
2.  **接受标准**：通常从 t1 开始，使用 **Rejection Sampling**（拒绝采样）。如果大模型在位置 i 的分布中，草稿 token 的概率满足 $P(token_i | history) > random / q$ （其中 q 是草稿模型的概率），则接受；否则拒绝并从大模型分布中重新采样该位置，该位置之后的所有草稿Token全部丢弃。
3.  **数学保证**：由于拒绝采样算法的数学性质，最终生成的序列分布与大模型自回归生成的分布完全一致（无损）。

📊 **性能数据**：
| 场景 | 加速比 | 接受率 | 备注 |
|------|--------|--------|------|
| 7B目标+1B草稿 | 2-3x | 60-70% | 小模型越强，接受率越高 |
| 70B目标+7B草稿 | 2-4x | 50-65% | Spec Decoding 越适合大模型 |

🔄 **变体技术**：
-   **Medusa**：不使用单独的小模型，而是在大模型Layer末尾加多个“预测头”，直接预测后续Token，减少模型切换开销。
-   **EAGLE**：在特征层进行预测，而非Logits层，速度更快。
-   **MTP (Multi-Token Prediction)**：DeepSeek-V3采用，训练时即预测多个未来Token，为推理加速打基础。

## 常见考点
1.  **加速比的瓶颈是什么**？
    *   答案：**接受率**。如果草稿模型质量太差，接受率低，大模型频繁重采样，加速效果会归零甚至变负。通常需要大模型和小模型在训练数据上有一定对齐。
2.  **Tree Attention** 是什么？
    *   答案：为了进一步提高效率，草稿模型不只生成一条线，而是生成一个树状的候选集。大模型使用Tree Mask并行验证所有分支，这也是Lookahead Decoding或某些Speculative Decoding优化的核心技术。
3.  **为什么需要额外的显存开销**？
    *   答案：需要同时加载大模型和小模型的权重，且大模型推理时需要处理更长序列（Prefix + Draft Tokens），这会增加KV Cache的峰值占用。

## 记忆要点

- 核心原理：小模型快速生成草稿，大模型并行验证，通过接受草稿Token加速生成。
- 验证机制：使用拒绝采样，若大模型概率分布接受草稿则保留，否则重采样并丢弃后续。
- 无损保证：数学上保证最终分布与大模型自回归生成一致，不牺牲质量。
- 关键瓶颈：加速比取决于草稿模型的接受率，接受率低则加速效果差。
- 变体技术：Medusa（加预测头）、EAGLE（特征层预测）、MTP（训练时预测多Token）。
