# 多模态大模型的架构有哪些？

多模态LLM让模型理解和生成文本、图像、音频等多种模态，核心在于如何将非文本模态的特征与LLM的文本空间对齐。以下是目前主流的架构分类与详解：

**1. 视觉-语言模型（VLM）：**

**架构一：投影层（线性连接，最主流）**
- **原理**：利用预训练的 Vision Encoder（如 CLIP, SigLIP, ViT）提取图像特征图（通常 shape 为 [HxW, D]），通过一个轻量级的 Adapter（如 MLP, Q-Former, Linear）将视觉特征投影到 LLM 的词嵌入空间。
- **数据流**：图像token与文本token直接拼接，作为 LLM 的输入序列。
- **代表模型**：LLaVA（MLP Adapter）、Qwen-VL（C-Abstractor）、MiniGPT-4（Q-Former）。
- **优缺点**：训练参数少，部署简单；但视觉特征压缩导致细节丢失，且在需要高频像素级交互的任务上表现较弱。

**架构二：交叉注意力**
- **原理**：冻结 LLM 主干，在 Transformer 层之间插入可训练的 Cross-Attention 模块。视觉特征作为 Key (K) 和 Value (V)，LLM 的上文作为 Query (Q)。
- **代表模型**：Flamingo。
- **优缺点**：支持更灵活的视觉信息检索，支持变长图像输入；但推理显存开销大（KV Cache占用高），且通常难以完全保留 LLM 原有的泛化能力。

**架构三：原生多模态**
- **原理**：不使用独立的 Encoder，直接从像素或离散 VAE tokens 开始，与文本 token 在同一个统一的 Transformer 架构中进行端到端训练。
- **代表模型**：GPT-4V（推测架构）、Gemini（原生多模态）、Chameleon（Early Fusion）。
- **特点**：真正打通了模态壁垒，能够生成多模态 interleaved 的内容，推理时可以全模态思考。

**架构四：Pix2Struct（基于屏幕解析）**
- **原理**：不使用标准坐标系统，而是将截图转换为一系列 token 适合处理 UI 图表等。
- **代表**：Pix2Struct。

**2. 音频-语言模型：**
- **架构**：Audio Encoder（Whisper, CFormer）提取声学特征 -> Projector -> LLM。
- **特殊点**：语音通常包含时间维度，特征序列很长，常需下采样处理以适配 LLM 的上下文窗口限制。
- **代表**：SALMONN (Whisper Encoder + BEATS + LLaMA)。

**3. 全模态：**
- **Gemini Ultra**：原生支持文本、图像、音频、视频。使用 MoE（混合专家）架构处理海量数据。
- **AnyGPT**：利用离散表示（语音 Codec tokens, 图像 VAE tokens）统一所有模态为下一个 Token 预测任务。

### 架构对比表格

| 特性 | 投影层架构 | 交叉注意力架构 | 原生多模态 |
| :--- | :--- | :--- | :--- |
| **实现复杂度** | 低（只需训练Adapter） | 高（需修改LLM结构） | 极高（需从头训练） |
| **推理显存/KV Cache** | 标准LLM开销 | 较高（需缓存视觉KV） | 标准LLM开销 |
| **视觉细节保留** | 中等（受限于压缩） | 高（保留原始特征） | 极高（直接处理像素） |
| **多模态生成** | 仅支持文本输出 | 仅支持文本输出 | 支持图文混合生成 |
| **训练稳定性** | 高（冻结LLM） | 中（容易出现过拟合） | 低（数据配对难度大） |
| **代表模型** | LLaVA, Qwen-VL | Flamingo | GPT-4V, Chameleon |

### 实战案例
在构建电商商品图表理解Agent时，我们曾尝试使用投影层架构，发现模型在理解密集表格的小数字时容易出错（特征压缩导致）。后来改用基于 **C-Abstractor**（类似Qwen-VL）的架构，增加视觉Token数量，并配合“Crop-resize”数据增强策略，才解决了OCR精度不足的问题。

### 代码示例
这是使用PyTorch实现简单的 **MLP Projector**（投影层）的关键代码片段，展示了如何将视觉特征对齐到LLM的词嵌入维度：

```python
torch
import torch.nn as nn

class VisionProjector(nn.Module):
    def __init__(self, vision_dim=1024, llm_dim=4096, hidden_dim=4096):
        super().__init__()
        # 使用两层MLP将CLIP特征映射到LLM嵌入空间
        self.net = nn.Sequential(
            nn.Linear(vision_dim, hidden_dim),
            nn.GELU(),
            nn.Linear(hidden_dim, llm_dim)
        )

    def forward(self, image_features):
        # image_features shape: [Batch, Num_Patches, Vision_Dim]
        return self.net(image_features) # Output: [Batch, Num_Patches, LLM_Dim]
```

## 记忆要点

- 投影层（主流）：Vision Encoder提取特征，通过MLP/Q-Former投影到LLM词嵌入空间。
- 交叉注意力：在LLM层间插入Cross-Attention，视觉特征做KV，保留细节但显存高。
- 原生多模态：不使用独立Encoder，像素/Token端到端训练，支持图文混合生成。
- 音频处理：Audio Encoder提取特征后同样需Projector对齐，注意时间序列下采样。
