# 什么是Function Calling（函数调用）？LLM如何实现工具使用？

🎯 **本质**：Function Calling让LLM能够调用外部函数/API，获取实时信息或执行操作，突破了LLM「只有静态知识」的限制。

📊 **工作流程**：
1. 定义可用函数（名称、参数、描述，JSON Schema格式）。
2. LLM分析用户意图，决定是否需要调用函数。
3. 如果需要，LLM输出函数名和参数。
4. 应用层执行实际函数调用，获取返回结果。
5. 将返回值送回LLM，LLM基于结果生成最终回复。

**示例（天气查询）**：
用户：北京今天天气怎么样？
LLM输出：需要调用 get_weather(city=Beijing)
应用执行：调用天气API -> {temp: 25, condition: sunny}
LLM回复：北京今天天气晴朗，气温25度。

**主流实现**：
1. OpenAI Function Calling：原生支持。
2. Anthropic Tool Use：Claude的tool_use。
3. 开源：Gorilla、Llama 3.1+、GLM-4+。
4. Agent框架：LangChain Tools、AutoGen、CrewAI。

**关键挑战**：
1. 函数选择准确性——选对函数。
2. 参数提取准确性——正确提取参数。
3. 多步骤编排——多个函数组合调用。
4. 错误处理——函数执行失败。
5. 并行调用——同时调用多个独立函数。

**最佳实践**：函数描述清晰、参数类型明确、提供错误处理、限制可用函数数量。

💥 **实战案例**：在开发邮箱助手时，LLM经常在日期格式上出错（如输入"5月12日"，API需"2023-05-12"）。我们在Prompt中增加了**Few-Shot Examples**展示日期转换，并配合Pydantic进行强制类型校验，将参数解析错误率降低了90%。

💻 **代码示例 (Python + OpenAI)**：
```pythonnimport json
from openai import OpenAI

tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get the current weather in a given location",
        "parameters": {
            "type": "object",
            "properties": {
                "location": {"type": "string", "description": "City and state, e.g. San Francisco, CA"},
                "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
            },
            "required": ["location"]
        }
    }
}]

# 关键：强制模型调用工具 (tool_choice)
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "What's the weather in Tokyo?"}],
    tools=tools,
    tool_choice={"type": "function", "function": {"name": "get_weather"}}
)
```

## 常见考点
1. **并行调用**：当用户的请求需要调用多个独立工具时（如同时查天气和股票），LLM 应该如何输出？（答案：LLM 可以输出一个包含多个 tool_call 的数组，客户端应并发执行这些调用）。
2. **强制函数调用**：如何确保模型一定会调用某个函数？（答案：在 API 参数中设置 `tool_choice={"type": "function", "function": {"name": "my_function"}}`）。
3. **Token 消耗**：Function Calling 会显著增加 Token 消耗吗？（答案：是的，工具定义和返回结果都会计入Context Window，需注意Prompt管理）。
