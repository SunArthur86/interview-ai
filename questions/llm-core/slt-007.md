# 预训练数据处理有哪些关键步骤？

高质量预训练数据是大模型涌现能力的基石。数据处理流程通常包含以下关键步骤：

```text
[Raw Data] -> [Deduplication] -> [Filtering] -> [Mixing] -> [Tokenization] -> [Training]
```

**1. 数据收集**
- **来源**：网页爬取（Common Crawl, RefinedWeb）、书籍（Books3, Project Gutenberg）、代码（GitHub, StarCoder）、论文（arXiv, S2ORC）、百科（Wikipedia）。
- **权属**：重视数据的License合规性（如Apache 2.0, MIT, CC-BY）。

**2. 数据清洗**
- **去重**：防止“死记硬背”和过拟合。
  - **精确去重**：去除完全相同的文档。
  - **模糊去重**：使用MinHash + LSH（局部敏感哈希）检测高相似度文档（Near-duplicates）。
- **质量过滤**：
  - **启发式规则**：过滤过短文本、包含过多HTML标签/特殊符号的文本。
  - **语言模型过滤**：使用轻量级模型计算困惑度（PPL），去除语法混乱的“垃圾”文本（如KenLM）。
  - **有害内容过滤**：基于关键词或分类器去除色情、暴力、仇恨言论。
- **PII去除**：去除个人隐私信息（如邮箱、电话、身份证号）。

**3. 数据配比**
- **科学配比**：不同数据源对能力贡献不同。一般比例参考：
  - 网页文本（50-60%）：通用知识和世界模型。
  - 代码（10-20%）：显著提升逻辑推理和思维链能力。
  - 书籍与长文本（10-15%）：提升长文本理解和文学性。
  - 学术论文（5%）：提升专业知识和科学推理。
- **退火**：在训练最后阶段，使用高质量、教科书级的数据进行“退火”，显著提升Benchmark表现。

**4. Tokenization**
- 使用BPE/Unigram/BBPE编码。
- 通常在预训练前对数据分片并打包以提高IO效率。

**5. 课程学习**
- **Warmup**：初期使用较小且简单的数据，稳定梯度。
- **逐步提升**：随着训练步数增加，逐步增加更难、更专业的数据。
- **数据量级**：Llama 3使用了超过15T tokens的高质量数据，且大幅提升了代码和数学数据的比例。数据质量（Quality > Quantity）是目前的核心竞争点。

### 实战深化

**实战案例**：
在处理大规模网页数据时，若不进行MinHash模糊去重，模型容易重复记忆热门新闻或网页的Footer/Header信息，导致推理时出现严重的复读现象。通过MinHash LSH去重后，模型在生成多样性上有明显提升，且过拟合损失下降得更慢但泛化性更好。

**代码示例 (Python - MinHash 去重思路)**：
```python
from datasketch import MinHash, MinHashLSH

def dedupe_docs(docs, threshold=0.8):
    # 创建LSH索引
    lsh = MinHashLSH(threshold=threshold, num_perm=128)
    minhashes = {}
    
    for idx, doc in enumerate(docs):
        # 计算MinHash签名
        m = MinHash(num_perm=128)
        for d in doc:
            m.update(d.encode('utf8'))
        lsh.insert(idx, m)
        minhashes[idx] = m
        
    # 查询近似重复
    for idx in minhashes:
        result = lsh.query(minhashes[idx])

```

**对比表格**：

| 步骤 | 技术/方法 | 作用 | 常见工具/库 |
| :--- | :--- | :--- | :--- |
| **去重** | Exact Dedup (精确去重) | 去除完全相同的数据，节省算力 | SHA-256, Spark
| | MinHash + LSH | 快速检测海量数据中的近似重复（洗稿、转载） | datasketch, Spark-MinHash
| **过滤** | Heuristic Rules (规则) | 去除HTML标签、过短文本、过多标点符号 | Regex, BeautifulSoup
| | Perplexity (困惑度) | 去除低质量、生成式或乱码文本 | KenLM, FastText
| **配比** | Curriculum (课程学习) | 从简单到难训练，提升收敛稳定性 | Custom Dataloader
| | Data Annealing (退火) | 训练末期用高质量数据微调，提升指令遵循能力 | High-Quality Books/Code |

## 记忆要点

- 流程：去重（精确/模糊MinHash）-> 过滤（规则/PPL）-> 配比 -> 分词。
- 去重：防死记硬背，MinHash+LSH处理近似重复（洗稿/转载）。
- 配比：网页为主，代码提升逻辑，退火（Annealing）用高质量数据收尾。
