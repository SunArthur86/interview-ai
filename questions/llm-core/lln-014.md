# 什么是Self-consistency和Best-of-N

两种方法都是基于聚合的测试时计算方法，即在大模型推理阶段通过生成多个回答并聚合以获得最终输出。

### Self-consistency（自一致性）
**原理**：让大模型生成多个 CoT（Chain of Thought）推理路径及多个答案，通过“多数投票”选出最终答案。
*   **核心逻辑**：假设虽然推理路径不同，但针对逻辑严密的数学或常识问题，正确的推理路径往往汇聚于同一个正确答案。
*   **流程**：
    1. Prompt 中加入“Let's think step by step”。
    2. 设定采样参数（如 temperature=0.7），生成 N 个不同的解题轨迹。
    3. 统计 N 个轨迹中最终答案的分布，取众数。

### Best-of-N
**原理**：让大模型生成 N 个回答，使用 Verifier、ORM (Outcome Reward Model) 或 PRM (Process Reward Model) 给每个回答评分，选择分数最高的作为最终回答。
*   **核心逻辑**：显式引入奖励模型来筛选质量，而非依赖答案的随机分布一致性。
*   **区别**：Self-consistency 依赖“投票”，适合有唯一正确解的场景；Best-of-N 依赖“打分”，适合开放域生成或代码生成等需要质量优化的场景。

```text
   输入 Prompt
      │
      ▼
┌─────────────────┐
│  多次采样生成    │ (N次, 如N=20)
│  (Sample T times)│
└────────┬────────┘
         │
    ┌────┴────┬────────────┐
    ▼         ▼            ▼
┌───────┐ ┌──────────┐ ┌──────────┐
│Answers│ │Answers...│ │Answers   │
└───┬───┘ └────┬─────┘ └────┬─────┘
    │          │           │
    │   Self-consistency   │
    │   (投票/众数)        │
    ▼                      │
 最终答案                  │
                          │ Best-of-N
                          │ (Verifier/PRM打分)
                          ▼
                       最终答案
```

### 深化内容

#### 1. 实战案例
在代码生成场景中，直接使用 Best-of-N (N=10) 配合一个轻量级的 Code Verifier（通过运行单元测试），可以从 10 个候选代码中选择唯一通过测试的版本，比单纯增加模型规模更能有效提升通过率。Self-consistency 则常用于数学竞赛题（如 MATH 数据集），通过 40 次采样投票大幅提升准确率。

#### 2. 代码示例 (Self-consistency 投票逻辑)
```python
import collections

def self_consistency(answers):
    """统计答案列表中的众数"""
    # 假设 answers 是提取出的最终答案列表 ['A', 'B', 'A', 'C', 'A']
    counter = collections.Counter(answers)
    # 返回出现次数最多的答案
    return counter.most_common(1)[0][0]
```

#### 3. 对比表格

| 维度 | Self-consistency | Best-of-N |
| :--- | :--- | :--- |
| **核心机制** | 投票 | 裁判打分 |
| **所需模型** | 仅需 LLM | LLM + Reward Model (额外开销) |
| **适用任务** | 数学、逻辑推理、有标准答案的任务 | 代码生成、开放域问答、创意写作 |
| **计算成本** | 中 (仅需生成) | 高 (生成 + 评分) |
| **鲁棒性** | 依赖群体共识，容易受系统性错误影响 | 依赖 RM 的准确性，抗群体性强 |
| **输出形式** | 唯一确定 (众数) | 最优解 (Score Max) |

## 记忆要点

- Self-consistency：生成多条CoT路径，通过多数投票（众数）选出最终答案。
- Best-of-N：生成N个回答，利用Verifier或Reward Model打分，选最高分。
- 区别：自一致性依赖投票适合标准答案题；Best-of-N依赖打分适合开放域或代码。
- 成本：两者均增加推理计算量，Best-of-N需额外RM评分开销。
