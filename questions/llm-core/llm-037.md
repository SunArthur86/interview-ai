# 为什么大模型用GQA(Group Query Attention)

大模型使用 **GQA (Grouped-Query Attention)** 是为了在 **推理速度** 和 **模型性能** 之间取得更好的平衡，主要解决推理时的显存带宽瓶颈。

### 1. 背景机制
*   **MHA (Multi-Head Attention)**：每个注意力头都有独立的 Key (K) 和 Value (V) 矩阵。
    *   *优点*：表达能力强，捕捉特征丰富。
    *   *缺点*：KV Cache 占用显存大，读取 KV 带宽开销高，推理慢。
*   **MQA (Multi-Query Attention)**：所有头共享同一组 K 和 V 矩阵，仅 Query (Q) 独立。
    *   *优点*：KV Cache 极小，推理速度极快。
    *   *缺点*：表达能力受限，模型性能下降明显。

### 2. GQA 的原理
GQA 是 MHA 和 MQA 的折中方案。它将查询头分为若干组，**组内**的所有头共享同一组 K 和 V 矩阵。

*   *机制*：假设有 $H$ 个 Query 头，将其分为 $G$ 组。每组包含 $H/G$ 个头，它们共享同一个 KV 矩阵对。
*   *特例*：当 $G=1$ 时，退化为 MQA；当 $G=H$ 时，退化为 MHA。

### 3. 实战案例与代码
**实战案例**：在 LLaMA 3 70B 的推理服务中，从 MHA 迁移到 GQA 后，长文本（Context Length 8k+）场景下的首字延迟（TTFT）降低了约 30%，同时并没有像 MQA 那样出现模型指令遵循能力明显下降的问题。但在部署时要注意，如果 GPU 显存带宽极低（如 P40），GQA 的加速效果会比 A100/H100 更明显，因为瓶颈主要卡在内存读写上。

**代码示例**：
```python
import torch
import torch.nn.functional as F

def grouped_query_attention(q, k, v, n_heads, n_groups):
    """
    q: [batch, seq_len, n_heads, head_dim]
    k, v: [batch, seq_len, n_groups, head_dim]
    """
    batch, seq_len, _, head_dim = q.shape
    # 将 K/V 重复以匹配 Q 的头数 (Upsampling)
    # n_heads_per_group = n_heads // n_groups
    k = k.repeat_interleave(n_heads // n_groups, dim=2) 
    v = v.repeat_interleave(n_heads // n_groups, dim=2)
    
    # 标准 Attention 计算 (转置为 [batch, heads, seq, dim])
    q, k, v = q.transpose(1, 2), k.transpose(1, 2), v.transpose(1, 2)
    attn_scores = torch.matmul(q, k.transpose(-2, -1)) / (head_dim ** 0.5)
    attn_weights = F.softmax(attn_scores, dim=-1)
    output = torch.matmul(attn_weights, v)
    return output.transpose(1, 2).contiguous() # [batch, seq_len, n_heads, head_dim]
```

### 4. 架构对比

| 特性 | MHA (Multi-Head) | GQA (Grouped-Query) | MQA (Multi-Query) |
| :--- | :--- | :--- | :--- |
| **KV Cache 独立性** | 每个头独立 | 组内共享 | 所有头共享 |
| **显存占用 (KV)** | 最大 ($H$) | 中等 ($G$, $1 < G < H$) | 最小 ($1$) |
| **推理带宽瓶颈** | 高 (Read heavy) | 中 (优化明显) | 低 (Compute bound) |
| **模型表达能力** | 最强 | 接近 MHA | 较弱 (易丢失细节) |
| **工程实现复杂度** | 低 | 中 (需 Upsampling) | 低 |
| **典型代表模型** | LLaMA 1, GPT-3 | LLaMA 2/3, Mistral, PaLM | Falcon, BLOOM |

### 5. 关键细节补充
*   **KV Cache 显存计算**：对于 Batch Size=1，序列长度 $L$，Hidden Size $d$，层数 $L_{layer}$。
    *   MHA 显存占用：$2 \times L_{layer} \times H \times L \times d_{k}$ ($H$ 为头数，$d_k$ 为每头维度)。
    *   GQA 显存占用：$2 \times L_{layer} \times G \times L \times d_{k}$ ($G$ 为组数，通常 $G \ll H$)。
    *   MQA 显存占用：$2 \times L_{layer} \times 1 \times L \times d_{k}$。
*   **Upsampling 机制**：在计算 Attention 分数时，共享的 K 和 V 需要复制（Repeat） $H/G$ 次以匹配 Q 的头数，或者在矩阵乘法时利用广播机制实现。
*   **性能回归**：从 MHA 迁移到 GQA 时，通常需要进行 **Up-training（增量训练）** 以恢复模型性能，直接使用 GQA 权重可能会导致效果略有下降。
