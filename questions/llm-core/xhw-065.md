# NTK长度外推方法

### NTK长度外推方法

#### 1. 背景与问题
**位置编码与外推难题**：
Transformer 模型（如 Llama、ChatGLM）使用位置编码（如 RoPE）注入序列信息。通常训练时的序列长度有上限（如 4096）。当推理时遇到更长的序列（如 16k），位置索引 $p$ 超出了模型见过的范围。这会导致位置编码对应的角度（旋转角度）过大，模型无法理解这种“陌生”的分布，导致性能骤降。

#### 2. NTK (Neural Tangent Kernel) 视角
NTK 描述了神经网络在无限宽度假设下，输入之间的相似度度量。如果位置的变化导致网络内部（特别是注意力机制）的相似度计算发生剧变（即 NTK 性质改变），模型就会失效。
**核心思路**：通过调整位置索引的计算方式，使得即使在更长的序列下，网络感受到的“相对距离”或“频率分布”仍然保持与训练时相似的平滑性。

#### 3. NTK-Aware 缩放方法
为了解决上述问题，NTK-Aware 方法主张对位置索引 $p$ 进行变换，将大的 $p$ 映射回模型见过的“有效范围”内，或者改变频率的增长速度。

**3.1 线性缩放**
最简单的做法，按比例压缩位置索引。
公式：$p_{eff} = p \cdot \frac{T_{train}}{T_{test}}$
- **含义**：推理时将实际位置 $p$ 缩放，使得最大位置 $T_{test}$ 对应的训练位置仍然是 $T_{train}$。
- **优点**：实现简单。
- **缺点**：仅保证边界一致，中间位置的关系可能失真。

**3.2 NTK-Aware 缩放**
不仅仅是缩放位置，而是缩放 RoPE 中的频率基数，或者对不同的频率分量进行不同程度的缩放。
- **原理**：通过放大旋转频率的基数（例如 $\theta_i' = \theta_i \cdot s$），使得模型在相同物理距离下的旋转角度变化更慢，从而在不修改位置索引 $p$ 的情况下，让模型感知到的“相对距离”变近了。
- **公式**：将原本的 $\theta_i = 10000^{-2i/d}$ 修改为 $\theta_i' = b \cdot 10000^{-2i/d}$（其中 $b$ 为缩放因子），或者结合插值方法动态调整。

**频率变换图示**：
```text
RoPE 频率分布 (高频 -> 低频):
Channel 0: f0 (高频)
Channel 1: f1
...
Channel k: fk (低频)

NTK-Aware Scaling:
所有频率乘以因子 alpha (>1)
结果: 旋转速度变慢，同样的距离 d 产生的角度变化变小
=> 模型误以为距离较短 => 能够处理更长的实际序列
```

#### 4. 小结
NTK 长度外推的核心在于**保持网络注意力分布的数学性质**。通过调整位置编码的计算方式（缩放位置或缩放频率），使得长序列输入在经过处理后，激活的分布与短序列训练时尽可能保持一致，从而实现性能稳定的外推。

#### 5. 实战深化

**实战案例**：
在 Llama-2-7B 推理服务中，直接将 Context Length 从 4k 扩展到 32k 时，模型在生成内容末尾开始出现重复和乱码（Attention Score 塌陷）。应用 NTK-Aware Scaling (alpha=4) 后，模型在长文本的“大海捞针”测试中召回率从 15% 提升至 92%。

**代码示例 (NTK-Aware RoPE 缩放)**：
```python
import torch

def apply_ntk_aware_scaling(freqs, alpha, actual_seq_len):
    """
    freqs: 原始频率张量 (head_dim / 2)
    alpha: 缩放因子 (通常设置为 actual_seq_len / training_len)
    """
    # 只在位置索引超出训练长度时应用 NTK 缩放
    # 这里简单演示修改 Base 的计算逻辑
    
    # 原始 RoPE base = 10000
    # NTK scaling: base' = base * alpha
    new_base = 10000 * alpha
    
    # 重新计算频率
    head_dim = freqs.shape[-1]
    exponents = torch.arange(0, head_dim, 2, dtype=torch.float32) / head_dim
    new_freqs = 1.0 / (new_base ** exponents)
    
    return new_freqs.to(freqs.device)
```

## 记忆要点

- NTK外推解决RoPE位置编码超出训练长度导致性能骤降的问题。
- 核心思路是缩放RoPE的频率基数，而非简单的线性位置插值。
- 通过放大频率基数，使模型感知的相对距离变近，从而适应更长序列。
