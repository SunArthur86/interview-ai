# MHA、MQA和GQA有什么区别？

**MHA (Multi-Head Attention)**：每个注意力头有独立的Q、K、V投影矩阵。标准Transformer使用。N个头意味着N份KV Cache，计算量大且显存带宽占用高。

**MQA (Multi-Query Attention)**：所有头共享同一组K和V投影矩阵（即K、V的头数为1），只有Q保留多头。极端省显存，大幅降低带宽压力，但可能损害模型表达能力。如PaLM、Falcon。

**GQA (Grouped-Query Attention)**：MHA和MQA的折中方案。将Q头分成G组，每组内的头共享一对K、V。当G=N时等价于MHA，当G=1时等价于MQA。如Llama 2 70B、Llama 3、Mixtral。

**架构对比图**：
```text
Input X
  │
  ├─────────────────────────────────────────────────────┐
  │                                                       │
  ▼ (MHA)                                                ▼ (MQA)
┌─────────────┐                                       ┌─────────────┐
│ Q1 K1 V1    │ (3 Projections)                       │ Q1 ... QN   │ (N Projs for Q)
│ Q2 K2 V2    │                                       │             │
│ ...         │                                       │ K1 V1       │ (Shared 1 Proj)
│ QN KN VN    │                                       │             │
└─────────────┘                                       └─────────────┘
  │                                                       │
  ▼                                                       ▼
N Independent Heads                                    N Heads share 1 KV Pair

  │ (GQA) ────────────────────────────────────────────────┘
  ▼
┌─────────────────────────────┐
│ Q1...Qk │ K1 V1 (Shared)     │
│ Qk+1...Qm │ K2 V2 (Shared)   │
│ ...      │ ...               │
└─────────────────────────────┘
  │
  ▼
G Groups (Each has 1 KV pair)
```

**KV Cache显存对比**（假设总头数为N，Hidden Size为d，GQA分组数为G）：
- MHA: KV Cache = 2 × N × d × seq_len（每头独立KV）
- MQA: KV Cache = 2 × 1 × d × seq_len（所有头共享1份KV，减少N倍）
- GQA: KV Cache = 2 × G × d × seq_len（减少N/G倍）

**性能与效果权衡**：
GQA在保持接近MHA质量的同时，大幅减少推理时的KV Cache内存占用。更重要的是，它显著降低了Decode阶段对显存带宽的需求，因为每次读取KV Cache的数据量减少了，从而成为吞吐量受限场景下的首选优化。

**实战案例**：
在将Llama-2-Chat从MHA转为GQA（如Llama 3）进行量化部署时，若直接加载量化权重不进行Up-training，模型会出现严重的逻辑混乱（"语无伦次"）。这是因为原本分离的Head特征被强行聚合，需要少量的对齐训练步数来恢复表达能力。

**代码示例（PyTorch，GQA投影简化示意）**：
```python
import torch
import torch.nn as nn

class GQA(nn.Module):
    def __init__(self, hidden_size, num_heads, num_kv_heads):
        super().__init__()
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads
        self.head_dim = hidden_size // num_heads
        
        # Q投影: hidden -> num_heads * head_dim
        self.q_proj = nn.Linear(hidden_size, num_heads * self.head_dim, bias=False)
        # KV投影: hidden -> num_kv_heads * head_dim (显存减少的关键)
        self.k_proj = nn.Linear(hidden_size, num_kv_heads * self.head_dim, bias=False)
        self.v_proj = nn.Linear(hidden_size, num_kv_heads * self.head_dim, bias=False)

    def forward(self, x):
        B, T, C = x.shape
        q = self.q_proj(x).view(B, T, self.num_heads, self.head_dim)
        k = self.k_proj(x).view(B, T, self.num_kv_heads, self.head_dim)
        v = self.v_proj(x).view(B, T, self.num_kv_heads, self.head_dim)
        
        # 实际推理中还需处理KV的Repeat以匹配Q的Head数进行Attention计算
        return q, k, v
```

**对比表格**：
| 特性 | MHA | MQA | GQA |
| :--- | :--- | :--- | :--- |
| **KV Cache 大小** | 100% (基准) | ~1% (1/N) | ~10-25% (取决于G) |
| **显存带宽压力** | 极高 (Decode瓶颈) | 极低 | 中低 |
| **模型表达能力** | 最强 | 较弱 (长文本/复杂推理略差) | 接近MHA (G=N/4~N/8时) |
| **算术强度 (ARITH INTENSITY)** | 低 (访存比高) | 高 | 较高 |
| **典型应用** | BERT, GPT-3 | PaLM, Falcon | Llama 3, Mixtral, Qwen2 |

## 常见考点
1. **Up-training（从MHA转GQA）**：如果在预训练时使用了MHA，如何在推理或微低成本时转换为GQA？（需进行少量的额外训练步数来对齐KV投影，如Llama 2/3的做法）
2. **带宽 vs 算力瓶颈**：MQA/GQA主要解决的是算力问题还是带宽问题？（主要是带宽瓶颈，特别是Decode阶段）
3. **Kernel实现**：MHA/GQA/MQA在FlashAttention等Kernel实现上的计算密度

## 记忆要点

- MHA：每头独立KV，显存大；MQA：所有头共享KV，显存极小。
- GQA：折中方案，Q分组共享KV，平衡性能与显存（Llama 3采用）。
- 收益：GQA/MQA大幅降低推理带宽压力，提升吞吐量。
