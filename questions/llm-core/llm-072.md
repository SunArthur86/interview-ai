# 什么是长度外推

### 1. 什么是长度外推
**长度外推** 是指模型在推理时能够处理比训练阶段所见过的最大序列长度更长的输入，且性能不出现严重衰减的能力。

### 2. 问题背景
*   **训练限制：** 受限于显存和计算资源，Transformer 通常在固定长度（如 512、1024 或 4096 tokens）上训练。
*   **位置编码局限：** 传统的绝对位置编码（如 Sinusoidal）对位置 ID 有范围限制，超出训练范围的位置可能产生未知的偏置，导致模型失效。

### 3. 为什么重要
*   **应用落地：** 实际场景中常出现长文档、长代码、长对话，超出模型预设长度。
*   **灵活性：** 具备外推能力的模型无需为了更长序列重新训练，降低了维护成本。
*   **处理长尾：** 能够适应输入长度的动态变化，提供更连贯的长文本理解。

### 4. 核心挑战与数学原理
长度外推的本质是让模型学习**相对位置**或**位置间的某种函数关系**，而非死记硬背绝对位置 ID。
*   **分布偏移：** 当输入长度 $L_{inference} > L_{train}$ 时，Attention Score 的分布会发生偏移。如果在训练时从未见过某个位置的组合（例如相对距离 > 1024），模型对其的注意力权重往往不可预测。
*   **Attention Extrapolation：** 理想的外推要求 $Attention(q, k, pos_i, pos_j)$ 在 $|i-j|$ 增大时能保持平滑衰减，而不是突变。

### 5. 常见解决方案对比

| 方法 | 类型 | 原理简述 | 外推能力 |
| :--- | :--- | :--- | :--- |
| **Sinusoidal (Transformer)** | 绝对位置 | 正弦/余弦函数固定频率 | 较弱，超出长度后位置含义未见过 |
| **Learned Absolute (BERT/GPT)** | 绝对位置 | 查表 Embedding | 无，需插值或直接截断 |
| **RoPE (Llama/GLM)** | 旋转相对位置 | 通过复数旋转将绝对位置编码转化为相对位置信息 | **较强**，通过 NTK-aware scaling 可增强 |
| **ALiBi** | 相对位置偏置 | 在 Score 上减去与距离成正比的惩罚 | **极强**，专门为此设计 |
| **YaRN** | 混合改进 | 结合 RoPE 和 NTK scaling 的改进版 | 最强，当前 LLM 长文本主流方案 |

### 6. 实战案例：NTK-Aware Scaling
在 LLaMA 推理时，若不修改代码直接将 Context 从 2k 扩到 8k，效果会断崖下跌。**NTK-Aware Scaling** 是工程中常用的“急救”技巧：在推理时放大 RoPE 的 Base Frequency（如从 10000 改为 40000），让位置编码的旋转频率更密，从而在不重新训练的情况下“骗”过模型，使其认为当前距离仍在训练范围内，显著提升外推效果。

### 8. 常见考点
1.  **RoPE 的外推原理**：为什么旋转位置编码比绝对位置编码更适合外推？（因为它将距离信息编码到了向量的点积中，具备了相对位置的性质）。
2.  **NTK-Aware Scaling**：在推理时延长上下文，除了简单的线性插值，为什么推荐使用 NTK Scaling？（通过调整 Base Frequency 来模拟更高频的正弦波，使模型能分辨更远的距离）。
3.  **评估指标**：如何衡量长度外推的效果？（通常使用 "Passkey Retrieval" 任务：在长文本中插入随机 Key，测试模型能否准确召回）。
