# GRPO(Group Relative Policy Optimization)的原理是什么?相比PPO有哪些改进

GRPO是DeepSeek提出的PPO改进算法,核心创新:去掉Critic网络,用组内相对奖励估计baseline.

- **PPO的问题:**
1. 需要训练Critic网络估计V(s),增加训练复杂度和显存
2. Critic估计不准会导致策略梯度方差大
3. 4个模型同时训练,工程复杂

- **GRPO的核心改进:**

**PPO的Advantage：** A(s,a) = r(s,a) + gamma*V(s') - V(s) 需要Critic
**GRPO的Advantage：** 对每个prompt采样G个回复,组内归一化
A_i = (r_i - mean(r_1,...,r_G)) / std(r_1,...,r_G)

- **GRPO流程:**
1. 对每个prompt采样一组G个回复
2. 用规则奖励(如数学正确性)或RM打分
3. 组内归一化得到相对优势
4. 用PPO-style目标函数(带clip和KL惩罚)更新策略

- **优势:**
- 去掉Critic网络,只需2个模型(Policy/Ref)
- 组内相对比较更稳定
- 特别适合有明确正确答案的任务(数学、代码)
- DeepSeek-R1使用GRPO实现了强大的推理能力

- **算法架构补充:**
GRPO 的关键假设是：对于同一个 Query，生成的多个 Outputs 构成一个组，该组内奖励的均值可以作为一个很好的 Baseline 替代 Critic 的 $V(s)$ 估计。
优势函数计算公式标准化：
$$ A_i = \frac{r_i - \text{mean}(\mathbf{r})}{\text{std}(\mathbf{r}) + \epsilon} $$
这种 Group Normalization 方式天然去除了 Reward Model 可能存在的偏差（只要 RM 对同一组输出的排序或相对打分是准的，绝对值偏差会被均值抵消）。

- **实战案例:** DeepSeek-R1 在强化学习阶段，对于一道数学题，模型会同时生成 64 个不同的解题路径。如果 Critic 网络难以训练（如代码环境交互奖励稀疏），PPO 会失效；而 GRPO 只要计算出这 64 个路径中多少个通过了测试用例（Reward 为 0 或 1），就能通过组内均值稳定地更新策略，奖励波动不会直接导致训练崩溃。

- **代码示例:**
```python
# GRPO Advantage 计算与 PPO 风格 Loss
def compute_grpo_loss(old_log_probs, new_log_probs, rewards, kl_coef, clip_ratio=0.2):
    # rewards: [batch_size, group_size] e.g. shape [4, 64]
    # 计算组内均值和标准差作为 baseline
    mean = rewards.mean(dim=-1, keepdim=True)
    std = rewards.std(dim=-1, keepdim=True) + 1e-6
    
    # 计算优势函数
    advantages = (rewards - mean) / std
    
    # PPO Ratio
    ratio = torch.exp(new_log_probs - old_log_probs)
    
    # PPO Clip Loss
    surr1 = ratio * advantages
    surr2 = torch.clamp(ratio, 1 - clip_ratio, 1 + clip_ratio) * advantages
    loss = -torch.min(surr1, surr2).mean() + kl_coef * (new_log_probs - old_log_probs).mean()
    return loss
```

## 记忆要点

- GRPO去掉Critic网络，用组内采样G个回复的均值作为Baseline
- 优势函数A_i = (r_i - mean) / std，通过组内归一化稳定训练
- 适合数学代码等有明确规则奖励的任务，DeepSeek-R1使用
