# 对于大模型复杂推理任务,有哪些主流方法

复杂推理任务是大模型目前讨论最多的基础能力，试图通过增强 LLM 的复杂推理能力来缓解幻觉问题，同时赋予 LLM 更多的可能性。

## 一、 Prompt Engineering
**一句话总结**：仅仅通过 Prompt 最大化地激发出 LLM 的知识，从而获得更好的推理结果。

### 1. CoT (Chain-of-Thought)
**原理**：让 LLM 通过分步推理生成答案，模仿人类思考过程，将复杂问题拆解为多个中间步骤。
- **实现**：
  - Zero-shot CoT：在 Prompt 中加入“Let's think step by step”。
  - Few-shot CoT：给予多个示例，展示如何分步解决问题。
- **优缺点**：
  - 优点：降低单步错误率，推理过程透明，便于验证。
  - 缺点：增加推理开销（计算和时间），对无需推理的任务效率低，依赖示例质量。
- **实战案例**：在处理复杂SQL生成时，直接生成SQL错误率高，强制要求模型先分析表结构、再拆解业务逻辑为自然语言步骤，最后写SQL，准确率提升约30%。

### 2. ToT (Tree of Thoughts)
**原理**：将单一线性 CoT 扩展为树状结构。允许模型生成多个可能的推理路径，并进行评估、回溯或剪枝。
- **关键实现**：
  - **节点生成**：生成多个中间思路。
  - **评估与剪枝**：对节点打分（如正确性、可能性），淘汰低质量分支。
  - **搜索策略**：采用广度优先 (BFS) 或深度优先 (DFS) 探索路径。

### 3. SC (Self-Consistency)
**原理**：通过生成多条推理路径，并采用“多数投票”机制选择最一致的答案，提升鲁棒性。
- **实现细节**：
  - 调高 Temperature 以生成多样化的推理路径（通常 5~40 条）。
  - 对多条路径的最终答案进行统计（如投票），选择出现频率最高的结果。

```text
Input: Q
  +-----------------+
  |  Sample 1       | ---> Answer A
  |  (CoT Path 1)   |
  +-----------------+
  |  Sample 2       | ---> Answer A (Winner)
  |  (CoT Path 2)   |
  +-----------------+
  |  Sample 3       | ---> Answer B
  |  (CoT Path 3)   |
  +-----------------+
```
- **实战案例**：金融风控场景中，利用SC对30条路径的决策结果进行投票，有效过滤掉了因偶然Token分布导致的“幻觉”拒贷判定。

### 4. PAL (Program-Aided Language Models)
**原理**：让 LLM 生成可执行的代码（如 Python）来解决推理问题，而非直接给出数值答案。
- **优势**：利用解释器保证计算的精确性，特别适合数学、逻辑和符号推理任务。
- **代码示例**：
```python
# Prompt: Write a Python function to solve... 
def solve_profit(cost, margin):
    return cost * (1 + margin)
# Execution Env runs this code and returns result
```

## 二、 Training
### 1. SFT (Supervised Fine-Tuning)
通过在特定的推理数据集上进行监督微调，让模型掌握推理模式。
- **常用数据集**：GSM8K, MATH, LogiQA, ARC。

### 2. RLHF / RLAIF (Reinforcement Learning)
- **流程**：生成回答 -> 评价 (人或AI) -> 训练 Reward Model -> PPO 优化。
- **Process Reward Model (PRM)**：对推理过程的每一步进行打分，而非只看结果。

## 三、 Advanced Methods
### 1. ReAct (Reason + Act)
结合推理与行动，模型先思考下一步动作，再执行（如调用搜索工具、计算器），然后观察结果继续思考。

### 2. Self-Refine
生成初始答案 -> 自我反思/批评 -> 根据批评修改答案。

## 常见考点
1.  **CoT 的涌现**：为什么只有大模型（>10B）才有效？（答案：小模型缺乏复杂的逻辑归纳能力，难以维持长程的推理链）。
2.  **ToT vs CoT 的区别**：ToT 引入了全局搜索和回溯，计算成本高得多，适合围棋、数学证明等复杂场景。
3.  **Agentic workflows**：现在流行的 Agent 如何利用这些方法？（答案：通常用 CoT 规划任务，用 Tools 执行任务，用 ReAct 循环迭代）。

## 记忆要点

- 线性CoT：分步拆解问题，降低单步错误率，是大模型复杂推理涌现的基础。
- 自洽性(SC)：采样多条推理路径并多数投票，有效过滤偶然幻觉。
- 思维树：将线性链扩展为树结构，支持节点评估、剪枝与回溯的全局搜索。
- 工具辅助(PAL/ReAct)：生成可执行代码解决精确计算，或结合外部工具进行思考-行动循环。
