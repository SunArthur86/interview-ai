# GRPO、PPO、DPO 三者的区别是什么?各自的优缺点

- **三者对比**

| 维度 | PPO | DPO | GRPO |
|------|-----|-----|------|
| 类型 | On-policy RL | Off-policy 偏好学习 | On-policy RL |
| 需要RM | ✅ 需要单独 RM | ❌ 隐式 reward | ❌ 组内相对优势 |
| 需要ref model | ✅ | ✅ | ✅ |
| 数据 | 实时采样 + RM 打分 | 偏好对 | 同一 prompt 多个采样 + 规则打分 |
| 训练稳定性 | 中(超参敏感) | 高(直接优化) | 高(组内归一化降方差)|

- **PPO 优缺点**
  - 优点: 理论成熟，可在线探索
  - 缺点: 4 个模型同时训练(policy/ref/reward/value)，显存大，超参敏感

- **DPO 优缺点**
  - 优点: 简单(只需偏好对)，训练稳定
  - 缺点: 依赖离线偏好数据质量，不能在线探索更好的策略

- **GRPO 优缺点**
  - 优点: 不需要单独 RM，用组内相对优势替代；减少方差；对推理任务特别有效(可用规则判定对错)
  - 缺点: 需要多次采样(组大小 G=4~64)，计算成本较高

- **选择建议**:
  - 通用对齐 → DPO
  - 推理能力强化 → GRPO
  - 在线探索需求 → PPO

*   **实战案例**：在数学题推理强化中，使用PPO时训练极不稳定，因为Reward模型对部分正确步骤的打分波动大，导致策略忽左忽右。切换到GRPO后，利用“答案是否正确”作为硬规则Reward，通过组内对比，模型迅速学会了避免常见的计算错误，准确率提升了15%。

*   **代码示例**
```python
# GRPO 优势计算示意图
def compute_grpo_advantage(rewards):
    # rewards: list of float, e.g., [0, 1, 0, 1] (0: wrong, 1: correct)
    rewards = torch.tensor(rewards)
    
    # 1. 计算组内均值
    mean_reward = rewards.mean()
    
    # 2. 计算组内标准差 (防止分母为0加一个小epsilon)
    std_reward = rewards.std(unbiased=False) + 1e-4
    
    # 3. 归一化优势
    advantages = (rewards - mean_reward) / std_reward
    
    return advantages
# Input: [0.0, 1.0, 0.0, 1.0] -> Output: [-0.707, 0.707, -0.707, 0.707]
```

**GRPO 与 PPO 优势计算对比图：**
```text
PPO (基于绝对 Value):
给定 prompt x:
  生成 y1, 计算 A1 = R(x,y1) - V(x)  <-- 需要 Critic (Value) Model
  生成 y2, 计算 A2 = R(x,y2) - V(x)  <-- 依赖 V(x) 的准确性
  Problem: V(x) 训练难，方差大

GRPO (基于组内相对优势):
给定 prompt x:
  生成组 G = {y1, y2, y3, y4}
  计算 Reward R_i (如结果是否正确 0/1)
  计算均值 R_mean = (R1+R2+R3+R4) / 4
  计算 Std R_std (组内标准差)
  优势 A_i = (R_i - R_mean) / R_std
  Benefit: 去掉了 Value Model，通过组内对比消除基准噪声
```

## 常见考点
1.  **KL 散度的约束位置**：PPO 的 Clip 目标和 DPO/SimPO 的 Loss 中都隐式或显式地包含了 KL penalty，它们在数学形式上是如何对应的？
2.  **GRPO 的组内采样代价**：GRPO 对同一个 prompt 采样 G 次，这会增加 decode 的计算量。在数学上，G 的大小与方差降低是什么关系？（答：方差通常随 1/G 减小）。
3.  **DPO 的局限性**：DPO 为什么不能用于强化学习中的“探索”？（答：因为 DPO 优化的是由静态数据集定义的分布，无法探索数据集中不存在的更好回答）。

## 记忆要点

- PPO：On-policy，需 RM 和 Value Model，4 模型训练，显存大。
- DPO：Off-policy，无需 RM，直接优化偏好对，稳定但不可在线探索。
- GRPO：On-policy，去 RM 和 Value，用组内相对优势降方差，适合推理。
- 选择口诀：通用对齐选 DPO，推理强化选 GRPO，在线探索选 PPO。
