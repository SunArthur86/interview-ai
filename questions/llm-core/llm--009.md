# GSPO 和 DAPO 是什么?它们与 GRPO 有什么区别

- **GSPO (Group Sequence Policy Optimization)**

是对 GRPO 的改进，核心变化:
1. **序列级重要性采样**: GRPO 在 token 级别做重要性采样，梯度更新粒度细但方差大；GSPO 改为序列级别，将整个序列作为一个更新单元。
2. **降低方差**: 序列级归一化减少训练不稳定性，利用 Group 内其他序列作为基准，计算相对优势。
3. **更稳定的 KL 控制**: 用序列级 KL 散度(新旧策略在整个输出分布上的差异)替代 token 级，防止策略漂移过快。

- **DAPO (Dynamic Adaptive Policy Optimization)**

另一个 GRPO 变体，针对长文本生成优化:
1. **动态优势权重**: 根据生成质量动态调整正负样本的权重。对于 Reward 极高的样本增加其梯度的贡献度，低分样本降低贡献度。
2. **自适应 clip 范围**: PPO 的 clip 范围 ε 固定，DAPO 根据训练进度和 Reward 方差自适应调整 ε，初期收敛快，后期稳。
3. **长序列友好**: 对长推理链(如数学证明)的奖励信号更敏感，解决了长序列中后期 Token 梯度消失的问题。

- **三者核心对比**

| 维度 | GRPO | GSPO | DAPO |
|------|------|------|------|
| 采样粒度 | Token 级 | 序列级 | 序列级 (动态加权) |
| KL 控制 | Token 级 | 序列级 | 自适应 |
| 优势估计 | 组内相对 (GAE风格) | 序列归一化 (整体排名) | 动态加权 (质量敏感) |
| 适用场景 | 通用短文本 | 稳定性优先/长文本 | 长推理链/数学证明 |
| 计算复杂度 | 较高 | 低 | 中等 |
| 梯度方差 | 高 | 低 | 中 |

- **总结**: 从 GRPO → GSPO/DAPO 的演进，本质上是在解决“如何更稳定、更有效地将序列级奖励分配到 token 级别”的信用分配问题。Token 级更新灵活但噪声大，序列级更新稳定但可能损失细节。

- **实战案例**：在训练代码生成模型时，使用 GRPO 常出现“奖励黑客”现象，模型通过生成冗长的无效代码骗取高 Reward，引入 GSPO 后，通过序列级 KL 惩罚强行约束模型保持原有代码风格，解决了模型“突然变啰嗦”的问题；在复杂数学题推理任务中，DAPO 表现出对“错误的第一步”极度敏感，相比 GRPO 能更快修正逻辑起点错误，收敛速度提升约 30%。

- **代码示例 (Python - 伪代码)**:
```python
# 关键差异：GSPO 使用 Sequence Level 的优势估计和 KL
# GRPO 使用 token level diff

def compute_gspo_loss(old_log_probs, new_log_probs, rewards, kl_coef):
    # 1. 序列级 Reward 归一化 (Group Normalization)
    mean_reward = rewards.mean()
    std_reward = rewards.std()
    normalized_rewards = (rewards - mean_reward) / (std_reward + 1e-8)
    
    # 2. 序列级 Ratio (区别于 GRPO 的 token-level ratio)
    # shape: (batch_size, seq_len) -> (batch_size,) sum over seq
    log_ratio = (new_log_probs - old_log_probs).sum(dim=1) 
    ratio = log_ratio.exp()
    
    # 3. 序列级 KL Penalty
    # 旧策略分布 vs 新策略分布
    kl_penalty = (old_log_probs.exp() * (old_log_probs - new_log_probs)).sum(dim=1)
    
    # 4. Loss 计算
    # 简化版 PPO-Clip 应用于序列级分数
    surrogate1 = ratio * normalized_rewards
    surrogate2 = torch.clamp(ratio, 1 - epsilon, 1 + epsilon) * normalized_rewards
    policy_loss = -torch.min(surrogate1, surrogate2).mean()
    
    final_loss = policy_loss + kl_coef * kl_penalty.mean()
    return final_loss
```
