# Transformer位置编码作用及局限性

Transformer 的位置编码主要用于注入序列顺序信息，解决自注意力机制固有的置换不变性问题。

**1. 作用**
*   **注入序列信息**：由于 Self-Attention 本身是并行计算所有词之间的关系，无法感知词序，位置编码通过给每个位置的嵌入向量添加特定的位置信息，使模型能够区分词语在序列中的顺序。
*   **捕捉相对位置**：通过特定的数学设计（如正弦余弦函数或 RoPE），模型不仅能知道绝对位置，还能学习到词与词之间的相对距离。
*   **外推能力**：正弦-余弦位置编码具有理论上的外推能力，可能有助于处理训练时未见过的序列长度。

**2. 局限性**
*   **长度限制**：固定的正弦位置编码在处理超出训练集长度的超长序列时，效果可能会下降，因为模型未曾见过那样的位置模式。
*   **缺乏语义适配**：固定编码是静态的，无法根据具体的语义或任务动态调整位置的重要性，而可学习的位置编码虽然灵活，但外推能力通常较差。
*   **信息干扰**：位置编码与词嵌入直接相加，可能会在某种程度上干扰语义信息的表达（尽管这是必要的妥协）。

**3. 原理细节与补充**
*   **绝对位置编码**：原始 Transformer 使用正弦和余弦函数：
    $$PE_{(pos, 2i)} = \sin(pos / 10000^{2i/d_{model}})$$
    $$PE_{(pos, 2i+1)} = \cos(pos / 10000^{2i/d_{model}})$$
    这种设计允许模型通过线性投影 $PE_{pos+k}$ 来学习相对位置，因为 $\sin(\alpha+\beta)$ 和 $\cos(\alpha+\beta)$ 可以由 $\sin\alpha, \cos\alpha, \sin\beta, \cos\beta$ 线性表示。
*   **相对位置编码 (RoPE/ALiBi)**：如 RoPE (Rotary Positional Embeddings) 通过旋转矩阵将相对位置信息注入到 Query 和 Key 的内积计算中，不仅具备相对位置感知能力，且在长文本外推上表现优于正弦编码。
*   **相加 vs 拼接**：位置编码通常与词向量相加，假设它们位于同一个语义空间。另一种方式是拼接，但这会增加维度和计算量。

**4. RoPE 注入位置信息原理图 (ASCII)**
```text
RoPE (旋转位置编码) 核心思想：将位置信息视作旋转角度

Step 1: 词向量 x (m维) 分组为复数
┌───────────────┐
│ x0, x1, x2, x3│  => (x0 + ix1), (x2 + ix3)
└───────────────┘

Step 2: 引入绝对位置 pos，构造旋转矩阵 mθ
对于位置 pos 的第 i 维，角度为 pos * θi

Step 3: 在 Attention 分数计算前旋转 Query 和 Key
┌──────────────────────┐     ┌──────────────────────┐
│ q (at position pos)  │  *  │   exp(i * pos * θ)   │ =  q_rot
└──────────────────────┘     └──────────────────────┘
                                       │
                                       ▼
                            (携带了绝对位置信息的 q)

Step 4: 计算内积
q_rot · k_trans = f(q, k, pos - pos')
结果仅取决于相对距离 (pos - pos')，而非绝对位置
```

**## 常见考点**
1.  **为什么正弦位置编码具有外推性？**（考察对线性性质和三角函数变换的理解）
2.  **RoPE 相比于绝对位置编码的优势是什么？**（考察对相对位置编码实际应用的理解）
3.  **BERT 和 GPT 在位置编码使用上的区别？**（BERT 使用可学习位置编码且有最大长度限制，GPT 也是可学习的，而 T5 等可能使用相对位置编码）

## 记忆要点

- 核心作用：因为Self-Attention具有置换不变性(无序)，所以需要位置编码注入词序信息。
- 绝对编码(正弦)：靠三角函数线性变换具有理论外推性，但静态固定且直接相加易干扰语义。
- 相对编码(RoPE/ALiBi)：只依赖相对距离且外推性极佳，是当前主流LLM长文本标配。
