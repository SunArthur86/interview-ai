# 什么是冷启动?为什么要冷启动

### 1. 什么是冷启动?

🫐在大型语言模型(LLM)的训练中，冷启动指在应用强化学习(RL)之前，先对模型进行一轮监督微调(SFT)。具体而言，这一过程涉及收集一小部分高质量的推理链数据，对基础模型进行微调，为后续的强化学习奠定基础。

### 2. 为什么要进行冷启动?

**🫐提升输出的可读性和一致性：** 直接通过强化学习训练的模型（如 DeepSeek-R1-Zero）可能在生成内容时出现可读性差、语言混杂等问题。通过冷启动阶段的监督微调，模型能够学习到更清晰、结构化的输出格式。

**稳定强化学习过程：** 在没有初始监督微调的情况下，直接进行强化学习可能导致模型训练过程不稳定。冷启动通过提供初始的监督信号，使模型在进入强化学习阶段时具备基本的推理能力，促进训练的稳定性。

**引导模型学习人类先验知识：** 冷启动数据通常包含人类设计的推理模式和总结，这些先验知识有助于模型更好地理解任务要求，提升推理能力。

### 3. 怎么做冷启动

🫐在实践中，冷启动数据的收集通常包括以下步骤:

*   **生成初始数据：** 使用未经过监督微调的模型（如 DeepSeek-R1-Zero）生成包含详细推理过程和反思的答案。
*   **人工后处理：** 对生成的内容进行人工审核和修改，确保其可读性和准确性。
*   **设计输出格式：** 采用统一的格式，例如在推理过程后添加总结部分，以提高输出的结构化和可读性。

通过上述方法，冷启动阶段为模型提供了一个良好的起点，使其在后续的强化学习中能够更有效地提升推理能力和输出质量。

### 4. 冷启动的数据构成与作用

在 DeepSeek-R1 的实践中，冷启动数据不仅仅是长推理链，还包含了特定格式的 prompt 响应模式：

*   **格式化训练**：教模型何时开始思考（``）、何时停止思考并输出最终答案。这避免了 RL 阶段模型在何时结束推理上浪费探索时间。
*   **思维链示范**：提供少量高质量的复杂问题解答步骤，类似于 In-Context Learning 的微调版。

### 5. 实战补充

**实战案例**：在实际工程中，如果直接跳过冷启动进行 RLHF（Reinforcement Learning from Human Feedback），模型极易出现“格式崩溃”，例如在需要 JSON 输出的任务中输出纯文本，或者根本不输出 `</think>` 结束符，导致下游解析服务大面积报错。

**代码示例**：构造冷启动训练数据的格式脚本
```python
def format_cold_start_data(question: str, reasoning: str, answer: str) -> str:
    """构造 R1 风格的冷启动 SFT 训练样本"""
    return (
        f"<｜User｜>{question}<｜Assistant｜>"
        f"<think>{reasoning}</think>\n\n"
        f"{answer}<｜end▁of▁thinking｜>"
    )

# 踩坑经验：必须强制包含 闭合标签
# 否则 RL 阶段模型会发现不结束思考也能获得高奖励（如果Reward Model主要看最终答案）
# 从而导致推理链无限膨胀，训练显存爆炸。
```

## 常见考点
1. **冷启动数据量一般需要多少？**：通常只需要非常少（几千到几万条），目的是“定调”和“格式对齐”，而不是灌输所有知识。
2. **没有冷启动会怎么样？**：参考 DeepSeek-R1-Zero，虽然能涌现推理能力，但输出不可读、语言混杂，甚至难以收敛，难以直接部署应用。
3. **冷启动与 SFT 的关系？**：冷启动是一种特殊的 SFT，专注于推理能力和输出格式，通常在通用指令微调（Instruction Tuning）之后、RL 之前进行。
