# 大模型幻觉(Hallucination)的分类、成因和缓解方法有哪些

幻觉是指 LLM 生成看似合理但事实上不正确的内容。

### 幻觉分类

| 类型 | 定义 | 示例 |
|------|------|------|
| **事实性幻觉** | 生成与客观事实不符的内容 | 编造不存在的论文/人物 |
| **忠实性幻觉** | 生成与输入/上下文矛盾的内容 | RAG中歪曲检索文档含义 |
| **推理幻觉** | 推理过程错误但结论可能正确 | 数学计算步骤错误 |

### 幻觉成因

1.  **数据层面**：
    *   **源头噪声**：训练数据包含错误信息、过时知识或偏见。
    *   **长尾缺失**：对于低频知识或复杂逻辑，模型覆盖不足，倾向于“脑补”。
    *   **知识冲突**：预训练数据中存在相互矛盾的信息，导致模型输出不确定。

2.  **训练层面**：
    *   **Next Token Prediction 机制**：自回归模型本质是概率拟合，而非事实检索。一旦某个词预测错误，后续为了语言通顺会“将错就错”。
    *   **SFT 对齐税**：有监督微调时，若模型本身不知道答案，可能会为了迎合指令格式而编造内容。

3.  **推理层面**：
    *   **解码策略**：Top-k 或 Top-p 采样增加了随机性，Temperature 过高极易引发幻觉。
    *   **Attention 缺陷**：在长上下文中，模型可能过度关注无关信息而忽略关键事实，导致“迷失中间”。

### 缓解方法

1.  **RAG (检索增强生成)**：
    *   **原理**：引入外部知识库，强制模型基于检索到的上下文生成。
    *   **关键点**：提高检索召回率和相关性是前提；需提示模型“仅依据上下文回答”。

2.  **Prompt Engineering (思维链)**：
    *   **原理**：通过 Chain-of-Thought (CoT) 让模型分步推理，降低一步到位的逻辑跳跃错误。
    *   **进阶**：Self-Consistency（自我一致性），采样多条推理路径，通过投票选出最一致答案。

3.  **RLHF / RLAIF**：
    *   **原理**：通过奖励模型或 AI 反馈，惩罚产生幻觉的输出，强化模型引用事实的行为。

4.  **解码控制**：
    *   **操作**：降低 Temperature（趋向 0），减少随机性；使用核采样 截断低概率 tokens。

5.  **Self-Check / Self-Reflection**：
    *   **流程**：先生成答案 -> 让模型自查事实性 -> 修正错误。或者生成多个草稿，相互比对矛盾点。

6.  **不确定性表达**：
    *   训练模型在置信度低时输出“我不知道”或拒绝回答，而非强行生成。

#### RAG 缓解幻觉的架构流程
```text
用户提问
   │
   ▼
┌──────────┐     检索      ┌───────────────┐
│ Retriever│──────────────▶│ Knowledge Base│
└──────────┘               └───────────────┘
   │
   ▼ (拼接 Context)
┌─────────────────────────┐
│    LLM (Prompt: 依据上下文)   │
│  1. 阅读 Context         │
│  2. 提取/推理             │
│  3. 生成 Answer (减少编造)    │
└─────────────────────────┘
```

### 实战案例与代码

**实战案例**：在一个企业级 RAG 系统中，我们发现模型经常将检索到的“竞争对手A”的参数张冠李戴给“用户提问的产品B”。通过引入**CoT 链式验证（验证步骤）**和降低 Temperature（从 0.7 降至 0.1），并将 Prompt 明确为“若上下文未提及直接回答未知”，幻觉率降低了 60%。

**代码示例**：
```python
# LangChain 实现的自验证 Chain 伪代码
from langchain.chains import LLMChain
from langchain.prompts import PromptTemplate

# 1. 初始生成
initial_prompt = PromptTemplate(
    input_variables=["question", "context"],
    template="Context: {context}\nQuestion: {question}\nAnswer:"
)
answer_chain = LLMChain(llm=llm, prompt=initial_prompt)

# 2. 事实性自检
check_prompt = PromptTemplate(
    input_variables=["question", "answer", "context"],
    template="Given the context: {context}, does the answer '{answer}' to the question '{question}' contain any hallucinations? Answer Yes/No and explain."
)
check_chain = LLMChain(llm=llm, prompt=check_prompt)

# 执行流
raw_answer = answer_chain.run(question=question, context=context)
validation = check_chain.run(question=question, answer=raw_answer, context=context)
```

### 缓解方法对比

| 方法 | 实施阶段 | 核心成本 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **RAG** | 推理时 | 向量数据库构建 + 延迟增加 | 知识密集型、时效性要求高 |
| **Prompting (CoT)** | 推理时 | Token 消耗翻倍 (推理变慢) | 逻辑推理、数学计算 |
| **RLHF** | 训练时 | 极高 (标注 + 训练算力) | 通用大模型基座对齐 |
| **Decoding Control** | 推理时 | 几乎无 (仅调参) | 创意性要求低、确定性要求高 |

### ## 常见考点
1.  **如何检测幻觉？**（追问：事实验证模型、基于 LogProbability 的不确定性评估、引用溯源）
2.  **RAG 为什么无法完全消除幻觉？**（追问：检索错误的内容会导致模型错误学习；模型仍可能产生忠实性幻觉）
3.  **Training-time 和 Inference-time 的缓解方法有何区别？**（追问：训练侧重知识内化与对齐，推理侧重约束与验证）
