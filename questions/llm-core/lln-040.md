# LongLora是怎么做的

LongLoRA 旨在高效扩展大语言模型的长上下文能力，同时保持低成本微调。

**核心机制**：
1. **Shifted Short Attention (S2-Attn)**：
   - 在训练时，将长序列划分为多个组，在每个组内进行注意力计算，即使用“短注意力”。
   - 通过 **Shift（偏移）** 操作，让相邻组的 Token 能够在半个窗口内交互，保证信息的局部流动和全局连续性。
   - 这种方式将计算复杂度从 $O(L^2)$ 降至接近 $O(L)$。

2. **高效微调**：
   - 结合 LoRA 进行训练，仅微调注意力相关的部分参数和 Embedding 层，大大降低了长上下文训练的显存需求。

3. **推理恢复**：
   - 在推理阶段，直接使用标准的 Full Attention（全局注意力），无需再分组。由于模型在训练时学到了长序列依赖模式，可以直接无缝迁移。

**ASCII 原理示意图**：
```text
训练阶段 (Shifted Group Attention):
序列: [T1 T2 T3 T4 | T5 T6 T7 T8 | T9 T10 T11 T12]

Group 1 关注范围:
[T1 T2 T3 T4] (常规)

Group 2 关注范围 (Shifted):
     [T5 T6 T7 T8] <-- 左移半个窗口，使得 T4, T5 能交互
     ^^^^

推理阶段 (恢复 Full Attention):
序列: [T1 ... T12] 全局互相可见
```

**实战案例**：
在处理 100k 长度的金融财报分析任务时，使用 FlashAttention-2 结合 LongLoRA 策略，使得单卡 80G 显存即可训练 7B 模型，且在 RAG 检索增强任务中，无需修改推理代码即可直接支持长文档上下文输入，解决了分段导致语义断裂的问题。

**代码示例 (HuggingFace Transformers)**：
```python
from transformers import AutoModelForCausalLM, TrainingArguments
model = AutoModelForCausalLM.from_pretrained("llama-7b", trust_remote_code=True)
# 启用 LongLoRA 配置：设置 group_size 和 shift 模式
model.config.longlora = True
model.config.group_size = 64  # 注意力分组大小

# 训练时建议微调 embed_tokens 和 lm_head
for name, param in model.named_parameters():
    if "embed" in name or "lm_head" in name:
        param.requires_grad = True
```

**## 常见考点**
1. LongLoRA 训练时用短注意力，推理时用长注意力，这会导致效果下降吗？（不会，甚至可能更好，因为训练的高效性允许更大的 Batch Size 和更长的上下文训练数据）
2. 为什么 LongLoRA 建议微调 Embedding 层？（为了适应位置编码在长序列下的外推性）

## 记忆要点

- 核心机制：训练时用移位短注意力(S2-Attn)分组计算，将复杂度降至接近$O(L)$。
- 信息流转：通过对序列进行半窗口Shift，使相邻分组Token交互，保证全局连续性。
- 推理无感切换：推理时直接恢复全局标准注意力，因模型已习得长序列模式故可直接迁移。
