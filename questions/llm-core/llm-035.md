# 介绍一下Bert的结构

BERT (Bidirectional Encoder Representations from Transformers) 是基于 Transformer 架构的预训练模型。其核心在于利用 Transformer 模型中的 **Encoder（编码器）** 部分，通过 **双向** 的上下文信息来理解句子中的词语，因此 BERT 能够更好地捕捉语言的语义关系。

### 1. BERT 的输入表示
BERT 的输入由三种嵌入求和而成，同时捕捉词汇、位置和句子间信息：
*   **WordPiece Embedding**：使用 WordPiece 分词，将单词划分为子词单元，处理未知词汇并提高泛化能力。
*   **Position Embedding**：由于 Transformer 不依赖序列结构（如 RNN），需通过位置嵌入为每个 token 添加位置特征。
*   **Segment Embedding**：用于区分两个句子（如句子 A 和句子 B），帮助模型理解句子间关系。

### 2. Transformer Encoder 结构
BERT 堆叠了多层 Transformer Encoder。每一层包含：
*   **Multi-Head Self-Attention**：计算句子中每个词与其他词的相关性，捕捉全局信息并解决长距离依赖问题。
*   **Feed-Forward Network (FFN)**：对每个词的表示进行非线性变换（通常包含两层线性变换和 GELU 激活函数）。
*   **Layer Normalization**：对每层输出进行归一化，稳定训练并加速收敛。
*   **Residual Connection**：将输入直接加到输出上，缓解梯度消失问题。

### 3. 模型规模
*   **BERT-Base**：12 层，12 个注意力头，隐藏层维度 768。
*   **BERT-Large**：24 层，16 个注意力头，隐藏层维度 1024。

### 4. BERT 整体架构图
```text
          Input: [CLS] Token A [SEP] Token B [SEP]
                    │         │         │
           ┌─────────┴─────────┴─────────┴─────────┐
           │      Token Embeddings (WordPiece)    │
           │    Segment Embeddings (A/B Sent)     │
           │     Position Embeddings (1...512)    │
           └─────────────────┬───────────────────┘
                             │ (Sum)
                             ▼
              ┌───────────────────────────────┐
              │    Transformer Encoder (x N)   │
              │  ┌─────────────────────────┐  │
              │  │ Multi-Head Self-Attn    │  │
              │  │ Add & Norm              │  │
              │  │ Feed Forward Network    │  │
              │  │ Add & Norm              │  │
              │  └─────────────────────────┘  │
              └───────────────┬───────────────┘
                              │
          ┌───────────────────┼───────────────────┐
          ▼                   ▼                   ▼
   [CLS] Output      Token A Output      Token B Output
 (For Classification)  (For Sequence Tagging)  (For QA/Sim)
```

### 5. 预训练任务（补充细节）
*   **Masked Language Model (MLM)**：随机掩盖 15% 的 Token，让模型根据上下文预测。包括 80% 替换为 `[MASK]`，10% 随机替换，10% 保持不变。
*   **Next Sentence Prediction (NSP)**：输入两个句子，预测第二个句子是否是第一个句子的下一句（用于理解句子间关系，虽然后续 RoBERTa 等模型证明此任务可能非必须）。

## 常见考点
1. **BERT 是双向的，GPT 是单向的，区别是什么？**（回答：BERT 的 Mask 允许看到左右上下文，适用于 NLU 理解任务；GPT 是因果掩码，只能看左边，适用于 NLG 生成任务）
2. **BERT 中的 [CLS] token 有什么作用？**（回答：作为整个序列的聚合表示，通常用于分类任务，其输出维度对应类别数）
3. **BERT 为什么有最大长度限制（512）？**（回答：Position Embedding 的学习限制和 Attention 的 $O(N^2)$ 复杂度限制）
4. **BERT 如何处理长文本？**（回答：滑动窗口 Sliding Window、截断等策略）

## 记忆要点

- 核心定位：仅用Transformer的Encoder，因双向注意力机制，擅捕捉全局上下文语义。
- 输入特征：Embedding由Token、Position、Segment三者直接相加而成。
- 内部结构：核心单元为多头自注意力+前馈网络(FFN)，配合残差连接与LayerNorm。
- 关键参数：Base版12层768维，Large版24层1024维，位置编码限长512。
