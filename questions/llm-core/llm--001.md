# 请解释 on-policy 和 off-policy 的区别,以及 online 和 offline RL 的含义.

- **On-policy vs Off-policy**

- **On-policy**:策略更新只能使用当前策略自己采集的数据.代表:PPO、REINFORCE.优点:理论清晰;缺点:数据利用率低(采一批用一次就丢).
- **Off-policy**:可以利用其他策略(或旧策略)采集的数据来更新当前策略.代表:DQN、SAC、Q-learning.优点:数据可复用(experience replay);缺点:需要重要性采样或函数近似修正偏差.

**核心区别：** 是否允许用旧数据训练新策略.

- **Online vs Offline RL**
- **Online RL**:训练过程中可以与环境持续交互采集新数据
- **Offline RL**:只有固定的历史数据集,不能与环境交互

在 RLHF 场景中:
- PPO 是 on-policy + online
- DPO 可视为 off-policy(直接利用偏好数据,不需要在线采样)
- GRPO 也是 on-policy,但通过组内相对优势减少方差

*   **实战案例**：在训练对话机器人的RLHF阶段，如果On-policy模型在某个话题下突然生成乱码，这部分“有毒”数据会直接污染下一轮更新，导致模型崩溃。而Off-policy方法可以人工清洗历史数据，剔除这些样本后再进行训练。

*   **代码示例**
```python
# PPO 的 On-policy 数据收集逻辑 (伪代码)
for epoch in range(num_epochs):
    # 1. 使用当前策略 采集数据
    trajectories = []
    for state in env:
        action = policy.act(state)
        reward, next_state = env.step(action)
        trajectories.append((state, action, reward))
    
    # 2. 使用当前 Batch 的数据更新策略
    # 重要：一旦更新，这里的 trajectories 就不能再用了（变为 Off-policy）
    ppo_update(policy, trajectories)
```

| 特性 | On-policy (PPO) | Off-policy (DQN/DPO) |
| :--- | :--- | :--- |
| **数据来源** | 必须是当前策略生成的 | 可以是旧策略或静态数据集 |
| **样本利用率** | 低（用完即弃） | 高（可重复采样） |
| **训练稳定性** | 较差，受近期采样影响大 | 较好，可由高质量数据平滑分布 |
| **适用场景** | 实时交互环境、探索性强 | 离线数据集、RLHF 偏好学习 |
| **实现难度** | 需维护交互循环 | 需处理重要性采样/OOD动作 |

## 常见考点
1. **重要性采样**：Off-policy 训练中，当行为策略与目标策略差异较大时，为什么要引入 Importance Sampling Ratio？它如何保证梯度的无偏性？
2. **分布漂移**：在 Offline RL 中，为什么简单的 Off-policy 算法（如 DQN）会失效？是因为“分布外”动作估计过高吗？
3. **RLHF 中的 On-policy 开销**：为什么 PPO 在 RLHF 中训练成本比 DPO 高得多？（答：因为 PPO 需要每步用当前模型生成数据，DPO 直接吃静态数据）。

## 记忆要点

- On-policy 只用当前策略数据，用完即弃，代表 PPO；Off-policy 复用旧数据，代表 DQN/DPO。
- Online 训练时可与环境交互；Offline 只能用固定历史数据集，不可交互。
- RLHF 场景：PPO 是 On+Online，DPO 是 Off+Offline，PPO 成本更高。
- Off-policy 核心难点：需重要性采样修正分布偏差，数据利用率高。
