# DPO的原理

DPO (Direct Preference Optimization，直接偏好优化) 是一种用于大模型对齐的算法。它无需训练显式的奖励模型，直接根据人类偏好数据优化策略。

### 1. 背景
传统方法（如 RLHF）需要先训练一个奖励模型 来模拟人类偏好，再使用强化学习（如 PPO）优化策略。这个过程复杂且不稳定。

### 2. 核心原理
DPO 证明了在 Bradley-Terry 偏好模型下，奖励函数和策略之间存在直接的解析关系：
$$ \log \frac{\pi(y_w|x)}{\pi(y_l|x)} = \beta \cdot (R(x, y_w) - R(x, y_l)) $$
这意味着优化奖励差值等价于优化策略的相对概率。DPO 利用这一关系，**直接重写奖励模型的优化目标为策略空间的优化目标**，从而消除了对奖励模型的需求。

### 3. DPO 损失函数
DPO 的目标是最大化偏好数据 $(x, y_w, y_l)$ 的对数似然，其中 $y_w$ 是偏好答案，$y_l$ 是拒绝答案。损失函数如下：
$$ L_{DPO}(\pi_\theta; \pi_{ref}) = -E \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right] $$
*   $\pi_\theta$：当前正在训练的策略。
*   $\pi_{ref}$：参考模型（通常是 SFT 模型），用于约束优化过程，防止模型崩溃或偏离过远。
*   $\beta$：温度系数，控制优化强度。

### 4. 优势
*   **无需奖励模型和强化学习**：简化了流程，避免了 RL 阶段的不稳定性。
*   **数学上更稳定**：直接在策略空间优化，避免了奖励模型拟合误差的累积。
*   **实现简单**：只需在 SFT 基础上增加一个简单的损失项即可训练。

### 5. RLHF vs DPO 流程对比

```text
RLHF 流程:
┌──────┐      ┌──────────┐      ┌──────────┐      ┌──────────┐
│ SFT  │ ───> │ 奖励模型 │ ───> │ PPO 训练 │ ───> │ 最终模型 │
└──────┘      │ (RM)     │      │ (RL)     │      │          │
             └──────────┘      └──────────┘      └──────────┘

DPO 流程:
┌──────┐      ┌─────────────────────┐      ┌──────────┐
│ SFT  │ ───> │  人类偏好数据收集    │ ───> │ DPO 训练 │ ───> 最终模型
└──────┘      │  (Prompt, Win, Lose)│      │ (无需RM) │
             └─────────────────────┘      └──────────┘
```

### 6. 关键细节补充
*   **参考模型 $\pi_{ref}$ 的作用**：$\pi_{ref}$ 是冻结的 SFT 模型。损失函数中的 $\log \frac{\pi_\theta}{\pi_{ref}}$ 本质上是计算当前策略与参考模型的 **KL 散度** 的隐式约束。如果没有这一项，DPO 可能会通过输出极其偏激的 token 来最大化对数概率，导致 Reward Hacking（如重复生成某些高概率词）。
*   **参数 $\beta$ 的影响**：$\beta$ 是 RLHF 中的 KL 惩罚系数的倒数。$\beta$ 越大，越倾向于让模型利用奖励（可能过拟合奖励模型，输出单一）；$\beta$ 越小，KL 约束越强，模型越接近 $\pi_{ref}$（可能对齐效果不明显）。
*   **Sigmoid 函数的意义**：将两个对数概率的差异映射到 [0, 1] 区间，表示模型选择 $y_w$ 优于 $y_l$ 的概率。

### 7. 实战案例与实现
**实战案例**：在微调 7B 模型时，若不加权重衰减，DPO 往往会出现“模式崩塌”，即对任意 Prompt 都输出极高的 Logits 以骗取 Loss 降低。此时需关注 `chosen_logps` 和 `rejected_logps` 的均值差，若差值过大（>20），需增大 `beta` 或减小学习率。

**代码示例**：
```python
# 基于 PyTorch 的简化 DPO Loss 计算片段
import torch.nn.functional as F

def dpo_loss(policy_chosen_logps, policy_rejected_logps, 
             reference_chosen_logps, reference_rejected_logps, beta):
    # 计算策略模型与参考模型的 log_prob 差值
    policy_logratios = policy_chosen_logps - policy_rejected_logps
    reference_logratios = reference_chosen_logps - reference_rejected_logps
    
    # DPO 核心公式：最大化策略差值与参考差值之间的差异
    losses = -F.logsigmoid(beta * (policy_logratios - reference_logratios))
    return losses.mean()
```
