# DPO的正负样本怎么构造

在 DPO (Direct Preference Optimization) 训练中，正负样本的构造是关键步骤，本质上是基于人类反馈构建“偏好对”。

### 1. 数据收集阶段
*   **Prompt (提示词)**：给定一个输入 $x$（如用户问题）。
*   **Generation (生成)**：让模型针对 $x$ 生成多个不同的回复。例如，生成 $y_1$ 和 $y_2$。

### 2. 人类标注与偏好形成
*   人类标注员对生成的回复进行比较，判断哪个更好。
*   假设标注员认为 $y_1$ 优于 $y_2$，则形成一个偏好对 $(y_w, y_l)$，其中 $y_w$ 是偏好回答，$y_l$ 是拒绝回答。

### 3. 正负样本定义
在 DPO 的 Loss 计算上下文中：
*   **正样本**：指 **偏好回答 $y_w$**。在训练中，模型需要提升生成该答案的概率。
*   **负样本**：指 **拒绝回答 $y_l$**。在训练中，模型需要降低生成该答案的概率（或者相对于 $y_w$ 降低其概率）。

### 4. 构造细节
*   **Pairwise (成对)**：必须保证 $y_w$ 和 $y_l$ 是基于同一个 Prompt $x$ 生成的，具有可比性。
*   **格式**：通常数据集格式为 `{ "prompt": x, "chosen": y_w, "rejected": y_l }`。
*   **来源**：除了生成后比较，也可以收集历史数据中的用户点赞（正样本）和点踩/重新生成（负样本）作为偏好对。

### 6. 实战采样策略
**实战案例**：在工程中，单纯用同一个模型生成两个回答往往差异太小（Score 靠近），导致 DPO 梯度信号微弱。**常用改进方案**是：$y_w$ 来自更强的 70B 模型或 GPT-4，$y_l$ 来自当前训练的 7B SFT 模型。这种“Expert vs Novice”的配对方式能提供极强的学习信号，显著加快收敛速度。

**代码示例**：
```python
# 构造偏好对的伪代码
def construct_preference_pair(dataset, generator_model, strong_model_ref):
    pairs = []
    for prompt in dataset:
        # 负样本：当前模型生成的
        y_l = generator_model.generate(prompt)
        
        # 正样本：标注数据或更强模型生成（模拟专家行为）
        y_w = strong_model_ref.generate(prompt) 
        # 或者: y_w = prompt['ground_truth']
        
        # 简单清洗：防止完全一致
        if y_l != y_w:
            pairs.append({
                "prompt": prompt,
                "chosen": y_w, 
                "rejected": y_l
            })
    return pairs
```
