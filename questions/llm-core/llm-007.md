# 解释一下prefix-tuning

### Prefix-tuning

**概念**：
Prefix-tuning 是一种参数高效的微调（PEFT）方法，主要用于自然语言处理任务。它通过在预训练语言模型（如 GPT-2、T5 等）的**每一层**输入序列前添加一段可训练的连续前缀向量，实现对下游任务的适应，而无需更新原有的大量模型参数。

**来源**：
出自 2021 年 Li and Liang 的论文《Prefix-Tuning: Optimizing Continuous Prompts for Generation Tasks》。

**核心原理**：
与仅在输入层添加提示的 Prompt Tuning 不同，Prefix-tuning 将可训练的前缀向量添加到了 Transformer 模型**每一层的隐藏状态**中。这意味着微调不仅发生在输入层，还深入到了网络的每一层自注意力模块中。

**技术细节**：
1.  **前缀构建**：给定模型层数 $L$，在每一层的自注意力模块输入前增加一组可训练的前缀 $P_i \in \mathbb{R}^{m \times d}$（$m$ 为前缀长度，$d$ 为隐藏维度）。实际上，这些前缀向量是由一个大的 Embedding 层生成的，然后针对不同层进行重参数化映射。
2.  **注意力计算**：这些前缀向量被视为额外的“虚拟 token”，与真实的输入 token 一起参与注意力计算。在计算 Key ($K$) 和 Value ($V$) 时，前缀向量也被拼接进去，从而影响后续 token 的注意力权重。
    $$h_i = [\text{Prefix}_i; \text{HiddenState}_i]$$
    $$Attention(Q, K, V) = \text{softmax}(\frac{QK^T}{\sqrt{d}})V$$
3.  **重参数化训练**：为了保证训练稳定性，作者使用了“重参数化”技巧。在训练时，不直接优化前缀向量，而是优化一个较小的 MLP，用 MLP 来生成前缀向量。这增加了前缀向量之间的复杂交互能力，防止训练不稳定。
4.  **训练过程**：仅优化这些前缀向量的参数（或生成前缀的 MLP 参数），冻结原始模型的所有权重。

**架构对比图：**
```text
Standard Prompt Tuning (Input Only):
Input: [Virtual Prefix] [Real Tokens]
  |
  +--> Layer 1 --> Layer 2 --> ... --> Output

Prefix-Tuning (Deep Prefix Injection):
Input: [Real Tokens]
  |
  v
Layer 1:  [Prefix_1] + Input ----> Attention
  |
  v
Layer 2:  [Prefix_2] + Layer1_Out --> Attention
  |
  ...
```

**优势**：
*   **参数效率极高**：对于大模型（如 GPT-3 175B），只需训练不到 0.1% 的参数。
*   **性能优异**：在生成任务上表现优于 Adapter 和 LoRA，尤其是在模型参数量较小或任务难度较高时。因为每一层都能受到“指令”的微调引导。

**局限**：
*   **推理变慢**：增加了推理时的计算量（因为每一层都要处理额外的虚拟 token），且受限于上下文长度，占用有效 Token 空间。
*   **可解释性差**：相比 Prompt Tuning，其可解释性稍弱，因为深层的前缀向量难以映射回自然语言。

## 常见考点
1. **Prefix-tuning vs Prompt Tuning**：核心区别在于 Prefix-tuning 影响每一层的 Attention，而 Prompt Tuning 只影响输入层。Prefix-tuning 表现更强但参数量和计算略增（因为多了每一层的参数）。
2. **为什么 Prefix-tuning 表现更好**：因为直接优化每一层的 Key/Value 比仅仅优化输入 Embedding 具有更强的表达能力，能够更直接地控制模型的生成行为和注意力分布。
3. **上下文长度限制**：Prefix-tuning 预留的 Prefix 长度会挤占用户输入的上下文长度。如果模型最大长度是 2048，Prefix 占了 10，用户就只能输 2038。

## 记忆要点

- 在Transformer每一层Key/Value前拼接可训练的连续前缀向量。
- 区别于Prompt Tuning：Prefix影响深层Attention，表达力更强。
- 使用重参数化（MLP）训练以增加稳定性，推理时直接用前缀。
- 优势是参数极省且性能好，缺点是占用上下文长度且推理变慢。
