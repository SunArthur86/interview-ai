# Transformer为什么使用多头注意力机制

📌Transformer 使用多头注意力机制是为了提升模型捕捉复杂依赖关系的能力,增强表达能 力,避免信息瓶颈,并且通过并行计算提高模型的计算效率和注意力分布的多样性.

1. 多头注意力机制的定义:
在 Transformer 中,多头注意力机制的核心思想是将输入通过多个独立的 注意力机制(即“头”)进行并行计算,然后将各个头的输出拼接起来,最终通过 线性变换生成最后的注意力输出.

$$MultiHead(Q, K, V ) = Concat(head_1, ... , head_h)W^O$$

每个 $head_i$ 代表一个注意力头,它们的并行计算能够为模型提供丰富的上下文信息.

2. 多头注意力的目的与优势:
*   **捕捉不同的特征信息**: 单个注意力头只能聚焦于输入序列中的某些特定关系，而通过多个头，模型能够并行地从不同的“视角”进行注意力计算。这些视角可能代表不同的上下文关系、语义层次或距离信息。换句话说，多头机制允许模型在同一层次上学到多种不同的依赖关系，提高了模型对复杂依赖的表达能力。
*   **增强模型的学习能力**: 每个注意力头会使用不同的投影矩阵 $W^Q, W^K, W^V$，这意味着每个头会生成不同的查询、键和值向量。这种多样化的投影可以帮助模型在更高维度上捕捉更多的信息特征，进而提升模型的表达能力。
*   **处理不同的关系层次**: 由于语言中的语义和句法结构具有多层次的特性，不同的头可以专注于不同的词汇或句法层次，进而使模型能够同时捕捉短程和长程的依赖关系。例如，一个头可能更关注局部词汇的联系（如形容词修饰名词），而另一个头可能会处理句子中远距离的依赖关系（如指代消解）。

3. 并行处理提升效率:
多头注意力机制允许不同的注意力头进行并行计算，这相较于传统的序列模型（如 RNNs）显著提高了训练和推理的效率。在多头机制中，模型能够在同一时间捕捉到多个依赖关系，而不是逐步处理，从而大大加速了计算。

4. 避免信息瓶颈:
使用单个注意力头可能会导致信息过于集中，模型在学习复杂上下文时难以捕捉到多维度的依赖关系。多头注意力通过引入多个注意力头，降低了单一头在学习复杂关系时可能遇到的信息瓶颈。

5. 平滑注意力分布:
在实际应用中，单个注意力头的权重分布可能过于集中，导致模型对某些特定词语的过度依赖。而多头注意力通过多个头的组合，能够使注意力分布更加均衡和平滑，从而提高模型对不同上下文的整体理解能力。

6. 总结:
📌Transformer 使用多头注意力机制是为了提升模型捕捉复杂依赖关系的能力，增强表达能力，避免信息瓶颈，并且通过并行计算提高模型的计算效率和注意力分布的多样性。

#### 实战案例：可解释性分析
在一个金融舆情分析项目中，我们发现 BERT 模型的第 5 层第 2 个头专门专注于否定词（如“不”、“无”）与其修饰对象的连接，而第 8 层第 6 个头则专注于股票代码与涨跌幅的关联。通过可视化这些不同的头，我们可以更直观地解释模型为何做出“看空”的判断。

#### 代码示例：PyTorch 多头实现
```python
import torch
import torch.nn as nn

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads  # 每个头的维度

        # 线性变换用于生成 Q, K, V
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, query, key, value, mask=None):
        batch_size = query.size(0)

        # 1. 线性变换并分割成多头
        # (batch, seq_len, d_model) -> (batch, seq_len, num_heads, d_k) -> (batch, num_heads, seq_len, d_k)
        Q = self.W_q(query).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(key).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(value).view(batch_size, -1, self.num_heads, self.d_k).transpose(1, 2)

        # 2. 缩放点积注意力
        scores = torch.matmul(Q, K.transpose(-2, -1)) / torch.sqrt(torch.tensor(self.d_k, dtype=torch.float32))
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attn_weights = torch.softmax(scores, dim=-1)

        # 3. 加权求和并拼接
        context = torch.matmul(attn_weights, V) # (batch, num_heads, seq_len, d_k)
        context = context.transpose(1, 2).contiguous().view(batch_size, -1, self.d_model)
        
        return self.W_o(context)
```
