# 什么是大模型的'幻觉'(Hallucination)?为什么会发生

- **幻觉** 指大模型生成看似合理但实际上不正确的内容.

- **幻觉类型**
  1. **事实幻觉**:说错事实('爱因斯坦生于1900年' → 实际1879年)
  2. **逻辑幻觉**:推理过程有误
  3. **来源幻觉**:编造不存在的引用/论文

- **产生原因**
  - 训练数据包含噪声、偏差或过时信息
  - 模型本质是'概率生成',不是'知识检索'
  - 缺乏外部实时信息或确定性机制

- **缓解方法**
  - **RAG**:检索增强生成，引入外部知识
  - **提示词约束**:设定'如果不确定请说不知道'的规则
  - **微调**:通过高质量数据减少编造倾向

- **一句话理解**:LLM 是'概率预测器'不是'数据库',所以会'一本正经地胡说'.

- **实战案例**：在搭建电商客服机器人时，针对“双十一是否有满 300 减 50 的活动”这类时效性问题，纯模型幻觉率高达 30%。接入向量数据库进行 RAG 检索后，要求模型“仅基于检索到的上下文回答”，将事实类问题的幻觉率降低到了 0.5% 以下。

- **缓解方案对比表**：
| 方案 | 原理 | 优点 | 缺点 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- |
| **Prompt 工程** | 增加约束指令（如“不知道说不知道”） | 实施简单，零成本 | 治标不治本，模型仍可能自信错误 | 轻量级应用，临时解决 |
| **RAG (检索增强)** | 引入外部权威文档作为上下文 | 可修正事实，支持实时信息 | 需维护知识库，增加检索延时 | 问答系统，知识库查询 |
| **微调 (SFT)** | 注入领域知识，校准模型输出概率 | 提升领域内语言的准确性 | 无法完全消除幻觉，成本高 | 特定风格，特定术语领域 |
| **CoT (思维链)** | 让模型分步骤推理，展示依据 | 提高逻辑类任务正确率 | 增加推理成本和耗时 | 数学计算，复杂逻辑推理 |

- **代码示例 (LangChain RAG 约束)**：
```python
from langchain.chains import RetrievalQA

prompt_template = """
使用以下上下文片段来回答最后的问题。
如果你不知道答案，就说不知道，不要试图编造答案。

上下文：{context}

问题：{question}
"""

# 设置 chain_type_kwargs 强制模型仅依据上下文
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=retriever,
    chain_type_kwargs={"prompt": prompt_template}
)
```
