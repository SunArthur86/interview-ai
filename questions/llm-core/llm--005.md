# SimCSE 的核心思想是什么?它如何解决句子嵌入的各向异性问题

- **SimCSE (Simple Contrastive Learning of Sentence Embeddings)**

- **背景问题: 各向异性**
  - **现象**: 预训练 BERT 的句向量分布在一个窄锥形空间中，所有向量挤在一个方向，平均余弦相似度极高。
  - **根因**: 预训练时的 MLM 任务并不关心全局句子表征，仅关注局部 token 上下文，导致生成的嵌入在空间中缺乏均匀分布的约束。
  - **后果**: 余弦相似度区分度低，任意两句话的相似度都偏高，无法有效区分语义差异。

- **SimCSE 核心思想: 通过对比学习拉开向量分布**
  利用对比学习中的 InfoNCE Loss，拉近正样本距离，推远负样本距离，迫使模型将不同语义的句子映射到空间的不同位置。

- **无监督 SimCSE (Unsupervised)**
  - **正样本**: 同一句子输入模型两次，利用 **Dropout** 作为数据增强（两次前向传播 Dropout 掩码不同），产生语义相同但表征略异的向量。
  - **负样本**: 同一个 Batch 内的其他句子（In-batch negatives）。
  - **目标函数**: $Loss = -\log \frac{\exp(\text{sim}(h_i, h_i^+) / \tau)}{\sum_{j=1}^{N} \exp(\text{sim}(h_i, h_j) / \tau)}$
  - **关键参数**: 温度系数 $\tau$（通常设为 0.05），控制分布的平滑程度。

- **有监督 SimCSE (Supervised)**
  - **数据来源**: SNLI 或 MNLI 数据集。
  - **正样本**: 蕴含对，即语义相近的句子。
  - **负样本**: 
    1. 矛盾对（Hard Negative，语义相悖但结构相似）。
    2. Batch 内其他句子。
  - **优势**: 显式利用语义关系，比无监督版本效果更好。

- **为什么有效**
  1. **缓解各向异性**: 对比学习 Loss 本质上倾向于最大化特征空间的**利用率**，使向量分布更加均匀（接近高斯分布），从而打破窄锥形分布。
  2. **Dropout 的巧妙利用**: Dropout 在这里充当了最简单的数据增强，既保持了语义不变性，又引入了足够的噪声让模型学习鲁棒特征。
  3. **不变性学习**: 学习到的特征对句子表面的细微变化不敏感，关注核心语义。

- **架构图**:
```
      Input Sentence: "A dog is running"
             │
             ├─────────────┐
             ▼             ▼
      Forward(Seed1)  Forward(Seed2)
      (Dropout Mask A) (Dropout Mask B)
             │             │
             ▼             ▼
      Embedding z_i   Embedding z_i+
             │             │
             └─────┬───────┘
                   │
                   ▼
        Contrastive Loss (InfoNCE)
                   │
    ┌────────────────┼────────────────┐
    ▼                ▼                ▼
(z_i, z_i)         (z_i, z_i+)       (z_i, z_j)
Ignore            Positive          Negative
(Batch内其他句子)
```

- **效果**: STS-B 等语义相似度任务上大幅提升(BERT: 0.48 → SimCSE: 0.81)。

- **## 常见考点**
  1. **温度系数 $\tau$ 的作用**: $\tau$ 越小，模型对正负样本的区分度要求越苛刻（分布越尖锐）；$\tau$ 越大，Softmax 越平滑。通常 $\tau \in [0.05, 0.2]$。
  2. **Hard Negative 的影响**: 在有监督模式下，如果只用 Batch 内随机负样本，模型可能只关注简单的语法差异，加入矛盾对能迫使模型学习深层语义理解。

- **## 深化实战**
  1. **实战案例**：在做电商搜索召回时，直接使用 SimCSE 无监督版会发现“同义词不同字”的词对（如“手机”vs“移动电话”）相似度不够高。引入行业数据（如用户点击 Query-Title 对）进行有监督微调，并混合 Hard Negative（如“手机壳”这种高点击但语义不同的负例），能显著提升点击率。
  2. **代码示例 (PyTorch)**:
```python
def simcse_unsup_loss(y_pred, temperature=0.05):
    # y_pred: [batch_size, 2, dim] (2个dropout view)
    z1, z2 = y_pred[:, 0], y_pred[:, 1]
    z = torch.cat([z1, z2], dim=0) # [2*batch, dim]
    sim = torch.matmul(z, z.t()) / temperature
    # 构建标签：对角线为正样本
    labels = torch.arange(z.size(0), device=z.device)
    labels[labels >= z.size(0)//2] -= z.size(0)//2
    loss = F.cross_entropy(sim, labels)
    return loss
```
