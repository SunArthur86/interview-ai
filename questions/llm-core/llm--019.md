# 大模型推理的输入和输出是什么?Temperature 参数有什么作用

- **推理** 是将输入文本送入模型,生成输出的过程.

- **输入**:一段文本序列
- **输出**:模型预测的下一个 Token(逐个生成)

- **Temperature 参数**
  - **Temperature = 0**:每次选概率最高的 token(确定性输出)
  - **Temperature = 1**:按原始概率分布采样(有创造性)
  - **Temperature > 1**:概率分布更平坦(更随机,可能不连贯)

- **实用建议**
  - 代码生成/事实问答:Temperature = 0~0.3
  - 创意写作/头脑风暴:Temperature = 0.7~1.0
  - 超过 1.2 通常不建议

- **一句话理解**:Temperature 低 = 严谨(选最优),Temperature 高 = 创意(加点随机性).

- **实战案例**：在构建 SQL 生成器时，设置 Temperature=0.7 导致生成的 SQL 语法正确但表名经常随机变换（如 users, user_list, app_users 混用），导致执行报错。将 Temperature 强制降为 0 后，模型对于同一问题的 SQL 输出变得完全确定，后端可以通过缓存结果大幅降低数据库压力。

- **解码策略参数对比表**：
| 参数 | 作用 | 推荐值 (严谨任务) | 推荐值 (创意任务) | 效果差异 |
| :--- | :--- | :--- | :--- | :--- |
| **Temperature** | 控制概率分布的平滑度 | 0.0 - 0.2 | 0.7 - 1.0 | 越高越随机，越低越确定 |
| **Top-p (Nucleus)** | 累积概率截断，保留质量高的词 | 0.9 - 0.95 | 0.9 - 1.0 | 防止采样到长尾低概率词 |
| **Top-k** | 仅从概率最高的 k 个词中选 | 40 - 50 | 40 - 100 | 限制候选词范围，提高稳定性 |
| **Frequency Penalty** | 惩罚重复出现的词 | 0.0 - 0.2 | 0.5 - 0.8 | 降低重复度，增加多样性 |

- **代码示例 (OpenAI API 调用)**：
```python
import openai

response = openai.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "写一首关于春天的诗"}],
    temperature=0.8,  # 创意调高
    top_p=0.9,        # 常用 0.9
    max_tokens=200
)
```

## 记忆要点

- 输入输出：输入文本序列，输出逐个生成的预测Token。
- Temperature=0：确定性输出，选概率最高词，适合代码/问答。
- Temperature>0：增加随机性，值越高越有创意，适合写作。
- Top-p：核采样，从累积概率达p的最小词集中采样。
