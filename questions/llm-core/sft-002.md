# SFT(Supervised Fine-Tuning)、RLHF、DPO三者有什么区别？如何选择？

🎯 **本质**：SFT教模型"怎么回答"（格式/指令遵循），RLHF/DPO教模型"怎么回答更好"（符合人类偏好/价值观）。三者在LLM训练流水线中扮演不同角色。

📊 **训练流水线全景图：**

```
┌─────────────────────────────────────────────────────────────────┐
│                  LLM Alignment Pipeline                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │

│  (海量文本自监督学习)                                            │
│       │                                                         │
│       ▼                                                         │
│  [SFT] <───────────────────┐                                   │
│  (指令微调)                 │ 数据: (Prompt, Response)         │
│  目标: 学会对话与格式       │                                   │
│       │                     │                                   │
│       ▼                     │                                   │
│  [DPO / RLHF]               │ 数据: (Prompt, Win, Lose)        │
│  (偏好对齐)                 │                                   │
│  目标: 人类偏好/安全/有用   │                                   │
│       │                     │                                   │
│       ▼                     │                                   │
│  [Production Model]         │                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**1. SFT（监督微调）**
  - **目标**：让预训练模型学会指令遵循和对话格式
  - **数据**：高质量配对数据，通常 10k - 100k 条
  - **方法**：标准 Cross-Entropy Loss，$L = -\log P(y|x)$
  - **类比**：教学生背诵标准答案，形成基本的语言和逻辑能力
  - **注意**：数据质量 > 数量，GPT-3.5 主要靠高质量的 SFT 数据

**2. RLHF（基于人类反馈的强化学习）**
  - **目标**：让模型输出更符合人类偏好（有用、无害、诚实）
  - **三步流程**：
    - **Step 1 (RM)**: 训练奖励模型 - 用人类标注的偏好数据 (Prompt, A, B, Better)
    - **Step 2 (PPO)**: 用 RM 作为奖励，PPO 算法优化策略模型
      - Objective: $\text{maximize } E[RM(p, y)] - \beta \cdot KL(\pi|\pi_{SFT})$
    - **Step 3 (KL Penalty)**: 防止模型偏离 SFT 模型太远（避免 Reward Hacking）

**3. DPO（Direct Preference Optimization）**
  - **目标**：直接优化偏好数据，无需训练 Reward Model
  - **原理**：利用推导出的解析解，将 Reward 的隐式最大化转化为 SFT Loss 的变体
  - **优势**：训练稳定，无需维护 RM，显存占用低
  - **公式**：$L_{DPO} = -\log \sigma(\beta \log (\pi_\theta(y_w|x)/\pi_\theta(y_l|x)))$

**对比表格：**

| 维度 | SFT | RLHF (PPO) | DPO |
| :--- | :--- | :--- | :--- |
| **数据类型** | (Prompt, Response) | (Prompt, Chosen, Rejected) | (Prompt, Chosen, Rejected) |
| **训练阶段** | 基础能力构建 | 价值观/偏好对齐 | 价值观/偏好对齐 |
| **训练难度** | 低 (标准 CE Loss) | 高 (需训练 RM + PPO) | 中 (修改 Loss 函数) |
| **资源消耗** | 低 | 高 (4个模型并行) | 低 (2个模型/或单模型) |
| **稳定性** | 非常稳定 | 难调参，易发散 | 稳定，类似 SFT |
| **工程落地** | 必做 | 逐步被 DPO 替代 | 推荐首选 |

**实战案例：**
在做智能客服项目时，先使用 5w 条历史工单数据进行 SFT，模型学会了业务流程但回答语气生硬。随后收集了 1w 条人工优化的“好回答 vs 坏回答”对进行 DPO 微调，模型回答的“人情味”和安全性显著提升，特别是拒绝回答恶意诱导问题的准确率从 60% 提升到了 95%。

**代码示例（DPO Loss 实现）：**
```python
import torch
import torch.nn.functional as F

def dpo_loss(policy_chosen_logps, policy_rejected_logps, 
             reference_chosen_logps, reference_rejected_logps, beta=0.1):
    """
    计算隐式奖励模型差异
    """
    pi_logratios = policy_chosen_logps - policy_rejected_logps
    ref_logratios = reference_chosen_logps - reference_rejected_logps
    
    losses = -F.logsigmoid(beta * (pi_logratios - ref_logratios))
    return losses.mean()
```

## 记忆要点

- 流水线三步：预训练打底 -> SFT教格式与指令(怎么答) -> 对齐教价值观(答得更好)
- 数据差异：SFT吃标准问答对，而RLHF与DPO均吃(Prompt, Chosen, Rejected)偏好对数据
- 对比RLHF与DPO：RLHF需训RM且PPO极不稳定，而DPO无需RM直接转为SFT形式训练更稳定
- 选型指导：算力足求泛化选RLHF(PPO)，缺显存求稳定极速对齐首选DPO
