# 为什么在attention中要进行scaled(为什么除以

在 Transformer 的多头注意力机制中，缩放点积注意力（Scaled Dot-Product Attention）是核心计算方法。计算过程中，查询向量（Q）和键向量（K）进行点积操作得到注意力分数，随后通过 Softmax 函数将这些分数转换为概率分布。

### 1. 问题背景
点积计算的公式如下：
$$Attention(Q, K, V) = \text{Softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$
其中 $d_k$ 是键向量的维度。

**问题 1：点积值过大**
高维向量的点积期望值会随着维度 $d_k$ 的增加而增大。当点积值过大时，Softmax 函数的输出会变得极端化：某些位置的输出非常接近 1，而其他位置接近 0。这导致注意力分布过于集中，模型难以从多个位置有效学习。

**问题 2：梯度消失**
当点积值过大时，Softmax 函数进入饱和区，其梯度会变得非常小，导致反向传播时梯度消失，使得模型训练难以收敛。

### 2. 缩放的解决方案
为了避免这些问题，Transformer 引入了缩放因子 $\frac{1}{\sqrt{d_k}}$。

**正则化效果**
两个独立随机向量的点积的标准差大约与 $\sqrt{d_k}$ 成比例。通过除以 $\sqrt{d_k}$，可以将点积值控制在合理的范围内，防止数值过大。

**保持数值稳定性**
缩放确保了 Softmax 的输入不会过大或过小，使注意力分布更加平滑，模型能够更稳定地训练。

**避免梯度消失**
通过限制输入值的范围，确保 Softmax 的梯度不会过小，保障了反向传播的有效性。

### 3. 数值举例
假设 $d_k = 64$，则 $\sqrt{d_k} = 8$。如果不进行缩放，点积结果可能在 64 量级，Softmax 输出极端；除以 8 后，数值被缩放至较小范围，Softmax 输出更加合理。

### 4. 总结
在 Softmax 之前对点积注意力分数进行缩放（除以 $\sqrt{d_k}$）是为了：
1. 防止数值不稳定，避免注意力权重极端化；
2. 避免梯度消失，确保训练效率；
3. 加快模型收敛，提升模型表现。

#### 实战案例：混合精度训练中的数值溢出
在早期使用 BFP16（半精度浮点数）进行 Transformer 模型训练时，如果不加缩放因子，$Q$ 和 $K$ 的点积经常超过 65504（float16 的最大值），导致计算结果变为 `NaN`。引入缩放因子后，数值被映射回安全范围，使得混合精度训练得以稳定进行，训练速度提升一倍。

#### 代码示例：Scaled Dot-Product 实现
```python
import torch
import torch.nn as nn
import math

class ScaledDotProductAttention(nn.Module):
    def forward(self, query, key, value, mask=None):
        d_k = query.size(-1)
        scores = torch.matmul(query, key.transpose(-2, -1)) 
        
        # 关键：除以 sqrt(d_k) 进行缩放
        # 这一步将方差控制在 1 左右，防止 Softmax 梯度消失
        scores = scores / math.sqrt(d_k)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        
        attention_weights = torch.softmax(scores, dim=-1)
        return torch.matmul(attention_weights, value), attention_weights
```
