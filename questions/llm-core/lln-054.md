# QLoRA的原理是什么?如何实现4-bit量化下的高效微调

QLoRA = Quantization + LoRA,在4-bit量化的基座模型上训练LoRA适配器.

- **核心创新:**

1. **NF4 (Normal Float 4) 量化**
- 基于权重的正态分布设计量化格子,信息论最优
- 量化过程:W -> NF4(4-bit) + 量化常数(fp16)

2. **双重量化**
- 不仅量化权重,还量化量化常数本身(fp32->fp8)
- 每参数平均节省0.37bit

3. **分页优化器**
- 使用NVIDIA统一内存,显存不足时自动转移到CPU
- 避免OOM

- **QLoRA训练流程:**
1. 加载预训练模型,权重量化为NF4 (4-bit)
2. 冻结量化权重,在fp16精度下训练LoRA适配器
3. 前向传播时反量化为fp16计算
4. 反向传播梯度只流经LoRA参数

- **效果:** 65B模型可在单张48GB GPU上微调,效果接近全参数微调(差距<1%),显存减少约75%.

- **架构与存储细节补充:**
QLoRA 的核心在于将大模型压缩至 4-bit 存储和读取，但在计算时保持 16-bit (BF16) 精度。
NF4 数据类型专门为正态分布的权重设计，通过分位量化将信息密度最大化。

```text
Memory Layout & Computation Flow

Disk/HDD            GPU Memory (VRAM)               Compute (CUDA Cores)
┌─────────┐         ┌───────────────────────────┐    ┌──────────────────────┐
│Weights  │───────▶ │ ┌───────────────────────┐ │    │  Dequantize (NF4->BF16)│
│(FP16)   │ Copy   │ │ W_quant (NF4, 4-bit)   │ │───▶│  Matrix Multiplication │
└─────────┘         │ │ [Frozen Base Model]   │ │    │  (Accumulate in BF16) │
                    │ └───────────────────────┘ │    └───────────┬──────────┘
                    │                           │                │
                    │ ┌───────────────────────┐ │                │
                    │ │ LoRA_A (BF16, Train.) │ │────────────────┘
                    │ │ LoRA_B (BF16, Train.) │ │
                    │ └───────────────────────┘ │
                    └───────────────────────────┘

Double Quantization Detail:
Original: Weight (NF4) + C_max (FP32) per block
DQ:       Weight (NF4) + C_max (FP8) + C_max_C (FP32)
```

## 常见考点
1. **反量化位置**：QLoRA 是在前向传播开始时一次性反量化整个模型，还是在计算层逐层反量化？（通常是在逐层计算时，实时将 NF4 反量化为 BF16 进行计算，这样不会占用额外的 FP16 权重显存，但计算开销略有增加）。
2. **NF4 vs Float4/Int4**：为什么需要专门的 NF4 而不是标准的 Int4？（LLM 权重通常呈现正态分布，标准 Int4 的均匀量化区间会导致大量权重落在截断区，丢失精度；NF4 的量化间隔也是非均匀的，能更好地匹配正态分布的尾部和峰值）。
3. **梯度计算**：量化的权重梯度如何计算？（对于冻结的基础模型，不计算梯度（或者梯度设为0），梯度只流经 LoRA 的低秩分解矩阵 A 和 B。由于 LoRA 参数量极少（<1%），显存占用主要在于激活值和优化器状态）。

- **实战案例:** 在单张 RTX 3090 (24GB) 显卡上微调 7B 模型时，使用全量 LoRA 仍可能因 Batch Size 过大导致显存溢出。启用 QLoRA 后，虽然推理速度略有下降（约慢 10%-15%），但显存占用降低至 12GB 左右，使得 Batch Size 可以翻倍，且最终的 perplexity 与全参数 LoRA 几乎一致。

- **代码示例:**
```python
from peft import LoraConfig, get_peft_model, prepare_model_for_kbit_training
from transformers import BitsAndBytesConfig

# 1. 配置 4-bit 量化 (NF4 + DQ)
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.float16,
    bnb_4bit_use_double_quant=True # 开启双重量化
)

# 2. 加载模型并准备微调
model = AutoModelForCausalLM.from_pretrained("llama-7b", quantization_config=bnb_config)
model = prepare_model_for_kbit_training(model) # 冻结基座，转换梯度

# 3. 添加 LoRA Adapter
lora_config = LoraConfig(r=8, lora_alpha=32, target_modules=["q_proj", "v_proj"])
model = get_peft_model(model, lora_config)
```
