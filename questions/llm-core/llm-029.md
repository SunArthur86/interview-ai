# 介绍一下Tokenizer怎么训练的

Tokenizer 的训练主要是指构建**词表** 的过程，最主流的方法是子词 分词算法，如 BPE、WordPiece 和 Unigram。

### 1. 为什么要训练 Tokenizer？
计算机无法直接处理文本，需将其转化为数字 ID。Tokenizer 的目标是：
*   **降低词表大小**：避免词级分词造成的词表过大和稀疏问题。
*   **处理未登录词 (OOV)**：通过子词组合表示生僻词，提高泛化能力。

### 2. 核心算法：BPE (Byte Pair Encoding)
BPE 是一种基于统计频率的贪心合并算法，也是 GPT、BERT 等模型的基础。

**训练步骤**：
1.  **准备语料**：准备大量文本，将单词拆分为字符序列（如 "apple" \rightarrow "a p p l e </w>"）。
2.  **统计频率**：统计语料中所有相邻字符对（如 "p p"）的出现频率。
3.  **合并最高频对**：找到频率最高的字符对（例如 "e r"），将其合并为一个新的子词（"er"），加入词表。
4.  **更新语料**：用新子词替换语料中对应的字符对。
5.  **迭代**：重复步骤 2-4，直到词表大小达到预设值（如 50k）。

**BPE 训练流程图**：
```text
初始状态: {'h u g </w>': 10, 'p u g </w>': 5, 'p u n </w>': 12, 'b u n </w>': 4}
统计频率: ('u', 'g')=15, ('u', 'n')=16, ...
合并 'u' 'n' -> 'un'
更新后:  {'h u g </w>': 10, 'p u g </w>': 5, 'p un </w>': 12, 'b un </w>': 4}
继续迭代... 直到达到目标词表大小。
```

### 3. 其他常见算法
*   **WordPiece**：与 BPE 类似，但合并标准不是“频率最高”，而是“能使训练语料似然概率最大化”的合并对。常用于 BERT。
*   **Unigram Language Model**：初始化一个巨大的词表，然后通过不断计算每个子词的损失，逐步删除对整体语言模型概率贡献最小的子词，直到词表缩小到目标大小。常用于 T5、ALBERT。

### 4. 现代 Tokenizer 的特点
*   **BBPE (Byte-level BPE)**：为了避免生僻字符（如 Emoji、生僻汉字）导致词表爆炸，现代 LLM（如 LLaMA, GPT-4）通常在字节级别进行 BPE 合并，支持任意 Unicode 字符。

### 5. 常见考点
1.  **BPE 的贪心性质**：BPE 是贪心算法，这可能导致什么问题？（答：局部最优，可能合并出次优的子词，例如可能优先合并了高频但语义不相关的组合）。
2.  **特殊Token**：训练时如何处理特殊标记（如 `<PAD>`, `<EOS>`, `<BOS>`）？（答：通常这些标记不参与合并过程，直接加入词表）。
3.  **BPE vs WordPiece**：二者在选择合并规则上的数学直觉区别是什么？（答：BPE 基于频次统计，WordPiece 基于语言模型似然最大化）。

### 6. 实战深化

**实战案例**
在实际业务中，如果直接使用通用 BPE（如 GPT-2 的 Tokenizer）处理垂直领域文本（如医疗、法律），会导致**切分颗粒度极细**，Token 序列过长，增加推理成本且丢失专业语义。正确做法是采集业务语料，使用 `sentencepiece` 或 `tokenizers` 库进行增量训练或从零训练专属 Tokenizer，通常能将序列长度缩短 20%-30%。

**代码示例**
```python
from tokenizers import Tokenizer, models, trainers, pre_tokenizers

# 1. 初始化 BPE 模型
tokenizer = Tokenizer(models.BPE(unk_token="[UNK]"))

# 2. 设置预分词器 (按空格和标点切分)
tokenizer.pre_tokenizer = pre_tokenizers.Whitespace()

# 3. 定义训练器
trainer = trainers.BpeTrainer(
    vocab_size=30000,   # 目标词表大小
    special_tokens=["[PAD]", "[UNK]", "[CLS]", "[SEP]", "[MASK]"]
)

# 4. 从文件训练
files = ["./data/corpus.txt"]
tokenizer.train(files, trainer)

# 5. 保存
tokenizer.save("./my_bpe_tokenizer.json")
```
