# Sparse Attention(稀疏注意力)是什么

### 87.1 概念与公式
**概念：** 在计算注意力权重时,不再对所有 的位置两两计算,而是只针对部分位置 进行注意力. 例如:局部块、滑动窗口、稀疏模式等。

**公式：** 虽然核心公式还是 $Attention(Q,K,V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}})V$，但 $QK^T$ 会被稀疏化为一个稀疏矩阵 $S$。例如只保留邻近若干位置或固定模式：
$$ S_{i,j} = \begin{cases} (QK^T)_{i,j} / \sqrt{d_k}, & \text{if } (i, j) \in \text{pattern} \\ -\infty, & \text{otherwise (masked)} \end{cases} $$
然后进行 $\text{softmax}(S)V$。被 Mask 掉的部分在 Softmax 后归零。

**应用：**
• 需要处理超长文本(成千上万 Token) 的 NLP 任务:长文摘要、长上下文 QA、法律/金融文档处理等。
• 序列长度很大的时序预测或推荐系统,也会考虑稀疏注意力。

### 87.2 稀疏注意力与全连接→Dropout 的思考
**背景对比：**
**全连接层到 Dropout:** 在传统神经网络中，所有神经元全连接，参数量大。Dropout 随机“断开”连接，通过训练时的稀疏性减少过拟合，提高泛化能力。
**自注意力到稀疏注意力：** 标准自注意力复杂度为 $O(N^2)$。稀疏注意力通过固定或可学习的“稀疏模式”（如局部块、滑动窗口）减少计算量，复杂度降至 $O(N)$ 或 $O(N \log N)$。主要目的是解决长序列的效率问题。

**“稀疏性”对比：**
• **Dropout 的稀疏性：** 训练阶段随机 Mask，推理阶段通常关闭（或缩放），为了正则化。
• **Sparse Attention 的稀疏性：** 遮蔽 $N \times N$ 矩阵中的大部分元素为 $-\infty$。模式是**固定**或**结构化**的（如滑动窗口），训练和推理时保持一致。为了降低 FLOPs 和显存。

### 87.3 稀疏注意力在大模型中的趋势
随着 LLM 上下文长度增长，稀疏注意力（如 Longformer, BigBird）变得至关重要，它使得处理万级 Token 成为可能。

**补充细节：**
常见的稀疏模式包括：
1. **Sliding Window (滑动窗口)**：每个 Token 只关注其前后 $w$ 个 Token。
2. **Dilated Sliding Window (空洞滑动窗口)**：以固定的步长（Dilation rate）选取 Token，扩大感受野。
3. **Global/Random Attention**：部分 Token（如分类符 [CLS]）作为全局节点关注所有 Token，或随机选取部分 Token 进行全局交互，以捕获长距离依赖。

**实战案例：Longformer 的踩坑**
在迁移 HuggingFace 的 Longformer 进行长文本法律合同审核时，发现模型偶尔会漏掉合同末尾的关键条款。排查发现是因为使用了 512 的滑动窗口，而合同关键条款恰好处于窗口断裂处。后来引入了 Global Attention 机制，强制将“签署日期”和“违约责任”等特定 Token 设为全局节点，解决了长距离依赖丢失的问题。

**关键代码**
```python
import torch
import torch.nn as nn

def create_sliding_window_mask(seq_len, window_size):
    """创建滑动窗口的 Attention Mask"""
    mask = torch.full((seq_len, seq_len), float('-inf'))
    for i in range(seq_len):
        start = max(0, i - window_size // 2)
        end = min(seq_len, i + window_size // 2 + 1)
        mask[i, start:end] = 0
    return mask
```

**稀疏注意力模式对比**

| 模式 | 复杂度 | 感受野 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **Full Attention** | $O(N^2)$ | 全局 | 短文本，需要精确建模所有关系 |
| **Sliding Window** | $O(N)$ | 局部 ($\pm w$) | 实时流式数据处理，文本局部连贯性 |
| **Dilated Window** | $O(N)$ | 扩大局部 | 图像处理，捕捉非邻近特征 |
| **Fixed Pattern (e.g. BigBird)** | $O(N)$ | 全局 (通过跳跃) | 长文档理解，理论上可近似全图 |

**## 常见考点**
1. **稀疏注意力如何处理长距离依赖？**
   单纯的滑动窗口无法捕捉距离超过窗口大小的信息。解决方案通常包括：引入“全局Token”（如每隔一段距离选一个作为全局锚点）、多层堆叠（一层滑动扩大感受野，层数加深后理论上能覆盖全长）或者混合注意力（局部+全局）。
2. **稀疏注意力在推理加速上的实际效果如何？**
   虽然计算量降低了，但在未针对稀疏算子优化的硬件上，由于显存访问不连续，实际运行时间可能不如理论快。需要使用专门的 Sparse Kernel（如 Triton 实现）才能发挥优势。

## 记忆要点

- 定义：只计算部分位置的注意力（如滑动窗口、全局节点），其余 Mask 掉。

- 模式：滑动窗口（局部）、Global Token（全局）、随机稀疏。
- 对比：Dropout 稀疏性用于正则化（训练随机），Sparse Attention 用于效率（模式固定）。
