# DPO 的 loss 是什么

DPO（Direct Preference Optimization）的损失函数直接基于 Bradley-Terry 模型构建，旨在利用人类偏好数据（$x, y_w, y_l$）优化模型，使其倾向于生成更好的回复。其损失函数形式如下：

$$ L(\theta) = -\mathbb{E}_{(x,y_w,y_l)\sim D} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_\theta(y_l|x)} - \beta \log \frac{\pi_{ref}(y_w|x)}{\pi_{ref}(y_l|x)} \right) \right] $$

其中：
- $\sigma$ 是 Sigmoid 函数。
- $\pi_\theta(y|x)$ 是当前策略模型生成的概率。
- $\pi_{ref}(y|x)$ 是参考模型（冻结的 SFT 模型）生成的概率。
- $y_w$ 是偏好回复，$y_l$ 是非偏好回复。
- $\beta$ 是温度系数（用于控制 KL 惩罚强度）。

**原理简述**：DPO 并没有显式训练奖励模型，而是通过数学推导，将奖励模型的差异转化为策略模型的对数概率比。最小化该损失函数意味着最大化 $y_w$ 和 $y_l$ 在策略模型与参考模型下的对数概率差之差，从而直接对齐人类偏好。

**补充关键细节**：
注意上面的完整公式中包含了两项对数概率的比值之差。简化的记忆方式是：策略模型希望让 chosen 和 rejected 拉开差距，而参考模型的作用是锚定 baseline，防止模型为了最大化这个差距而简单地增加所有词的概率（即导致模式崩塌或偏离原有语言能力）。Loss 的本质是最大化 $y_w$ 优于 $y_l$ 的概率。

### 实战推导与代码实现

**实战经验**：在代码实现中，通常不直接计算概率 $\pi(y|x)$（因为它是所有路径的连乘），而是计算 **Log Probability**（所有 token logits 的 log sum）。这在数学上是等价的（$\log \frac{a}{b} = \log a - \log b$）。另外，要注意 `padding` 部分 mask 掉，否则 Padding token 的高概率会误导模型。

**代码示例**：
```python
import torch
import torch.nn.functional as F

def compute_dpo_loss(policy_chosen_logps, policy_rejected_logps,
                     ref_chosen_logps, ref_rejected_logps, 
                     beta=0.1, label_smoothing=0.0):
    """
    计算 DPO Loss
    policy_chosen_logps: [Batch] 策略模型对正样本的 log_prob
    policy_rejected_logps: [Batch] 策略模型对负样本的 log_prob
    ref_*: 参考模型对应的 log_prob
    """
    # 计算策略模型和参考模型的 log ratio 差异
    # 公式: beta * (log_pi(y_w) - log_pi(y_l) - (log_ref(y_w) - log_ref(y_l)))
    logits = beta * (policy_chosen_logps - policy_rejected_logps - 
                     ref_chosen_logps + ref_rejected_logps)
    
    # Label Smoothing (可选): 防止模型过于自信，增加鲁棒性
    if label_smoothing > 0:
        logits = logits * (1 - label_smoothing)

    # Loss = -log(sigmoid(logits))
    # 等价于: log(1 + exp(-logits))
    losses = -F.logsigmoid(logits)
    
    return losses.mean()
```

## 记忆要点

- 公式核心：Sigmoid(策略对数比 - 参考模型对数比)，本质是二元分类似然。
- 关键参数：β控制KL惩罚强度，参考模型作为锚点防止模式崩塌。
- 计算技巧：代码中用Log Probability差值代替概率比值，注意Mask Padding。
- 物理意义：最大化Chosen优于Rejected的概率，同时限制偏离初始策略。
