# ALiBi原理

### ALiBi (Attention with Linear Biases) 原理

#### 1. 核心思想
ALiBi (Attention with Linear Biases) 提出了一种无需显式位置嵌入的方法。它在注意力机制的计算中，直接对注意力分数（attention logits）加入一种与位置距离成线性关系的偏置。

#### 2. 位置编码的局限
传统的 Transformer 模型（如 BERT、GPT）通常使用绝对位置嵌入或相对位置编码。然而，这些方法在扩展序列长度（例如，将训练中从未见过的长度应用到推断时）存在性能退化问题。模型对训练中序列最大长度存在隐含假设，一旦在推理阶段遇到更长的序列，效果会显著下降。

#### 3. ALiBi 的计算机制
ALiBi 的核心在于修改 Attention Score 的计算：

$$Attention(Q, K, V) = \text{softmax}(\frac{QK^T}{\sqrt{d_k}} + \text{Bias}) \cdot V$$

其中偏置项 $\text{Bias}$ 是一个非学习的静态矩阵，其元素 $b_{i,j}$ 定义为 Query 位置 $i$ 与 Key 位置 $j$ 的函数：

$$b_{i,j} = -\alpha \cdot |i - j|$$

- $|i - j|$：表示两个 Token 之间的距离。
- $\alpha$：可设置的超参数（通常随头数呈指数衰减，如 $\frac{1}{2^{h/4}}$，让不同头关注不同范围的距离）。

这意味着，Key 离 Query 越远，其被添加的负向惩罚越大，导致 Attention 分数越低。

**计算流程图**：
```
Query (Q)  Key (K)
   |         |
   +----+----+
        | MatMul (QK^T)
        v
   Score Matrix
        |
        +------------------+
        | 加上 Bias Matrix |
        | (非学习参数)      |
        | b_{ij} = -α|i-j| |
        v
   Score + Bias
        |
   Softmax(·)  ← 越远的距离，Score 被减得越多，概率越低
        |
   Attention Weight
        |
   MatMul with Value (V)
```

#### 4. 优势
- **无需训练位置参数**：不需要在长序列推理时修改参数，也无需重新训练。模型只需要在相对较短序列上完成训练。
- **优秀的长度外推能力**：由于偏置是线性且基于相对距离的，不依赖训练中出现过的最大长度。在推理阶段可直接将序列长度扩展到训练时未见过的范围，模型依然能保持较好的性能。
- **计算高效**：仅是在 $QK^T$ 矩阵上加上一个固定的偏置，计算开销极小。

#### 5. 实战深化

**实战案例**：
在金融长文档摘要生成任务中，我们曾尝试将最大序列长度从 2K 扩展到 8K。使用 Sinusoidal 位置编码时，PPL 在 3K 后急剧上升；切换为 ALiBi 后，无需重新训练模型，直接在 8K 长度下推理，PPL 曲线平滑，且摘要的关键信息召回率未下降。注意需将 `causal_mask` 与 ALiBi bias 相加时处理好数据类型，防止 `float16` 溢出。

**代码示例**：
```python
import torch
import torch.nn.functional as F

def get_alibi_bias(n_heads, seq_len, device='cuda'):
    # 生成基础斜率，随头数指数衰减
    slopes = torch.tensor([2 ** (-8 / n_heads * i) for i in range(1, n_heads + 1)], 
                          dtype=torch.float32, device=device)
    # 生成相对距离矩阵 |i - j|
    pos = torch.arange(seq_len).unsqueeze(1) - torch.arange(seq_len).unsqueeze(0)
    # 计算偏置: -slope * |i - j|
    bias = -slopes.unsqueeze(1).unsqueeze(1) * pos.unsqueeze(0).abs()
    return bias # Shape: (n_heads, seq_len, seq_len)

# 在 Attention 计算中使用

```

**ALiBi vs RoPE vs Sinusoidal 对比**：

| 特性 | ALiBi | RoPE (Rotary Pos) | Sinusoidal (Absolute) |
| :--- | :--- | :--- | :--- |
| **位置信息注入方式** | Logits 加法线性偏置 | Query/Key 向量几何旋转 | Input Embedding 加法拼接 |
| **外推能力** | **极强**（训练短，推理长无需微调） | 较强（需 NTK-aware scaling 等技巧） | 弱（受限于训练最大长度） |
| **计算开销** | 极低（加法） | 中等（逐元素旋转操作） | 低（查表或计算） |
| **实现复杂度** | 简单 | 中等（需处理复数或旋转向量） | 简单 |
| **主流采用场景** | BLOOM, MPT 等部分开源模型 | Llama 2/3, Qwen, PaLM (目前主流) | BERT, GPT-2 (早期模型) |

#### ## 常见考点
1.  **RoPE 对比**：ALiBi 与目前大模型主流的 RoPE（旋转位置编码）有什么核心区别？（RoPE 也是相对位置，但通过旋转矩阵注入位置信息，通常具有更好的远程衰减建模能力，且理论更优雅）。
2.  **多 Head 差异**：ALiBi 中不同 Attention Head 的 $\alpha$ 参数是如何设定的？（通常按头数指数递减，使不同头关注不同尺度的上下文）。
3.  **为何不需要 Embedding**：ALiBi 是如何隐式编码序列顺序信息的？（通过惩罚非对角线元素，迫使模型关注附近的 Token）。
4.  **训练与推理长度**：如果模型只在长度 1024 上训练，推理时输入长度 2048，ALiBi 的表现通常会比 Sinusoidal 或 Learned Absolute Encoding 好在哪里？
