# RLHF 最近的新算法有哪些,介绍几个

RLHF 及其变体算法发展如下：

1. **RLHF vs RLAIF**：标注来源是人类还是 AI（如 GPT-4）。
2. **PPO (Proximal Policy Optimization)**：经典算法，引入 Critic 网络和 Importance Sampling，但训练不稳定、资源消耗大。
3. **GRPO (Group Relative Policy Optimization)**：通过 Group 采样计算 Baseline，去掉了对 Critic 网络的依赖，简化训练。
4. **ReMax**：直接对齐 Reward Model，去掉 Critic，优化目标更为直接。
5. **DPO (Direct Preference Optimization)**：摒弃显式 Reward Model 和 RL 循环，直接在偏好数据 上优化，稳定性高。
6. **ORPO (Odds Ratio Preference Optimization)**：认为 SFT + RLHF 非 End-to-End，直接在 SFT 损失中加入 Alignment 的 Odds Ratio 损失项。
7. **KTO (Kahneman-Tversky Optimization)**：针对 DPO 需要成对数据 的痛点，支持单点数据及二元 Label，数据获取更容易。
8. **Step-DPO**：针对 Instance-level（结果级）优化的局限，对推理过程进行 Step-level（步骤级）的偏好优化。
9. **DPO 变体 (DPO+, DPOP)**：引入正则项（如长度惩罚）解决长度 Hacking 问题。
10. **Nash Preference Optimization**：针对偏好不可传递性，引入纳什均衡概念求解最优策略。
11. **Multi-turn DPO**：将 DPO 拓展至多轮对话场景，优化多轮交互中的累积偏好。

**实战案例**：
在训练 Llama-3-70B-Instruct 时，Meta 团队采用了迭代式的 DPO（Iterative DPO）。每轮 DPO 后，用模型生成新的偏好数据，再进行下一轮 DPO。相比于单轮 PPO，这种方法不仅训练吞吐量提升了 3 倍以上，而且避免了 PPO 中常见的 KL 散度突然爆炸导致模型“脑抽”的现象。

**对比表格**：主流 RLHF 算法选型对比

| 算法 | 核心特点 | 计算资源消耗 | 训练稳定性 | 数据要求 | 适用场景 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **PPO** | 在线 RL，需 Rollout | 极高 (需 Actor+Critc+RM) | 低 (超参敏感) | Preference + Online Reward | 追求极致对齐效果，资源充足 |
| **DPO** | 离线优化，解析解 | 中 (无需 RM 推理) | 高 | 成对偏好数据 | 标准对齐，工业界首选 |
| **KTO** | 单样本优化 | 中 | 高 | 单样本+好坏标签 | 数据收集难，难以构造成对数据时 |
| **ORPO** | 单阶段 SFT+Alignment | 低 (仅需 SFT Loss) | 高 | 成对偏好数据 | 快速迭代，减少多阶段训练开销 |
| **Step-DPO** | 过程级奖励 | 中 | 中 | 包含步骤标注的数据 | 数学推理、代码生成等强逻辑任务 |

**代码示例**：
```python
# 使用 TRL 库进行 DPO 训练的配置片段
from trl import DPOTrainer, DPOConfig

dpo_config = DPOConfig(
    beta=0.1,              # KL 散度系数，防止过度偏离 SFT
    loss_type="sigmoid",   # 损失函数类型
    max_length=512,
    max_prompt_length=128
)

dpo_trainer = DPOTrainer(
    model,
    ref_model,             # 参考模型，计算 KL 约束
    args=dpo_config,
    train_dataset=preference_dataset, # 包含 chosen, rejected 列
    tokenizer=tokenizer
)
dpo_trainer.train()
```

## 记忆要点

- DPO：摒弃显式RM和RL循环，直接在偏好数据上优化，稳定性高
- KTO：支持单样本二元标签，解决成对数据收集难的问题
- ORPO：在SFT损失中直接加入对齐项，实现单阶段训练
