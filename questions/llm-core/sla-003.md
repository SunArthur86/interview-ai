# GQA(Grouped-Query Attention)和MQA(Multi-Query Attention)是什么？

🎯 **本质**：GQA和MQA通过减少KV头的数量来减少KV Cache大小，降低推理时显存占用和访存带宽压力。

📊 **三种Attention对比**：

1.  **MHA(Multi-Head Attention)** - 标准Transformer
    -   Q有n个头，K有n个头，V有n个头。
    -   KV Cache = 2 * n * seq_len * head_dim。
    -   问题：解码阶段瓶颈在显存带宽，KV Cache读写量大。

2.  **MQA(Multi-Query Attention)** - 极致压缩
    -   Q有n个头，K有1个头，V有1个头。
    -   所有Q头共享同一组KV。
    -   KV Cache减少为1/n。
    -   缺点：表达能力下降，模型质量受损。

3.  **GQA(Grouped-Query Attention)** - 折中方案
    -   Q有n个头，K有g个头，V有g个头（1 < g < n）。
    -   每组Q头共享一对KV。
    -   优点：质量接近MHA，显存大幅减少，带宽压力显著降低。

🏗️ **结构对比图**：
```text
MHA (n=4)              GQA (g=2)              MQA (g=1)
──────────────────     ──────────────────     ──────────────────
Q1 Q2 Q3 Q4            Q1 Q2 | Q3 Q4            Q1 Q2 Q3 Q4
│  │  │  │             │  │  │  │  │  │        │  │  │  │
K1 K2 K3 K4            K1    | K2               K_shared
│  │  │  │             │     │                 │
V1 V2 V3 V4            V1    | V2               V_shared
``` 
*注：GQA中 Q1,Q2 共享 K1,V1；Q3,Q4 共享 K2,V2。* 

📋 **实际应用与数据**：
-   **Llama 2 70B**: GQA, g=8（原n=32）。
-   **Llama 3**: GQA, g=8（原n=32）。
-   **Mistral 7B**: GQA。
-   **PaLM**: 使用了MQA。

性能数据（Llama 2 70B 近似值）：
| 方案 | KV Cache大小 | 推理速度 | 模型质量 |
|------|--------------|---------|---------|
| MHA(32头) | 100% | 1.0x (Baseline) | 最好 |
| GQA(8组) | 25% | 1.7x | 接近MHA |
| MQA(1头) | 3% | 2.1x | 略差 |

💡 **面试要点**：GQA是MHA和MQA之间的甜蜜点——用很小的质量损失换取显著的推理性能提升（尤其是带宽受限场景），已成为现代LLM（如Llama 3）的标配。

## 常见考点
1.  **KV维度不匹配时的处理**：在GQA中，Q的维度是 [n, head_dim]，K/V是 [g, head_dim]。Attention计算时需要将K/V重复（Repeat）以匹配Q的头数，例如将K从 [g, d] 扩展到 [n, d]，或者通过广播机制实现。
2.  **训练与推理的一致性**：如果推理使用GQA，训练时必须使用GQA吗？（是的，架构必须一致，但可以使用Up-training从MHA微调到GQA）。
3.  **带宽 vs 计算**：面试官可能会问“为什么减少KV头能加速？”——回答关键在于：LLM推理是Memory-bound（受限于显存带宽），而不是Compute-bound。减少KV Cache体积直接减少了HBM的读写量，从而提速。

## 记忆要点

- MHA：Q、K、V头数相等，KV Cache大，推理受限于显存带宽。
- MQA：Q多头但K、V仅1头，Cache极小，速度最快，但模型质量受损。
- GQA：Q多头，K、V分组（1 < g < n），平衡了质量与速度，是现代LLM标配。
- 核心收益：推理是Memory-bound，减少KV Cache体积直接减少HBM读写，从而提速。
