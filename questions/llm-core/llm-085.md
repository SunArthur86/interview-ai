# 强化学习算法是怎么分类的

强化学习算法主要分为基于价值、基于策略和Actor-Critic三类。

**1. 基于价值**
*   **核心思想**：学习一个状态或状态-动作对的价值函数 $V(s)$ 或 $Q(s,a)$，通过最大化价值来选择动作。
*   **代表算法**：Q-Learning, DQN (Deep Q-Network)。
*   **适用场景**：通常适用于**离散动作空间**。因为在连续空间中，寻找 Argmax 的操作是连续优化的，计算困难。
*   **特点**：策略是确定性的（ greedy），只能处理离散动作。

**2. 基于策略**
*   **核心思想**：直接参数化策略函数 $\pi_\theta(a|s)$，输出动作的概率分布，通过梯度上升直接优化策略参数。
*   **代表算法**：REINFORCE, PPO (Proximal Policy Optimization), TRPO。
*   **适用场景**：适用于**连续动作空间**或**随机策略**场景。
*   **特点**：能够学习随机策略，收敛性较好，但方差通常较大，样本效率相对较低。

**3. Actor-Critic (AC)**
*   **核心思想**：结合上述两者。
    *   **Actor**：负责策略，即决定动作。
    *   **Critic**：负责评估，即计算状态价值 $V(s)$ 或优势函数 $A(s,a)$，用来指导 Actor 的更新方向，减少方差。
*   **代表算法**：A3C, A2C, SAC, PPO (虽然PPO主要是Policy Gradient，但利用了Value Function作为Critic做归一化)。
*   **特点**：平衡了方差和偏差，训练较为稳定。

**4. 其他分类维度**
*   **Model-free vs Model-based**：
    *   **Model-free**：不学习环境动力学模型，直接通过与环境交互学习。泛化能力强，但样本效率低。
    *   **Model-based**：学习环境模型（状态转移概率），利用模型进行规划或生成数据。样本效率高，但模型误差会累积。

**实战案例**：在推荐系统的多轮对话排序任务中，初期尝试使用 DQN（离散动作），导致动作空间爆炸且难以收敛。后改用 PPO（连续/随机策略），直接优化最终点击率的期望值，不仅收敛更快，且探索性更强，避免了陷入局部最优。

**代码示例**：
```python
import torch
import torch.nn as nn

class ActorCritic(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        # Actor: 输出动作概率分布
        self.actor = nn.Sequential(nn.Linear(state_dim, 128), nn.Tanh(), nn.Linear(128, action_dim), nn.Softmax())
        # Critic: 输出状态价值 V(s)
        self.critic = nn.Sequential(nn.Linear(state_dim, 128), nn.Tanh(), nn.Linear(128, 1))
```

**对比表格：**

| 维度 | Value-Based (DQN) | Policy-Based (PPO) | Actor-Critic (A2C/SAC) |
| :--- | :--- | :--- | :--- |
| **优化目标** | 最大化 $Q(s,a)$ | 最大化 $R(\theta)$ | 同时优化 $Q(s,a)$ 和 $\pi(a\|s)$ |
| **动作空间** | 仅限离散 | 离散 & 连续 | 离散 & 连续 |
| **探索机制** | $\epsilon$-greedy | 熵正则化 | 熵正则化 / 噪声 |
| **收敛速度** | 较慢，易震荡 | 较慢，方差大 | 较快，稳定性好 |
| **适用性** | Atari游戏 | 机器人控制 | 复杂控制任务 |

**## 常见考点**
1.  **DQN 的局限性**：为什么 DQN 难以处理连续动作空间？（无法对所有动作求 Argmax）。
2.  **Actor-Critic 的优势**：为什么结合 Critic 可以提升训练稳定性？（引入基线减少策略梯度的方差）。
3.  **On-policy vs Off-policy**：两者区别是什么？（On-policy 如 SARSA、PPO 必须用当前策略产生的样本；Off-policy 如 Q-Learning、DQN 可以用历史经验池的数据）。
4.  **探索与利用**：基于价值的方法（如 $\epsilon$-greedy）与基于策略的方法（如熵正则化）如何平衡探索？

## 记忆要点

- 基于价值：学习 Q(s,a)，输出确定动作，适合离散空间（如 DQN）。
- 基于策略：直接参数化策略 π(a|s)，适合连续/随机空间（如 PPO）。
- Actor-Critic：结合两者，Actor 决定动作，Critic 评估价值以减少方差。
- Model-free：不学环境模型，直接交互；Model-based：学环境模型进行规划。
