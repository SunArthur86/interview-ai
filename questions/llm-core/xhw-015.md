# 为什么用LayerNorm不用BatchNorm

在 Transformer 中使用 LayerNorm (LN) 而非 BatchNorm (BN)，主要是由 NLP 任务的特性和数据结构决定的。

**1. 数据维度与批次依赖**
*   **BN 的局限性**：BN 是对同一个 batch 中不同样本的**同一维度**特征进行归一化。在 NLP 中，输入通常是 `(batch, seq_len, embedding)`。由于序列长度往往不一致（需要 padding），或者 batch size 较小（如 1 或 2），BN 计算出的统计量（均值和方差）会非常不稳定，且受 padding 干扰严重。此外，推理时 BN 需要依赖训练时的全局统计量，增加了复杂度。
*   **LN 的优势**：LN 是对**单个样本内部**的所有特征进行归一化。它不依赖于 batch size，也不受序列长度变化的影响，计算出的均值和方差始终是针对当前句子的特征分布。

**2. 语义一致性**
*   在文本处理中，同一个句子在不同位置的特征分布应当具有一致性。LN 能够保持单个样本内部的特征分布稳定，更符合 NLP 任务对文本特征独立性的要求。

**3. 训练稳定性**
Transformer 结构深层且参数量大，使用 LayerNorm 能够在隐藏层维度上稳定数据的分布，有效缓解梯度问题，且无需像 BN 那样维护额外的 Running Mean/Var。

**4. 归一化方向细节**
*   **计算维度对比**：
    *   **BatchNorm**: 对 `Batch` 维度做统计，归一化 `Channel/Hidden` 维度。
    *   **LayerNorm**: 对 `Hidden/Channel` 维度做统计，归一化 `Batch` 维度。
*   **ResNet vs Transformer**：在 CV 任务（如图像分类）中，BN 效果好是因为不同样本的同一特征（如边缘、纹理）遵循相似的分布，且批量数据能提供稳定的统计量。而在 NLP 中，不同句子的填充位置差异巨大，强制在 Batch 维度归一化会破坏语义。

**5. 计算维度原理图 (ASCII)**
```text
输入张量 X 形状: [Batch, Seq_Len, Hidden_Dim]

BatchNorm (BN) 计算逻辑:
统计量计算: 沿着 Batch 轴聚合
┌──────────────────────────────────────────────┐
│ Batch1: Token1[0]  Token2[0] ... TokenN[0]  │<─ 均值1/方差1
│ Batch2: Token1[0]  Token2[0] ... TokenN[0]  │<─ 均值2/方差2
│ ...                                          │
│ 最终统计量: 整个 Batch 在同一 Channel 的均值  │
└──────────────────────────────────────────────┘
归一化: (x - mean_batch) / var_batch
结果: 拉长了不同样本在同一位置的分布差异

LayerNorm (LN) 计算逻辑:
统计量计算: 沿着 Hidden 轴聚合 (针对单个样本)
┌──────────────────────────────────────────────┐
│ Batch1: [Tok1_h0, Tok1_h1, ... Tok1_hd]     │ <─ 计算该 Token 的均值/方差
│         [Tok2_h0, Tok2_h1, ... Tok2_hd]     │ <─ 独立计算
│         [...                                │
└──────────────────────────────────────────────┘
归一化: (x - mean_hidden) / var_hidden
结果: 保持了每个 Token 自身的特征分布稳定性
```

**## 常见考点**
1.  **LayerNorm 和 BatchNorm 在计算均值和方差的维度上有什么具体区别？**（考察对数据轴的理解）
2.  **Transformer 中 LayerNorm 通常放在残差连接的前面还是后面？**（Post-LN vs Pre-LN，Post-LN 更稳定，Pre-LN 训练更深模型时梯度流更好）
3.  **为什么 CV 领域常用 BN，而 NLP 常用 LN？**（考察对图像数据（像素分布相对统一）和文本数据（语义分布随样本变化）差异的理解）

## 记忆要点

- 计算方向：BN跨样本算特征(NLP易被变长Padding干扰)，LN在单样本内部算特征。
- 因果句：因为NLP句子长度多变且Batch小，所以用LN保持单样本内部特征分布稳定。
- 优势对比：LN不依赖Batch Size，训练无需全局统计量，更适合深层Transformer。
