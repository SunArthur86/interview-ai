# LLMs复读机问题

**LLMs复读机问题**是指模型在生成文本时过度依赖复制输入内容，出现高频重复或缺乏创造性的现象，常表现为无限循环输出某短语。

### 核心成因与原理
1.  **曝光偏差**：训练时使用真实的上一词预测下一词，而推理时使用模型生成的上一词。如果模型生成一个概率稍低的词，后续分布可能偏移回常见的高频词，导致循环。
2.  **目标函数局限**：标准交叉熵损失鼓励模型拟合训练数据的共现频率。训练数据中若存在大量重复模式，模型会倾向于模仿这些高频但低信息量的序列。
3.  **解码策略缺陷**：贪婪搜索易陷入局部最优；Beam Search虽然保留多个候选，但若长度归一化处理不当，模型倾向于生成短且通用（甚至重复）的高概率序列以降低累积损失。

### 解决方案与参数细节
1.  **采样策略调整**：
    *   **温度参数**：提高温度（如 >1.0）拉平概率分布，增加低频词被选中的概率。
    *   **Top-k / Top-p (Nucleus) Sampling**：从概率最高的 k 个词或累积概率达 p 的词池中随机采样，打破确定性循环。
2.  **惩罚机制**：
    *   **重复惩罚**：对已经生成过的 token 在计算 logits 时减去一个惩罚值，降低其再次出现的概率。
    *   **频率惩罚**：根据 token 在当前回答中的出现频率进行线性惩罚。
3.  **训练阶段优化**：使用更高质量的去重数据，或在 RLHF 阶段对重复输出给予极低的奖励值。

### 深化内容

#### 1. 实战案例
在使用 Llama-2-7B 进行长篇小说生成时，如果不设置 `repetition_penalty`（通常设为 1.1~1.5），模型在生成几百字后往往会陷入“...然后他...然后他...”的死循环。实战中调整该参数是解决长文本生成崩溃的第一手段。

#### 2. 代码示例 (HuggingFace 采样参数)
```python
output = model.generate(
    input_ids,
    max_new_tokens=200,
    do_sample=True,          # 开启采样
    temperature=0.7,          # 控制随机性
    top_p=0.9,                # Nucleus Sampling
    repetition_penalty=1.2,   # 关键：惩罚重复 token
    no_repeat_ngram_size=3    # 强制禁止 3-gram 重复
)
```

#### 3. 对比表格

| 现象/机制 | 贪婪搜索 | Beam Search | 采样 | 重复惩罚 |
| :--- | :--- | :--- | :--- | :--- |
| **原理** | 选概率最高的 1 个 | 选概率最高的 N 条路径 | 按概率分布随机选 | 对已出现 token 降权 |
| **文本多样性** | 极低 | 低 | 高 | 中高 (配合采样使用) |
| **复读风险** | **极高** (易陷入死循环) | 高 (同质化严重) | 低 | 中 |
| **适用场景** | 逻辑推理、数学运算 | 机器翻译、摘要 | 创意写作、对话 | 所有长文本生成 |
| **确定性** | 完全确定 | 完全确定 | 随机 (受 Seed 控制) | 随机 |

## 记忆要点

- 成因：曝光偏差（训练推理不一致）、目标函数拟合高频重复模式、解码策略局部最优。
- 解码：贪婪搜索易死循环，需用Top-p/Top-k采样增加随机性。
- 惩罚：设置Repetition Penalty（1.1-1.5）或Frequency Penalty降低重复词概率。
- 实战：长文本生成必调参数，配合no_repeat_ngram_size强制禁止N-gram重复。
