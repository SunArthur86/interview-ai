# SFT、RLHF和DPO有什么区别？

**SFT（Supervised Fine-Tuning）：**
- **定义**：监督微调，用高质量的指令-回答对训练。
- **目的**：让模型从“续写模式”转为“指令回答模式”，学会遵循指令。
- **数据格式**：{"instruction": "...", "input": "...", "output": "..."}。
- **特点**：是对齐的第一步，决定了模型的基础能力和回答风格。

**RLHF（Reinforcement Learning from Human Feedback）：**
- **定义**：基于人类反馈的强化学习。
- **流程**：收集人类偏好数据 → 训练Reward Model（奖励模型） → 用PPO（近端策略优化）优化策略模型。
- **核心组件**：需要训练Reward Model来模拟人类的喜好。
- **缺点**：PPO训练复杂、不稳定、超参数多；需要同时维护4个模型（Policy, Reference, Reward, Critic），显存占用极大。

**DPO（Direct Preference Optimization）：**
- **定义**：直接偏好优化，隐式地学习Reward Model，无需显式训练。
- **数学原理**：利用Bradley-Terry模型，证明了最优Reward函数可以直接从Policy和Reference模型的概率比推导出来，从而将Reward函数消元，直接优化Policy的 odds ratio。
- **损失函数**：仅基于偏好数据计算Loss，不需要Reward Model打分。
- **优势**：流程简化，只需要Policy和Ref模型；训练更稳定，数学上避免了PPO的方差问题；资源消耗少。

**关系与选择：**
SFT → RLHF/DPO。先SFT让模型“会对话”，再RLHF/DPO让模型“说人话/符合人类价值观”。目前DPO因其实现简单和效果好，已成为RLHF的主流替代方案。

### 实战案例
在训练一个垂直领域的问答助手时，SFT阶段的模型虽然能回答问题，但回答往往缺乏“人情味”且容易产生幻觉。我们尝试引入 **RLHF**，但在训练PPO时经常出现KL散度爆炸导致模型退化。最后我们切换到 **DPO**，直接利用SFT好的模型作为Reference，仅用10K条偏好数据就成功解决了“复读机”和“语气生硬”的问题，训练耗时从3天缩短至6小时。

### 详细对比表格

| 维度 | SFT (监督微调) | RLHF (强化学习) | DPO (直接偏好优化) |
| :--- | :--- | :--- | :--- |
| **数据类型** | Question-Answer (QA对) | Preference (Chosen/Rejected) | Preference (Chosen/Rejected) |
| **训练目标** | 最大化生成概率 | 最大化奖励信号 | 最大化偏好对概率差 |
| **模型数量** | 1个 | 4个 | 2个 (Policy + Ref) |
| **显存占用** | 低 | 极高 (PPO需Rollout) | 低 (同SFT)
| **训练稳定性** | 高 | 低 (超参数敏感) | 高
| **核心难点** | 构建高质量SFT数据 | 训练稳定的Reward Model | 避免Reference过拟合 |
| **主要作用** | 注入知识，学会指令 | 对齐人类价值观/安全性 | 对齐人类价值观/风格 |

### 代码示例
使用 `transformers` 库中的 `DPOTrainer` 进行训练的极简代码片段：

```python
from transformers import AutoModelForCausalLM, TrainingArguments
from trl import DPOTrainer

# 模型与Reference模型 (通常初始化为同一个SFT模型)
model = AutoModelForCausalLM.from_pretrained("path/to/sft_model")
model_ref = AutoModelForCausalLM.from_pretrained("path/to/sft_model")

# DPO 训练配置
training_args = TrainingArguments(
    output_dir="./dpo_output",
    per_device_train_batch_size=4,
    learning_rate=1e-5,
    beta=0.1 # DPO的温度系数，控制对Reference的偏离程度
)

# 初始化Trainer
dpo_trainer = DPOTrainer(
    model=model,
    ref_model=model_ref,
    args=training_args,
    train_dataset=preference_dataset, # 数据需包含 'prompt', 'chosen', 'rejected'
    tokenizer=tokenizer
)

dpo_trainer.train()
```

## 记忆要点

- SFT：监督微调，用QA对让模型学会遵循指令，是对齐基础。
- RLHF：需训练Reward Model，用PPO优化，流程复杂、显存高、不稳定。
- DPO：直接偏好优化，无需Reward Model，数学上消元Reward函数，仅用Policy和Ref模型。
- 关系：SFT是地基，RLHF/DPO是精装修；目前DPO因稳定简单逐渐替代RLHF。
