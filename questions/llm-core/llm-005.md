# 解释一下Adapter

大模型中的 Adapter 是一种轻量级的模型微调技术，旨在在大规模预训练模型的基础上，通过插入少量可训练的参数来实现特定任务的微调，而无需重新训练整个模型。Adapter 方法有效地降低了微调大模型的计算和存储成本，同时保留了预训练模型的能力和知识。

### 1. 背景
大模型（如 BERT、GPT 等）经过大量无监督数据的预训练，拥有广泛的语言理解或生成能力。传统的微调方式需要更新模型的全部参数，导致微调过程的存储和计算开销巨大。Adapter 技术应运而生，通过向模型中插入小规模的可训练参数模块，只微调这些模块，而冻结原有的大量预训练参数。

### 2. Adapter 的工作原理
Adapter 通常是在 Transformer 架构中的每个层之间插入的一些可训练层。一个典型的 Adapter 模块设计如下：
1.  **输入降维**：通过一个小型的全连接层将输入的特征维度从原始的高维度降低到一个更小的维度（通常称为 bottleneck dimension）。
2.  **非线性激活**：通常使用 ReLU 或 GELU 等激活函数对降维后的特征进行非线性变换。
3.  **输出升维**：通过另一个全连接层将降维后的特征重新映射回原始的高维空间。

**Adapter 数据流示意图：**
```text
Layer Input (x)
    |
    v
+-----------------------+
|   Original Transformer  |
|        Sub-Layer        | <--- 冻结参数
+-----------+-----------+
            |
            v
+-----------------------+
|    Adapter Module      |
|  +-----------------+   |
|  | Down (d -> r)   |   |
|  +--------+--------+   |
|           |            |
|  +--------v--------+   |
|  |  Non-Linearity  |   |
|  +--------+--------+   | <--- 可训练参数
|           |            |
|  +--------v--------+   |
|  |   Up (r -> d)   |   |
|  +--------+--------+   |
+-----------+-----------+
            |
            +-----> (+) <--- Residual Connection (Skip)
            |
            v
      Layer Output
```

**具体数学表示如下：**
给定输入特征 $h \in \mathbb{R}^d$，Adapter 模块的操作可以表示为:
$$z = \text{ReLU}(hW_{\text{down}})$$
$$h_{\text{adapter}} = zW_{\text{up}}$$
$$h_{\text{out}} = h_{\text{adapter}} + h$$
其中，$W_{\text{down}} \in \mathbb{R}^{d \times r}$ 和 $W_{\text{up}} \in \mathbb{R}^{r \times d}$ 是 Adapter 中可训练的参数，$r$ 是降维后的中间维度，通常取值远小于 $d$（例如 $d=768, r=64$），因此引入的参数量非常少。公式中最后加上 $h$ 表示残差连接，确保在 Adapter 训练初期模型至少能维持原有力能。

### 3. Adapter 的优势
*   **参数效率高**：Adapter 只微调少量新引入的参数，避免了对整个大模型的微调。例如，BERT-base 有 1.1 亿参数，而典型的 Adapter 模块可能只有几百万参数（约 3-4%）。
*   **任务间参数共享**：Adapter 使得不同任务的参数模块可以独立微调，但基础的预训练模型参数仍然保持共享。这种方法允许多个任务共享相同的大模型，只需存储各个任务的 Adapter 参数。
*   **训练稳定性**：由于 Adapter 只更新小部分参数，相比全参数微调，训练过程更为稳定，且不容易发生过拟合。

### 4. Adapter 的典型架构与变种
1.  **Houlsby Adapter**：这是最早的 Adapter 之一，设计在 Transformer 的每一层注意力块和前馈网络之间插入 Adapter 模块。每个 Adapter 包含两个线性投影层和一个非线性激活函数。
2.  **Parallel Adapter**：不同于将 Adapter 插入到主网络中（串行），平行结构的 Adapter 与 Transformer 层并行工作。这意味着输入 $x$ 同时进入原始层和 Adapter，最后将两者的输出相加。这种结构适合需要保持原始层计算完整性，同时快速适应新任务的场景。
3.  **Compacter**：使用了低秩参数分解技术（类似 LoRA 思想，即 $W = P \cdot \text{diag}(H) \cdot Q$）来减少 Adapter 模块中的参数量，进一步压缩模型。
4.  **LoRA**：与 Adapter 类似的思想，但 LoRA 直接在权重矩阵上进行旁路更新（$W = W_0 + BA$），推理时可无缝融合（$W' = W_0 + BA$ 为单个矩阵），无额外推理延迟。而 Adapter 通常需要额外的网络层数计算，推理会有额外延迟。

### 5. 应用场景
*   **自然语言处理 (NLP)**：任务包括情感分析、命名实体识别、机器翻译等。
*   **跨语言模型微调**：Adapter 被证明可以适应多语言环境中的任务（如为每种语言或语言对训练特定的 Adapter）。
*   **多任务学习**：Adapter 能够有效实现同一个大模型在多个任务之间的共享。

### 6. 总结
Adapter 通过引入少量额外参数以微调预训练大模型，极大降低了微调的计算和存储成本。它为大模型的部署和任务扩展提供了一种高效、灵活的解决方案。然而，相比 LoRA，其在推理阶段的额外计算开销是主要劣势。

## 常见考点
1. **Adapter vs LoRA 的区别**：核心区别在于 Adapter 增加了网络层数（推理有延迟），而 LoRA 是修改权重矩阵（推理可重参数化，无延迟）。Adapter 更适合对模型结构有特定隔离要求的场景。
2. **Adapter 的位置**：Houlsby 插在 Attention 之后、FFN 之前（或之后）。为什么这个位置？通常认为这是特征变换的关键位置。
3. **Bottleneck 的大小**：中间维度 $r$ 的选择对性能和参数量的平衡至关重要，通常设为原始维度的 1/10 或更小（如 64）。

## 记忆要点

- 轻量级微调技术，冻结原模型参数，仅插入少量可训练模块。
- 结构为降维-非线性-升维的瓶颈层，通常接残差连接。
- 参数量极少（约3-4%），适合多任务共享同一大模型。
- 缺点是推理有额外延迟，无法像LoRA那样合并权重。
