# 什么是LLM的幻觉(Hallucination)问题？有哪些缓解方案？

🎯 **本质**：幻觉是指LLM生成看似合理但事实上不正确的内容。分为事实性幻觉（与客观事实不符）和忠实性幻觉（与输入/上下文不符）。

📊 **幻觉产生原因**：
1. 训练数据包含错误信息、统计偏差、知识过时。
2. 自回归生成基于概率采样，可能偏离事实。
3. 长尾知识分布不均。
4. 高温度采样引入随机性。

**缓解方案**：

1. **RAG（检索增强生成）——最有效**
   - 提供外部知识源，模型基于检索到的真实文档生成答案。
   - 减少对模型内部记忆的依赖。

2. **对齐训练**
   - RLHF/DPO：训练模型在不确定时承认不知道。
   - Constitutional AI：模型自我审查输出。

3. **解码策略优化**
   - 降低temperature（事实性任务用0-0.3）。
   - Contrastive Search：对比搜索。
   - 对比不同层的输出（DoLa）。

4. **后处理验证**
   - Self-Check：模型自我检查事实准确性。
   - External Verification：用知识库验证关键事实。
   - FActScore：细粒度事实评估。

5. **架构改进**
   - 多模态输入辅助验证。
   - Tool Use：让模型调用搜索引擎/计算器。
   - 内置安全检查。

6. **Prompt工程**
   - 明确要求引用来源。
   - 要求标注不确定性。
   - 分步推理(CoT)减少跳步错误。

**RAG 原理架构图**：
```
┌──────────┐     Query     ┌───────────────┐
│  用户    │──────────────▶│  Query Embed  │
└────┬─────┘               └───────┬───────┘
     │                              │
     │                              ▼
     │                    ┌──────────────────┐
     │                    │ 向量数据库/检索器│◀─────┐ 知识库
     │                    └────────┬─────────┘      │
     │                             │               │
     │                             ▼               │
     │                    ┌──────────────────┐      │
     └────────────────────│ Prompt Augmented│──────┘
                          │  (上下文注入)    │
                          └────────┬─────────┘
                                   │
                                   ▼
                          ┌──────────────────┐
                          │     LLM 生成     │
                          │ (基于事实回答)   │
                          └──────────────────┘
```

💥 **实战案例**：在一个医疗问答RAG系统中，模型曾凭空捏造不存在的药物组合。引入**Self-RAG**机制后，模型会先评估检索文档的相关性，如果相关性低于阈值，会直接回答“基于提供的信息无法回答”，而非强行编造。

💻 **代码示例**：
```python
# 强制模型基于上下文回答的Prompt模板
prompt_template = """
You are a helpful assistant. 
Answer the question using ONLY the context below. 
If the answer is not in the context, say "I don't know".

Context:
{context}

Question: {question}
Answer:"""
# 关键点：明确的负向约束和Scope限定
```

## 常见考点
1. **RAG 的局限性**：如果检索到的文档本身是错误的怎么办？（答案：可以通过重排序或增加检索源的权威性权重来缓解）。
2. **Temperature 设置**：在需要极高准确性的场景下，Temperature 应该如何设置？（答案：接近 0，如 0.1 或 0，以减少随机性）。
3. **幻觉检测指标**：如何量化评估幻觉？（答案：使用 FactScore、TruthfulQA 等基准测试）。
4. **KL 散度惩罚**：在 RLHF 中，KL 散度如何帮助减少幻觉？（答案：防止模型在优化奖励时过度偏离原始策略，从而避免产生奇怪的内容）。
