# LLaMA、Qwen和DeepSeek系列的架构有什么主要区别

### 三大主流开源模型系列的架构对比

| 特性 | LLaMA-3 | Qwen-2.5 | DeepSeek-V2 |
|------|---------|----------|-------------|
| 注意力 | GQA (Grouped Query Attention) | GQA (Grouped Query Attention) | **MLA** (Multi-head Latent Attention) |
| FFN | SwiGLU | SwiGLU | **DeepMoE** (混合专家) |
| Norm | RMSNorm | RMSNorm | RMSNorm |
| Norm位置 | Pre-Norm | Pre-Norm | Pre-Norm |
| 位置编码 | RoPE | RoPE (支持长上下文) | **Decoupled RoPE** |
| MoE | 否 (Dense) | 否 (Dense) | 是 (细粒度 + 共享专家) |
| 词表 | 128K | 152K | 100K |
| 上下文 | 8K → 1M (推理扩展) | 128K (支持 32K/128K) | 128K |

### DeepSeek-V2 的独特设计 (极致推理性价比)

1.  **MLA (Multi-head Latent Attention) - 关键显存优化**：
    *   **原理**：传统的 Multi-Head Attention 中，KV Cache 显存占用随 Head 数量和 Hidden Size 线性增长。MLA 引入低秩分解，将 Key 和 Value 压缩到一个潜在的低维向量（Latent Vector）上。
    *   **效果**：KV Cache 显存减少至原来的 1/4 ~ 1/8，极大提升了推理吞吐量和长文本处理能力。
    *   **Decoupled RoPE**：为了配合 MLA 的压缩特性，将位置编码与内容向量解耦。RoPE 仅作用于未被压缩的部分，保留位置信息。

2.  **DeepSeekMoE (细粒度专家路由)**：
    *   **架构**：160 个专家（远超 Mixtral 的 8 个），分为 2 个共享专家 和 158 个路由专家。
    *   **策略**：每个 Token 激活 6 个路由专家 + 2 个共享专家。
    *   **优势**：细粒度分工更明确，知识密度更高；共享专家保留通用知识，防止路由坍塌。

3.  **Yarn (Yet Another RoPE extension)**：支持动态长上下文扩展。

### Qwen 系列的特色
*   **词表优化**：152K 词表，大幅提升中文和多语言编码效率，降低 Token 消耗。
*   **数据优势**：代码 和数学 数据配比高，逻辑推理能力强。
*   **统一多模态**：Qwen-VL 系列通过 C-Former 或类似的抽象层统一视觉和文本表征。

### LLaMA 系列的特色
*   **GQA (Grouped Query Attention)**：在 LLaMA 3 中全面采用 GQA，平衡了 MQA 的速度和 MHA 的质量。Key/Value Head 数量小于 Query Head，减少 KV Cache。
*   **极简主义**：架构最标准，研究生态最完善，社区适配度最高。

### 实战案例与代码

**实战案例**：在部署 70B 参数模型进行长文档摘要时，使用 MHA (如 LLaMA 2) 会导致单张 A100 显存迅速爆满（KV Cache 占用过大），只能支持极短的并发。切换至 DeepSeek-V2 (MLA 架构) 或 LLaMA 3 (GQA 架构) 后，**Batch Size 提升了 3 倍**，吞吐量显著提高，且无需复杂的 PagedAttention 优化即可支持 32K 上下文。

### 注意力机制对比 (MQA vs GQA vs MLA)

| 特性 | MHA (Multi-Head) | MQA (Multi-Query) | GQA (Grouped-Query) | MLA (Multi-head Latent) |
| :--- | :--- | :--- | :--- | :--- |
| **KV Cache 大小** | `Num_Heads × D` (最大) | `1 × D` (最小) | `Num_Groups × D` (中等) | `Rank × D` (极小, 压缩) |
| **推理速度** | 慢 (内存带宽瓶颈) | 最快 | 快 | 最快 (显存占用极低) |
| **模型质量** | 基准 | 下降 (表达能力受限) | 接近 MHA | 接近 MHA (低秩重构) |
| **代表模型** | LLaMA 1, GPT-3 | PaLM | LLaMA 3, Qwen2 | DeepSeek-V2/V3 |
