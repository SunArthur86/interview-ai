# 手撕 K-Means聚类

实现K-Means 聚类,要求包含Fit和Predict两个函数. 以下是优化后的版本，补充了空簇处理和向量化加速。

```python
import numpy as np

class KMeans:
    def __init__(self, k=3, max_iters=100, tol=1e-4):
        self.k = k
        self.max_iters = max_iters
        self.tol = tol
        self.centroids = None

    def fit(self, X):
        # 初始化质心：随机选择 k 个样本点
        indices = np.random.choice(X.shape[0], self.k, replace=False)
        self.centroids = X[indices]
        
        for _ in range(self.max_iters):
            # 1. 计算距离并分配簇 (向量化计算，避免循环)
            # distances shape: [m_samples, k]
            # 利用广播机制计算所有点到所有质心的欧氏距离平方
            distances = np.linalg.norm(X[:, np.newaxis] - self.centroids, axis=2)
            labels = np.argmin(distances, axis=1)
            
            # 2. 计算新的质心
            new_centroids = np.array([X[labels == i].mean(axis=0) for i in range(self.k)])
            
            # 3. 处理空簇：如果某个簇为空，随机重置该簇的质心（防止 Crash）
            for i in range(self.k):
                if np.isnan(new_centroids[i]).any():
                    new_centroids[i] = X[np.random.choice(X.shape[0])]
            
            # 4. 检查收敛
            if np.allclose(self.centroids, new_centroids, atol=self.tol):
                break
            self.centroids = new_centroids

    def predict(self, X):
        distances = np.linalg.norm(X[:, np.newaxis] - self.centroids, axis=2)
        return np.argmin(distances, axis=1)

# 示例使用
if __name__ == "__main__":
    from sklearn.datasets import make_blobs
    import matplotlib.pyplot as plt
    
    X, _ = make_blobs(n_samples=300, centers=4, cluster_std=0.60, random_state=0)
    kmeans = KMeans(k=4)
    kmeans.fit(X)
    y_pred = kmeans.predict(X)
    
    print("Centroids:", kmeans.centroids)
    plt.scatter(X[:, 0], X[:, 1], c=y_pred, cmap='viridis', marker='o', edgecolor='k')
    plt.scatter(kmeans.centroids[:, 0], kmeans.centroids[:, 1], c='red', s=200, alpha=0.75)
    plt.show()
```

### 实战案例
*   **K值选择（K=5陷阱）**：在用户画像分群中，业务方常指定将用户分为5群（VIP、高潜、流失等）。但数据本身可能只有3个自然聚类中心，强行K=5会导致K-Means将一个自然的簇硬生生切成两半，得出的业务结论不具备泛化性。通常需结合手肘法或Silhouette Coefficient确定K。
*   **离群点敏感**：如果数据集中存在极端离群点（如电商交易金额中的刷单异常值），K-Means的质心会被严重拉偏。实战中常先做3-sigma剔除或改用K-Medoids（基于中位数，抗噪性强）。

### 对比表格

| 维度 | 手撕基础版 (Loop) | 工业优化版 (Vectorized) | K-Medoids |
| :--- | :--- | :--- | :--- |
| **距离计算** | Python for循环 | NumPy广播 (C加速) | 距离矩阵 |
| **质心更新** | 均值 | 均值 | 中位数 |
| **抗噪能力** | 弱 | 弱 | 强 |
| **性能** | 慢 (O(N*k*Iter)) | 快 (并行化) | 较慢 |
| **空簇处理** | 未处理 | 随机重置/最远点采样 | - |
