# 手撕绝对位置编码

```python
import torch
import torch.nn as nn
import math

class PositionalEncoder(nn.Module):
    def __init__(self, d_model, max_seq_len=5000, dropout=0.1):
        super().__init__()
        self.dropout = nn.Dropout(p=dropout)
        
        # 1. 计算位置矩阵
        pe = torch.zeros(max_seq_len, d_model)
        # position: (max_seq_len, 1)
        position = torch.arange(0, max_seq_len, dtype=torch.float).unsqueeze(1)
        
        # 2. 计算频率项 div_term: (d_model / 2)
        # 10000^(2i/d_model) -> exp(log(10000) * 2i / d_model) 的倒数
        div_term = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model))
        
        # 3. 填充正弦和余弦
        # 偶数维度 (0, 2, 4...) 用 sin
        pe[:, 0::2] = torch.sin(position * div_term)
        # 奇数维度 (1, 3, 5...) 用 cos
        pe[:, 1::2] = torch.cos(position * div_term)
        
        # 4. 增加batch维度并注册为 buffer (不参与梯度更新，但随模型保存/移动)
        pe = pe.unsqueeze(0) # (1, max_seq_len, d_model)
        self.register_buffer('pe', pe)

    def forward(self, x):
        # x shape: (batch_size, seq_len, d_model)
        # 原答案中 `x = x + math.sqrt(self.d_model)` 是错误的，通常不需要额外缩放 Embedding，
        # 除非在初始化 Embedding 时权重范围较小，但标准 Transformer 论文中提到 PE 的方差要调整。
        # 这里保持标准写法，直接相加。
        
        # 截取与当前序列长度一致的位置编码
        x = x + self.pe[:, :x.size(1)]
        return self.dropout(x)
```

### 深化实战内容

**实战案例**：
在 Fine-tune BERT 处理超长文档（如 2k+ tokens）时，若直接截断会导致尾部关键信息丢失。一种实战技巧是使用“滑动窗口 + 绝对位置编码”，将文档切分为多段，输入模型前对每一段重置位置索引（0-511），虽然丢失了全局绝对位置，但保留了局部相对位置关系，往往比直接截断效果更好。

**技术对比：绝对位置编码 vs RoPE (旋转位置编码)**

| 特性 | Sinusoidal (Sin/Cos) | Learnable (可学习 Embedding) | RoPE (Llama/Qwen默认) |
| :--- | :--- | :--- | :--- |
| **外推性** | 较差 (需插值如 YARN) | 极差 (无法处理超过 max_len 的位置) | **较好** (利用旋转矩阵性质) |
| **位置表示** | 绝对位置相加 | 绝对位置相加 | 绝对位置注入到 Attention Score |
| **相对位置** | 理论上可通过线性变换捕获 | 难以显式捕获 | **天然具有相对位置感知** ($PE_{m+n}/PE_m$) |
| **参数量** | 0 | $L \times d_{model}$ (随长度增加) | 0 |
| **主要应用** | Transformer (Original) | GPT-2, BERT | LLaMA, Mistral, Qwen2, PaLM |

**## 常见考点**
*   **正弦余弦的选择**：为什么要用 Sin/Cos 而不是可学习的 Embedding？优势是什么？（外推性）
*   **奇偶维度**：为什么维度索引 $2i$ 用 Sin，$2i+1$ 用 Cos？这种交替排列有何数学含义？
*   **相对位置关系**：绝对位置编码 PE 如何通过线性变换来表达相对位置关系？

## 记忆要点

- 正弦余弦：偶数维Sin，奇数维Cos，交替编码位置信息
- 频率计算：div_term = exp(-log(10000) * 2i/d)，指数级衰减
- 外推性：固定频率无需学习，理论上可处理超长序列
- 相对位置：通过线性变换PE可表达相对位置关系
- Buffer注册：位置编码不参与梯度更新，需register_buffer
