# 什么是warmup_ratio?训练过程中怎么设置

### 1. 什么是 Warmup & warmup_ratio
**Warmup** 是指在训练初期使用较小的学习率，经过一定步数后线性（或非线性）增加到目标 LR 的策略。
**warmup_ratio** 指的是 Warmup 阶段占总训练步数的比例。

### 2. 为什么需要 Warmup？
*   **稳定性：** 训练初期模型参数随机初始化，若使用较大 LR，容易导致梯度爆炸或 Loss 震荡。
*   **适应优化器：** 对于 Adam 等自适应优化器，初期梯度方差大，Warmup 有助于优化器更准确地估计一阶、二阶矩（特别是 $v$ 项，即二阶矩估计）。
*   **LayerNorm/Layer Scale**：在深层网络中，初期输出幅度不稳定，Warmup 缓解了数值溢出风险。

### 3. warmup_ratio 设置建议
*   **预训练场景（1 Epoch）：** 建议设为 **0.01**（1%），即前 1% 步数用于升温。
*   **SFT 场景（多 Epoch）：** 可设为 **0.03 - 0.1**。因为 SFT 数据量小，训练步数相对较少，Warmup 占比可适当提高以确保平稳。
*   **高 LR 或大 Batch Size：** LR 越大、Batch 越大，通常需要更长的 Warmup（更大的 ratio）来缓冲。

### 4. 实战案例：大 Batch 训练崩坏
在使用 4k Batch Size 进行 MoE 模型预训练时，工程师照搬了 0.001 的 Warmup ratio，导致训练前 500 步 Loss 出现 `NaN`。分析发现大 Batch 导致初期梯度方差极大。将 warmup_ratio 调整至 0.05 后，梯度更新平滑，训练正常收敛。

### 5. 关键代码
```python
from torch.optim.lr_scheduler import LambdaLR

def get_warmup_scheduler(optimizer, warmup_steps, total_steps):
    def lr_lambda(step):
        if step < warmup_steps:
            return float(step) / float(max(1, warmup_steps))
        # 此后接余弦衰减或常数
        return 0.5 * (1.0 + math.cos(math.pi * (step - warmup_steps) / (total_steps - warmup_steps)))
    return LambdaLR(optimizer, lr_lambda)
```

### 6. 不同场景设置对比

| 场景 | 推荐比例 | 原因 |
| :--- | :--- | :--- |
| **从头预训练** | 0.01 - 0.05 | 数据量极大，初期不稳定，需较长缓冲但占比极小 |
| **指令微调 (SFT)** | 0.03 - 0.1 | 数据量少，总步数少，需占比稍大以快速稳定 | 
| **LoRA 微调** | 0.05 - 0.1 | 参数量少，易受初始梯度冲击，需更充分预热 |
| **Resume 训练** | 0 或极小 | 模型已处于收敛轨道，无需重新预热 |

### 8. 常见考点
1.  **Warmup 类型**：除了 Linear Warmup，还有哪些常见的 Warmup 策略？（Constant Warmup（先保持小 LR 不变再升），Cosine Warmup（直接接在 Cosine Decay 前面））。
2.  **重启训练**：如果从 checkpoint 恢复训练，需要重新做 Warmup 吗？（通常不需要，可以直接从当前 LR 或从较小值开始，避免已经收敛的模型因为 LR 突变而发散）。
3.  **微调时的 Warmup**：在全参数微调时，Warmup ratio 应该比预训练大还是小？（通常设置较小或设为 0，因为模型已经有较好的参数初始化，过长的 Warmup 会浪费计算资源）。
