# 解释一下AdaLora和QLora的原理

这两种技术分别解决了不同阶段的优化问题，分别在模型微调和部署上表现出色。下面将详细介绍它们的原理。

### 1. AdaLoRA 原理

**原理简介：** AdaLoRA (Adaptive Low-Rank Adaptation) 通过在微调模型时将权重矩阵分解为低秩矩阵，并且引入了自适应调整低秩 的机制。在此过程中，模型在不同层可以根据梯度信息自适应地调整分配给每层的秩值。这个机制通过动态调整秩值，在保持模型性能的同时有效减少了计算和内存需求。

**矩阵分解：** 给定原始模型权重矩阵 $W_0$，通过 LoRA，我们将其分解为两个低秩矩阵 $A \in \mathbb{R}^{m \times r}$ 和 $B \in \mathbb{R}^{n \times r}$：
$$W \approx W_0 + \Delta W = W_0 + AB^T$$
其中 $r$ 是低秩值。

AdaLoRA 引入的改进是通过自适应调整 $r$ 来使得不同层的权重矩阵分解具有不同的秩值。具体来说，使用重要性评分（如基于梯度的 Frobenius 范数）来动态调整：
- 对于重要性高的层，分配更高的秩值，保留更多信息。
- 对于重要性低的层，降低秩值，节省计算资源。

**关键细节：** AdaLoRA 并不是独立调整 A 和 B 的秩，而是通过奇异值分解（SVD）将增量矩阵 $\Delta W$ 分解为 $P \Lambda Q^T$，其中 $\Lambda$ 是对角矩阵包含奇异值。通过裁剪不重要的奇异值（即令其对角线元素为 0）来实现秩的动态调整。这意味着每层的秩是变化的，且在训练过程中可以增减。

**用途和优势：**
- **用途：** AdaLoRA 主要用于模型微调，适合针对特定任务进行细化微调。
- **优势：** 动态调整秩值减少了资源浪费，同时保留了模型的强大性能。

### 2. QLoRA 原理

**原理简介：** QLoRA (Quantized Low-Rank Adaptation) 结合了量化和 LoRA 的优势。它通过对模型进行 4-bit 量化，将模型权重的精度降低，从而显著减少计算和存储需求。随后，QLoRA 在量化后的模型上应用 LoRA 微调技术，保持低秩矩阵分解的效率和适应性。

**关键技术：**
1. **4-bit 量化 (NF4)**: 使用 NormalFloat4 (NF4) 数据类型，这是一种专为正态分布权重优化的数据类型，相比标准 4-bit 量化（如 Integer 4-bit）能更好地保留模型精度。它通过将权重分布归一化到标准正态分布，然后等间距量化来实现。
2. **双重量化**: 对量化常数本身进行再次量化。通常量化常数是 FP32，QLoRA 将其进一步量化为 FP8，平均每个参数可节省约 0.37 bit 的显存，这对于 65B 模型来说节省量非常可观。
3. **分页优化器**: 利用 NVIDIA 统一内存特性，将优化器状态从 GPU 显存转移到 CPU RAM 中，并在需要时自动分页传输。这有效解决了梯度更新过程中显存峰值导致的 OOM（Out of Memory）问题。

**训练流程图：**
```text
+-------------------+
|    Pre-trained    |
|     Weights       |
|     (FP16)        |
+---------+---------+
          | 1. 4-bit Quantization (NF4)
          v
+-------------------+
| Quantized Weights | <--- Dequantize (FP16/BF16) during Forward
|      (4-bit)      |          Only for current block
+---------+---------+
          | 2. Inject LoRA Adapters (FP16)
          v
+-------------------+
|   Active Model    |
| (Base + LoRA)     |
+---------+---------+
          | 3. Forward Pass & Compute Loss
          v
+-------------------+
|   Backward Pass   |
| (Update LoRA only)|
+-------------------+
```

**用途和优势：**
- **用途：** QLoRA 主要用于模型部署。特别是在资源有限的硬件环境下，它能够将大模型在 4-bit 精度下运行。
- **优势：** 通过量化技术大幅降低内存占用，使得单张 GPU 上微调 65B 模型成为可能。

### 3. 用途上的区别
*   **AdaLoRA 用于模型微调：** 侧重于在训练阶段动态调整模型结构，提升微调效率。
*   **QLoRA 用于模型部署：** 侧重于在资源受限的环境中进行低资源部署和微调，通过极致的压缩降低硬件门槛。

### 4. 总结
*   **技术核心不同：** AdaLoRA 是通过动态低秩分解来提高微调效率；QLoRA 则结合了 4-bit 量化和低秩微调，降低部署成本。
*   **解决问题不同：** AdaLoRA 解决微调效率和性能平衡的问题；QLoRA 解决显存不足导致无法部署/微调大模型的问题。

## 常见考点
1. **QLoRA 的反量化时机**：QLoRA 并不是全程将模型反量化回 FP16，而是仅在当前计算 Block 进行前向传播时，实时将 4-bit 权重反量化为 FP16 (BF16) 计算，计算完即可释放，这样极大节省了显存。
2. **AdaLoRA 的秩调整策略**：面试官可能会问如何决定某层的重要性高/低。答案是基于梯度信息（如 Sensitivity）或者增量矩阵的奇异值大小来裁剪。
3. **NF4 vs Float4**：为什么不用普通的 Float4？因为普通 Float4 的动态范围太小，而 LLM 权重通常服从正态分布，NF4 通过分位数量化更好地适配了这种分布。
