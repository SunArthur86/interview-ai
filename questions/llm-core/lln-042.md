# 讲讲你了解的scaling law有哪些

🌟目前关于LLM的Scaling Law研究主要集中在模型性能与规模(参数、数据、计算量)之间的关系、涌现能力的发现、推理 优化以及实际应用中的挑战等方面.

### 一、 基础Scaling Law研究 [Kaplan et al., Chinchilla]
**1. Kaplan等人的经典理论**
- **幂律关系**：2020年提出的Scaling Law指出,模型性能(如测试损失)随参数数量(N)、数据集大小(D)和计算量(C)的指数增长呈线性提升。
- **协同扩展**：三者需协同扩展以避免瓶颈。例如，模型参数增加约8倍时，训练数据需增加约5倍以维持性能提升。

**2. Chinchilla Scaling Laws (Hoffmann et al.)**
- **计算最优**：纠正了Kaplan关于“参数优先”的观点，指出对于固定的计算预算，模型大小和数据量应当按比例同步增长（约 1:1）。
- **推论**：以前的模型（如GPT-3）通常训练不足（Under-trained），即参数量过大而数据量不足。

```text
性能 (Loss)
  ^
  |            _______ (Chinchilla Optimal)
  |           /
  |          /
  |         /
  |________/__________ (Kaplan - Data Heavy)
  |       /
  |      / 
  |     /   (Kaplan - Param Heavy, Under-trained)
  |------------------------------------> Compute
```

### 二、 推理 Scaling Law (Inference-time Compute)
**核心思想**：在模型参数固定的情况下，增加推理时的计算量（计算更多次、搜索更广）来提升性能。

**1. 策略分类**
- **Best-of-N (BoN)**：采样 N 个答案，选择奖励最高的。问题在于成本随 N 线性增长。
- **Beam Search / Tree Search**：维护多个候选路径。
- **Multi-step Verification (如 Process Reward Model)**：在推理步骤中引入验证器进行剪枝。

**2. 关键发现**
- **缩放法则**：推理计算收益遵循幂律分布。在数据较难时，增加测试时计算比增加训练时参数更有效（OpenAI o1 的核心逻辑）。

### 三、 微调 Scaling Law
- **幂律联合缩放**：微调性能与微调数据量、模型大小呈幂律关系。
- **PEFT vs FMT**：
  - 数据少：PEFT (LoRA) 更优，泛化强，不易遗忘。
  - 数据多：FMT (Full Fine-tuning) 上限更高。

**实战案例**：
在进行 7B 到 70B 模型的资源规划时，发现简单的 Kaplan 定律严重高估了数据需求。依据 Chinchilla 定律重新调整 Token 数量后，训练出的模型在同等算力下 Loss 降低了 15%，且避免了欠训练导致的泛化差问题。

**Scaling Laws 对比表**：

| 维度 | Kaplan Laws (2020) | Chinchilla Laws (2022) | 推理 Scaling (OpenAI o1) |
| :--- | :--- | :--- | :--- |
| **核心观点** | 参数优先 | 数据与参数 1:1 同步增长 | 推理时计算可替代训练规模 |
| **最优配比** | 约 1.5:1 (参数:Token) | 1:1 (参数:Token) | 依赖任务难度 (难任务重推理) |
| **训练状态** | 倾向于 Under-trained | Compute Optimal (充分训练) | 训练后强化 (Post-training) |
| **适用场景** | 算力受限需快速迭代 | 追求最佳性能效率 | 复杂推理任务 (数学/代码) |

## 常见考点
1.  **Chinchilla Laws 与 Kaplan Laws 的区别**：重点在于“数据与参数的配比”以及“训练是否充分”。
2.  **推理计算的权衡**：什么情况下应该增加推理算力（如复杂数学任务），什么情况下应该增加模型参数（如知识密集型任务）？
3.  **涌现能力**：是否真的存在“相变”，还是仅仅是评估指标的平滑变化？

## 记忆要点

- 经典Kaplan定律：主张参数优先，模型参数、数据量、计算量遵循幂律提升关系。
- Chinchilla定律：主张计算最优，纠正前者的重参数轻数据，强调参数与数据量应1:1同步增长。
- 推理期缩放：参数固定下，增加推理算力(如o1模型的多路搜索验证)可显著提升复杂任务性能。
