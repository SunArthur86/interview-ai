# 讲讲attention的各个变体

Attention 机制的变体主要围绕计算效率、容量和长程依赖的捕捉能力演进。

1.  **加性Attention**：
    *   *来源*：Bahdanau et al. (2014)。
    *   *计算*：$score(h_t, \bar{h}_s) = v^T \tanh(W_1 h_t + W_2 \bar{h}_s)$。
    *   *特点*：引入了可学习参数 $v$，适用于 Query 和 Key 维度不同的情况。计算复杂度较高，未利用矩阵乘法加速。

2.  **点积Attention**：
    *   *来源*：Luong et al. (2015)。
    *   *计算*：$score(h_t, \bar{h}_s) = h_t^T \bar{h}_s$。
    *   *特点*：计算速度快，但在维度较高时，点积数值过大，导致 Softmax 梯度极小（进入饱和区）。

3.  **缩放点积Attention**：
    *   *来源*：Vaswani et al. (Transformer, 2017)。
    *   *计算*：$Attention(Q, K, V) = softmax(\frac{QK^T}{\sqrt{d_k}})V$。
    *   *原理*：除以 $\sqrt{d_k}$（$d_k$为向量维度）将方差拉回 1，解决梯度消失问题，稳定训练。

4.  **多头注意力**：
    *   *机制*：将 $Q, K, V$ 线性投影到 $h$ 个不同的子空间，分别计算 Attention 后拼接再线性输出。
    *   *优势*：允许模型在不同的表示子空间中关注不同位置的信息（例如一个头关注语法，一个头关注语义）。

5.  **其他重要变体**：
    *   **Sparse Attention (如 Longformer, BigBird)**：为降低 $O(N^2)$ 复杂度，只关注局部窗口 + 少量全局 Token，将复杂度降至 $O(N)$。
    *   **FlashAttention (v1/v2/v3)**：**工程优化**。不计算显存中的巨大 Attention Matrix，通过 Tiling 和 IO 感知计算，在 GPU HBM/SRAM 间高效流转数据，大幅提升速度并降低显存占用。

**实战案例**：
在做长文本摘要（如100k上下文）时，标准Transformer的Attention矩阵会导致显存溢出（OOM）。工程上我们常使用Longformer或Sliding Window Attention，将Self-Attention限制在局部窗口内，仅对CLS等特殊Token做全局Attention。在实现层面，使用FlashAttention-2内核替换标准的PyTorch `nn.MultiheadAttention`，通常能在A100上将训练吞吐量提升2-3倍，并支持更长的Batch Size。

**变体对比表格**：
| 特性 | Additive Attention | Dot-Product | Scaled Dot-Product | Multi-Head | FlashAttention |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **计算公式** | $v^T \tanh(W_1h + W_2\bar{h})$ | $q \cdot k$ | $q \cdot k / \sqrt{d}$ | $Concat(head_1...head_h)$ | Softmax(qk/\sqrt{d})v (Tiled) |
| **计算复杂度** | $O(T^2 \cdot D)$ | $O(T^2)$ | $O(T^2)$ | $O(T^2 \cdot h)$ | $O(T^2)$ (更快常数) |
| **显存占用** | $O(T^2)$ | $O(T^2)$ | $O(T^2)$ | $O(T^2 \cdot h)$ | $O(T)$ (不存Atten Map) |
| **主要优势** | 适应不同维度 | 速度快 (矩阵运算) | 解决高维梯度消失 | 多子空间表达 | IO感知，极致速度快 |
| **适用场景** | RNN时代/Seq2Seq | 早期Seq2Seq | Transformer架构 | 现代LLM主流 | 所有高性能LLM训练/推理 |

**代码示例 (Scaled Dot-Product)**：
```python
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(query, key, value, mask=None):
    d_k = query.size(-1)
    # Scores: [batch, heads, seq_len, seq_len]
    scores = torch.matmul(query, key.transpose(-2, -1)) / math.sqrt(d_k)
    
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)
    
    attention_weights = F.softmax(scores, dim=-1)
    return torch.matmul(attention_weights, value), attention_weights
```

**常见考点**：
1. 为什么要除以 $\sqrt{d_k}$？（为了解决维度过大导致点积数值过大，Softmax 梯度消失的问题）
2. Multi-head Attention 中头数 $h$ 的设置对性能有何影响？（头数过多导致每头维度变小，可能削弱表达能力，通常 $d_k = d_{model} / h$ 不能太小）
3. FlashAttention 的核心优化点是什么？（减少 HBM 读写的次数，即 IO 感知）

## 记忆要点

- 公式核心：因为高维点积数值过大会导致梯度消失，所以要除以根号dk
- 多头机制：将QKV映射到不同子空间并行计算再拼接，以此捕捉多维语义特征
- 长文本优化：Sparse Attention用局部窗口加全局Token，将复杂度降至线性
- 工程加速：FlashAttention是IO感知优化，通过分块计算避开了显存中巨型矩阵
