# 如何提升合成数据的多样性

提升合成数据多样性的核心在于打破模型生成的模式化倾向。仅依靠简单提示往往导致数据分布狭窄。除了明确维度外，还可以通过以下增强策略实现：

1.  **维度解耦与枚举**：
    *   不仅询问ChatGPT维度，而是强制要求模型构建「本体树」，确保覆盖语义空间的正交维度（如：领域、意图、实体类型、情感、句式结构）。
    *   对每个维度的值集合进行笛卡尔积组合，避免仅生成常见的强关联组合。

2.  **多阶段生成流程**：
    *   **Phase 1: 元数据生成**：先生成JSON格式的元数据，包含所有约束维度。
    *   **Phase 2: 条件生成**：基于随机采样的元数据生成具体文本，强制模型适应罕见组合。
    *   **Phase 3: 逆向变异**：对已有样本进行改写、回译或风格迁移，增加样本的鲁棒性。

3.  **多样化解码策略**：
    *   在生成阶段使用高Temperature或Top-p采样，鼓励探索低概率区域。
    *   引入Simulated Annealing（模拟退火）机制，初期注重多样性，后期注重质量。

4.  **自洽性与去重**：
    *   使用MinHash或LSH（局部敏感哈希）进行海量去重。
    *   训练轻量级判别器过滤语义重复但表达不同的样本。

**实战案例**：
在构建行业知识库问答对时，直接让GPT-4生成“用户提问”往往集中在“How to reset password”等Top 20高频问题上。实战中，我们引入了“复杂度螺旋”策略，先让模型生成“用户画像”（如：暴躁的运维、新入职的实习生），再结合“罕见场景”（如：断网时的证书配置）组合生成指令，使合成数据的指令长尾覆盖率从30%提升至85%。

**代码示例 (Python/Self-Instruct)**：
```python
import json
import random

# 维度解耦生成元数据
def generate_metadata_prompt(domain):
    return f"""
    Domain: {domain}. 
    Generate 10 orthogonal attributes (e.g., intent, tone, complexity) 
    and 5 unique values for each in JSON format.
    """

# 模拟Phase 1 & 2 的采样逻辑
def sample_pair(metadata):
    instruction = f"Write a {random.choice(metadata['tone'])} {random.choice(metadata['intent'])} about {random.choice(metadata['topic'])}."
    constraint = f"Constraint: Must be less than {random.choice(metadata['length'])} words."
    return {"instruction": instruction, "constraint": constraint}
```

**常见考点**：
1. 如何保证合成数据的质量？（引入强模型作为判别器，或基于Self-Instruct的过滤机制）
2. 当数据维度过多时如何组合？（采用分层采样或优先采样新颖组合）
3. 什么是「Catastrophic Forgetting」在合成数据中的体现？（合成数据分布若覆盖不全，模型在真实分布长尾上性能下降）

## 记忆要点

- 核心痛点：简单提示会导致数据模式化，因此需维度解耦构建本体树做笛卡尔积
- 生成策略：采用多阶段生成（元数据采样->条件生成->逆向变异），强制覆盖罕见组合
- 解码参数：因为要打破高频词，所以用高Temperature或Top-p采样探索低概率区域
- 质量控制：用MinHash/LSH海量去重，因为合成数据极易出现语义重复陷阱
