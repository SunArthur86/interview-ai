# LoRA的变体有哪些?DoRA、AdaLoRA、LoRA+各自有什么改进

LoRA变体通过不同角度改进原始LoRA的局限性.

- **主要变体对比:**

| 变体 | 核心改进 | 解决的问题 |
|------|---------|-----------|
| AdaLoRA | 动态分配rank | 固定rank不合理 |
| DoRA | 分解为大小和方向 | LoRA缺少权重幅值调整 |
| LoRA+ | A和B使用不同学习率 | A/B学习不均衡 |
| GaLore | 梯度低秩投影,全参数训练 | LoRA只能训练少量参数 |
| LongLoRA | Shifted Sparse Attention | 原始LoRA不支持长上下文 |

- **DoRA详解(最值得关注):**
- LoRA: W = W₀ + BA
- DoRA: W = m · (W₀ + BA) / ||W₀ + BA||
- 将权重分解为大小和方向
- 更接近全参数微调的更新模式
- 效果显著优于标准LoRA,尤其在低rank时

- **AdaLoRA详解:**
- 基于重要性分数动态调整每层的rank
- 使用SVD分解替代随机初始化
- 训练过程中裁剪不重要的奇异值,分配给重要层

```text
      Original LoRA                  DoRA (Weight-Decomposed)
┌──────────────────────┐      ┌──────────────────────────────────────┐
│        Input         │      │               Input                 │
└──────────┬───────────┘      └──────────────┬───────────────────────┘
           │                                 │
           ▼                                 ▼
┌──────────────────────┐      ┌──────────────────────────────────────┐
│       Layer (W₀)     │      │           Direction (d)              │
│     (Frozen)         │      │  d = (W₀ + △W) / ||W₀ + △W||         │
└──────────┬───────────┘      └──────────────┬───────────────────────┘
           │                                 │
           │            ┌────────────────────┴────────────────────┐  
           │            │   Magnitude (m) <- Learnable Scalar   │  
           │            │   (New Component in DoRA)             │  
           │            └────────────────────┬────────────────────┘  
           │                                 │
           ▼                                 ▼
┌──────────────────────┐      ┌──────────────────────────────────────┐
│   W = W₀ + B·A       │      │        Output = m · d                │
│   (Additive Update)  │      │      (Magnitude * Direction)        │
└──────────────────────┘      └──────────────────────────────────────┘
```

- **LoRA+详解:**
- **核心痛点**: LoRA中A矩阵通常随机高斯初始化，B矩阵初始化为0。由于初始梯度主要由B承担，A和B若使用相同学习率会导致B更新过快而A滞后，导致次优收敛。
- **改进方案**: 对矩阵A使用相对较高的学习率，B使用较低的学习率（通常 A 的学习率是 B 的 2 倍），并重新缩放初始化方差。

**【实战案例】**
在一次医疗垂类微调中，发现标准LoRA在指令遵循能力上提升明显，但医学知识准确率下降。切换到 **DoRA** 后，通过保留幅值信息，模型在维持指令跟随的同时，医学实体识别的F1 score提升了约 5%，解决了LoRA在知识密集型任务上的遗忘问题。

**【代码示例】(Python/PEFT)**
```python
from peft import get_peft_model, LoraConfig, TaskType

# LoRA+ 配置核心：通过 modules_to_save 或自定义优化器组实现不同LR
config = LoraConfig(
    task_type=TaskType.CAUSAL_LM, 
    r=16, 
    lora_alpha=32, 
    target_modules=["q_proj", "v_proj"]
)
model = get_peft_model(base_model, config)

# 实践中手动设置 LoRA+ 的学习率差异
optimizer = torch.optim.AdamW([
    {'params': model.model.parameters(), 'lr': 1e-4}, # A矩阵部分
    {'params': [p for n, p in model.named_parameters() if 'lora_B' in n], 'lr': 2e-4} # B矩阵部分
])
```

## 记忆要点

- DoRA将权重分解为幅值m和方向d，解决LoRA缺少幅值调整的问题
- AdaLoRA动态分配rank，重要层rank高；LoRA+对A/B矩阵设不同学习率
- DoRA在低rank时效果显著优于LoRA，更接近全参数微调
