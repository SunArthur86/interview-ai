# 手撕flash-attention

FlashAttention 是一种通过分块计算和在线更新优化注意力机制的算法，减少内存访问开销并加速计算，特别适合长序列任务。核心步骤包括：1）将 Q、K、V 分块处理；2）在线计算 softmax（逐步更新最大值和指数和）；3）逐块累加输出。

### 核心原理：Tiling 与 Online Softmax
标准 Attention 计算 $O = \text{softmax}(QK^T)V$ 的时间瓶颈在于从 HBM (High Bandwidth Memory) 读写巨大的 $N \times N$ 注意力矩阵。FlashAttention 通过分块将计算局限在 SRAM (Static RAM) 中进行，极大减少了 HBM 访问次数。

为了在分块中计算 Softmax，必须使用 **Online Softmax** 算法，即在不看到所有数据的情况下，分步更新 Softmax 的分子和分母。

### 算法流程图

```text
Input: Q, K, V (分块加载到 SRAM)

Initialize: O = 0, m = -∞, l = 1 (用于存储分块累积的输出、最大值、归一化因子)

Loop over K, V blocks (inner loop):
  ┌─────────────────────────────────────────────┐
  │  Load block of Q (fix in SRAM)              │
  │  Loop over blocks of K, V (traverse cols):  │
  │    1. Load block K_j, V_j from HBM -> SRAM  │
  │    2. Compute S_ij = Q_i * K_j^T (dot prod) │
  │    3. Compute P_ij = softmax(S_ij)          │
  │       (使用 Online Softmax 更新 m, l)       │
  │    4. Compute O_i = O_i + P_ij * V_j        │
  │    5. Discard S_ij, P_ij (keep in SRAM)     │
  │    6. Write O_i back to HBM (if needed)     │
  └─────────────────────────────────────────────┘

Output: O (Attention Result)
```

### 实战案例
在实际工程（如 Llama 推理）中，显存带宽往往比计算能力更是瓶颈。FlashAttention 通过减少 HBM 访问（IO bound 级优化），使得长文本训练（如 32k context）成为可能。但需注意，FlashAttention 对序列长度有一定的对齐要求（通常需 padding 到 block size 的倍数），且在极短序列（<512）下，由于 Kernel 启动开销，速度可能不如未优化的 Attention。

### 代码示例：Flash Attention 核心 (Triton Pseudo-code)
注意：生产环境通常使用 CUDA C++ 或 Triton 编写。以下展示关键的 Online Softmax 更新逻辑。

```python
import torch

def flash_attention_forward(q, k, v, block_size):
    # 输入形状: (Batch, Heads, SeqLen, HeadDim)
    # 仅演示核心计算逻辑，省略精确的 SRAM/HBM 模拟
    B, H, N, D = q.shape
    M = k.shape[2]
    
    O = torch.zeros_like(q)
    l = torch.zeros(B, H, N, 1) # 归一化因子 sum(exp)
    m = torch.full((B, H, N, 1), -float('inf')) # 最大值

    # 分块循环
    for start_j in range(0, M, block_size):
        end_j = min(start_j + block_size, M)
        k_block = k[:, :, start_j:end_j, :] # (B, H, block, D)
        v_block = v[:, :, start_j:end_j, :]
        
        # 计算 QK^T
        # q: (B, H, N, D) @ k_block.T: (B, H, D, block) -> (B, H, N, block)
        s_block = torch.matmul(q, k_block.transpose(-2, -1)) * (1.0 / (D ** 0.5))
        
        # --- Online Softmax Update ---
        # 新块的最大值
        m_block = s_block.max(dim=-1, keepdim=True)[0]
        # 全局最大值更新
        m_new = torch.maximum(m, m_block)
        # 计算指数和修正因子
        p = torch.exp(s_block - m_new)
        l_new = torch.exp(m - m_new) * l + p.sum(dim=-1, keepdim=True)
        
        # --- Output Update ---
        # O = (l * O + exp(s - m_new) * V) / l_new
        # 这里需要简化的数学推导实现，核心是加权累加
        O_scaled = torch.exp(m - m_new) * O
        v_contrib = torch.matmul(p, v_block) # (B, H, N, D)
        O = (O_scaled + v_contrib) / l_new
        
        # 更新状态
        m, l = m_new, l_new
        
    return O
```

## 记忆要点

- 核心优化：分块计算减少HBM访问，IO感知而非计算感知
- Online Softmax：分步更新最大值m和指数和l，无需全量Score矩阵
- Tiling技术：Q、K、V分块加载到SRAM，避免显存瓶颈
- 计算流程：Q固定，遍历K、V块，累加输出O并更新统计量
- 适用场景：长序列训练加速，解决显存带宽瓶颈
