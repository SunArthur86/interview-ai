# 什么是warmup_ratio

### 什么是 Warmup Ratio？

#### 1. 定义
**Warmup Ratio**（热身比例）是指在训练总步数中，用于学习率预热阶段的比例。

例如：`warmup_ratio = 0.01` 表示整个训练过程的前 1% 的步数里，学习率会从一个极小值线性增长到设定的最大学习率。

#### 2. 为什么需要 Warmup？

- **稳定性**：训练初期模型参数随机初始化（或微调时参数刚加载），梯度往往不稳定。如果直接使用大学习率，容易导致模型参数剧烈震荡、Loss 爆炸或梯度消失。
- **平滑收敛**：先让模型用小步长找到一个相对合理的优化方向，再加速收敛，通常能获得更好的最终效果。

#### 3. 设置建议

- **训练时长**：
  - **1 Epoch（如大规模预训练）**：建议 `0.01`。
  - **3 Epochs（如 SFT）**：建议 `0.03`。

- **学习率大小**：
  - 若计划使用较大的峰值学习率，建议适当增大 `warmup_ratio`（如 0.05~0.1），给模型更长的缓冲期。
  - 若学习率本身较小，`warmup_ratio` 可相应调低。

- **Batch Size**：
  - 当 Batch Size 很大时，梯度统计更稳定但也可能累积过快，适当增加 Warmup 步数有助于平稳起步。

#### 4. 调度策略
通常采用**线性 Warmup**：在 Warmup 阶段，LR 从 0（或很小值）线性增加到 `max_lr`。Warmup 结束后，配合 Cosine Decay 或 Linear Decay 进行衰减。

#### 5. 学习率变化示意图

```text
Learning Rate
  ^
  |          /\¯¯¯¯¯¯\ (Cosine Decay after warmup)
  |         /  \      \
  |        /    \       \
  |       /      \        \
  |      /        \         \
  |_____/          \__________\____
  |    Warmup       Training    End
  +----------------------------------> Step
  (0)          (warmup_steps)
```

#### 6. 实战补充

**实战案例**：在使用 PEFT（如 LoRA）微调 Llama 3 时，如果 `warmup_ratio` 设置过高（如 0.2），模型在训练初期几乎只学到了 LoRA 适配器的初始化偏差，导致后期收敛变慢甚至陷入次优解；对于 QLoRA（4bit 量化），建议 `warmup_ratio` 设为 0.03~0.05 以适应量化噪声。

**代码示例**：
```python
from transformers import get_linear_schedule_with_warmup

# 计算总步数
num_training_steps = len(train_dataloader) * num_epochs
# 计算 warmup 步数
warmup_steps = int(num_training_steps * warmup_ratio)

scheduler = get_linear_schedule_with_warmup(
    optimizer, num_warmup_steps=warmup_steps, num_training_steps=num_training_steps
)
```

## 常见考点
1.  **Warmup Steps 和 Warmup Ratio 如何换算？代码实现中要注意什么？**
    *   **回答思路**：公式为 `warmup_steps = max_steps * warmup_ratio`。在代码实现（如 HuggingFace Trainer）中，如果 `max_steps` 未指定而是指定了 `num_train_epochs`，需要先根据数据集大小和 Batch Size 算出总步数，再计算 warmup_steps，否则会报错或比例错误。
2.  **除了 Linear Warmup，还有其他的 Warmup 形式吗？**
    *   **回答思路**：有，如 **Constant Warmup**（先保持一个小常数不变，再突变到最大值，较少用）、**Cosine Warmup**（按 Cosine 曲线上升）等，但**线性增长**是最通用且效果最稳定的默认选择。
3.  **在增量预训练中，Warmup Ratio 应该比从头训练大还是小？**
    *   **回答思路**：通常建议**稍大一些**或至少保持一致。因为模型参数已经处于一个极小值区域，直接开始训练梯度可能较大（分布差异导致），更长的 Warmup 有助于平滑地从旧知识区域过渡到新知识优化路径。
