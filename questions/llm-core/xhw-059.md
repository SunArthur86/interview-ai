# 不同Attention之间的区别

### 不同 Attention 之间的区别

#### 1. Self-Attention vs. Cross-Attention
- **Self-Attention**：Q、K、V 均来自**同一个序列**。用于捕捉序列内部的依赖关系（如句子中词与词的关系）。
- **Cross-Attention**：Q 来自**一个序列**，K、V 来自**另一个序列**。用于融合两个不同来源的信息（如 Transformer 的 Decoder 查询 Encoder 的结果）。

**计算细节**：
Self-Attention 计算公式：$Attention(Q,K,V) = softmax(\frac{QK^T}{\sqrt{d_k}})V$
Cross-Attention 通常是将 Decoder 的输出作为 $Q$，Encoder 的输出作为 $K, V$，实现源序列到目标序列的信息对齐。

```text
Self-Attention (Seq -> Seq):
Input X -> [Q, K, V] -> Attention -> Output

Cross-Attention (Seq1 x Seq2 -> Seq1):
Query (Seq1)     +     Key/Value (Seq2)
       |                      |
       +---- Attention Output ----+
                  |
             Output (Seq1)
```

#### 2. Sparse Attention (稀疏注意力)
- **问题**：标准 Attention 计算复杂度是 $O(n^2)$，序列长时计算慢。
- **解决**：通过限制每个 Token 只关注部分 Token（如附近的局部窗口、固定的随机模式），将复杂度降低到接近 $O(n)$。
- **代表**：Longformer, BigBird。
- **模式**：
  - **Window Attention**：只关注窗口内的邻居（$w$）。
  - **Dilated Sliding Window**：带空洞的滑动窗口，增大感受野。
  - **Global Attention**：特定 Token（如 [CLS]）关注所有 Token，反之亦然。

#### 3. Linear Attention (线性注意力)
- **思路**：利用核技巧或矩阵分解，将 Attention 的计算顺序重排，避免计算巨大的 $N \times N$ 矩阵，将复杂度降为 $O(n)$。
- **原理**：利用结合律 $(QK^T)V \rightarrow Q(K^T V)$。
  - 原始 Softmax 不支持直接分解，Linear Attention 通常使用 $\text{sim}(Q,K) = \phi(Q)\phi(K)^T$，其中 $\phi$ 是核映射（如 ELU+1）。
  - 计算变为 $S(K^T V)$，先计算 $K^T V$ （复杂度 $n \times d^2$ 或 $d \times n \times d$，取决于实现），再与 $Q$ 相乘。
- **代表**：Performer, Linformer。

#### 4. Multi-head / Multi-query / Grouped-query Attention
- **Multi-head (MHA)**：每个头有独立的 Q、K、V 投影。表达能力最强，但计算和存储开销大（KV Cache 最大）。
- **Multi-query (MQA)**：所有头共享同一组 K、V（只有 Q 是分头的）。极大减少 KV Cache，提升推理速度，但可能损失部分精度。
- **Grouped-query (GQA)**：折中方案。将头分组，每组内共享 K、V。平衡了 MHA 的表达能力和 MQA 的效率。

**架构对比图**：
```text
Head Count: 4 (q1-q4)

MHA (KV Cache = 4x):
q1 -- k1 -- v1
q2 -- k2 -- v2
q3 -- k3 -- v3
q4 -- k4 -- v4

MQA (KV Cache = 1x):
q1 --+
q2 --+-- k_shared -- v_shared
q3 --+
q4 --+

GQA (KV Cache = 2x, Groups=2):
q1 --+
q2 --+-- k_g1 -- v_g1
q3 --+
q4 --+-- k_g2 -- v_g2
```

#### 5. 实战补充：关键属性对比表

| 特性 | MHA (Multi-Head) | MQA (Multi-Query) | GQA (Grouped-Query) | MLA (Latent) |
| :--- | :--- | :--- | :--- | :--- |
| **KV Cache 大小** | $H \times d \times L$ (最大) | $1 \times d \times L$ (最小) | $G \times d \times L$ (中等) | $d_{latent} \times L$ (极小) |
| **计算复杂度 (Attention)** | $O(L^2 \cdot H)$ | $O(L^2)$ | $O(L^2 \cdot G)$ | $O(L^2 \cdot H)$ (解压开销) |
| **表达能力** | 最强 | 较弱 (信息瓶颈) | 中等 | 强 (通过上投影恢复) |
| **推理速度** | 慢 (显存带宽瓶颈) | 快 | 较快 | 快 (显存大幅节省) |
| **典型模型** | LLaMA-1, GPT-3 | PaLM, Claude 1.x | LLaMA-2/3, Mistral | DeepSeek-V2/V3 |

## 常见考点
1.  **为什么 LLaMA-2/3 选择使用 GQA 而不是 MQA？**
    *   **回答思路**：MQA 虽然节省显存最多，但会严重影响模型的“知识召回能力”，导致推理质量下降明显。GQA 提供了一个折中点（通常设 $G=H/8$ 或 $H/4$），在推理吞吐量和模型精度之间取得了最佳平衡。
2.  **Linear Attention 为什么在实际大模型预训练中不如 Sparse Attention 流行？**
    *   **回答思路**：Linear Attention 通常需要对核函数 $\phi(x)$ 进行近似（如 Elu+1），这在极深层的网络和大规模数据上容易出现数值不稳定和收敛困难，导致最终效果难以超越标准的 Softmax Attention；而 Sparse Attention 更贴近物理意义，更容易优化。

## 记忆要点

- Self vs Cross：Self 捕捉内部依赖（QKV 同源），Cross 融合外部信息（Q 异于 KV）。
- MHA vs MQA vs GQA：MHA 独立 KV（最强），MQA 共享 KV（最快），GQA 分组共享（折中）。
- Sparse Attention：限制每个 Token 只关注局部或全局 Token，复杂度接近 O(n)。
- Linear Attention：利用核技巧重排计算顺序 (Q(K^T)V))，避免 N^2 矩阵。
- MLA：通过低秩隐变量压缩 KV，极致节省显存，以算换存。
