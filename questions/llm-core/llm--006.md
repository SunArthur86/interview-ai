# Llama 3 相比 Llama 2 有哪些关键技术改进

- **Llama 3 核心技术升级**

1. **Tokenizer 升级**
   - Llama 2: 32K vocab (SentencePiece BPE)
   - Llama 3: **128K vocab** (tiktoken-based BPE)
   - **细节**: 
     - 增加了对多语言和非自然语言（如代码、数学符号）的分词效率。
     - 优化了特殊 token 处理，更适配长上下文。
   - **效果**: 相同序列长度下编码的信息密度更高，压缩率提升约 15%，推理速度加快。

2. **Grouped-Query Attention (GQA)**
   - Llama 2 (70B): MHA → GQA (70B 独有，小模型用 MHA)
   - Llama 3 (全系 8B/70B/405B): **全部使用 GQA**
   - **原理**: 将 Query 分组，每组共享一组 Key 和 Value Head。
   - **优势**: 
     - KV Cache 显存占用大幅减少（推理吞吐量显著提升）。
     - 推理延迟降低，尤其在长序列场景下。

3. **RoPE 基频调整 (Long Context)**
   - Llama 2: $\theta = 10000$
   - Llama 3: $\theta = 500000$
   - **细节**: 提高基频降低了位置编码在高维位置上的衰减速度，从而**改善了外推能力**。
   - **效果**: 虽然原生训练长度主要为 8K，但 RoPE 调整使其能更好地通过插值扩展到 128K 甚至更长（如 Llama 3.1）。

4. **SwiGLU 激活函数**
   - 与 Llama 2 一致: FFN 使用 SwiGLU，虽然参数量增加约 1.5 倍（因为有三个线性层），但性能提升显著。

5. **训练数据**
   - Llama 2: 2T tokens
   - Llama 3: **15T+ tokens** (Llama 3 规模)
   - **数据清洗**: 极度强调数据质量，使用了启发式过滤器、语义去重、以及基于模型的文本质量评分。
   - **配比**: 大幅增加了代码和数学推理数据的比例（>5%），这也是其逻辑能力大幅提升的关键。

6. **模型规模与算力效率**
   - 规模: 8B / 70B / 405B (405B 是目前最大开源稠密模型之一)。
   - **训练细节**: 
     - 使用了超过 16K H100 GPU。
     - 并非严格遵循 Chinchilla Optimal，而是采用了“推算最优”，即在推理预算有限的情况下，训练更多 Token 比增加参数量更划算。

7. **对齐技术**
   - **RLHF + DPO 组合**: 使用 PPO 进行早期强化学习，后期主要使用 DPO (Direct Preference Optimization)，DPO 更稳定且无需显式训练奖励模型。
   - **能力增强**: 显著增强了多轮对话、指令遵循和代码生成能力。
   - **工具调用**: 对 Function Calling 进行了专门的指令微调。

- **## 常见考点**
  1. **为什么 405B 模型没有使用 MoE (Mixture of Experts)**？
     Meta 选择稠密模型是为了推理稳定性和简化工程栈，405B 虽然推理成本高，但在复杂任务上的泛化性能通常优于同等参数量的稀疏模型。
  2. **GQA 与 MQA 的区别**？
     MQA (Multi-Query Attention) 是所有 Query Head 共用一组 KV，极度压缩显存但可能损害表达能力；GQA 折中，将 Q 分组（如 8 个 Q 共享一组 KV），平衡了性能与效率。
  3. **RoPE $\theta$ 调大如何影响模型**？
     基频变大使得位置 embedding 在高频分量上变化更慢，减缓了长距离的位置衰减，使得模型在不重新训练的情况下，直接通过位置插值扩展上下文时效果更好，PPL 上升更慢。

- **## 深化实战**
  1. **实战案例**：在 vLLM 中部署 Llama 3-8B 时，由于 GQA 的存在，KV Cache 的显存占用只有 Llama 2-7B 的一半左右。实测在单卡 A10 上，Llama 3 的并发吞吐量（TPS）比 Llama 2 提升了 30-40%，但这要求推理框架必须显式支持 GQA 的内核优化，否则性能提升不明显。
  2. **代码示例 (RoPE 基频差异)**:
```python
# Llama 2 vs Llama 3 RoPE 计算差异
def get_rope_freqs(dim, theta=10000, seq_len=2048):
    # theta: Llama2=10000, Llama3=500000
    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[: (dim // 2)].float() / dim))
    t = torch.arange(seq_len, device=freqs.device)
    freqs = torch.outer(t, freqs).float()
    return torch.polar(torch.ones_like(freqs), freqs) # 复数编码
```
