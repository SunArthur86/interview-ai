# 大模型推理时为什么会出现重复输出?有哪些缓解方法

- **重复原因:**

1. **概率分布尖锐化** - 模型对某些 token 概率过高
2. **训练数据偏差** - 训练数据中有重复模式（如 JSON 重复字段）
3. **上下文强化** - 之前生成的重复内容强化后续重复（Degenerative Loop）

- **实战案例：**
在生成 JSON 格式数据时，模型曾陷入死循环，反复输出 `"status": "success", "status": "success"...`。这是因为训练数据中大量包含单调的状态字段。通过引入 `frequency_penalty=1.2` 并在后处理中加入 JSON Schema 强校验，成功解决了此问题。

- **缓解方法:**

| 方法 | 原理 | 典型参数 | 副作用 |
|------|------|------|--------|
| **repetition_penalty** | 降低已出现 token 的原始 logits | 1.1-1.5 | 过高会导致语句不通顺 |
| **no_repeat_ngram_size** | 硬性禁止 n-gram 重复 | 3-4 | 逻辑性要求高的场景慎用 |
| **frequency_penalty** | 按出现次数累加惩罚 | 0.1-0.5 | - |
| **presence_penalty** | 出现过即惩罚(与次数无关) | 0.1-0.5 | 鼓励话题发散，可能跑题 |
| **Temperature 调高** | 增加随机性，拉平概率分布 | 0.7-1.0 | 降低逻辑一致性 |
| **Top-p 调大** | 扩大候选集 | 0.9-0.99 | 引入更多低概率噪声 |

- **算法逻辑详解 (Repetition Penalty):**
```text
logits[t] = logits[t] / penalty_score (if token_t has been generated)

惩罚系数计算:
score = 1.0  (token 出现 0 次)
score = 1.15 (token 出现 1 次, 假设 penalty=1.15)
score = 1.3  (token 出现 2 次, 指数或线性累加, 取决于具体实现)
```

- **no_repeat_ngram_size 原理:**
- 设 n=3
- 每次生成新 token 时，检查是否与已生成内容形成 3-gram 重复
- 如果重复，将该 token 的 logits 置为 -inf

- **代码示例 (Transformers 应用采样参数):**
```python
from transformers import AutoModelForCausalLM, AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("deepseek-ai/deepseek-llm-7b-chat")
model = AutoModelForCausalLM.from_pretrained("deepseek-ai/deepseek-llm-7b-chat")

input_text = "Please introduce the history of AI."
input_ids = tokenizer(input_text, return_tensors="pt").input_ids

# 关键参数组合：平衡重复与随机性
outputs = model.generate(
    input_ids,
    max_new_tokens=256,
    repetition_penalty=1.1,    # 防止重复
    temperature=0.7,           # 增加多样性
    top_p=0.9,                # 核采样
    no_repeat_ngram_size=3     # 禁止3-gram重复
)
```

- **生产建议:** repetition_penalty=1.1 + temperature=0.7 + top_p=0.92。针对代码生成等强逻辑场景，应避免使用 no_repeat_ngram_size，仅使用轻微的 repetition_penalty。

## 记忆要点

- 原因：概率尖锐化、数据偏差、上下文强化导致循环
- 核心解法：repetition_penalty(1.1-1.5)降logits，no_repeat_ngram_size硬性禁止
- 辅助解法：调高Temperature(0.7-1.0)或Top-p(0.9-0.99)增加随机性
- 注意：代码生成慎用ngram限制，主要靠轻微惩罚
