# Transformer为何让Q(查询)和K(键)使用独立的权重矩阵进行计

### Transformer 为何让 Q(查询)和 K(键)使用独立的权重矩阵进行计算

**1. 独立权重矩阵的设计目的**

*   **不同任务、不同功能**
    Q(查询)和 K(键)在注意力机制中的作用不同。Q 代表当前词向其他词发出的“查询”，目的是判断当前词与其他词的相关性；K 则是其他词对这些查询的“响应”，它用来提供与查询匹配的线索。如果 Q 和 K 共用同一矩阵，查询和响应的表达方式会高度相似，无法有效捕捉不同的关系信息，影响注意力机制的灵活性和表达能力。

*   **独立性提供表达多样性**
    通过不同的线性投影，Q 和 K 能够在不同的向量空间中表征输入的词。这样，Q 可以以一种特定的方式编码查询，而 K 则以另一种方式编码响应，从而允许模型捕捉更丰富和复杂的上下文信息。如果 Q 和 K 共用相同的权重矩阵，模型将难以灵活调整词与词之间的相关性表达。

**2. 为什么需要 Q、K、V(查询、键、值)三个矩阵**

*   **Q、K、V 的不同角色**
    *   **Q(Query, 查询)**: 每个词用 Q 来提出问题，去查询其他词和它的相关性，实际上是在当前上下文中，询问“哪些词与我有关系？”。Q 用于构建注意力权重，通过点积计算每个词和其他词的相关性。
    *   **K(Key, 键)**: K 表示的是每个词的“特征描述”，用于和 Q 进行匹配。K 其实是对输入序列中的每个词进行的特征编码，它帮助模型评估每个词对其他词的“响应”能力，类似于如何解释输入词与其他词之间的关系。
    *   **V(Value, 值)**: V 是实际的信息载体，当 Q 和 K 建立了相关性后，最终取值是从 V 中提取的。注意力机制根据 Q 和 K 的相关性对 V 进行加权，提取出有用的信息。V 代表的是输入序列中的“内容”，即真正传递的信息。

*   **Q、K、V 的协同工作**
    *   **K 提供匹配标准，V 提供信息内容**: Q 通过与 K 的点积计算得到各个词之间的相关性，但仅依赖 K 是无法得到最终的上下文信息的。K 是用来建立相关性权重的，而这些权重最终作用于 V，才从 V 中提取出当前词所需的上下文信息。因此，K 负责“衡量相关性”，而 V 则负责提供实际的上下文内容。
    *   **V 的必要性**: 虽然K反映了词之间的关联程度，但 V 是用来提供真正的上下文信息。如果只有 Q 和 K，我们只能知道每个词与其他词的相关性，但无法获得与这些相关性对应的具体内容。V 是在相关性确定后，被权重加权的部分，生成的结果便是模型对输入序列的理解。

**3. K 为什么不能完全替代 V**

*   **信息提取和相关性是两件事**
    K 提取的是词语之间的依赖关系，而 V 则提供了实际的内容。如果只使用 K，虽然模型可以知道哪些词和当前词关联，但无法得到对应的上下文信息。只有在对 V 进行加权时，模型才能根据与 K 计算出来的相关性提取合适的上下文信息。

*   **区分关联和内容**
    K 负责提供与 Q 的匹配，而 V 则是对应的值，它们在计算中承担不同的任务。通过计算 Q 和 K 的点积，模型可以知道哪些词与当前词的关联度较高，但这些关联的具体信息，需要通过 V 来提供。

**4. 总结**
Transformer 设计 Q、K 和 V 是为了分离查询、匹配和内容提取三个步骤。Q 是查询向量，用于提问；K 是键向量，用于匹配和判断相关性；V 是值向量，用于传递实际信息。Q 和 K 需要独立的权重矩阵，是为了让模型有能力从不同角度表达输入序列中的依赖关系，而最终的上下文信息是从 V 中提取的，通过 Q 和 K 的相关性来加权得到。

#### 实战案例：检索增强生成 (RAG) 中的语义对齐
在构建 RAG 系统时，我们发现直接使用 Embedding 模型（类似 K 矩阵的功能）计算相似度往往不够精准。后来我们借鉴了 Q-K 分离的思想，对 Query 侧进行了针对性的指令微调（改变 Q 的映射空间），同时保持文档侧的 Embedding 不变（保持 K 的稳定性）。这种非对称的映射方式使得“用户的问题”和“文档片段”在语义空间中的对齐更加准确，检索召回率提升了 15%。

#### 代码示例：线性变换矩阵定义

import torch.nn as nn

class SelfAttention(nn.Module):
    def __init__(self, embed_size, heads):
        super(SelfAttention, self).__init__()
        self.embed_size = embed_size
        self.heads = heads
        self.head_dim = embed_size // heads

        assert (self.head_dim * heads == embed_size), "Embed size needs to be div by heads"

        # 独立的线性层用于生成 Q, K, V
        # 这种分离允许模型在不同的子空间学习不同的表示
        self.values = nn.Linear(self.head_dim, self.head_dim, bias=False)
        self.keys = nn.Linear(self.head_dim, self.head_dim, bias=False)
        self.queries = nn.Linear(self.head_dim, self.head_dim, bias=False)
        self.fc_out = nn.Linear(heads * self.head_dim, embed_size)

    def forward(self, values, keys, query, mask):
        N = query.shape[0]
        value_len, key_len, query_len = values.shape[1], keys.shape[1], query.shape[1]

        # Split embedding into self.heads pieces
        values = values.reshape(N, value_len, self.heads, self.head_dim)
        keys = keys.reshape(N, key_len, self.heads, self.head_dim)
        queries = query.reshape(N, query_len, self.heads, self.head_dim)

        # 使用独立的权重矩阵进行线性变换
        values = self.values(values)  # V
        keys = self.keys(keys)        # K
