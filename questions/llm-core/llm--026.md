# 预训练数据的质量如何影响模型能力?高质量数据管道的关键步骤有哪些

**数据质量 = 模型质量** (Data is all you need)

**核心原理**：Llama 3 的实验表明，用 15T tokens 的高质量数据训练 70B 模型，效果远超用 2T tokens 训练的更大参数模型。数据质量的边际效益在后期远高于数据规模。

**高质量数据管道的关键步骤**

1.  **数据采集与源**
    - **网页**：Common Crawl、Dolma 等。特点：量大但噪声极高，需重度清洗。
    - **高价值源**：书籍、学术论文、Wikipedia。特点：逻辑性强、语言规范。
    - **代码**：GitHub (Star 过滤)。特点：提升模型逻辑推理和长程依赖能力。
    - **合成数据**：利用强模型生成。特点：用于填补长尾分布，但需防范模型坍塌。

2.  **去重**
    - **URL 级**：完全相同的网页链接只保留一份。
    - **文档级**：使用 MinHash LSH (Locality Sensitive Hashing) 或 SimHash 进行近似去重，去除高度相似的文章。
    - **子串/行级 (Llama 3 创新)**：去除网页中通用的 Header、Footer、License 模板。经验：此步骤可去除 30-50% 的无效 token。

3.  **质量过滤**
    - **启发式规则**：去除过短（<512 tokens）、过长（上下文截断）、标点符号比例异常、乱码文本。
    - **模型打分**：训练一个小型的 Quality Classifier（如基于 FastText 或 BERT），给文本打分，低分丢弃。
    - **安全过滤**：正则匹配及 NER 模型去除 PII（个人隐私）、暴力、色情等有害内容。
    - **语言识别**：利用 fastText 等工具识别并过滤非目标语言（如只保留中文和英文）。

4.  **数据配比**
    - **Llama 3 经验配比**：~50% 通用网页 + ~25% 代码 + ~17% 数学/推理数据 + ~8% 多语言。
    - **关键点**：代码和数学数据对模型的逻辑推理能力至关重要，配比需通过消融实验确定。

5.  **课程学习**
    - **阶段一**：初始预训练，主要使用简单、通用的高质量数据。
    - **阶段二**：加入长文本数据，提升上下文窗口能力。
    - **阶段三 (退火/Annealing)**：最后阶段使用最高质量的数据（如教科书、QA 对），降低学习率，提升指令遵循能力。

**数据管道 ASCII 图**
```text
[原始数据源] (Web, Books, Code)
      │
      ▼
[去重] (URL -> MinHash -> 行级去重)
      │
      ▼
[质量过滤] (规则打分 -> 模型打分 -> 安全过滤)
      │
      ▼

      │
      ▼
[课程学习] (Pretrain -> Long Context -> Annealing)
      │
      ▼
[最终 Token 流] -> 分词器 -> 模型训练
```

**实战案例：PII 去重后的灾难性遗忘**
在某金融领域微调项目中，为了隐私合规，团队对训练数据进行了极高强度的正则去重（去除所有数字、邮箱、身份证号）。结果发现模型在数学计算能力上断崖式下跌（甚至无法做 1+1）。**教训**：过度激进的清洗会破坏上下文的语义结构。解决方案是改用实体替换，将敏感信息替换为 `<SSN>`, `<PHONE>` 等占位符，既保护隐私又保留了数值逻辑。

**代码示例：MinHash 去重流程**
```python
from datasketch import MinHash, MinHashLSH

def deduplicate_docs(documents, threshold=0.8):
    # 创建 LSH 索引
    lsh = MinHashLSH(threshold=threshold, num_perm=128)
    minhashes = {}
    
    for idx, doc in enumerate(documents):
        # 计算 MinHash 签名
        m = MinHash(num_perm=128)
        for d in doc.split():
            m.update(d.encode('utf8'))
        lsh.insert(idx, m)
        minhashes[idx] = m
    
    # 查询并移除重复文档
    unique_indices = []
    for idx in minhashes:
        # 返回相似度大于 threshold 的文档索引（包含自己）
        near_duplicates = lsh.query(minhashes[idx])
        if min(near_duplicates) == idx: # 仅保留索引最小的作为代表
            unique_indices.append(idx)
            
    return [documents[i] for i in unique_indices]
```

## 常见考点
1.  **数据配比原则**：为什么代码数据有助于提升推理能力？（代码具有严格的逻辑结构和执行反馈，能强制模型学习精确的思维链）。
2.  **去重策略**：精确去重和模糊去重（MinHash）的区别是什么？（精确去重（MD5）耗时长但准，仅限内容完全一致；模糊去重基于 Jaccard 相似度，能处理改写或复制粘贴的内容）。
3.  **合成数据风险**：什么是模型坍塌？如何避免？（模型反复学习自己生成的数据导致错误放大和分布退化；解决方案是严格控制合成数据占比，如 <5%-10%，并主要在高质量领域使用）。
4.  **退火训练**：为什么最后阶段要降低学习率并使用高质量数据？（为了在保留泛化能力的同时，微调输出分布以符合人类偏好，类似指令微调的预热）。

## 记忆要点

- 核心原则：数据质量决定模型上限，高质量数据的边际效益远高于规模。
- 关键步骤：去重（URL/MinHash/子串） -> 质量过滤（规则/模型打分） -> 安全过滤。
- 数据配比：代码和数学数据对逻辑推理至关重要，需通过消融实验确定比例。
- 课程学习：分阶段训练，从简单通用数据到长文本，最后用高质量数据退火。
- 避坑指南：过度激进的清洗（如全去数字）会破坏语义结构，导致能力灾难性遗忘。
