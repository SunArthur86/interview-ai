# BERT的结构

BERT 的结构基于 Transformer 的**Encoder（编码器）**部分，采用了深度双向的架构。

**1. 核心组件**
*   **Embedding 层（输入层）**：由三部分相加而成。
    *   **Token Embedding**：将词转换为向量（通常使用 WordPiece）。
    *   **Position Embedding**：标记词在序列中的位置（可学习的向量）。
    *   **Segment Embedding**：区分两个句子（用于句对任务，如判断下一句）。
*   **Encoder 层（主体）**：堆叠了多个相同的 Transformer Encoder Block（BERT-Base 有 12 层）。每个 Block 包含：
    *   **Multi-Head Self-Attention**：让当前词同时关注上下文所有词。
        *   **参数细节**：BERT-Base 的 Hidden Size 为 768，Attention Heads 为 12（每个 Head 维度 64）。FFN 维度为 3072（4倍 Hidden Size）。
    *   **Position-wise Feed-Forward Networks (FFN)**：对每个位置的向量进行非线性变换。
    *   **Add & Norm**：残差连接和层归一化，用于稳定训练。
        *   **结构细节**：BERT 使用 Post-LN（即 Attention/FFN 输出后再做 Norm），这与某些现代大模型使用的 Pre-LN（先 Norm 再 Attention）不同，这使得 BERT 训练初期较为困难，通常需要 Warmup。

**2. 关键特性**
*   **双向性**：由于使用了 Encoder 的 Mask 为空（全可见），BERT 能够同时利用左侧和右侧的上下文，生成深度融合的词义表示。
*   **预训练任务**：结构支持 **MLM (Masked Language Modeling)** 和 **NSP (Next Sentence Prediction)** 两种自监督训练方式。

**模型结构图**：
```text
Inputs: [CLS] Token A Token B [SEP]
  │
  ├─> Token Embeddings
  ├─> Position Embeddings
  └─> Segment Embeddings
  │
  ▼ (Sum & Normalize)
┌─────────────────────────────────┐
│       Transformer Encoder       │
│  ┌───────────────────────────┐  │
│  │ Multi-Head Self-Attention  │  │ <--- 全双向交互
│  └───────────┬───────────────┘  │
│              ▼                  │
│  ┌───────────────────────────┐  │
│  │    Add & Layer Norm       │  │
│  └───────────┬───────────────┘  │
│              ▼                  │
│  ┌───────────────────────────┐  │
│  │   Feed-Forward Network    │  │
│  └───────────┬───────────────┘  │
│              ▼                  │
│  ┌───────────────────────────┐  │
│  │    Add & Layer Norm       │  │
│  └───────────┬───────────────┘  │
│      (Repeat L times)          │
└──────────────┬──────────────────┘
               │
               ▼
      Output: [CLS] T1 T2 ... [SEP]
```

**3. 实战补充**
*   **实战案例**：在文本分类任务微调 BERT 时，若直接取每个 Token 的 hidden state 进行平均池化，效果往往不如只取 [CLS] 向量。这是因为 BERT 的预训练任务（NSP 和 MLP Head）专门优化了 [CLS] 聚合全局语义的能力，强行使用平均池化反而破坏了这种预训练的对齐。
*   **代码示例**：使用 `transformers` 获取 BERT 隐状态的代码片段，展示如何区分 [CLS] 和 Token 输出：
```python
from transformers import BertModel, BertTokenizer

model = BertModel.from_pretrained("bert-base-uncased")
tokenizer = BertTokenizer.from_pretrained("bert-base-uncased")

inputs = tokenizer("Hello world", return_tensors="pt")
outputs = model(**inputs)

# last_hidden_state: [batch_size, seq_len, hidden_size]
last_hidden_state = outputs.last_hidden_state 

# 获取 [CLS] 向量 (用于分类任务)
cls_embedding = last_hidden_state[:, 0, :] 
```

**4. BERT (Encoder-only) vs GPT (Decoder-only) 对比**

| 特性 | BERT (Encoder) | GPT (Decoder) |
| :--- | :--- | :--- |
| **注意力机制** | 双向 | 单向 (Causal Mask) |
| **上下文感知** | 同时看到过去和未来 | 只能看到过去 |
| **预训练目标** | MLM (填空) + NSP (下句预测) | CLM (自回归预测下一个词) |
| **适用任务** | NLU (理解、分类、实体抽取) | NLG (生成、续写) |
| **结构特点** | Post-LN，收敛慢 | Post/Pre-LN，生成流畅 |
| **典型模型** | BERT, RoBERTa, DeBERTa | GPT-3/4, LLaMA |
