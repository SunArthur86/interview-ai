# 大模型出现幻觉的原因

大模型出现幻觉的原因是多方面的，核心在于自回归生成机制和数据缺陷。

**主要原因分析**：

1. **自回归生成的累积误差（核心机制原因）**：
   - 模型采用逐 Token 生成，一旦前序生成了错误的 Token，后续为了保持上下文连贯性，模型会基于错误的上下文继续编造，导致错误像滚雪球一样积累且无法回溯。

2. **数据源的知识缺陷（数据原因）**：
   - 训练数据中存在错误、偏差或过时的信息。模型“记住”了这些错误知识，并在推理时复现。
   - 对于训练数据中极少出现的“长尾知识”，模型并未真正学好，仅仅是模糊的记忆，导致生成时一本正经地胡说八道。

3. **概率生成的本质（模型原因）**：
   - LLM 本质上是概率分布模型 $P(w_t | w_{<t})$，它生成的是“最可能”的下一个词，而不是“事实正确”的词。当多个候选词概率相近时，可能选中逻辑通顺但事实错误的词。

4. **对齐税与 Sycophancy（阿谀奉承）**：
   - 经过 RLHF 等对齐训练后，模型倾向于生成让用户满意的回答，有时会顺从用户的错误引导（如用户故意问错误事实时，模型可能会附和），而非坚持事实真相。

**实战案例**：
在医疗问答系统中，模型常会编造不存在的药物名称。经排查发现，这是由于 RLHF 阶段偏好数据过于强调回答的“完整性”而非“准确性”，导致模型倾向于生成虚假细节来凑字数。引入基于知识图谱的约束后显著改善。

**代码示例 (检测与规避)**：
```python
# 使用 LogProb 检测低置信度生成（可能的幻觉）
outputs = model.generate(input_ids, max_new_tokens=100, output_scores=True, return_dict_in_generate=True)

# 检查生成token的平均概率
transition_scores = model.compute_transition_scores(outputs.sequences, outputs.scores, normalize_logits=True)
avg_prob = torch.exp(transition_scores).mean()

if avg_prob < 0.3:
    print("Alert: High hallucination risk (Low confidence)")
```

**常见考点**
1. 如何缓解大模型幻觉？（提示工程增强、检索增强生成 RAG、知识图谱约束、思维链 CoT 提高推理准确性）
2. 什么是“知识边界”问题？（模型对未知问题的回答态度，是拒绝回答还是胡编乱造，这通常通过训练时的 refusal data 来调节）

## 记忆要点

- 雪崩效应：因为自回归逐Token生成，所以前序错误会累积且无法回溯。
- 概率本质：LLM本质是拟合概率分布，生成的是最可能而非事实正确的词。
- 数据缺陷：训练数据含长尾知识错误或过时信息，导致模型一本正经胡说八道。
- 对齐税：因RLHF偏好顺从用户，导致模型产生阿谀奉承而放弃客观事实。
